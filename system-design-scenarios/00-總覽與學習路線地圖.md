# 00 · 總覽與學習路線地圖

> 這一章不教任何一道題,它教你**怎麼把已有的散材料,走成一條能上白板的路**。
>
> 你的問題從來不是「材料不夠」——你有 `system-design/` L0–L9 零件庫、`financial-consistency`、`perf`、`observability`、`concurrency-capacity` 一堆世界級深礦。你的問題是:**它們是一張 hub-and-spoke 的 reference 網,不是一條線。** 趕面試時你不知道「先讀哪、讀到哪算夠、哪些題用哪些零件」。這章就是那條線。

---

## 1. 先分清:兩種「系統設計」別混為一談

| | 概念深挖(零件) | 場景整題(裝配) |
|---|---|---|
| 代表 | `system-design/` L0–L9 | **這個 track** |
| 形式 | 「講講 CAP / 一致性哈希 / 熔斷」 | 「**設計一個 Twitter / 限流器 / 支付**,限時 45 分鐘」 |
| 練的肌肉 | 被追問時的**彈藥**——reason about 單個零件的 tradeoff | 主導一道題的**流程**——把零件裝成整機並講清 |
| 失敗長相 | 「知道熔斷,但說不出何時不該用」 | 「零件都會,但 45 分鐘下來架構是亂的、估算沒做、tradeoff 沒暴露」 |

**面試兩種都考**,但這個 track 只練第二種——因為第一種你已經有了(零件庫),缺的是把它們**裝起來、限時講清**的能力。零件不夠時,順著每題第 7 段的**錨點表**回零件庫補,不重學。

---

## 2. 一道 SDI 面試長什麼樣(45 分鐘解剖)

完整方法在 `01 七步框架`,這裡先給你骨架和**時間感**,好讓後面的路線地圖有座標:

| 步 | 在做什麼 | 時間 | 不做會怎樣 |
|---|---|---|---|
| 1 澄清 | 功能邊界 / 非功能 SLA / 規模 / 約束 | ~5 min | 設計錯題,後面全白做 |
| 2 估算 | QPS / 存儲 / 帶寬 / 連接數 | ~5 min | 架構拍腦袋,撐不撐得住說不清 |
| 3 API + 數據模型 | 對外接口 + 核心表/KV + 分片鍵 | ~8 min | 高層架構沒落點 |
| 4 高層架構 | 請求一鏡到底走完 | ~10 min | — 這是主菜 |
| 5 deep dive | 挑 2–3 個最會被追問的點下鑽 | ~12 min | 顯得只會畫框、不懂細節 |
| 6 瓶頸 / 取捨 | 主動暴露 tradeoff、瓶頸、演進 | ~5 min | **最大扣分項**——面試官等的就是這個 |

> **評分的真相:** 面試官不是在數你畫對幾個框,是在聽你**能不能 reason about tradeoff、能不能主動暴露代價**。沿用 `system-design/README` 那張標準線——

| 角色 | 在這道題上的表現 |
|---|---|
| 一般工程師 | 畫得出一個能跑的架構 |
| **資深(你的目標)** | 畫得出 + **講得清每個選擇的 why 和代價**,主動暴露 2–3 個 tradeoff |
| 架構師 | **擁有這個決策**——「要 P99<200ms → 必須加緩存 → 換來一致性窗口 → 這業務能接受」這句話 |

## 3. 學習路線地圖(本章核心)

「能獨立走完一道題」可以拆成 6 個**能力包**。每包指向你**已有**的材料——這就是把散礦串成線:

| 能力包 | 你要會什麼 | 去哪讀(已有) |
|---|---|---|
| **A · 框架 + 數字感** | 七步骨架、容量估算的背誦級數字 | 本 track `01`、`02`(新寫) |
| **B · 存儲與數據規模化** | 選型決策樹、分庫分表/分片鍵、讀寫分離、緩存四件套 | `system-design/06`、`07` + `redis-handson/` + `mysql-handson/` |
| **C · 韌性與流量** | 超時/熔斷/重試/隔離/限流/降級、背壓、削峰、容量定容 | `system-design/01 韌性` + `concurrency-capacity/` + `distribution/限流算法/` |
| **D · 一致性與複製** | 一致性模型、複製拓撲→RPO、quorum、冪等/exactly-once、對賬 | `system-design/00`、`02`、`03` + `financial-consistency/` |
| **E · 服務化與通信** | RPC vs 消息 vs 事件驅動、MQ 扇出、服務發現/配置 | `system-design/04`、`05` |
| **F · 容災/可觀測/發布** | RPO/RTO、多活、SLO/告警、藍綠灰度 | `system-design/08`、`09` + `observability/` |

## 4. 兩條標準線:IC 面試線 vs 架構師決策線

你說的「面試 + 長遠架構師」是兩個目標,在每道題裡走**不同的深度**:

| | IC 面試線 | 架構師決策線 |
|---|---|---|
| 目標 | 45 分鐘講清一個合理設計 + 主動暴露 2–3 個 tradeoff | **擁有**這個決策:成本誰買單、容量怎麼規劃、怎麼演進、會怎麼掛 |
| 讀每題的 | 第 1–5 段(澄清→架構→deep dive) | **第 6 段「架構師深挖」+ 第 7 段錨點**往深礦鑽 |
| 一句話 | 「裝得出 + 講得清」 | 「為什麼是**這個**裝法,代價誰買單,何時不該這麼裝」 |

> 你的路徑:**先把 IC 線走順**(它是面試的及格+加分線),每題收尾時順手讀架構師深挖層,長期把「面試答法」升級成「真實決策」。兩條線共用同一道題,不另起爐灶。

## 5. 診斷:你該從哪開始

三個自問,分流到不同節奏:

1. **七步框架 + 容量估算,我能空手走一遍嗎?** → 不能就先吃 `01`/`02`,這是地基,別跳。
2. **L0–L9 的零件,我大致 reason 得動嗎?** → (你的情況)能 → **跳過 T1 精讀**,T1 只當「練框架的暖身」快走,直接從 T2 開始練整題打法,缺零件回錨點補。
3. **我練題時計時、蓋答案了嗎?** → 沒有的話,先看第 6 節——**SDI 是肌肉,只讀不練等於沒練**。

**為你(概念脊柱已熟)定制的順序:**
`00`(本章,確認哪些能力包已會)→ `01`/`02` 快過建立框架與數字感 → 直接 **T2 三高頻**(`News Feed` / `IM` / 再加 T3 的 `支付`)整題實練 → 補完 T2/T3 → `20 模式速查` 收口 → `21 playbook` 臨場 → `99` 抽卡。T1 五題作為框架不熟時的回爐暖身。

---

## 6. 怎麼用這個 track(學習節奏)

每道場景章,**這樣練才有效**:

1. **蓋住答案,計時 45 分鐘**,自己空手走完七步(在紙/白板上畫)。
2. 對照本章的「高層架構 + deep dive」,看你漏了哪步、哪個 tradeoff 沒想到。
3. 讀「**架構師深挖層**」,把答案升級到決策層。
4. 順「**錨點表**」回零件庫,補掉這題暴露的知識空洞。
5. 最後用「**面試卡**」抽查,確認能脫稿說出。

> ❌ 最常見的無效學習:把 17 章當小說讀一遍。SDI 是**輸出型**技能——不計時、不空手走一遍,讀得再多上白板還是會卡。**讀 1 題不如練 1 題。**

---

## 一句話總綱(背起來)

> **這個 track 把你散落的零件庫(L0–L9 + 深礦)串成一條線:6 個能力包撐起 17 道整題,每題用固定七步「裝 + 講」,IC 線練「裝得出講得清」、架構師線練「代價誰買單」。練法只有一條——蓋住答案、計時、空手走一遍,再回錨點補洞。**
