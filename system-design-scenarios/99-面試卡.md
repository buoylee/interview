## 一 · 方法與數字(Part A 抽查)

| 正面 | 背面 |
|---|---|
| SDI 七步是哪七步? | 需求**澄清** → 容量**估算** → **API** → **數據**模型 → 高層**架構** → **deep dive** → 瓶頸**取捨**。(`01`) |
| 半場(~22 min)的硬指標? | **必須已畫出高層架構**;還在澄清/估算 = 危險,立刻往前推。(`21`) |
| QPS 怎麼估? | `日操作數 ÷ 1e5(一天秒數)= 平均 QPS`;`峰值 = 平均 × 3`。(`02`) |
| 存儲怎麼估? | `寫 QPS × 單條大小 × 86400 × 保留期 × 副本3 × 索引開銷`。(`02`) |
| 背誦級數字(至少報得出 5 個)? | 一天 **1e5 s**;峰值 **×3**;副本 **×3**;行記錄 **~1KB**;單 Redis **~10 萬 QPS**;單 MySQL 寫 **~幾千**;跨洲 RTT **~100ms**。(`02`) |
| 估算完必須做什麼? | 接一句「**所以架構上要……**」——把數字焊到決策(讀寫比大→緩存;TB級→分庫;媒體→CDN)。(`01`/`02`) |
| 七步裡最大加分項?最大扣分項? | 加分=**主動暴露 tradeoff**(第7步);扣分=**假裝方案沒缺點 / 沒澄清就開畫**。(`01`/`21`) |

## 二 · 17 題「一句話題眼」速記(蓋住右欄自測)

> 每道題最值錢的那一句。面試報出題眼 = 立刻顯出你抓住了這題的本質矛盾。

| 題 | 一句話題眼 |
|---|---|
| `03` 短鏈 | **QPS 騙局**——寫 40 讀千級單機全扛,真瓶頸是 5 年 TB 級存儲增長 + 熱鏈接傾斜;短碼=發號器+序號打散(兼得無碰撞與不可枚舉)。 |
| `04` 限流器 | 固定窗有**邊界雙倍突刺**;分散式靠 **Redis Lua 原子**;大規模用**本地預扣配額 + 中心對賬**兩級。 |
| `05` 唯一 ID | Snowflake 怕**時鐘回撥**;**趨勢遞增**對 B+ 樹順序寫友好(vs UUID 頁分裂);嚴格遞增會暴露業務量。 |
| `06` 緩存 | 數據分布=**一致性哈希 vs slot**;四件套防穿透/擊穿/雪崩;真難點是**熱點 key 與大 key**。 |
| `07` 排行榜 | ZSet = **跳表 + dict**,O(log N) 排名;億級單 key 無法分片 → **分桶/分片歸併**。 |
| `08` News Feed | **寫擴散 vs 讀擴散 vs 混合**;大 V 走讀擴散;feed **只存 ID** 省 40 倍且可改可刪。 |
| `09` IM | **連接數才是關鍵約束**(不只 QPS);連接 = **常駐成本**,不按 QPS 縮容;重連風暴是招牌失效。 |
| `10` 通知 | 扇出 + **渠道適配層**;**去重 vs 限頻 fail-safe 方向相反**;成本大頭在外部網關。 |
| `11` 自動補全 | 補全 QPS = 搜索 × 每鍵一請求(~20 倍);**Trie 預存 Top-K**;離線算 + **只讀快照原子切換**。 |
| `12` 網盤 | **內容尋址去重 = 天然冪等 + 秒傳**;metadata/blob 分離;衝突保留 **conflicted copy** 雙版本。 |
| `13` 爬蟲 | **禮貌 > 速度**;布隆**只假陽性 = 只漏爬**(良性);**按 host 分片**把分散式限速消解成本地串行。 |
| `14` 支付 | **反規模題,一分不能錯**;冪等四防線 + **本地消息表/TCC/Saga** 最終一致 + **對賬兜底**(對賬是設計一部分)。 |
| `15` 派單 | **不對稱流量**(位置高頻寫可丟 vs 叫車低頻讀敏感);地理索引 GeoHash;並發搶同一司機**靠存儲原子寫**。 |
| `16` 秒殺 | **99% 請求注定失敗**;**多層漏斗**把洪峰擋在庫存前;Redis Lua 預扣 + 分段庫存防單行鎖。 |
| `17` 視頻 | 先切開 **VOD vs Live**;**5 Tbps → CDN 入場券**;轉碼流水線 + 直播源站放大三招收斂。 |
| `18` 廣告聚合 | **一題兩張臉**(看板近似 vs 計費精確)→ **Lambda 雙層**;HLL ~12KB 估億級 UV;watermark 容亂序。 |
| `19` 協同編輯 | **難在並發收斂不在規模**;**OT(中心 transform)vs CRDT(無中心收斂)**;要離線 → CRDT。 |

## 三 · 六大模式族快問快答(Part C `20` 抽查)

| 正面 | 背面 |
|---|---|
| 寫擴散 vs 讀擴散怎麼選? | 讀多寫少 + 接收方不多 → **寫擴散**(讀快);超級節點(大 V/大群)→ **讀擴散**(避免寫放大)。標準答案是**混合**。(`08`/`09`) |
| 緩存四件套? | 穿透→**布隆/緩存空值**;擊穿→**互斥鎖/邏輯過期**;雪崩→**隨機 TTL/多級**;旁路→**Cache-Aside**。(`06`) |
| 為什麼不用 2PC 做分散式事務? | 同步阻塞 + 協調者單點 + 數據庫鎖佔用長。改用**本地消息表/TCC/Saga** 最終一致。(`14`) |
| 防超賣的核心手段? | **存儲層原子操作**(Redis Lua 預扣 / DB 樂觀鎖 `where stock>0`),絕不「先查再寫」(TOCTOU)。(`16`/`15`/`03`) |
| 冪等怎麼做? | **冪等鍵 + 防重表 + 狀態機條件 UPDATE + 唯一索引兜底**;靠存儲原子性,不靠先查。(`14`) |
| 近似算法各算什麼? | **HLL** 估基數(UV);**Count-Min** 估頻次;**布隆** 判存在性。用精度換內存。(`18`/`13`/`06`) |
| 海量「找附近」用什麼? | **地理空間索引**:GeoHash(編碼成前綴可查)/ QuadTree / S2;Redis GEO 底層=ZSet+GeoHash。(`15`) |
| 拿到新題第一步做什麼? | 先判**本質類型**(讀密集?扇出?強一致?地理?流式?協同?)→ 調對應**模式族**,別背題。(`20`) |

## 四 · 高頻 deep dive 快答(最常被追問的點)

| 正面 | 背面 |
|---|---|
| feed 為什麼只存 ID 不存整貼? | 省內存(~40 倍)+ 貼文**可改可刪**(改一處 vs 改千萬份拷貝);讀時按 ID 批量回查。(`08`) |
| IM 一條消息怎麼從 A 到 B? | A 的連接 → 接入 gateway → 查**用戶→gateway 路由表**定位 B 在哪台 → 轉發到 B 的長連接;B 離線則寫離線消息待 PULL。(`09`) |
| 短鏈 301 vs 302 怎麼選? | 301 瀏覽器緩存→省負載但**統計丟失**;302 每次回源→**統計準 + 可改指向**。統計是核心價值 → 選 **302**。(`03`) |
| 緩存集群掛了怎麼辦? | 讀全回源 DB,DB 扛不住 10 倍 → 必須**降級 + 對 DB 限流**,寧可少數變慢別讓 DB 雪崩。(`06`/`03`) |
| OT 和 CRDT 的根本區別? | OT=**中心服務器**對並發 op 做 transform 收斂(實現複雜、依賴中心);CRDT=**數據結構本身**滿足交換律/冪等,任意順序收斂(支持離線去中心,代價是墓碑膨脹)。(`19`) |
| 秒殺怎麼把洪峰擋住? | **多層漏斗**:前端限流/答題 → 網關限流 → MQ 異步下單 → DB;600 萬 QPS 逐層銳減到幾百。(`16`) |
| 廣告計費怎麼又快又準? | **Lambda 雙層**:近似層(HLL/流式)給秒級看板,精確層(批 + 對賬)給計費;乱序用 **watermark** 容忍遲到。(`18`) |
| 分散式爬蟲怎麼做到禮貌? | **按 host 分片**——同域 URL 路由到同一 worker,把「分散式限速」消解成**單機本地串行限速**。(`13`) |
