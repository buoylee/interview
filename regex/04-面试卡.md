## A. 概念问答(高频)

**Q1. 贪婪和懒惰量词的区别?**
一句话:贪婪(`*`)**先吃到最多再回吐**,懒惰(`*?`)**先吃到最少再多吃**。
展开:`<.*>` 配 `<a><b>` 得整串(贪婪吃到最后一个 `>`);`<.*?>` 得 `<a>`(到第一个 `>` 停)。提取最短片段用懒惰,或更稳地用排除类 `<[^>]*>`。

**Q2. 什么是回溯(backtracking)?**
引擎一条路走到黑,某步匹配失败时**退回上一个量词,让它少吃一个字符再试别的路**。回溯是 Python/Java/PCRE 这类 NFA 引擎的核心机制,也是灾难性回溯的根源。

**Q3. 什么是灾难性回溯 / ReDoS?怎么防?**
一句话:**嵌套量词等结构**让引擎在不匹配的输入上回溯组合**指数级爆炸**,几十字符就能跑满 CPU。
怎么发生:`(a+)+$` 配 `"aaaa...!"`。怎么防:① 改写消除嵌套(`(a+)+`→`a+`)+ 用排除类替代 `.*`;② 占有量词/原子组 `a++` `(?>...)`(Py3.11+);③ 不可信输入用 RE2(`google-re2` / Go);④ `regex` 模块 `timeout=`。

**Q4. DFA 和 NFA(回溯)引擎的区别?Go 为什么没有前瞻?**
NFA(Python/Java)一条路试、撞墙回溯,**功能强**(前瞻、反向引用)但**可能指数爆炸**。DFA/RE2(Go/grep)同时跟踪所有可能路径,**线性时间**但**放弃**前瞻和反向引用(它们依赖回溯)。Go 选 RE2 是用功能换"永不卡死"的安全保证,适合处理不可信输入。

**Q5. 捕获组 `(...)` 和非捕获组 `(?:...)` 的区别?**
捕获组占编号、能被 `\1` 引用、能在结果里单独取出;非捕获组只做分组(如配合 `|` 或量词)。不需要取值时用 `(?:...)`,略快且不打乱编号。

**Q6. `re.match` / `re.search` / `re.fullmatch` 区别?**
`search` 任意位置找第一个;`match` 从**开头**匹配(但**不要求到结尾**);`fullmatch` 整串完全匹配。
⚠️ 坑:`re.match(r'\d+', '12ab')` **成功**(匹配 `12`)。Java 的 `matches()` 是整串,对应 Python 的 `fullmatch`,不是 `match`——Java 转 Python 必栽。

**Q7. 什么是零宽断言?举个有用的例子。**
匹配"位置"而非"字符",成功后指针不前进(零宽)。前瞻 `(?=)`/`(?!)`、后顾 `(?<=)`/`(?<!)`。例:`\B(?=(?:\d{3})+(?!\d))` 给整数加千分位——只定位插入点,不消耗字符。

**Q8. 为什么 Python 的后顾 `(?<=...)` 要定长?**
后顾要从当前位置往左固定回退再比对,变长无法确定回退几位。`(?<=ab)` 行,`(?<=a+)` 报错。需要变长后顾用第三方 `regex` 模块。

**Q9. `re.MULTILINE` 和 `re.DOTALL` 分别改了什么?**
`MULTILINE`(`re.M`)让 `^ $` 匹配**每行**首尾(默认只整串首尾)。`DOTALL`(`re.S`)让 `.` 也匹配**换行**(默认不含)。两个独立,常被混。

**Q10. `re.findall` 返回什么?有什么坑?**
无捕获组→整串列表;**1 个组→只返回该组的列表**;**≥2 个组→元组列表**。想要整串又分组,用 `finditer` + `m.group()`,别用 `findall`。

**Q11. `\d` 一定是 `[0-9]` 吗?**
不是。Python3 默认 `\d` 含 Unicode 数字(如全角 `１`)。要纯 ASCII 用 `re.A` 或显式 `[0-9]`。`\w` 同理默认含中文等。

**Q12. `re.compile` 有必要吗?**
`re` 内部有编译缓存(约512条),功能上常可省;但热路径显式 `compile` 到模块级更清晰、可复用 `Pattern` 方法、也避免缓存被挤掉。推荐对固定正则显式编译。

## B. 手写题(常考,先归桶再写)

**T1. 校验密码:≥8 位,含大小写字母和数字。**(经典前瞻题)
```python
r'^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$'
```
三个前瞻各查一类字符(零宽,不消耗),最后 `.{8,}` 管长度。考点:用前瞻把"多个独立条件"叠在一起。

**T2. 提取字符串里所有整数。**
```python
re.findall(r'-?\d+', s)
```

**T3. 找出连续重复的单词(如 "the the")。**
```python
re.findall(r'\b(\w+)\s+\1\b', text)     # 反向引用 \1
```

**T4. 整数加千分位。**
```python
re.sub(r'\B(?=(?:\d{3})+(?!\d))', ',', '1234567')    # 1,234,567
```

**T5. 驼峰转下划线。**
```python
re.sub(r'(?<=[a-z0-9])(?=[A-Z])', '_', 'getUserName').lower()    # get_user_name
```
注意:这个简版处理不了连续大写缩写(`parseHTTPResponse` → `parse_httpresponse`,缩写被吞)。要保留缩写需更复杂的两段式正则——面试能说出这个局限即可。

**T6. 校验 IPv4(每段 0–255)。**
```python
r'^(?:(?:25[0-5]|2[0-4]\d|1?\d?\d)\.){3}(?:25[0-5]|2[0-4]\d|1?\d?\d)$'
```
考点:`25[0-5]|2[0-4]\d|1?\d?\d` 把 0–255 拆成三段表达。

**T7. 写个"够用"的邮箱正则,并说明为什么不追求完美。**
```python
r'^[^@\s]+@[^@\s]+\.[^@\s]+$'
```
答题要点:完整 RFC 5322 正则几百字符无人用;实务用宽松校验拦明显错误 + 发确认邮件真验证。**主动讲这个权衡就是得分点。**

## C. 一眼看穿题(给正则问匹配/风险)

- `^(cat|dog)$` vs `^cat|dog$`:前者整串是 cat 或 dog;**后者**因 `|` 优先级极低,等于 `(^cat)|(dog$)`——开头是 cat **或** 结尾是 dog。**记住给 `|` 加组。**
- `(a+)+$`:典型 ReDoS,不匹配输入上指数爆炸。
- `<.*>`:贪婪,会吞掉中间所有标签。
- `[a-z]{2,4}?`:懒惰,优先只匹配 2 个。
