# 03 · 配方与陷阱(抄就能用 + 别踩的坑)

> 上半部分是**可直接抄的配方**;下半部分是**会让你 debug 半天或线上爆炸的坑**。
> 所有配方默认 Python `r"..."`。能用字符串方法(`str.strip` `str.split`)解决的别上正则——简单且快。

---

## Part A · 配方库

### 数字

```python
r'-?\d+'                      # 整数(带可选负号)
r'-?\d+(?:\.\d+)?'           # 浮点(整数也匹配)
r'-?\d+(?:\.\d+)?(?:[eE][+-]?\d+)?'   # 科学计数法
r'\d{1,3}(?:,\d{3})*'        # 带千分位的整数,如 1,299,000
```

### 邮箱(务实版,别追求完美)

```python
r'[^@\s]+@[^@\s]+\.[^@\s]+'   # 够用:非空白非@ + @ + 域名 + .后缀
```

> ⚠️ 完整 RFC 5322 邮箱正则有几百字符且没人用。**实务做法**:用上面这种"长得像邮箱"的宽松校验拦掉明显错误,真正验证靠**发确认邮件**。面试被问"写个邮箱正则"时,先写务实版,再说明这个权衡——这正是考点。

### URL / IP / 日期时间

```python
r'https?://[^\s/$.?#]\S*'                # URL(务实版)
r'\b(?:\d{1,3}\.){3}\d{1,3}\b'           # IPv4(宽松,不校验 0-255)
# IPv4 严格版(每段 0-255):
r'\b(?:(?:25[0-5]|2[0-4]\d|1?\d?\d)\.){3}(?:25[0-5]|2[0-4]\d|1?\d?\d)\b'
r'\d{4}-\d{2}-\d{2}'                      # 日期 ISO,如 2026-06-15
r'\d{2}:\d{2}(?::\d{2})?'                 # 时间 HH:MM[:SS]
```

### 中文 / 手机号

```python
r'[一-鿿]+'          # 连续中文(基本汉字区)
r'1[3-9]\d{9}'               # 中国大陆手机号(11位,1开头第二位3-9)
```

### 清洗文本

```python
re.sub(r'\s+', ' ', s).strip()           # 多个空白压成一个 + 去首尾
re.sub(r'<[^>]+>', '', html)             # 粗暴去 HTML 标签(仅应急)
```

### 提取键值 / 解析日志

```python
dict(re.findall(r'(\w+)=([^;]+)', 'a=1;b=2;c=3'))    # {'a':'1','b':'2','c':'3'}

# Nginx/Apache 访问日志(命名组,直接 .groupdict())
log = re.compile(
    r'(?P<ip>\S+) \S+ \S+ \[(?P<time>[^\]]+)\] '
    r'"(?P<method>\S+) (?P<path>\S+)[^"]*" '
    r'(?P<status>\d{3}) (?P<size>\d+)'
)
```

### 命名转换(常考)

```python
# 驼峰转下划线:camelCase -> camel_case
re.sub(r'(?<=[a-z0-9])(?=[A-Z])', '_', 'getUserName').lower()    # get_user_name

# 整数加千分位(零宽断言定位插入点,见 01 第5节)
re.sub(r'\B(?=(?:\d{3})+(?!\d))', ',', '1234567')               # 1,234,567

# 匹配成对 HTML 标签内容(反向引用 \1 + 懒惰)
re.findall(r'<(\w+)>(.*?)</\1>', '<b>hi</b><i>yo</i>')          # [('b','hi'),('i','yo')]
```

## Part B · 陷阱清单(高频翻车点)

### 1. 贪婪吃过头(最常见)

```python
re.findall(r'<.*>', '<a><b>')      # ['<a><b>']   ← 贪到最后一个 >,吞了中间
re.findall(r'<.*?>', '<a><b>')     # ['<a>', '<b>'] ← 懒惰,到第一个 > 就停 ✅
re.findall(r'<[^>]*>', '<a><b>')   # ['<a>', '<b>'] ← 更稳:压根不让它吃 >
```

> 提取"到第一个结束符为止"的片段,要么用懒惰 `.*?`,要么用排除字符类 `[^>]*`(更快、无回溯)。**优先用排除字符类**。

### 2. 灾难性回溯 / ReDoS(会让线上 CPU 跑满)

某些模式在**不匹配**的输入上,回溯组合数**指数级**爆炸,一个几十字符的字符串能让引擎跑几秒到几分钟。

```python
re.match(r'(a+)+$', 'aaaaaaaaaaaaaaaaaaaaaaaa!')   # ← 卡死!指数级回溯
```

**爆炸的根源——三种危险结构:**
- **嵌套量词**:`(a+)+`、`(a*)*`、`(\d+)*`
- **重叠的或**:`(a|a)*`、`(a|ab)*`
- **相邻的贪婪通配**:`.*.*`、`\s*\s*`

**怎么防:**
1. **改写,消除嵌套/重叠**:`(a+)+` → `a+`;能用 `[^x]*` 排除类就别用 `.*`。
2. **占有量词 / 原子组**(Python 3.11+,关掉回溯):`(?>a+)+` 或 `a++`。
3. **处理不可信输入用非回溯引擎**:`google-re2` 库(RE2,线性时间);或第三方 `regex` 模块的 `timeout=` 参数兜底。
4. Go 的 `regexp` 是 RE2,**天生免疫**——处理用户输入/日志洪流时这是优势。

> **面试高频:** 能讲清"ReDoS 怎么发生(回溯指数爆炸)+ 怎么定位(嵌套量词)+ 怎么修(改写/占有量词/RE2)"是加分项。

### 3. `.` 默认不匹配换行

```python
re.search(r'a.b', 'a\nb')              # None ← . 不含 \n
re.search(r'a.b', 'a\nb', re.S)        # 匹配 ← re.S/DOTALL 让 . 含 \n
```

### 4. `^ $` 默认是整串首尾,不是每行

```python
re.findall(r'^\d+', '1\n2\n3')         # ['1'] ← 只匹配串首那行
re.findall(r'^\d+', '1\n2\n3', re.M)   # ['1','2','3'] ← re.M 后每行首 ✅
```

### 5. `\d \w` 默认匹配 Unicode,不止 ASCII

```python
re.findall(r'\d', '１２３')            # ['１','２','３'] ← 全角数字也算!
re.findall(r'\d', '１２３', re.A)       # [] ← re.A 限定 ASCII
re.findall(r'[0-9]', '１２３')          # [] ← 或直接用 [0-9] 更明确
```

> 同理 `\w` 默认匹配中文等 Unicode 字母。要"纯 ASCII 单词"用 `re.A` 或显式字符类。

### 6. `findall` 有捕获组时,返回的是**组**不是整串

```python
re.findall(r'(\d{4})-\d{2}', '2026-06')    # ['2026'] ← 只给第1组,不是整串!
re.findall(r'(\d{4})-(\d{2})', '2026-06')  # [('2026','06')] ← 多组返回元组
re.findall(r'(?:\d{4})-\d{2}', '2026-06')  # ['2026-06'] ← 非捕获组,返回整串
```

> 想要整串又要分组,用 `finditer` 取 `m.group(0)` / `m.group(1)`,别用 `findall`。

### 7. `split` 带捕获组会把分隔符也塞进结果

```python
re.split(r',', 'a,b')          # ['a','b']
re.split(r'(,)', 'a,b')        # ['a',',','b'] ← 捕获组让分隔符进结果(有时正是你要的)
```

### 8. 转义地狱(Java 转 Python 注意)

- Python:**永远 `r"\d+"`**(原始串),否则 `"\d"` 在某些情况下触发警告/出错。
- Java:字符串里要写 `"\\d+"`(双重转义)。别把 Java 习惯带进 Python。
- 拼接用户输入务必 `re.escape()`(见 `02`)。

## Part C · 性能要点

| 做法 | 说明 |
|---|---|
| **预编译复用** | 热路径/循环里的正则提到模块级 `PAT = re.compile(...)`,别每次现编 |
| **排除字符类 > 贪婪通配** | `[^"]*` 比 `.*?` 更快(无回溯);能限定就别用 `.` |
| **锚定起点** | `^` 让引擎少试很多起点 |
| **非捕获组** | 不需要捕获时用 `(?:...)`,省一点开销 |
| **不可信输入** | 用 RE2(`google-re2` / Go `regexp`),或 `regex` 模块 `timeout=`,防 ReDoS |

> Python `re` 内部有约 512 条的编译缓存,所以不显式 `compile` 也不会每次重编;但**显式 `compile` 更清晰**,且能拿到 `Pattern` 对象复用方法。
