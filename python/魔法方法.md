# 魔法方法

## 什么是魔法方法

- 魔法方法也叫特殊方法（special methods）或 dunder 方法（double underscore）。
- 典型命名形式是 `__init__`、`__repr__`、`__iter__`。
- 它们不是给你手动随便调用的普通方法，而是让 Python 解释器在特定语义下自动调用。
- 例如：创建对象时调 `__init__`，打印对象时调 `__str__` 或 `__repr__`，`for` 循环时调 `__iter__`。

## 资深工程师应该掌握的核心方法

### 对象生命周期

- `__new__`：负责创建对象，先于 `__init__` 执行。
- `__init__`：负责初始化对象，是最常见的魔法方法。
- `__del__`：对象销毁时调用，但不建议依赖它做资源释放。

### 字符串表示

- `__repr__`：面向开发者，尽量提供可调试信息。
- `__str__`：面向用户，适合输出友好文本。

### 容器与迭代协议

- `__len__`：支持 `len(obj)`。
- `__getitem__`：支持 `obj[key]`。
- `__setitem__`：支持 `obj[key] = value`。
- `__contains__`：支持 `x in obj`。
- `__iter__`：支持 `for x in obj`。
- `__next__`：配合 `__iter__` 实现迭代器。

### 比较与哈希

- `__eq__`：支持 `==`。
- `__lt__` / `__gt__`：支持大小比较。
- `__hash__`：支持放入 `set`、作为 `dict` key。

### 属性与调用

- `__getattr__`：访问不存在的属性时触发。
- `__getattribute__`：访问任何属性都会先触发，功能强但更危险。
- `__setattr__`：给对象设置属性时触发。
- `__call__`：让对象像函数一样被调用。

### 上下文管理

- `__enter__` / `__exit__`：支持 `with`。
- `__aenter__` / `__aexit__`：支持 `async with`。

## `__init__` 是什么

`__init__` 是初始化方法。实例创建后，Python 会自动调用它来初始化实例属性。

```python
class User:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

u = User("Alice", 18)
print(u.name)  # Alice
```

注意：

- `__init__` 不是严格意义上的“构造器本体”。
- 真正负责创建对象的是 `__new__`。
- 日常业务开发里，通常只需要写 `__init__`。

## 常见例子

### 1. `__repr__` / `__str__`

```python
class User:
    def __init__(self, name: str):
        self.name = name

    def __repr__(self):
        return f"User(name={self.name!r})"

    def __str__(self):
        return f"User<{self.name}>"

u = User("Alice")
print(u)         # User<Alice>
print(repr(u))   # User(name='Alice')
```

### 2. `__len__` / `__getitem__`

```python
class Team:
    def __init__(self, members):
        self.members = members

    def __len__(self):
        return len(self.members)

    def __getitem__(self, index):
        return self.members[index]

team = Team(["A", "B", "C"])
print(len(team))  # 3
print(team[0])    # A
```

### 3. `__iter__`

```python
class Team:
    def __init__(self, members):
        self.members = members

    def __iter__(self):
        return iter(self.members)

for name in Team(["A", "B"]):
    print(name)
```

### 4. `__call__`

```python
class Multiplier:
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, value):
        return value * self.factor

mul2 = Multiplier(2)
print(mul2(5))  # 10
```

### 5. `__enter__` / `__exit__`

```python
class DbSession:
    def __enter__(self):
        print("open")
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        print("close")

with DbSession() as session:
    print("query")
```

## 魔法方法是不是都作用在实例上

不是。

- 大多数常见魔法方法作用在实例对象上，比如 `__init__`、`__str__`、`__len__`。
- 也有一部分作用在类上，比如 `__init_subclass__`、`__class_getitem__`。
- 更底层还会涉及元类上的魔法方法，比如控制类创建和实例化流程。

## 面试回答建议

如果被问“资深工程师要掌握哪些魔法方法”，回答时抓住下面这些就够了：

- `__init__`、`__new__`
- `__repr__`、`__str__`
- `__eq__`、`__hash__`
- `__len__`、`__iter__`、`__getitem__`
- `__getattr__`、`__getattribute__`
- `__call__`
- `__enter__`、`__exit__`

再补一句：

- 日常业务开发主要用实例级魔法方法。
- 元类和更底层协议只有在框架开发、ORM、SDK 设计时才需要深入。
