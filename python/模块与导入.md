# 模块与导入

## Python 的模块是什么

在 Python 里，一个 `.py` 文件就是一个模块（module）。

可以把它粗略理解成：

- Java 主要靠 `class` 做第一层组织。
- Python 主要靠 `module` 做第一层组织，`class` 做第二层组织。

所以这两种写法在定位上很接近：

```java
StringUtils.trim(s)
```

```python
import string_utils
string_utils.trim(s)
```

## `import` 时到底会发生什么

Python 第一次 `import` 一个模块时，会：

- 执行模块顶层代码
- 创建模块对象
- 放入 `sys.modules` 缓存

后面同一个进程里再次 `import` 同一个模块，通常直接复用缓存，不会把顶层代码完整再跑一遍。

## 顶层代码和函数体不是一回事

例如：

```python
print("module loaded")

x = 1


def foo():
    print("run foo")
```

第一次导入这个模块时：

- `print("module loaded")` 会执行
- `x = 1` 会执行
- `def foo(): ...` 会执行“定义函数”这个动作
- 但 `foo()` 函数体不会自动执行

只有你显式调用 `foo()` 时，`run foo` 才会打印。

## 为什么模块顶层副作用危险

下面这些动作不建议写在普通工具模块顶层：

- 配日志
- 建 Redis 连接
- 建数据库连接池
- 建 OpenAI client
- 启动线程、调度器、订阅器

原因不是它们“绝对不能初始化”，而是：

- 谁 import 了模块，谁就触发这些动作
- 初始化顺序依赖 import 顺序
- 排查副作用来源很难
- 测试时只想 import 一个函数，也会把这些副作用带进来

## 多 worker / 多进程时会怎样

以 gunicorn 为例，不同 worker 是不同进程。

这意味着：

- 每个 worker 都会有自己的模块导入过程
- 顶层代码通常会在每个 worker 各执行一次
- 数据库连接、Redis client、HTTP client 这类资源，通常也应该每个 worker 自己初始化一份

这里的重点不是“不能初始化”，而是：

- 可以在每个 worker 初始化
- 但不应该在 `import` 阶段偷偷初始化
- 更合适的位置是应用启动生命周期里显式初始化

## 什么时候初始化更合理

更推荐放在这些位置：

- FastAPI 的 `lifespan` / `startup`
- 应用工厂函数
- 依赖注入容器
- 工厂函数或 lazy init

例如：

```python
from redis import Redis


def create_redis_client(settings):
    return Redis(host=settings.redis_host, port=settings.redis_port)
```

然后在应用启动时显式创建：

```python
app.state.redis = create_redis_client(settings)
```

## `from x import y` 为什么更难追踪

这类写法：

```python
from utils import foo

foo()
```

问题是调用点里只剩 `foo()`，你很难一眼看出它来自哪个模块。

相比之下，这种更容易追踪：

```python
import utils

utils.foo()
```

或者：

```python
import utils as app_utils

app_utils.foo()
```

## 工程建议

- 纯工具函数可以放模块里。
- 不要把模块写成大杂烩 `utils.py`。
- 不要在普通模块顶层做重副作用初始化。
- 日志配置放应用入口，不要散落在工具模块。
- 对外部资源初始化使用明确生命周期，而不是靠 import 触发。
- 想提高可追踪性时，优先用 `import module`，少用大面积 `from module import x`。

## 面试回答建议

可以直接这样答：

- Python 第一次 `import` 模块时会执行模块顶层代码，并缓存模块对象。
- 后续同进程再次导入通常不会重复执行。
- 顶层代码和函数体不同，函数体不会因 import 自动执行。
- 多 worker 场景下，每个 worker 是独立进程，通常会各自导入并各自初始化资源。
- 数据库、Redis、HTTP client 等资源应该显式放在应用启动生命周期里初始化，而不是放在工具模块顶层。
