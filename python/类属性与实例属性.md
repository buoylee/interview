# 类属性与实例属性

## 类属性怎么定义

类属性直接写在类体里，属于类本身，所有实例共享。

```python
class User:
    role = "guest"
```

使用：

```python
print(User.role)  # guest
u1 = User()
u2 = User()
print(u1.role)    # guest
print(u2.role)    # guest
```

## 实例属性怎么定义

实例属性通常在 `__init__` 里通过 `self.xxx` 定义，属于每个实例自己的状态。

```python
class User:
    def __init__(self, name: str):
        self.name = name
```

使用：

```python
u1 = User("Alice")
u2 = User("Bob")
print(u1.name)  # Alice
print(u2.name)  # Bob
```

## 能不能不定义，使用时直接写

可以，Python 支持动态加属性。

```python
class User:
    pass

u = User()
u.name = "Alice"
User.version = "1.0"
```

但工程上不建议这么做，原因很直接：

- 阅读代码时很难知道对象到底有哪些字段。
- 类型检查和 IDE 补全效果会变差。
- 后期维护时容易写出隐藏 bug。

更推荐：

- 类属性显式写在类体里。
- 实例属性显式写在 `__init__` 里。

## 类属性和实例属性同名会不会冲突

会发生“遮蔽”，但不会像 Java 一样直接编译报错。

```python
class User:
    role = "guest"

u = User()
print(u.role)   # guest，先从实例找，找不到再到类上找

u.role = "admin"
print(u.role)   # admin，实例属性遮蔽了类属性
print(User.role)  # guest，类属性本身没变
```

查找顺序可以简单记成：

- 先找实例属性
- 再找类属性
- 再找父类属性

## Python 里怎么理解 Java 的 `static`

最接近 Java `static` 变量的有两种：

### 1. 类属性

```python
class Counter:
    total = 0

    def __init__(self):
        Counter.total += 1
```

这和 Java 的类级共享字段最接近。

### 2. 模块变量

```python
# config.py
APP_NAME = "demo"
```

然后别处直接：

```python
import config
print(config.APP_NAME)
```

在 Python 里，很多本来 Java 会放在 `static final` 里的常量，会直接放在模块里。

## `@staticmethod` 和类属性不是一回事

`@staticmethod` 是静态方法，不是静态变量。

```python
class MathUtil:
    @staticmethod
    def add(a, b):
        return a + b
```

但 Python 里如果只是普通工具函数，通常更推荐直接写成模块函数，而不是为了模仿 Java 硬塞进类里。

## 面试回答建议

可以直接这样答：

- 类属性定义在类体里，所有实例共享。
- 实例属性通常定义在 `__init__` 里，每个实例独立。
- Python 支持运行时动态加属性，但工程上不建议这么做。
- 同名时不会报错，而是实例属性遮蔽类属性。
- Python 里 Java `static` 的常见对应物是类属性和模块变量。
