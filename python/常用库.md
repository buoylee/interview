可以把 Python 里那些“几乎每个工程都会用的强力库”按场景分成几类，类似 Java 世界里 Guava、Lombok 的定位，帮助补齐标准库的不足或简化开发工作。

---

## 1. 语言增强 / 工具集（对标 Guava）

- **`more-itertools`**：扩展版的 `itertools`，提供切片、窗口、分组、无限序列等高阶迭代工具。
- **`boltons`**：一套纯 Python 的实用函数集合，包括容器扩展、迭代器、路径操作、序列处理、缓存等。
- **`toolz` / `cytoolz`**：函数式编程工具箱，支持组合、柯里化、管道等；`cytoolz` 为 C 加速版。
- **`funcy`**：函数式工具集，简单易用，提供操作列表/字典/迭代器的常用函数。
- **`attrs`**：自动生成 `__init__`、`__repr__`、验证等，类似 Lombok；更早出现于 `dataclasses` 之前。
- **`pydantic`**：数据模型与校验工具（v2 支持 dataclasses 语法），适合配置、API、数据处理。

### 标准库替代
- `dataclasses`（内置）可以满足简单的数据类需求，但 `attrs`/`pydantic` 提供更多验证、默认值、转换等能力。
- `itertools` + `functools` + `collections` 已经很强，但 `more-itertools` 这类库在实际工程中经常用到。

---

## 2. 常用基础库（HTTP / CLI / 日志）

- **HTTP**：`requests`（同步）、`httpx`（同步+异步）
- **CLI**：`click` / `typer`（声明式命令行，非常常用）
- **终端美化**：`rich`（彩色输出、表格、进度条）、`tqdm`（进度条）
- **日志增强**：`structlog`（结构化日志）、`loguru`（简化 logging 配置）
- **配置管理**：`dynaconf`、`python-decouple`、`pydantic-settings`

---

## 3. Web / API / 后端

- **FastAPI** / **Starlette**：现代异步 Web 框架，Schema 和文档自动生成。
- **Flask**：轻量级 Web 框架，生态成熟。
- **Django**：全功能框架（ORM + Admin + Auth）。
- **HTTP 客户端/服务端**：`aiohttp`、`websockets`。
- **数据模型与 ORM**：`SQLAlchemy`（最常用）、`SQLModel`（基于 SQLAlchemy + Pydantic）、`Tortoise ORM`（异步）。

---

## 4. 数据处理 / 科学计算

- **NumPy**：基础数组计算库，Python 科学计算基石。
- **Pandas**：数据分析、表格处理。
- **Polars**：新兴 DataFrame 库，Rust 实现，性能极高。
- **PyArrow**：列式内存格式，跨语言数据互操作。
- **scikit-learn**：机器学习基础库。

---

## 5. 测试 / 质量控制

- **pytest**：主流测试框架，语法简洁、生态丰富。
- **hypothesis**：基于属性的测试（Property-based testing）。
- **coverage.py**：测试覆盖率统计。
- 质量工具：
  - **black**（代码自动格式化）
  - **isort**（import 排序）
  - **flake8 / ruff**（静态分析、lint）
  - **mypy / pyright**（类型检查）
  - **pre-commit**（集中管理这些工具的 git hook）

---

## 6. 并发 / 任务调度

- **concurrent.futures**（标准库）：线程池、进程池。
- **asyncio**（标准库）：异步编程核心。
- **trio** / **anyio**：替代 asyncio 的现代异步调度库。
- **celery**：分布式任务队列 / 延迟任务。
- **rq / huey / dramatiq**：简化版任务队列。

---

## 7. 序列化 / 数据交换

- **pydantic**（再次强调）：模型 + 序列化 + 校验。
- **marshmallow**：Schema 管理、序列化/反序列化。
- **orjson / ujson**：快速 JSON 处理。
- **msgpack**：二进制数据交换格式。

---

## 8. 缓存 / 队列 / 消息

- **redis-py**：Redis 客户端。
- **aioredis**：Redis 异步客户端。
- **kafka-python / confluent-kafka**：Kafka 客户端。
- **kombu**：抽象消息队列层，Celery 的基础。

---

## 9. 构建 / 包管理 / 项目模板

- **pipx**：隔离安装 CLI 工具。
- **poetry**：依赖管理 + 打包。
- **pip-tools**：生成锁定的 requirements.txt。
- **cookiecutter**：项目脚手架生成器。

---

## 10. 其它备选工具

- **typer**：基于 type hints 的 CLI（click 之上的再封装）
- **pytz** 或 **zoneinfo**（时区处理）
- **pendulum**（增强版 datetime）
- **arrow**（友好的 datetime 包装）
- **fastapi-utils / starlite**：FastAPI 补充
- **faker**：生成模拟数据
- **dateutil**（标准库 datetime 的拓展）

---

## 关键对照：Guava / Lombok

| Java 功能                             | Python 对应                                                  |
| ------------------------------------- | ------------------------------------------------------------ |
| Guava（集合工具、缓存、字符串处理等） | `collections`（标准库），`itertools`、`more-itertools`、`toolz`、`boltons` |
| Lombok（简化数据类、构造器、Builder） | `dataclasses`（标准库），`attrs`、`pydantic`                 |
| Apache Commons Lang / IO              | Python 标准库 + `pathlib`、`shutil`、`boltons`、`click` 等   |
| Guava 的 Cache                        | `functools.lru_cache`（标准库）、`cachetools`（第三方）      |

---

## 总结建议

- **基础增强**：`more-itertools` / `boltons` + `attrs`（或 `pydantic`）
- **CLI + 输出**：`click` + `rich` + `tqdm`
- **HTTP**：`requests` / `httpx`
- **配置/序列化**：`pydantic`、`yaml`、`orjson`
- **测试 + 质量**：`pytest` + `black` + `ruff` + `mypy`
- **Web/API**：`FastAPI` + `SQLAlchemy`
- **任务队列**：`celery` / `rq`
- **数据分析**：`pandas` + `numpy`
