# 02 · 方法集与接收者:为什么 `*T` 能满足接口而 `T` 不能

> 上一章说「编译器检查方法集是否覆盖接口」。这一章把**方法集规则**讲透——它是 Go 面试最容易答错的点:**值接收者 vs 指针接收者**,以及由此决定的「`T` 和 `*T` 谁能满足哪个接口」。
>
> 桥接锚点:Java 方法都挂在对象上、`this` 永远是引用,没有「值/指针接收者」之分。Go 的方法是「带接收者参数的函数」,接收者按值还是按指针,直接决定能不能改原值、以及谁能进接口。

---

## 1. 核心问题

```go
type Counter struct{ n int }
func (c Counter) Get() int  { return c.n }   // 值接收者
func (c *Counter) Inc()      { c.n++ }        // 指针接收者

type Incrementer interface{ Inc() }

var x Incrementer = Counter{}    // ❌ 编译报错!为什么?
var y Incrementer = &Counter{}   // ✅ 这个就行,为什么?
```

- 值接收者 `(c Counter)` 和指针接收者 `(c *Counter)` 到底差在哪?什么时候用哪个?
- 为什么 `Counter{}`(值)**不能**赋给 `Incrementer`,但 `&Counter{}`(指针)可以?
- `c := Counter{}; c.Inc()` 明明能直接调,为什么装进接口就不行?

---

## 2. 直觉理解

### 接收者 = 方法的第一个隐藏参数

`func (c Counter) Get()` 本质是 `func Get(c Counter)`——`c` 是**拷贝**,改它不影响原值。
`func (c *Counter) Inc()` 本质是 `func Inc(c *Counter)`——`c` 是**指针**,改 `c.n` 改的是原值。

所以:**要改原值、或避免拷贝大结构体 → 指针接收者;只读小值 → 值接收者**。

### 方法集规则:`*T` 的方法集更大

「谁能调哪些方法」由**方法集**定义:

| 类型 | 方法集包含 |
|---|---|
| 值 `T` | 只有**值接收者**方法 |
| 指针 `*T` | **值接收者 + 指针接收者**方法(全都有) |

记忆:**`*T` 包含 `T` 的全部,外加指针接收者方法。** 反过来 `T` 的方法集**不含**指针接收者方法。

### 为什么 `Counter{}` 进不了接口

`Incrementer` 要 `Inc()`,而 `Inc` 是**指针接收者**方法 → 只在 `*Counter` 的方法集里,**不在** `Counter` 的方法集里。所以:

- `var y Incrementer = &Counter{}` ✅(`*Counter` 有 `Inc`);
- `var x Incrementer = Counter{}` ❌(`Counter` 没有 `Inc`,方法集不覆盖)。

### 那为什么 `c.Inc()` 直接调又可以?

```go
c := Counter{}
c.Inc()          // ✅ 编译器自动改写成 (&c).Inc()
```

因为 `c` 是个**可寻址(addressable)**的变量,编译器自动取地址。但**接口装值时没有这个自动取地址**——接口里存的是值的拷贝,拷贝**不可寻址**,编译器无法 `&` 它去满足指针接收者方法。这就是「直接调行、装接口不行」的根因。

---

## 3. 原理深入

### 3.1 可寻址性(addressability)决定能否自动取址

编译器只会对**可寻址**的东西自动 `&`:

- ✅ 可寻址:局部变量、指针解引用 `*p`、slice 元素 `s[i]`、可寻址 struct 的字段。
- ❌ 不可寻址:**map 元素**、函数返回值、**接口里的值**、字面量、常量。

```go
m := map[int]Counter{0: {}}
m[0].Inc()        // ❌ 编译错误:map 元素不可寻址,不能自动取址调指针方法
v := m[0]; v.Inc(); m[0] = v   // 绕法:取出→改→写回
```

> 这把 `00` 章的「值语义」和接口/方法集串起来了:能不能调指针方法,取决于这个值**有没有地址**。

### 3.2 编译期怎么判「实现了接口」

`var x I = v` 时编译器查:`typeof(v)` 的方法集 ⊇ `I` 的方法集?

- `v` 是 `T` → 用 `T` 的方法集(仅值接收者方法);
- `v` 是 `*T` → 用 `*T` 的方法集(全部)。

缺任一方法 → 编译报错(还会贴心提示「`Inc` 方法有指针接收者」)。覆盖 → 生成 itab(上一章)。

### 3.3 一致性铁律:别混用值/指针接收者

```go
// ✅ 推荐:一个类型的方法,接收者风格统一
func (c *Counter) Get() int { return c.n }
func (c *Counter) Inc()      { c.n++ }
```

准则:**只要有一个方法需要指针接收者(要改值,或类型含锁/大),就把该类型所有方法都用指针接收者**。理由:

- 统一后只有 `*T` 满足接口,心智简单、不会踩「值进不了接口」;
- 含 `sync.Mutex` 的类型用值接收者会**拷贝锁**(bug,`go vet copylocks` 报);
- 避免「一半方法改得动原值、一半改不动」的混乱。

### 3.4 什么时候用值接收者

- 类型是**小且不可变**的(如 `time.Time`、小 struct、基本类型别名);
- 方法**只读**、不改状态;
- 想让 `T` 和 `*T` **都能**满足接口(值接收者方法两边方法集都有)。

> 标准库 `time.Time` 全用值接收者:它小、语义上是不可变值,值传递安全又免逃逸。

---

## 4. 日常开发应用

- **默认指针接收者**,除非类型明确是「小的不可变值」。尤其:要改状态、含锁/含大数组、或类型会增长。
- **构造函数返回 `*T`**:`func NewCounter() *Counter`,调用方拿到指针,自然满足含指针方法的接口。
- **往接口里放就放指针**:`var w io.Writer = &buf`,既满足方法集又免装值逃逸(上一章)。
- **map 里要改的值用 `map[K]*V`** 而非 `map[K]V`:后者元素不可寻址、不能原地改、不能调指针方法。
- **slice 元素可寻址**:`s[i].Inc()` 可以(`s[i]` 可寻址),但 `range` 出来的副本 `v.Inc()` 改不到原值。

---

## 5. 生产&调优实战

- **值接收者 = 每次调用拷贝接收者**:大 struct 用值接收者,每次方法调用都拷贝一份,热路径上有成本 → 指针接收者免拷贝。
- **指针接收者可能让接收者逃逸**:指针被存起来/传入接口会让对象逃逸到堆;但通常比「值方法 + 大对象反复拷贝」划算。用 `-gcflags=-m` 权衡。
- **拷贝含锁结构体是真 bug**:值接收者方法 + 含 `sync.Mutex` → 方法内操作的是锁的副本,失去互斥。务必指针接收者 + `go vet`。
- **接口 + 值接收者的隐藏拷贝**:`var s fmt.Stringer = bigStruct`(值满足接口)会把整个大结构体拷进堆;放指针避免。

---

## 6. 面试高频考点

- **值接收者 vs 指针接收者区别?** 值接收者操作拷贝(改不到原值、每次调用拷贝);指针接收者操作原值(能改、免拷贝)。要改状态/类型大/含锁 → 指针。
- **`T` 和 `*T` 的方法集?** `T` 只含值接收者方法;`*T` 含值接收者 + 指针接收者方法(全部)。
- **为什么值类型可能满足不了接口?** 接口要的方法是指针接收者时,只在 `*T` 方法集里,`T` 不含 → `T` 的值赋接口编译失败,得用 `*T`。
- **`c.Inc()` 能调但 `var x I = Counter{}` 不行,为什么?** 局部变量可寻址,编译器自动 `(&c).Inc()`;接口里的值不可寻址,无法自动取址满足指针方法。
- **map 元素为什么不能调指针方法/改字段?** map 元素不可寻址(扩容会搬迁),不能取址;取出改了写回,或用 `map[K]*V`。
- **接收者该统一用指针还是值?** 一致性铁律:有一个要指针就全用指针;只读小不可变值可全用值(如 `time.Time`)。
- **值接收者 + 含锁 struct 有什么问题?** 拷贝锁导致互斥失效,`go vet copylocks` 报。

---

## 7. 一句话总结

> **接收者是方法的第一个隐藏参数:值接收者操作拷贝(改不到原值)、指针接收者操作原值。方法集规则:`T` 只含值接收者方法,`*T` 含全部**——所以接口要的方法若是**指针接收者**,只有 `*T` 能满足,`T` 的值赋接口编译失败。`c.Inc()` 能直接调是因为局部变量**可寻址**、编译器自动取址;接口里的值**不可寻址**,所以装值满足不了指针方法(map 元素同理不可寻址)。一致性铁律:**有一个方法要指针接收者就全用指针**(还能避免拷贝含锁结构体的 bug);只读、小、不可变的类型才全用值(如 `time.Time`)。
