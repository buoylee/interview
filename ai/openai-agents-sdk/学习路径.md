## 第一阶段：理解核心 Loop（1-2 小时）

> 目标：搞懂 Agent 的本质 —— 一个 while 循环

### 1.1 先看这张图

```
Runner.run(agent, input)
         │
         ▼
    ┌─────────┐
    │ LLM Call │ ◀──────────────────────────┐
    └────┬────┘                              │
         │                                   │
         ▼                                   │
    ┌─────────────┐                          │
    │ 判断 response │                         │
    └──┬──┬──┬────┘                          │
       │  │  │                               │
       │  │  └── has_tool_calls ──▶ 执行工具 ─┘
       │  │                      结果追加到 input
       │  │
       │  └── has_handoff ──▶ 切换 agent ────┘
       │                    继续循环
       │
       └── has_final_output ──▶ return 结束
```

### 1.2 核心 Loop 伪代码

```python
async def run_loop(agent, input, max_turns):
    turns = 0
    while turns < max_turns:
        # 1. 调 LLM（带上 agent 的 instructions、tools、历史消息）
        response = await llm.call(
            model=agent.model,
            instructions=agent.instructions,
            tools=agent.tools,
            input=input
        )

        # 2. 有 tool calls → 执行工具 → 结果追加 → 继续
        if response.has_tool_calls:
            results = await execute_tools(response.tool_calls)
            input.append(results)
            tool_choice = "auto"  # 防无限循环：执行完工具后重置
            turns += 1
            continue

        # 3. 有 handoff → 切换到另一个 agent → 继续
        if response.has_handoff:
            agent = response.handoff_target
            turns += 1
            continue

        # 4. 有最终输出 → 结束
        if response.has_final_output:
            return response.output

    raise MaxTurnsExceeded()
```

**就这么多。** 这是所有 agent 的共同基础。

### 1.3 源码阅读入口

```
openai-agents-python/
  src/agents/
    ├── run.py              ← Runner 类，公开 API（run / run_sync / run_streamed）
    ├── _run_impl.py        ← 内部执行逻辑
    └── run_internal/
        └── run_loop.py     ← ⭐ 核心 loop 在这里，最重要的文件
```

**只读 `run_loop.py` 就够。** 关注这几个函数：

| 函数 | 作用 |
|------|------|
| `run_single_turn()` | 执行一轮：LLM 调用 → 判断 → 工具执行 |
| `run_single_turn_streamed()` | 同上，但支持流式输出 |
| `get_new_response()` | 获取 LLM 响应 |

### 1.4 需要理解的关键问题

- [ ] Loop 什么时候停？（final_output 且无 tool_calls）
- [ ] Tool call 结果怎么喂回 LLM？（append 到 input）
- [ ] 怎么防止无限循环？（max_turns + reset tool_choice to "auto"）
- [ ] Handoff 时历史消息怎么处理？（默认继承全部）

## 第二阶段：理解 5 个核心原语（半天）

> 目标：掌握 SDK 的所有构建块

### 2.1 Agent —— 一个配置好的 LLM

```python
from agents import Agent

agent = Agent(
    name="客服助手",
    instructions="你是一个客服助手，帮用户解决问题",  # system prompt
    model="gpt-4o",                                   # 可换成任何模型
    tools=[search_kb, create_ticket],                  # 工具列表
    handoffs=[billing_agent, refund_agent],            # 可交接的其他 agent
    output_type=ResponseSchema,                        # 结构化输出（Pydantic）
    hooks=MyHooks(),                                   # 生命周期钩子
)
```

源码：`src/agents/agent.py`

关键点：
- Agent 只是**配置**，不持有运行状态
- 支持泛型 `Agent[ContextType]`，用于依赖注入
- `instructions` 可以是字符串或 `Callable`（动态生成 system prompt）

### 2.2 Tool —— Agent 的手

5 种类型：

| 类型 | 说明 | 在哪执行 |
|------|------|---------|
| **Function Tool** | `@function_tool` 装饰的 Python 函数 | 你的服务器 |
| **Hosted Tool** | WebSearch、FileSearch、CodeInterpreter | OpenAI 服务器 |
| **Local Tool** | ShellTool、ComputerTool | 你的本地环境 |
| **Agent as Tool** | `agent.as_tool()` 把 agent 当工具用 | 嵌套调用 |
| **MCP Tool** | 外部 MCP Server 提供的工具 | MCP Server |

```python
from agents import function_tool

@function_tool
def search_knowledge_base(query: str) -> str:
    """搜索知识库，返回相关结果"""  # docstring 自动变成工具描述
    results = db.search(query)
    return format_results(results)

# 需要人工审批的工具
@function_tool(needs_approval=True)
async def delete_account(user_id: str) -> str:
    """删除用户账号（不可恢复）"""
    await db.delete_user(user_id)
    return f"已删除用户 {user_id}"

# 条件审批
@function_tool(needs_approval=lambda ctx, args, call_id: args["amount"] > 10000)
async def transfer_funds(account_id: str, amount: float) -> str:
    """转账"""
    ...
```

源码：`src/agents/tool.py` + `src/agents/function_schema.py`

内部流程：
1. `function_schema.py` 从函数签名 + type hints + docstring 自动生成 JSON Schema
2. LLM 收到 Schema 后决定什么时候调用
3. LLM 返回 tool_call → Pydantic 验证参数 → 执行函数 → 结果追加到历史

### 2.3 Handoff —— Agent 间交接

```python
from agents import Agent, handoff

# 简单写法：直接传 agent
triage = Agent(
    name="Triage",
    instructions="根据用户问题路由到对应的专家",
    handoffs=[billing_agent, refund_agent, support_agent]
)
# LLM 看到的工具：transfer_to_billing_agent, transfer_to_refund_agent, ...

# 高级写法：自定义 handoff 行为
custom = handoff(
    agent=specialist,
    tool_name_override="escalate_to_specialist",   # 自定义工具名
    input_type=EscalationData,                      # 结构化输入
    input_filter=lambda items: filter_sensitive(items),  # 过滤历史消息
    on_handoff=async_callback,                      # handoff 时的回调
    is_enabled=lambda ctx: ctx.user_tier >= "premium"  # 条件启用
)
```

源码：`src/agents/handoff.py`

关键理解：
- Handoff 在 LLM 看来就是一个 tool（`transfer_to_xxx`）
- Handoff 后，**对话历史默认全部传递给新 agent**
- 可以用 `input_filter` 过滤敏感信息
- 可以用 `nest_handoff_history` 把历史折叠成一条消息

### 2.4 Guardrail —— 安全护栏

```python
from agents import input_guardrail, output_guardrail, GuardrailFunctionOutput

@input_guardrail
async def check_jailbreak(input: str, context: RunContextWrapper) -> GuardrailFunctionOutput:
    """检查是否是 jailbreak 攻击"""
    is_safe = await safety_check(input)
    return GuardrailFunctionOutput(
        output_info="Jailbreak check",
        tripwire_triggered=not is_safe  # True = 立即阻断
    )

@output_guardrail
async def check_hallucination(output: str, context: RunContextWrapper) -> GuardrailFunctionOutput:
    """检查输出是否有幻觉"""
    has_hallucination = await fact_check(output)
    return GuardrailFunctionOutput(
        output_info="Hallucination check",
        tripwire_triggered=has_hallucination
    )

agent = Agent(
    input_guardrails=[check_jailbreak],     # 输入护栏
    output_guardrails=[check_hallucination], # 输出护栏
    ...
)
```

源码：`src/agents/guardrail.py`

3 种护栏：

| 类型 | 执行时机 | 执行模式 |
|------|---------|---------|
| Input Guardrail | agent 执行前 | 默认并行（和 LLM 调用同时），可设为 blocking |
| Output Guardrail | agent 执行后 | 始终 blocking |
| Tool Guardrail | tool 执行前后 | 包装在 tool 上 |

触发 tripwire 后：抛出异常，**立即中断整个 run**。

### 2.5 Context —— 依赖注入

```python
from dataclasses import dataclass
from agents import Agent, Runner, function_tool, RunContextWrapper

@dataclass
class AppContext:
    user_id: str
    database: DatabaseConnection
    logger: Logger

@function_tool
async def get_order(context: RunContextWrapper[AppContext], order_id: str) -> str:
    """查询订单"""
    # 类型安全地访问上下文
    db = context.context.database
    user_id = context.context.user_id
    return await db.get_order(user_id, order_id)

agent = Agent[AppContext](
    name="Order Agent",
    tools=[get_order],
    ...
)

# 运行时注入上下文
ctx = AppContext(user_id="123", database=db, logger=log)
result = await Runner.run(agent, "查一下我的订单", context=ctx)
```

源码：`src/agents/run_context.py`

关键：Context **不会发送给 LLM**，纯粹是本地的依赖注入机制。

## 第三阶段：掌握 Session 和状态管理（2-3 小时）

> 目标：理解多轮对话和状态持久化

### 3.1 Session —— 自动管理对话历史

```python
from agents import Runner
from agents.memory import SQLiteSession

session = SQLiteSession("conversation_123", db_path="sessions.db")

# 第一轮
result = await Runner.run(agent, "我叫张三", session=session)
# 内部：存储对话历史到 SQLite

# 第二轮（agent 自动拿到之前的历史）
result = await Runner.run(agent, "我叫什么？", session=session)
# agent 回答："你叫张三"
```

内部流程：
```
run 开始 → session.get_history() → 拼接到 input 前面 → LLM 调用 → run 结束 → session.save(new_items)
```

### 3.2 可用的 Session 实现

| 实现 | 适用场景 | 存储 |
|------|---------|------|
| `SQLiteSession` | 本地开发 | SQLite 文件 |
| `AsyncSQLiteSession` | 异步本地开发 | SQLite 文件 |
| `RedisSession` | 分布式系统 | Redis |
| `SQLAlchemySession` | 生产环境 | 任何关系数据库 |
| `DaprSession` | 云原生 | Dapr sidecar |
| `EncryptedSession` | 安全要求高 | 任何后端 + 加密 |
| `OpenAIConversationsSession` | OpenAI 托管 | OpenAI 服务端 |

### 3.3 RunState —— 中断和恢复

```python
# 场景：tool 需要人工审批，run 被中断
result = await Runner.run(agent, "删除我的账号")

if result.interruptions:
    # 序列化状态（可以存数据库、发到前端）
    state_str = result.to_state().to_string()

    # ... 等待人工审批 ...

    # 反序列化并恢复
    state = RunState.from_string(state_str)
    state.approve(result.interruptions[0])  # 或 state.reject(...)

    # 从断点继续执行
    result = await Runner.run(agent, state)
```

源码：`src/agents/run_state.py`

这个能力让你可以：
- 跨进程/跨服务恢复 agent 执行
- 实现异步人工审批流程
- 持久化长时间运行的任务

## 第四阶段：掌握高级模式（半天）

### 4.1 Streaming —— 流式输出

```python
result = await Runner.run_streamed(agent, "写一篇关于 AI 的文章")

async for event in result.stream_events():
    match event.type:
        case "raw_response_event":
            # token 级别的流式输出
            token = event.data.response.output_text.delta
            print(token, end="", flush=True)

        case "run_item_stream_event":
            match event.name:
                case "tool_called":
                    print(f"\n[调用工具: {event.item.tool_name}]")
                case "tool_output":
                    print(f"[工具结果: {event.item.output}]")
                case "handoff_requested":
                    print(f"[交接到: {event.item.agent_name}]")

        case "agent_updated_stream_event":
            print(f"\n[当前 Agent: {event.new_agent.name}]")

# 重要：必须等待完成（后台有 session 保存等收尾工作）
await result.stream.completed
```

Stream 事件类型：

| 事件类型 | 说明 |
|---------|------|
| `raw_response_event` | LLM 逐 token 输出 |
| `run_item_stream_event` | 语义级事件（tool_called / tool_output / handoff 等） |
| `agent_updated_stream_event` | agent 切换 |

### 4.2 Lifecycle Hooks —— 生命周期钩子

```python
from agents import RunHooksBase, AgentHooksBase

# 全局钩子（所有 agent 共享）
class GlobalHooks(RunHooksBase):
    async def on_llm_start(self, agent, context):
        print(f"[LLM 开始] {agent.name}")

    async def on_llm_end(self, agent, context):
        print(f"[LLM 结束] {agent.name}")

    async def on_tool_start(self, tool_name, context):
        print(f"[工具开始] {tool_name}")

    async def on_tool_end(self, tool_name, result, context):
        print(f"[工具结束] {tool_name}")

    async def on_handoff(self, from_agent, to_agent, context):
        print(f"[交接] {from_agent.name} → {to_agent.name}")

# Agent 级钩子
class AgentHooks(AgentHooksBase):
    async def on_start(self, agent, context):
        ...
    async def on_end(self, agent, context):
        ...

agent = Agent(hooks=AgentHooks(), ...)
await Runner.run(agent, "task", hooks=GlobalHooks())
```

### 4.3 Structured Output —— 结构化输出

```python
from pydantic import BaseModel

class AnalysisResult(BaseModel):
    summary: str
    key_points: list[str]
    sentiment: str  # positive / negative / neutral
    confidence: float

agent = Agent(
    name="Analyst",
    instructions="分析用户提供的内容",
    output_type=AnalysisResult,  # 强制 LLM 返回这个结构
)

result = await Runner.run(agent, "分析一下最近的市场趋势...")

# 类型安全
assert isinstance(result.final_output, AnalysisResult)
print(result.final_output.summary)
print(result.final_output.key_points)
```

### 4.4 Agent as Tool vs Handoff

两种多 agent 协作方式：

```python
# 方式 1：Handoff（转移控制权）
# 适合：用户直接和目标 agent 对话
triage = Agent(
    handoffs=[billing_agent, support_agent]
)
# triage 把控制权交给 billing_agent，后续对话由 billing_agent 处理

# 方式 2：Agent as Tool（嵌套调用）
# 适合：主 agent 调用子 agent 完成子任务，结果返回给主 agent
manager = Agent(
    tools=[
        researcher.as_tool(name_override="do_research"),
        writer.as_tool(name_override="write_draft"),
    ]
)
# manager 保持控制权，researcher 只是被调用的工具
```

| | Handoff | Agent as Tool |
|--|---------|---------------|
| 控制权 | 转移到目标 agent | 始终在主 agent |
| 对话上下文 | 全部传递 | 不传递 |
| 用户感知 | 在和另一个 agent 说话 | 无感，主 agent 做了一次内部调用 |
| 适合场景 | 路由/分流 | 编排/协调 |

async def pipeline(task):

import asyncio

results = await asyncio.gather(

final = await Runner.run(

### 8.2 FastAPI 集成示例

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from agents import Agent, Runner, function_tool
from agents.memory import RedisSession

app = FastAPI()

agent = Agent(name="Assistant", instructions="...", tools=[...])

@app.post("/chat")
async def chat(user_id: str, message: str):
    session = RedisSession(f"session:{user_id}")

    result = await Runner.run_streamed(agent, message, session=session)

    async def event_stream():
        async for event in result.stream_events():
            if event.type == "raw_response_event":
                delta = event.data.response.output_text.delta
                if delta:
                    yield f"data: {delta}\n\n"
        await result.stream.completed
        yield "data: [DONE]\n\n"

    return StreamingResponse(event_stream(), media_type="text/event-stream")
```

```
src/agents/
├── run.py                  ← Runner 类（入口）
├── _run_impl.py            ← 内部执行逻辑
├── run_internal/
│   ├── run_loop.py         ← ⭐ 核心 loop（最重要）
│   └── oai_conversation.py ← 服务端会话追踪
├── agent.py                ← Agent 定义
├── tool.py                 ← Tool 抽象和执行
├── function_schema.py      ← 自动 JSON Schema 生成
├── handoff.py              ← Handoff 实现
├── guardrail.py            ← Guardrail 实现
├── result.py               ← RunResult / RunResultStreaming
├── run_context.py          ← Context 依赖注入
├── run_state.py            ← RunState 序列化/反序列化
├── stream_events.py        ← 流式事件类型定义
├── strict_schema.py        ← JSON Schema 工具
├── exceptions.py           ← 异常定义
├── logger.py               ← 日志
├── models/
│   ├── openai_responses.py     ← Responses API 模型（推荐）
│   └── openai_chatcompletions.py ← Chat Completions 模型
├── extensions/
│   └── models/
│       └── litellm_provider.py ← LiteLLM 集成
├── mcp/                    ← MCP 协议支持
├── memory/
│   └── session.py          ← Session 接口和实现
├── tracing/                ← 可观测性
└── realtime/               ← 语音 / WebSocket
```

### 推荐精读顺序

```
第 1 轮（核心 loop）：
  run_loop.py → run.py → _run_impl.py

第 2 轮（Agent + Tool）：
  agent.py → tool.py → function_schema.py

第 3 轮（协作机制）：
  handoff.py → guardrail.py → run_context.py

第 4 轮（状态管理）：
  result.py → run_state.py → memory/session.py

第 5 轮（流式 + 可观测）：
  stream_events.py → tracing/
```
