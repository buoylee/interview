# 00 - 架构总览：OpenClaw Agent 与 Memory 系统

> OpenClaw 是一个运行在用户设备上的个人 AI 助手，支持 20+ 消息渠道（WhatsApp/Telegram/Slack/Discord/Signal/iMessage 等）。
> 核心代码量巨大（src/ 约 328K 行），但 AI 智能的核心集中在 **agents**（95K）和 **memory**（11K）两个模块。
> 本系列聚焦这两个模块，按数据流剖析 Agent 执行引擎和 Memory 搜索引擎的设计。

---

## 一句话定位

OpenClaw 的 Agent 是一个 **多层重试 + 流式工具循环**：消息进来 → 解析模型/认证 → 组装 prompt → 调 LLM → 流式执行工具 → 重试/回退/压缩 → 返回结果。

Memory 是一个 **双后端混合搜索引擎**：文件监听 → 分块 → 嵌入 → SQLite 存储 → 向量 + 关键词混合搜索 → MMR 去重 → 时间衰减。

## 5 个关键设计决策

### 1. 基于 pi-coding-agent 的嵌入式执行

OpenClaw 没有自己实现 Agent 循环，而是嵌入了 `@mariozechner/pi-coding-agent` 库。该库提供：
- `SessionManager`：会话文件（JSONL）的读写管理
- `createAgentSession`：创建带工具循环的 Agent 会话
- `activeSession.prompt()`：发送消息并自动执行工具循环

OpenClaw 在此之上加了大量包装：多层 streamFn 适配、认证管理、重试策略、上下文压缩等。

→ 详见 [01-Agent主循环](./01-Agent主循环.md)

### 2. 多层重试与优雅降级

```
一次请求最多经历:
  外层: runWithModelFallback() → 模型候选链逐一尝试
    中层: while(true) in run.ts → 最多 160 次迭代
      内层: runEmbeddedAttempt() → 单次 LLM 调用

降级链:
  上下文溢出 → compaction（最多 3 次）→ 工具结果截断
  认证失败 → 刷新 token → 切换 profile
  thinking 不支持 → 降级 thinking level
  模型失败 → FailoverError → 切换模型
  超时 → 切换模型或报错
```

→ 详见 [02-Prompt与模型路由](./02-Prompt与模型路由.md)

### 3. 子 Agent 树形生命周期

子 agent 不是简单的函数调用，而是完整的生命周期管理：

```
派生 → 注册 → 执行 → 完成 → 通知父 agent → 清理
  │                           │
  ├── 深度限制（防止无限递归）  ├── 级联 kill（父死子亡）
  ├── 并发上限（每 session）   ├── steer（中途改方向）
  └── 沙箱继承                └── 指数退避重试通知
```

→ 详见 [03-子Agent系统](./03-子Agent系统.md)

### 4. 双后端混合搜索

Memory 系统有两个搜索后端：

| | Builtin（SQLite） | QMD（外部 CLI） |
|---|---|---|
| 向量搜索 | sqlite-vec 扩展 | QMD 内置 |
| 关键词搜索 | FTS5 | QMD 内置 |
| 嵌入生成 | OpenAI/Gemini/Voyage/Ollama | QMD 管理 |
| 优点 | 零依赖、内嵌 | 高性能、多集合 |
| 降级 | 作为 QMD 的 fallback | — |

搜索结果通过 **混合合并 + MMR 去重 + 时间衰减** 三层后处理。

→ 详见 [04-Memory存储与同步](./04-Memory存储与同步.md) 和 [05-Memory搜索引擎](./05-Memory搜索引擎.md)

### 5. CJK 原生支持的查询扩展

Memory 搜索不是简单的全文匹配。对于中日韩文本：
- **中文**：字符 unigram + bigram（"记忆系统" → "记", "忆", "系", "统", "记忆", "忆系", "系统"）
- **日文**：汉字/假名/ASCII 分离 + bigram
- **韩文**：尾缀助词剥离 + 词干提取

→ 详见 [05-Memory搜索引擎](./05-Memory搜索引擎.md)

## 与 OpenCode 的设计对比

| 维度 | OpenClaw | OpenCode |
|------|----------|----------|
| **定位** | 多渠道个人助手产品 | AI 编程 CLI 工具 |
| **核心代码量** | agents 95K + memory 11K | 全部核心 ~7K |
| **Agent 循环** | 嵌入 pi-coding-agent 库 | 自研 while(true) 循环 |
| **状态管理** | JSONL 会话文件 | SQLite 消息持久化 |
| **模型支持** | 通过 pi-ai 多 provider | Vercel AI SDK 20+ provider |
| **重试机制** | 3 层嵌套（模型回退 + 循环重试 + 单次尝试） | 单层 processor 重试 |
| **子 Agent** | 完整生命周期（spawn/steer/kill） | 简单 task 委派 |
| **上下文管理** | 压缩 + 工具结果截断 | Prune + Compaction 双层 |
| **Memory** | 独立搜索引擎（向量 + FTS5 + MMR） | 无内置记忆系统 |
| **工具系统** | bash/web/browser/memory/cron/session 等 | bash/edit/read/grep/glob 等 |

**核心差异**：OpenCode 是一个专注编程的轻量 CLI，设计简洁。OpenClaw 是一个全功能助手平台，需要处理多渠道、多模型、子 agent 编排、持久化记忆等产品级问题，复杂度高一个量级。
