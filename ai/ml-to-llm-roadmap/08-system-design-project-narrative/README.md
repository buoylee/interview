## 这个模块的主线

AI 系统设计不是把模型 API 接进后端就结束。你需要把不确定的模型行为变成可定义、可评估、可上线、可迭代的工程链路：

```text
任务目标 -> 输入输出契约 -> 模型与控制策略 -> 评估与安全 -> 服务与成本 -> 监控迭代
```

本模块默认例子是结构化客服工单分类和回复草稿生成。它不依赖 RAG，也不依赖 Agent 工具循环，适合先学习通用系统设计方法。RAG 与 Agent 系统设计会在后续模块单独处理。
