# 6.6 微调(LoRA) + 知识蒸馏 + 模型合并（Day 12-13）⭐⭐⭐

> **一句话定位**：LoRA 是大模型微调的事实标准，知识蒸馏把大模型能力压缩到小模型，模型合并可以零成本组合模型能力。

---

## 1. LoRA 详解 ⭐⭐⭐

### 1.1 核心数学

```
原始: Y = Wx                    W: (d × d), 如 4096 × 4096
LoRA: Y = Wx + ΔWx = Wx + BAx   A: (d × r), B: (r × d), r << d

r = rank (通常 8, 16, 32, 64)

例: d=4096, r=16
  W: 4096 × 4096 = 16.7M 参数 (冻结)
  A: 4096 × 16   = 65K 参数  ↘
  B: 16 × 4096   = 65K 参数  → 共 130K (0.78% of W!)
```

### 1.2 为什么低秩有效？

```
预训练后，微调时权重变化 ΔW 是低秩的：
  W_fine = W_pretrain + ΔW
  rank(ΔW) << rank(W)

直觉：微调是在预训练基础上做微小调整
  → 调整量（ΔW）的有效维度很低
  → 低秩矩阵就够表达这个调整
```

### 1.3 QLoRA

```
QLoRA = 4-bit 量化 + LoRA

1. 基础模型 4-bit 量化（GPTQ/NF4）→ 70B 模型 ~35GB
2. 在量化后的模型上加 LoRA adapter（FP16）
3. 只训练 LoRA 参数（0.1-1%）

效果：在单块 A100-80G 上微调 70B 模型！
```

### 1.4 LoRA 变体

| 变体 | 改进 | 用途 |
|------|------|------|
| **LoRA+** | A 和 B 用不同学习率 | 效果略好 |
| **DoRA** | 分解方向+大小 | 更接近全参数微调 |
| **rsLoRA** | 缩放因子调整 | 大 rank 时更稳定 |
| **AdaLoRA** | 自适应分配 rank | 不同层用不同 rank |

## 2. 知识蒸馏

### 2.1 核心方法

```
教师(大模型) → 学生(小模型)

方法 1: Logits 蒸馏
  学生学习教师的 soft label（softmax 输出）
  L = α × L_hard + (1-α) × T² × D_KL(P_s || P_t)

方法 2: 特征蒸馏  
  学生模仿教师的中间层表示

方法 3: 数据蒸馏 ⭐ LLM 时代主流
  用教师模型生成高质量训练数据 → 训练学生模型
  例：用 GPT-4 生成指令数据 → 训练小模型
  Alpaca, Vicuna 都是这样做的
```

### 2.2 LLM 时代的蒸馏

```
传统蒸馏：需要访问教师模型内部（logits/特征）→ 闭源模型无法使用
数据蒸馏：只需要教师模型的输出 → API 调用即可

实际操作：
1. 用 GPT-4 / Claude 生成大量高质量 (指令, 回答) 对
2. 用这些数据 SFT 训练小模型
3. 小模型学到了大模型的部分能力
```

## 3. 模型合并

### 3.1 核心思想

```
不用训练！直接在权重空间组合多个模型：

模型 A（擅长代码）+ 模型 B（擅长数学）→ 合并后两者都擅长
```

### 3.2 主要方法

| 方法 | 做法 | 特点 |
|------|------|------|
| **Linear Merge** | W = α×W_A + (1-α)×W_B | 最简单 |
| **SLERP** | 球面线性插值 | 保持向量范数 |
| **TIES** | 去掉冲突参数再合并 | 处理参数冲突 |
| **DARE** | 随机丢弃 + 缩放合并 | 适合多模型 |

### 3.3 实际应用

```
MergeKit (开源工具) 实现一键合并
Hugging Face 上大量合并模型排名靠前
零训练成本 → 快速实验
```

## 4. 面试常问

### Q1: LoRA 的原理是什么？为什么有效？

**答**：冻结预训练权重 W，用两个低秩矩阵 A(d×r) 和 B(r×d) 近似权重变化 ΔW=BA。有效因为微调时的权重变化本身是低秩的（调整量的有效维度远小于参数空间维度）。

### Q2: LoRA rank 怎么选？

**答**：通常 8-64。rank 越大表达力越强但参数越多。简单任务 r=8 够用，复杂任务用 r=32-64。关键不是 rank 大小，而是训练数据质量。

### Q3: 全参数微调 vs LoRA？

**答**：数据多+算力充足 → 全参数微调；数据少或算力有限 → LoRA/QLoRA；需要多任务切换 → LoRA（多个 adapter 共享基模型）。实际中大模型几乎一律用 LoRA。
