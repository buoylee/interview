# 6.4 分布式训练（Day 8-9）⭐ 工程必考

> **一句话定位**：大模型一块 GPU 放不下，必须用多 GPU/多机训练。DP、TP、PP、ZeRO 是四种核心并行策略。

---

## 1. 为什么需要分布式

```
LLaMA-70B 参数占用（FP16）：
  参数: 70B × 2 bytes = 140 GB
  优化器(Adam): 参数 × 8 = 560 GB（FP32 参数 + 一阶矩 + 二阶矩）
  梯度: 140 GB
  激活值: 取决于 batch size

总计: ~840 GB → 单块 A100 (80GB) 放不下！
```

## 2. 四种并行策略

### 2.1 数据并行 (DP)

```
同一个模型复制到 N 块 GPU，每块 GPU 处理不同的数据

GPU 0: 模型副本 + Batch 0 → 梯度 0 ↘
GPU 1: 模型副本 + Batch 1 → 梯度 1  → AllReduce 平均梯度 → 更新
GPU 2: 模型副本 + Batch 2 → 梯度 2 ↗

优点：实现简单，加速比接近线性
缺点：每块 GPU 都要放完整模型 → 模型太大放不下
```

### 2.2 张量并行 (TP) ⭐

```
把模型的矩阵拆分到多块 GPU：

例: 线性层 Y = XW，W 是 (4096 × 4096)
  GPU 0: W 的左半 → Y₀ = X × W₀  (4096 × 2048)
  GPU 1: W 的右半 → Y₁ = X × W₁  (4096 × 2048)
  → 拼接: Y = [Y₀, Y₁]

优点：单层拆分，显存均匀分配
缺点：需要频繁 GPU 间通信（每层都要同步）→ 只适合同机多卡
```

### 2.3 流水线并行 (PP)

```
不同层放不同 GPU，像流水线一样传递：

GPU 0: 层 1-10  → 输出传给 GPU 1
GPU 1: 层 11-20 → 输出传给 GPU 2
GPU 2: 层 21-30 → 输出传给 GPU 3
GPU 3: 层 31-40 → 计算损失

问题：前面的 GPU 算完要等后面的 → "气泡"（空闲时间）
解决：Micro-batch（把一个 batch 分成多个 micro-batch，交错执行）
```

### 2.4 ZeRO (DeepSpeed) ⭐

```
ZeRO = Zero Redundancy Optimizer

发现：DP 中每块 GPU 存了完整的 参数+梯度+优化器状态 → 大量冗余！

ZeRO-1: 分片优化器状态 → 显存省 4x
ZeRO-2: 分片优化器状态 + 梯度 → 省 8x
ZeRO-3: 分片优化器状态 + 梯度 + 参数 → 省 N×
         代价：需要时从其他 GPU 取参数（通信增加）
```

## 3. 实际组合 ⭐

```
大模型训练通常组合使用：

3D 并行 = DP + TP + PP

例: 用 64 块 A100 训练 70B 模型：
  TP = 8 (8 卡做张量并行，同一台机器内)
  PP = 4 (4 组做流水线并行，跨机器)
  DP = 2 (2 份数据并行)
  8 × 4 × 2 = 64 GPUs

+ 混合精度 (BF16) + 梯度检查点 + Flash Attention
```

## 4. 其他关键技术

| 技术 | 作用 |
|------|------|
| **混合精度** | FP32 主权重 + BF16 计算 → 省一半显存 |
| **梯度检查点** | 不存中间激活值，反向传播时重算 → 时间换显存 |
| **FSDP (PyTorch)** | PyTorch 原生的 ZeRO-3 实现 |
| **梯度累积** | 小 batch × 多次 = 等效大 batch |

## 5. 面试常问

### Q1: DP、TP、PP 的区别？

**答**：
- **DP**：同模型不同数据 → 加速训练，不解决模型放不下问题
- **TP**：一层的矩阵拆分到多卡 → 解决单层放不下，通信频繁需同机
- **PP**：不同层放不同卡 → 解决总层数太多，有气泡问题
- 实际中三者组合使用（3D 并行）

### Q2: ZeRO 是什么？三个阶段分别做什么？

**答**：ZeRO 消除数据并行中的冗余存储。ZeRO-1 分片优化器状态，ZeRO-2 再分片梯度，ZeRO-3 再分片参数。逐步减少每块 GPU 的显存占用。
