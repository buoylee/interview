## 1. Autoencoder 自编码器

### 1.1 结构

```
输入 x → [编码器 Encoder] → 潜在表示 z → [解码器 Decoder] → 重建 x̂

目标: 让 x̂ ≈ x（重建尽可能好）
损失: L = ||x - x̂||²（MSE 重建损失）

关键：z 的维度 << x 的维度 → 信息瓶颈 → 迫使模型学习压缩表示
```

### 1.2 直觉

```
把"大象放进冰箱"分三步：
  1. 编码：大象 → 压缩成"4条腿灰色大动物"（关键特征）
  2. 潜在空间：存储这个压缩描述
  3. 解码：从描述重新画出大象

如果能重建 → 说明 z 保留了最重要的信息 → z 是好的表示
```

### 1.3 应用

| 应用 | 说明 |
|------|------|
| 降维 | 类似 PCA 但非线性 |
| 去噪 (Denoising AE) | 输入加噪声，重建干净版本 → 学到鲁棒特征 |
| 特征学习 | 无监督学习有意义的表示 |
| 异常检测 | 正常数据重建好，异常数据重建差 |

## 2. VAE 变分自编码器

### 2.1 与 AE 的区别

```
AE:  编码器输出一个确定的向量 z
VAE: 编码器输出一个分布 (μ, σ)，z 从这个分布中采样

z ~ N(μ, σ²)   ← 不是固定值，是随机采样！
```

### 2.2 为什么要用分布？

```
AE 的问题：潜在空间不连续、不规整
  → 随机采一个 z 解码出来可能是乱码
  → 不能用来生成新数据

VAE 的解决：强制潜在空间接近标准正态分布
  → 潜在空间平滑、连续
  → 可以随机采样生成新数据
```

### 2.3 损失函数

```
L_VAE = 重建损失 + KL 散度约束
      = ||x - x̂||² + D_KL(q(z|x) || N(0, I))

重建损失：让重建好
KL 散度：让潜在分布接近标准正态 → 空间规整化
```

> 🔑 **ML 关联**：KL 散度在这里==约束分布不偏离太远，和 RLHF 中的用法一模一样！（回顾 [04-信息论](../00-math-foundations/04-information-theory.md)）

### 2.4 局限

```
VAE 生成的图像通常比较模糊（MSE 损失的副作用）
→ 后来被 GAN 和 Diffusion Model 在图像生成上超越
→ 但 VAE 的思想（潜在空间 + KL 约束）在 LLM 领域仍然重要
```

## 3. GAN 生成对抗网络

### 3.1 核心思想

```
两个网络对抗训练：

生成器 (Generator, G)：造假 → 从噪声 z 生成假数据
判别器 (Discriminator, D)：验真 → 判断数据是真是假

       噪声 z → [G] → 假数据 ↘
                                [D] → 真/假？
                真实数据    ↗

G 的目标：骗过 D（让假数据像真的）
D 的目标：识别 G 的假数据

博弈过程：G 和 D 互相提升，最终 G 生成的数据几乎以假乱真
```

### 3.2 直觉

```
G = 造假币的
D = 验钞机

造假币的越来越厉害 → 验钞机也越来越厉害 → 最终造出"完美"的假币
```

### 3.3 用途和局限

| 优点 | 缺点 |
|------|------|
| 生成质量高（图像清晰）| 训练不稳定（模式崩溃）|
| 理论优美 | 难以评估生成质量 |
| | 不提供概率密度 |

### 3.4 现状

```
2014-2020：GAN 是图像生成的王者
2020+：Diffusion Model (Stable Diffusion, DALL-E) 逐渐取代 GAN
原因：Diffusion 训练更稳定，生成质量更好，多样性更好
```

> 🔑 面试中 GAN 了解思想即可，不需要深入细节。Diffusion Model 是可选扩展方向（阶段 6.20）。

## 4. 对比学习 (Contrastive Learning) ⭐⭐⭐

### 4.1 为什么对比学习最重要？

```
对比学习是以下技术的理论基础：
  ✅ CLIP（图文对齐）→ 多模态
  ✅ Sentence-BERT → RAG 中的 Embedding 模型
  ✅ SimCLR / MoCo → 自监督视觉表示
  ✅ text-embedding-ada-002 / BGE / E5 → 你用的 Embedding 都是对比学习训练的
```

### 4.2 核心思想

```
拉近「相似样本」的表示，推远「不相似样本」的表示

输入空间:     嵌入空间:
"猫在沙发上"  ──→  ●  (近)
"小猫躺着"    ──→  ●  (近)
"股市涨了"    ──→        ● (远)
```

### 4.3 训练方式

```
正样本对 (Positive Pair)：语义相似的两个样本
负样本对 (Negative Pair)：语义不同的两个样本

训练目标：
  - 正样本对的 Embedding 距离 → 拉近（余弦相似度 → 1）
  - 负样本对的 Embedding 距离 → 推远（余弦相似度 → 0 或 -1）
```

### 4.4 InfoNCE Loss ⭐

```
L = -log [exp(sim(z_i, z_j⁺)/τ) / Σₖ exp(sim(z_i, z_k)/τ)]

z_i = anchor 的 Embedding
z_j⁺ = 正样本的 Embedding
z_k = 所有样本（1 个正 + N-1 个负）的 Embedding
τ = 温度参数
sim = 余弦相似度

直觉：softmax 分类 —— 从 N 个样本中正确识别出配对的那个
```

> 🔑 **和交叉熵的关系**：InfoNCE 本质上就是一个 N 路分类的交叉熵，其中正确类别是正样本对。

### 4.5 代表方法

#### SimCLR (视觉)

```
同一张图 → 两种数据增强 → 两个 view → 作为正样本对
不同图 → 负样本对

图 → [增强1] → [编码器] → z₁ ↘
                              → 对比损失
图 → [增强2] → [编码器] → z₂ ↗

核心：不需要标签！数据增强自动构造positive pair
```

#### CLIP (图文对齐) ⭐

```
图像-文本对 = 正样本
不匹配的图文 = 负样本

"猫的照片" ←→ "a photo of a cat"    → 拉近
"猫的照片" ←→ "the stock market"     → 推远

图像 → [Image Encoder] → 图像 Embedding ↘
                                         → 对比损失
文本 → [Text Encoder]  → 文本 Embedding ↗
```

> 🔑 **CLIP 的重要性**：
> - 实现了图像和文本在同一嵌入空间的对齐
> - 是多模态 LLM（如 LLaVA、GPT-4V）的基础
> - Zero-shot 图像分类：用文本描述类别，找最匹配的图像

### 4.6 硬负样本挖掘 (Hard Negative Mining)

```
问题：大多数负样本太简单（"猫" vs "量子物理"），模型学不到东西
解决：专门找"难区分"的负样本

简单负样本："猫" vs "量子物理"  → 太容易，没信息量
困难负样本："猫" vs "猫科动物"  → 很像但不同 → 学到更细粒度的区分
```

> 🔑 **RAG 关联**：Embedding 模型的质量很大程度上取决于硬负样本的质量。这就是为什么领域适配 Embedding 需要构造领域相关的困难负样本。

## 6. 面试常问

### Q1: 对比学习和监督学习的区别？

**答**：
- 监督学习需要人工标签，对比学习不需要（自监督构造正负对）
- 对比学习学到的是好的表示（Embedding），而不是直接预测标签
- 对比学习训练的 Embedding 可以迁移到多种下游任务

### Q2: CLIP 是怎么训练的？为什么重要？

**答**：
- CLIP 用 4 亿图文对做对比学习，让匹配的图文 Embedding 接近，不匹配的远离
- 实现了图像和文本的统一嵌入空间
- 重要性：(1) Zero-shot 图像理解 (2) 多模态模型的基础 (3) 图像检索

### Q3: 你用的 Embedding 模型是怎么训练的？

**答**：
- 基于对比学习（InfoNCE Loss）
- 训练数据：大量的（query, relevant_document）正样本对
- 负样本：in-batch negatives + 硬负样本挖掘
- 架构：通常是 BERT 类 Encoder + 平均池化/[CLS] token
- 这就是 Sentence-BERT / BGE / E5 等模型的训练方式

### Q4: VAE 中 KL 散度的作用？

**答**：
- 限制编码器输出的分布接近标准正态分布 N(0,I)
- 让潜在空间平滑连续 → 可以插值、采样、生成
- 和 RLHF 中的 KL 约束同理：防止分布偏离太远
