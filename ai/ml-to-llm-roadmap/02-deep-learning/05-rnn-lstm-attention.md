## 1. RNN 循环神经网络

### 1.1 为什么需要 RNN？

```
MLP 的问题：输入输出长度固定，不能处理序列

语言是序列：
  "我 爱 机器 学习" → 4 个词，有先后顺序
  "机器 学习 我 爱" → 同样 4 个词，但意思不同！

RNN 解决：引入「记忆」→ 当前输出依赖于前面的所有输入
```

### 1.2 RNN 结构

```
每个时间步 t：
  hₜ = f(W_hh × hₜ₋₁ + W_xh × xₜ + b)

  xₜ = 当前输入
  hₜ₋₁ = 上一步的隐藏状态（"记忆"）
  hₜ = 当前的隐藏状态

展开：
  x₁ → [RNN] → h₁
  x₂ → [RNN] → h₂ (依赖 h₁)
  x₃ → [RNN] → h₃ (依赖 h₂)
  ...

同一个 [RNN] 模块在每个时间步共享权重
```

### 1.3 RNN 的致命问题

#### 无法并行 ❌

```
h₃ 依赖 h₂，h₂ 依赖 h₁
→ 必须按顺序计算
→ GPU 并行能力完全浪费
→ 长序列训练极慢

对比 Transformer：所有 token 同时计算 → 并行性完美
```

#### 长期依赖问题 ❌

```
"我在北京出生，在上海上学，在深圳工作，......，所以我的母语是[中文]。"

问题：预测"中文"需要记住很远以前的"北京"
RNN 的信息通过 h₁ → h₂ → ... → hₙ 逐步传递
信息在传递过程中不断被覆盖/衰减 → 记不住久远的信息
```

#### 梯度消失/爆炸 ❌

```
反向传播时：
  ∂L/∂h₁ = ∂L/∂hₙ × (∂hₙ/∂hₙ₋₁ × ... × ∂h₂/∂h₁)
                        ↑ 同一矩阵 W_hh 的 n 次连乘

W_hh 的特征值 < 1 → 梯度消失
W_hh 的特征值 > 1 → 梯度爆炸
```

## 2. LSTM 长短期记忆

### 2.1 核心思想

```
RNN 只有一个隐藏状态 h → 又要记忆又要输出，负担太重
LSTM 加了一个"细胞状态" c → 专门用来存储长期记忆

LSTM = RNN + 三个门（gate）来控制信息流
```

### 2.2 三个门 ⭐

```
遗忘门 (Forget Gate):  fₜ = σ(W_f × [hₜ₋₁, xₜ] + b_f)
  → 决定忘掉旧记忆的哪些部分 (0=全忘, 1=全记)

输入门 (Input Gate):   iₜ = σ(W_i × [hₜ₋₁, xₜ] + b_i)
  → 决定写入新记忆的哪些部分

输出门 (Output Gate):  oₜ = σ(W_o × [hₜ₋₁, xₜ] + b_o)
  → 决定输出记忆的哪些部分

细胞状态更新：
  c̃ₜ = tanh(W_c × [hₜ₋₁, xₜ] + b_c)    ← 候选新记忆
  cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ              ← 旧记忆部分遗忘 + 新记忆部分写入
  hₜ = oₜ ⊙ tanh(cₜ)                      ← 从记忆中选择性输出
```

### 2.3 直觉类比

```
想象你在读一本小说，做笔记本：

遗忘门：翻到新章节 → 决定抹掉哪些旧笔记（不重要的角色）
输入门：读到新内容 → 决定记下哪些新信息（重要情节）
输出门：有人问你问题 → 决定从笔记中取出哪些信息回答
```

### 2.4 LSTM 解决了什么？

```
✅ 长期依赖：细胞状态 c 是线性传递（cₜ = fₜ ⊙ cₜ₋₁ + ...），梯度不容易消失
✅ 选择性记忆：门控机制让模型学会什么该记什么该忘
❌ 但仍然不能并行！(hₜ 依赖 hₜ₋₁)
❌ 序列太长时效果仍然不够好
```

## 3. GRU 门控循环单元

### 3.1 简化的 LSTM

```
GRU 只有两个门：
  更新门 (Update Gate): zₜ → 合并了遗忘门和输入门
  重置门 (Reset Gate):  rₜ → 控制使用多少旧状态

  没有独立的细胞状态 c（只有隐藏状态 h）
```

### 3.2 LSTM vs GRU

| | LSTM | GRU |
|--|------|-----|
| 门数 | 3（遗忘/输入/输出）| 2（更新/重置）|
| 状态 | h + c | 只有 h |
| 参数量 | 更多 | 更少（约 3/4 LSTM）|
| 效果 | 通常略好 | 接近 LSTM，更快 |
| 使用 | NLP 经典方案 | 参数敏感时 |

## 4. Seq2Seq 编码器-解码器

### 4.1 问题：输入输出长度不同

```
翻译："I love AI" (3词) → "我爱人工智能" (4词)
摘要：一篇文章 (500词) → 一句话 (20词)

RNN 做不到（输入输出等长），需要新架构
```

### 4.2 Seq2Seq 结构 ⭐

```
编码器 (Encoder)：读入整个输入 → 最终隐藏状态 = "压缩表示"
  "I love AI" → [Encoder RNN] → context vector c

解码器 (Decoder)：从 c 开始逐步生成输出
  c → [Decoder RNN] → "我" → "爱" → "人工智能" → <EOS>
```

### 4.3 瓶颈问题 ⭐

```
整个输入的所有信息 → 压缩到一个固定长度的向量 c
  → 信息瓶颈！长序列信息丢失严重

类比：读了一本 500 页的书 → 用一句话总结 → 再从这句话还原 → 信息损失巨大
```

> 🔑 **这个瓶颈就是 Attention 被发明的直接原因！**

## 5. Attention 注意力机制

### 5.1 核心思想

```
Seq2Seq 问题：解码器只能看到一个 context vector
Attention 改进：解码每个词时，让解码器"回头看"编码器的所有隐藏状态

解码"我"时 → 重点看 Encoder 中 "I" 对应的隐藏状态
解码"爱"时 → 重点看 Encoder 中 "love" 对应的隐藏状态
```

### 5.2 计算步骤

```
1. 计算注意力分数 (Attention Score):
   eᵢⱼ = score(sⱼ, hᵢ)
   sⱼ = 解码器第 j 步的状态
   hᵢ = 编码器第 i 步的状态

2. 归一化成注意力权重:
   αᵢⱼ = softmax(eᵢⱼ)  → Σαᵢⱼ = 1

3. 加权求和得到上下文向量:
   cⱼ = Σ αᵢⱼ × hᵢ     → 加权组合编码器输出

4. 用 cⱼ 辅助解码器预测：
   output = f(sⱼ, cⱼ)
```

### 5.3 Attention 的直觉 ⭐

```
Attention = 加权平均

权重不是固定的，而是动态计算的：
  - 当前需要什么信息 (Query)
  - 哪里有相关信息 (Key-Value)
  - 相关性 = Query 和 Key 的相似度
  - 加权取 Value = 提取最相关的信息
```

> 🔑 这就是 Transformer 中 Q, K, V 的来源！
> ```
> 原始 Attention            Transformer
> Query = 解码器状态        Q = X × W_Q
> Key = 编码器状态          K = X × W_K
> Value = 编码器状态        V = X × W_V
> ```

### 5.4 Attention 解决了什么？

```
✅ 信息瓶颈：不再压缩到单个向量，可以动态访问所有输入
✅ 长距离依赖：直接连接到任意位置，不需要逐步传递
✅ 可解释性：注意力权重可以可视化（哪些词对齐了）
❌ 仍然是 RNN + Attention → 不能并行
```

### 5.5 从 Attention 到 Self-Attention

```
原始 Attention (2014): 编码器和解码器之间
  → Query 来自解码器，Key/Value 来自编码器
  
Self-Attention (2017 Transformer): 序列和自己之间
  → Query, Key, Value 都来自同一输入
  → 每个 token 可以和所有其他 token 交互
  → 不需要 RNN！→ 可以并行！
```

> 🔑 **Transformer = Self-Attention + FFN + Residual + LayerNorm，完全去掉了 RNN！**

## 6. Beam Search 解码策略

### 6.1 为什么需要解码策略？

```
自回归生成：每步选一个 token → 选择策略影响最终质量

贪婪 (Greedy): 每步选概率最高的 → 可能不是全局最优
Beam Search:  同时保持 K 个候选 → 更可能找到全局最优
```

### 6.2 Beam Search 原理

```
Beam Width = K（保持 K 个候选序列）

例: K=2

Step 1:  P("我")=0.6  P("今")=0.3
         保持: ["我", "今"]

Step 2:  "我"→ P("爱")=0.5, P("是")=0.4
         "今"→ P("天")=0.7, P("日")=0.2
         综合分数排序，保持 top-2:
         ["今天"(0.21), "我爱"(0.30)]

Step 3: 继续扩展...
```

### 6.3 解码策略总览

```
Greedy → Beam Search → Top-k → Top-p → Temperature

确定性 ←→ 多样性
```

| 策略 | 特点 | 用途 |
|------|------|------|
| Greedy | 每步选最高概率 | 确定性任务(翻译) |
| Beam Search | 保持 K 个候选 | 质量优先 |
| Top-k | 从前 k 个中采样 | 多样性 |
| Top-p (Nucleus) | 从累积概率 ≥ p 中采样 | 动态范围 |
| Temperature | 调整概率分布平坦度 | 控制创造性 |

> 这些会在阶段 3 的「解码策略」章节详细展开。

## 7. 演进总结

```
时间线：

RNN (1986)
  ↓ 问题：梯度消失，记不住长距离
LSTM (1997)
  ↓ 问题：仍然无法并行
GRU (2014) ← LSTM 简化版
Seq2Seq + Attention (2014)
  ↓ 解决：信息瓶颈
Self-Attention (2017)
  ↓ 突破：去掉 RNN，完全并行
Transformer (2017) "Attention is All You Need"
```

### 每一步解决的核心问题

| 阶段 | 解决了什么 | 还有什么问题 |
|------|-----------|------------|
| RNN | 处理序列数据 | 梯度消失、无法并行 |
| LSTM | 长期依赖 | 无法并行 |
| Attention | 信息瓶颈 | 仍依赖 RNN |
| **Transformer** | **去掉 RNN，完全并行** | O(n²) 复杂度 |

## 8. 面试常问

### Q1: RNN 为什么会梯度消失？LSTM 怎么解决的？

**答**：
- RNN：反向传播时对同一权重矩阵 W_hh 做 n 次连乘，特征值 <1 时指数衰减
- LSTM：引入细胞状态 c，通过线性传递 cₜ = fₜ⊙cₜ₋₁ + iₜ⊙c̃ₜ
  - 遗忘门 fₜ 接近 1 时，梯度可以沿 c 直接传回 → 类似 Residual Connection
  - 门控机制学习性地保留/遗忘信息

### Q2: 什么是 Attention？为什么需要 Attention？

**答**：
- Attention = 动态加权平均，让模型在需要时关注输入的不同部分
- 原因：Seq2Seq 把整个输入压缩成一个固定向量 → 信息瓶颈
- Attention 让解码器在每一步动态访问编码器的所有输出 → 解决了瓶颈

### Q3: Self-Attention 和传统 Attention 的区别？

**答**：
- **传统 Attention**：Query 和 Key/Value 来自不同序列（解码器→编码器）
- **Self-Attention**：Query、Key、Value 都来自同一序列（每个 token 和所有其他 token 交互）
- Self-Attention 不依赖 RNN → 可以完全并行 → Transformer 的基础

### Q4: 为什么 Transformer 取代了 LSTM？

**答**：
1. **并行性**：LSTM 必须按顺序处理，Transformer 所有 token 同时计算
2. **长距离依赖**：LSTM 信息需要逐步传递，Self-Attention 直接连接任意距离
3. **训练效率**：并行性使得 Transformer 训练快得多 → 可以用更大数据
4. **可扩展性**：Transformer + 大数据 + 大算力 = Scaling Law → 大模型时代
