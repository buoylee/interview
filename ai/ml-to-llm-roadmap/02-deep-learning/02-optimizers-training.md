## 1. 优化器演进

### 1.1 演进路线

```
SGD → Momentum → RMSProp → Adam → AdamW
      (加惯性)   (自适应)   (两者结合) (修正Weight Decay)
```

### 1.2 SGD（随机梯度下降）

```
θ = θ - α × ∇L

每一步只用一个/一批数据计算梯度
```

**问题**：
- 震荡严重（在峡谷式损失面上来回弹）
- 所有参数用同一个学习率

### 1.3 SGD + Momentum（动量）

```
v = β × v + ∇L          (动量 = 历史梯度的指数加权平均)
θ = θ - α × v
```

**直觉**：下山时滚雪球
- 同一方向的梯度会累积 → 加速
- 来回震荡的方向会抵消 → 减震
- β 通常 = 0.9

### 1.4 RMSProp（自适应学习率）

```
s = β × s + (1-β) × (∇L)²    (梯度平方的指数加权平均)
θ = θ - α × ∇L / (√s + ε)
```

**直觉**：
- 梯度大的方向 → s 大 → 步长小（别跑太快）
- 梯度小的方向 → s 小 → 步长大（别太慢）
- 每个参数有自己的「有效学习率」

### 1.5 Adam ⭐

```
m = β₁ × m + (1-β₁) × ∇L        (一阶矩：梯度的均值 → Momentum)
v = β₂ × v + (1-β₂) × (∇L)²     (二阶矩：梯度的方差 → RMSProp)

m̂ = m / (1-β₁ᵗ)                  (偏差修正)
v̂ = v / (1-β₂ᵗ)

θ = θ - α × m̂ / (√v̂ + ε)

默认超参数: β₁=0.9, β₂=0.999, ε=1e-8
```

**优点**：结合 Momentum + RMSProp，自适应学习率，对大多数问题开箱即用。

### 1.6 AdamW ⭐⭐ LLM 标配

```
Adam 中加 L2 正则化有 bug：
  Adam:  θ = θ - α × (m̂/(√v̂+ε) + λθ)  ← L2 被自适应学习率影响了！
  AdamW: θ = θ - α × m̂/(√v̂+ε) - α×λ×θ  ← Weight Decay 和梯度更新解耦

区别：AdamW 的权重衰减不受自适应学习率的干扰
```

**为什么 LLM 都用 AdamW？**
- 正确实现了权重衰减 → 正则化效果更好
- 大模型对正则化更敏感

### 1.7 对比总结

| 优化器 | 自适应学习率 | 动量 | 适用 |
|--------|------------|------|------|
| SGD | ❌ | ❌ | 简单问题 |
| SGD+Momentum | ❌ | ✅ | 传统 DL |
| RMSProp | ✅ | ❌ | RNN |
| Adam | ✅ | ✅ | 通用 |
| **AdamW** | ✅ | ✅ | **LLM 训练** |

## 2. 学习率调度

### 2.1 为什么需要调度？

```
固定学习率的问题：
  太大 → 后期震荡，无法精细收敛
  太小 → 训练太慢

理想策略：先大后小
  前期：快速探索 → 大学习率
  后期：精细收敛 → 小学习率
```

### 2.2 常见调度策略

#### Warmup ⭐

```
前 N 步：学习率从 0 线性增大到目标值

为什么需要：
  - 训练初期参数随机，梯度方向不可靠
  - Adam 的二阶矩估计还不准确
  - 大学习率 + 不准确的梯度 → 参数跑偏
  - Warmup 让模型先「慢慢适应」
```

#### Cosine Decay ⭐

```
学习率按余弦函数从峰值降到接近 0：

α(t) = α_min + (α_max - α_min) × (1 + cos(πt/T)) / 2
```

**直觉**：前期下降慢（维持较高学习率多学一会），后期下降快。

#### Linear Decay

```
学习率线性从峰值降到 0：
α(t) = α_max × (1 - t/T)
```

### 2.3 LLM 标配组合

```
Warmup（前 1-10% 步） + Cosine Decay（剩余步）

       α_max ─────╮
                    ╲
    Warmup          Cosine Decay
  ╱                  ╲
0 ─                    ─── α_min
  |-------|--------------------------|
  0     warmup                     总步数
```

## 3. 混合精度训练

### 3.1 数值精度

```
FP32 (32位浮点): 精度高，显存占用大，计算慢
FP16 (16位浮点): 精度减半，显存减半，计算快 2-3x
BF16 (Brain Float 16): Google 设计，范围和 FP32 一样大，精度和 FP16 一样
```

### 3.2 为什么不全用 FP16？

```
FP16 的值域: ±65504
FP32 的值域: ±3.4×10³⁸

问题：
  - 梯度值很小时 → FP16 下溢为 0 → 训练失败
  - 损失值很大时 → FP16 溢出为 Inf → NaN
```

### 3.3 混合精度策略 ⭐

```
                  FP32         FP16/BF16
主权重存储        ✅
前向计算                       ✅ (快)
梯度计算                       ✅ (快)
权重更新          ✅ (精确)
优化器状态        ✅

流程：
1. 用 FP16 做前向和反向（快）
2. 得到 FP16 梯度
3. 转成 FP32 更新主权重（精确）
4. 主权重转回 FP16 给下一轮使用
```

### 3.4 BF16 vs FP16

```
FP16:  1 符号 + 5 指数 + 10 尾数 → 精度高但范围小
BF16:  1 符号 + 8 指数 + 7 尾数  → 精度低但范围大

BF16 范围和 FP32 一样 → 不容易溢出 → 更适合大模型训练
LLM 训练标配：BF16（如果硬件支持）
```

## 4. 梯度累积与梯度裁剪

### 4.1 梯度累积 (Gradient Accumulation)

```
问题：想用大 batch size 但显存不够

解决：分多次算梯度，累加后再更新

伪代码：
for i in range(accumulation_steps):
    loss = model(batch[i])
    loss = loss / accumulation_steps
    loss.backward()                    # 梯度累积
optimizer.step()                       # 累积完再更新一次
optimizer.zero_grad()

效果等价的 batch size = 实际 batch × accumulation_steps
```

### 4.2 梯度裁剪 (Gradient Clipping) ⭐

```
问题：梯度突然变得很大 → 参数大幅更新 → 训练崩溃

解决：限制梯度的最大范数

if ||∇L|| > max_norm:
    ∇L = ∇L × max_norm / ||∇L||

常用 max_norm = 1.0
```

**LLM 训练必备**：防止偶尔出现的巨大梯度破坏训练。

## 5. 面试常问

### Q1: Adam 和 SGD 的区别？什么时候用哪个？

**答**：
- **Adam**：自适应学习率 + 动量，开箱即用，收敛快 → 大多数情况首选
- **SGD+Momentum**：泛化可能更好（有论文声称），但需要精细调参 → CV 竞赛中有时用
- **LLM**：一律 AdamW

### Q2: 为什么需要 Warmup？

**答**：
- 训练初期模型参数随机，梯度方向很不准
- Adam 的二阶矩估计需要一些步骤来预热
- 直接用大学习率 → 早期不准确的大步更新 → 可能跑到很差的区域，后面很难恢复
- Warmup 让模型先慢慢适应，避免早期灾难性更新

### Q3: 混合精度训练的原理？为什么 LLM 用 BF16？

**答**：
- 前向和反向用 FP16/BF16（快），权重更新用 FP32（精确）
- BF16 的值域和 FP32 一样大，不容易溢出
- FP16 值域小（最大 65504），大模型训练容易溢出
- 混合精度能省一半显存，提速 2-3x

### Q4: 梯度累积的作用？

**答**：用小 batch 模拟大 batch 训练。多次前向/反向传播后累积梯度，然后做一次参数更新。效果等价于更大的 batch size，但显存只需要小 batch 的量。
