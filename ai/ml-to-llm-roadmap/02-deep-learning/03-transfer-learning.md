## 1. 迁移学习 (Transfer Learning)

### 1.1 核心思想

```
在任务 A 上学到的知识 → 迁移到任务 B

例：
  ImageNet 上学到「边缘、纹理、形状」→ 迁移到医疗图像分类
  互联网文本上学到「语言规律」→ 迁移到客服对话、代码生成
```

**直觉**：先成为「通才」，再成为「专才」。不需要每次都从零学起。

### 1.2 为什么迁移可以工作？

```
底层特征是通用的：
  CV：边缘、纹理、颜色 → 任何图像任务都需要
  NLP：语法、语义、常识 → 任何语言任务都需要

高层特征是任务相关的：
  CV：「猫的耳朵」vs「X 光中的肿瘤」
  NLP：「客服话术」vs「代码模式」
```

### 1.3 迁移学习的范式

```
传统 ML：每个任务从零训练
预训练+微调：先在大数据上预训练，再在小数据上微调

时间线：
2012 ImageNet (CV)
  → 2018 BERT/GPT (NLP)
    → 2020+ 大模型 (通用)
```

| 阶段 | 数据量 | 学到什么 |
|------|--------|---------|
| **预训练** | TB 级无标注数据 | 通用知识（语言规律、世界知识）|
| **微调** | KB-GB 级标注数据 | 任务特定能力 |

### 1.4 微调策略

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| **全参数微调** | 所有参数都更新 | 数据多、算力充足 |
| **冻结底层** | 只更新顶层，底层冻结 | 数据少、底层特征够通用 |
| **LoRA** | 冻结全部 + 插入低秩矩阵 | 大模型微调标配 ⭐ |
| **Adapter** | 冻结全部 + 层间小模块 | 类似 LoRA |
| **Prompt Tuning** | 冻结全部 + 可学习前缀 | 极少参数 |

> 🔑 **关键洞察**：为什么大模型微调不全参数更新？
> 1. 全参数微调太贵（GPT-3 有 175B 参数）
> 2. 数据少时全参数更新容易**灾难性遗忘**
> 3. LoRA 只更新 0.1% 的参数但效果接近全参数

## 2. 灾难性遗忘 (Catastrophic Forgetting)

### 2.1 什么是灾难性遗忘？

```
模型在任务 A 上训练好了
再在任务 B 上微调
→ 任务 A 的能力急剧下降！

例：
  GPT 预训练学会了通用对话
  在客服数据上微调 → 变得只会客服话术，忘了怎么写代码
```

**直觉**：新知识覆盖了旧知识，像用新笔迹盖住了原来的字。

### 2.2 为什么会发生？

```
神经网络的知识存储在权重中
微调时更新权重 → 新知识写入
但同时 → 旧知识的权重被改变 → 旧能力丢失

特别是：
  - 如果微调数据和预训练数据分布差异大 → 更严重
  - 如果微调步数多 → 更严重
  - 如果学习率大 → 更严重
```

### 2.3 解决方案 ⭐

| 方法 | 原理 | 效果 |
|------|------|------|
| **LoRA / QLoRA** ⭐ | 冻结原始权重，只训练低秩增量 | 原始能力完全保留 |
| **冻结层** | 只更新最后几层 | 底层通用特征不变 |
| **小学习率** | 减少对旧权重的改变 | 简单有效 |
| **EWC (Elastic Weight Consolidation)** | 重要权重加大惩罚 | 理论优美但计算贵 |
| **Replay / 混合训练** | 微调时混入预训练数据 | 简单有效 |
| **正则化 (KL 约束)** | 微调后不能偏离原模型太远 | RLHF/DPO 中使用 |

### 2.4 LoRA 为什么能解决灾难性遗忘？

```
原始模型: y = Wx                    (W 冻结，原始能力保留)
LoRA:     y = Wx + ΔWx = Wx + ABx   (只训练 A, B)

ΔW = A × B  (低秩)
  - A: (d × r), B: (r × d), r << d
  - 原始 W 完全不动 → 不会遗忘！
  - 如果不需要了，把 LoRA 去掉就恢复原来的模型
  - 多个 LoRA 可以切换（不同任务不同 LoRA）
```

### 2.5 EWC 的直觉

```
思路：不是所有权重对旧任务都同等重要
  - 找出对旧任务「重要的权重」（Fisher Information 矩阵）
  - 微调时：这些重要权重不许大幅改变
  
Loss = L_new_task + λ × Σ Fᵢ × (θᵢ - θ*ᵢ)²
                        ↑          ↑
                   重要度      不许偏离旧值太多
```

## 3. Domain Adaptation

### 3.1 什么是域适应？

```
源域 (Source Domain) → 有大量标注数据（如互联网文本）
目标域 (Target Domain) → 少量或无标注数据（如医疗/法律文本）

问题：两个域的分布不同（Distribution Shift）
目标：让源域训练的模型在目标域也能工作
```

### 3.2 类型

| 类型 | 说明 | 例子 |
|------|------|------|
| 有监督域适应 | 目标域有少量标注 | 少量医疗标注 + 大量通用预训练 |
| 无监督域适应 | 目标域无标注 | 只有未标注的法律文档 |
| 零样本域适应 | 目标域无任何数据 | 从没见过的语言/领域 |

### 3.3 在 LLM 中的体现

```
通用预训练 (Source) → 领域微调 (Target)

方法：
1. 领域继续预训练 (Domain-Adaptive Pretraining)
   - 在领域文本上继续做 CLM → 适应语言分布
2. 指令微调 (Instruction Tuning)
   - 用领域特定的指令-回复对微调
3. RAG (检索增强)
   - 不改模型，用检索注入领域知识
```

> 🔑 **实用关联**：你在做 RAG 时，本质上就是在做无参数的域适应——用检索的方式补充模型缺乏的领域知识，而不用微调模型。

## 4. 面试常问

### Q1: 什么是迁移学习？为什么 LLM 需要预训练+微调？

**答**：
- 迁移学习 = 在大规模通用数据上学到的知识迁移到特定任务
- 预训练学到通用的语言能力（语法、语义、常识）
- 微调让模型适应特定任务（对话、代码、特定领域）
- 从零训练每个任务不现实（数据量和算力都不够）

### Q2: 什么是灾难性遗忘？怎么解决？

**答**：
- 微调新任务时，模型忘掉预训练学到的旧知识
- 因为新数据的梯度更新覆盖了旧知识对应的权重
- **解决**：LoRA（冻结原始权重）、小学习率、混入旧数据、KL 约束
- LoRA 是最优雅的解决方案：原始权重不动，遗忘=0

### Q3: LoRA 和全参数微调，什么时候用哪个？

**答**：
| 条件 | 选择 |
|------|------|
| 数据多、算力充足、需要最大性能 | 全参数微调 |
| 数据少 | LoRA（防过拟合和遗忘）|
| 算力有限（无法加载全部梯度）| LoRA 或 QLoRA |
| 需要多个任务版本切换 | LoRA（多个 adapter 共享基模型）|
| 实际中 | 大模型几乎一律 LoRA/QLoRA |

### Q4: RAG 和微调哪个好？

**答**：
| | RAG | 微调 |
|--|-----|------|
| 知识更新 | 实时（换文档就行）| 需要重新训练 |
| 幻觉 | 有据可查（可引用来源）| 可能编造 |
| 成本 | 推理时多加检索步骤 | 训练一次，推理无额外成本 |
| 适用 | 知识密集型任务 | 风格/格式适应 |
- 很多场景下两者结合最好
