# 3.4 语言模型 & 解码（Day 7-8）

> **一句话定位**：语言模型的本质是预测下一个 token 的概率分布，解码策略决定了如何从这个分布中选 token。理解这些才理解 LLM 生成文本的机制。

## 1. 语言模型的两种范式

### 1.1 自回归 (Autoregressive, AR) — GPT

```
给定前面的词，预测下一个词：

P(w₁, w₂, ..., wₙ) = P(w₁) × P(w₂|w₁) × P(w₃|w₁,w₂) × ...

"今天" → P("天气"|"今天") = 0.3
"今天天气" → P("真"|"今天天气") = 0.4
"今天天气真" → P("好"|"今天天气真") = 0.6

从左到右，一个一个生成
```

**特点**：
- 单向（只看左边）
- 天然适合生成任务
- **大模型主流范式** → GPT、LLaMA、Qwen 都是 AR

### 1.2 自编码 (Autoencoding, AE) — BERT

```
掩码语言模型 (MLM)：完形填空

"今天[MASK]真好" → P("天气") = 0.7

同时看左边和右边 → 双向理解
```

**特点**：
- 双向
- 适合理解任务（分类、NER、问答）
- 不适合生成（完形填空和生成是不同的）

### 1.3 对比

| | 自回归 (AR) | 自编码 (AE) |
|--|------------|------------|
| 代表 | GPT | BERT |
| 方向 | 单向（左→右） | 双向 |
| 训练任务 | 下一个词预测 (CLM) | 完形填空 (MLM) |
| 适合 | **生成** | 理解 |
| 大模型趋势 | ⭐ **主流** | 用于 Embedding/分类 |

## 2. 自回归模型 GPT 系列

### 2.1 训练目标：Causal Language Modeling (CLM)

```
输入: "今天 天气 真 好"
目标: 预测每个位置的下一个 token

位置 0: "今天" → 预测 "天气"
位置 1: "今天 天气" → 预测 "真"
位置 2: "今天 天气 真" → 预测 "好"

损失 = -Σ log P(wₜ | w₁, ..., wₜ₋₁)  = 交叉熵
```

### 2.2 Causal Mask

```
为什么自回归模型只看左边？→ 用 Causal Mask（因果掩码）

          今天  天气  真  好
今天       ✅    ❌   ❌  ❌
天气       ✅    ✅   ❌  ❌
真         ✅    ✅   ✅  ❌
好         ✅    ✅   ✅  ✅

每个 token 只能 attend 到自己和之前的 token
→ 防止"作弊"（看到未来的词）
→ 训练和推理时行为一致
```

### 2.3 推理过程（生成）

```
输入: "请介绍一下"
Step 1: 输入 → P(next) → 选 "人工" → 输出 "人工"
Step 2: "请介绍一下人工" → P(next) → 选 "智能" → 输出 "智能"
Step 3: "请介绍一下人工智能" → P(next) → 选 "。" → 停止
```

> 🔑 每次推理只产生 1 个 token，且需要重新处理所有之前的 token → 这就是为什么需要 KV-Cache 加速（阶段 6）

## 3. 自编码模型 BERT 系列

### 3.1 训练任务

#### MLM (Masked Language Model)

```
输入: "今天 [MASK] 真 好"
目标: 预测 [MASK] = "天气"

15% 的 token 被选中，其中：
  80% → 替换为 [MASK]
  10% → 替换为随机 token
  10% → 保持不变
```

#### NSP (Next Sentence Prediction)

```
判断两个句子是否是连续的：
  "我喜欢猫。" + "猫很可爱。" → IsNext
  "我喜欢猫。" + "今天股市涨了。" → NotNext

后来 RoBERTa 证明 NSP 没用 → 去掉更好
```

### 3.2 BERT 在 LLM 时代的角色

```
BERT 不是生成模型 → 不能聊天、写文章
但 BERT 的 Encoder 很擅长理解 → 用于：
  1. Embedding 模型（Sentence-BERT）
  2. 分类任务（情感分析、意图识别）
  3. NER（命名实体识别）
  4. Cross-Encoder（Reranking）
```

## 4. Perplexity 困惑度

### 4.1 定义

```
PPL = exp(-1/N × Σ log P(wₜ | w₁, ..., wₜ₋₁))
    = exp(交叉熵)
```

### 4.2 直觉

```
PPL = 模型在每个位置平均面对的"等可能选项数"

PPL = 1    → 完美预测（每个位置只有 1 个可能）
PPL = 10   → 在 10 个词之间犹豫
PPL = 100  → 在 100 个词之间犹豫（很困惑）

PPL 越低 → 模型越好
```

### 4.3 注意事项

```
PPL 只衡量语言建模能力，不等于"好用"：
  - PPL 低不代表回答准确（可能流畅地说错话）
  - 不同 tokenizer 的 PPL 不可比（token 数量不同）
  - 通常在特定数据集（如 WikiText）上评测
```

## 5. 解码策略详解

### 5.1 概览

```
模型输出: logits → softmax → 概率分布 P(next_token | context)

      P("猫") = 0.3
      P("狗") = 0.2
      P("鸟") = 0.15
      P("鱼") = 0.1
      ...

解码策略 = 怎么从这个分布中选 token
```

### 5.2 Greedy Decoding（贪婪解码）

```
每步选概率最高的 token

P("猫")=0.3, P("狗")=0.2, P("鸟")=0.15 → 选"猫"

优点：确定性、简单
缺点：可能陷入重复循环、缺乏多样性
适用：确定性任务（数学推理、代码）
```

### 5.3 Beam Search（束搜索）

```
同时保持 B 个候选序列，每步扩展所有候选

Beam Width = 3:
  Step 1: ["猫"(0.3), "狗"(0.2), "鸟"(0.15)]
  Step 2: 每个候选扩展 → 取 top-3 → [...]
  ...
  最终选总概率最高的序列

优点：比 Greedy 更可能找到全局最优
缺点：仍然偏向高频短序列
适用：翻译、摘要（质量优先）
```

### 5.4 Top-k Sampling ⭐

```
只从概率最高的 k 个 token 中随机采样

k=3:
  P("猫")=0.3, P("狗")=0.2, P("鸟")=0.15
  → 归一化: P("猫")=0.46, P("狗")=0.31, P("鸟")=0.23
  → 按新概率随机采样

优点：避免采到概率极低的 token
缺点：k 固定，不够灵活（有时合理选项只有 2 个，有时有 50 个）
```

### 5.5 Top-p (Nucleus) Sampling ⭐

```
取概率累积和 ≥ p 的最小 token 集合

p=0.9:
  按概率排序: P("猫")=0.3, P("狗")=0.2, P("鸟")=0.15, P("鱼")=0.1, ...
  累积: 0.3 → 0.5 → 0.65 → 0.75 → ... → ≥0.9
  只从这些 token 中采样

优点：自适应候选集大小！
  - 分布集中时（明确答案）→ 候选少 → 更确定
  - 分布分散时（开放话题）→ 候选多 → 更多样

常用: p = 0.9 或 0.95
```

### 5.6 Temperature ⭐

```
P_adjusted(xᵢ) = exp(logit_i / T) / Σ exp(logit_j / T)

T = 1.0: 原始分布
T > 1.0: 分布更平坦 → 更随机、更有创造力
T < 1.0: 分布更尖锐 → 更确定、更保守
T → 0:   等价于 Greedy（选最高概率）
T → ∞:   等价于均匀随机

例：原始 P("猫")=0.3, P("狗")=0.2, P("鱼")=0.01

T=0.5: P("猫")=0.55, P("狗")=0.30, P("鱼")=0.0001  → 集中
T=2.0: P("猫")=0.25, P("狗")=0.22, P("鱼")=0.05     → 分散
```

### 5.7 组合使用

```
实际 LLM API 中通常组合使用：

ChatGPT: temperature + top_p（二选一调节）
通用推荐:
  - 创意写作: T=0.8, top_p=0.95
  - 代码/数学: T=0.2, top_p=0.9（或 T=0, greedy）
  - 对话: T=0.7, top_p=0.9

重复惩罚 (Repetition Penalty):
  降低已出现 token 的概率 → 避免重复
```

## 6. 面试常问

### Q1: 自回归和自编码的区别？为什么大模型用自回归？

**答**：
- 自回归（GPT）：左到右预测，适合生成
- 自编码（BERT）：双向完形填空，适合理解
- 大模型用自回归因为：(1) 生成是通用能力(2) 自回归训练简单（不需要设计 mask 策略）(3) Scaling 效果更好

### Q2: Temperature 怎么影响输出？

**答**：Temperature 调整 softmax 前的 logits。T 高 → 分布平坦 → 输出多样但可能不准；T 低 → 分布尖锐 → 输出确定但单调。T=0 等价于贪婪解码。

### Q3: Top-k 和 Top-p 的区别？为什么 Top-p 更好？

**答**：
- Top-k 固定候选数量（如 50）→ 不够灵活
- Top-p 自适应候选数量 → 分布集中时候选少（确定），分散时候选多（多样）
- Top-p 更好因为它能自动适应不同位置的确定性程度

### Q4: 什么是 Perplexity？怎么解释？

**答**：困惑度 = 模型平均在多少个等可能 token 中犹豫。PPL=10 表示模型在每个位置有 10 个等可能的选项。是交叉熵的指数形式。PPL 越低模型越好，但不同 tokenizer 的 PPL 不可比。
