## 1. 从词向量到句子向量

### 1.1 问题

```
Word2Vec/BERT 给的是词级向量
RAG 需要的是句子/段落/文档级向量

怎么从词向量得到句子向量？
```

### 1.2 简单方法（效果差）

| 方法 | 做法 | 问题 |
|------|------|------|
| 平均池化 | 所有词向量取平均 | 信息丢失（"不好"和"好"平均了）|
| [CLS] token | 直接用 BERT 的 [CLS] | BERT 原始 [CLS] 不适合句子相似度 |
| Max Pooling | 每个维度取最大值 | 信息丢失 |

### 1.3 为什么原始 BERT 做不好句子相似度？

```
BERT 原始训练目标是 MLM + NSP
[CLS] 向量是为 NSP 设计的，不是为语义相似度设计的

实验发现：直接用 BERT [CLS] 算余弦相似度
→ 效果甚至不如 GloVe + 平均！
→ 因为 BERT 的语义信息分布在所有 token 上，不集中在 [CLS]
```

> 🔑 这就是为什么需要 Sentence-BERT 等专门的 Embedding 模型。

## 2. Sentence-BERT 架构

### 2.1 核心改进

```
BERT: 两个句子拼在一起 → [CLS] s1 [SEP] s2 [SEP] → 交叉注意力 → 分类
  → 慢！10000 个文档两两比较需要 ~5000 万次推理

Sentence-BERT: 每个句子独立编码 → 固定长度的句子向量 → 直接比较
  → 快！10000 个文档只需 10000 次编码

架构：
  句子 → BERT → 所有 token 向量 → 池化(平均/[CLS]) → 句子 Embedding
```

### 2.2 Bi-Encoder 架构 ⭐

```
      句子 A                句子 B
        ↓                     ↓
    [BERT 编码器]          [BERT 编码器]  ← 同一个模型，共享参数
        ↓                     ↓
    向量 a (768D)          向量 b (768D)
        ↓                     ↓
         cosine_similarity(a, b)

特点：两个句子独立编码 → 可以预先计算所有文档的 Embedding → 检索时只算 query
```

> 🔑 这就是你在 RAG 中用的架构：文档离线编码存入向量数据库，查询时只编码 query。

## 3. 对比学习训练

### 3.1 训练数据

```
正样本对 (Positive Pairs):
  - (问题, 相关回答)
  - (标题, 正文)
  - (query, relevant document)
  - NLI 中的蕴含对

负样本:
  - In-batch Negatives: 同 batch 中其他样本的配对
  - Hard Negatives: 语义接近但不相关的样本
```

### 3.2 训练损失

#### InfoNCE Loss（回顾）

```
L = -log [exp(sim(q, d⁺)/τ) / Σ exp(sim(q, dₖ)/τ)]

q = query embedding
d⁺ = 正样本 embedding
dₖ = 所有样本（1 正 + N-1 负）
τ = 温度
```

#### Multiple Negatives Ranking Loss

```
Sentence-BERT 常用变体：
给定 (anchor, positive)，batch 中其他 positive 作为 negative

Batch:
  (q₁, d₁⁺)  → q₁ 和 d₁⁺ 是正样本对
  (q₂, d₂⁺)  → q₂ 和 d₂⁺ 是正样本对
  ...

对于 q₁：d₁⁺ 是正样本，d₂⁺, d₃⁺, ... 是负样本
→ 更大的 batch size = 更多负样本 = 效果更好
```

### 3.3 硬负样本挖掘 ⭐

```
简单负样本："如何做饭" vs "量子物理论文"  → 太容易
硬负样本：  "如何做饭" vs "厨房设备推荐"  → 相似但不是答案

硬负样本挖掘方法：
1. BM25 检索 top-K 但非正样本 → 关键词相似但语义不同
2. 用先前训练的模型检索接近但错误的结果
3. 交叉编码器重排后取高分但非正样本

硬负样本越好 → 模型学到越细粒度的区分 → Embedding 质量越高
```

> 🔑 **RAG 关联**：如果你做领域 Embedding 微调，构造好的硬负样本是最关键的步骤。

## 4. 相似度度量

### 4.1 三种度量

| 度量 | 公式 | 值域 | 特点 |
|------|------|------|------|
| **余弦相似度** | cos(θ) = a·b / (\|\|a\|\|×\|\|b\|\|) | [-1, 1] | 只看方向，忽略长度 |
| **点积** | a·b = Σaᵢbᵢ | (-∞, +∞) | 考虑方向和长度 |
| **欧式距离** | \|\|a-b\|\| = √Σ(aᵢ-bᵢ)² | [0, +∞) | 长度越小越相似 |

### 4.2 为什么 RAG 用余弦？⭐ 面试常问

```
原因：
1. 归一化效果：长文本和短文本的 Embedding 长度不同
   - 点积会偏向长文本（向量长度大 → 点积大）
   - 余弦只看方向 → 公平比较
   
2. Embedding 模型通常训练时用余弦距离
   → 用余弦检索和训练目标一致

3. 如果向量已经归一化（单位向量），余弦 = 点积
   → 很多向量数据库存归一化向量 + 用点积（更快）
```

### 4.3 归一化后

```
如果 ||a|| = ||b|| = 1（单位向量）:
  cosine(a, b) = a·b              (余弦 = 点积)
  ||a-b||² = 2 - 2×(a·b)          (欧式距离 ∝ 1 - 余弦)

三者等价！实践中常对 Embedding 做 L2 归一化后用点积（最快）
```

## 5. Embedding 维度

### 5.1 常见维度

| 模型 | 维度 | 说明 |
|------|------|------|
| Word2Vec | 100-300 | 词向量 |
| BERT-base | 768 | 标准 |
| text-embedding-ada-002 | 1536 | OpenAI |
| text-embedding-3-small | 1536 | OpenAI v3 |
| text-embedding-3-large | 3072 | OpenAI v3 |
| BGE-large | 1024 | 开源 |
| E5-large | 1024 | 开源 |

### 5.2 维度越高越好吗？

```
不一定！

维度高的优点：
  - 表达能力更强（能编码更多信息）
  - 在困难任务上可能更好

维度高的缺点：
  - 存储和计算成本更高（向量数据库占用更多空间）
  - 可能过拟合（维度太高但训练数据不够）
  - 检索速度变慢

实践：
  768-1024 对大多数 RAG 场景够用
  Matryoshka Embedding：训练时让模型在多个维度都有效 → 可以按需截断
```

## 6. Embedding 微调

### 6.1 为什么需要微调？

```
通用 Embedding 模型：在通用数据上训练
你的领域：医疗/法律/金融 → 术语和语义可能不同

例："discharge"
  通用理解：释放、排出
  医疗领域：出院

微调让 Embedding 适应你的领域
```

### 6.2 微调方法

```
1. 准备训练数据：(query, positive_doc, hard_negative_doc) 三元组
2. 加载预训练模型（如 BGE-base）
3. 用对比学习继续训练
4. 评估：Recall@K, MRR 等检索指标

关键：硬负样本的质量决定微调效果
```

### 6.3 Matryoshka Representation Learning (MRL)

```
训练时同时优化多个维度的 Embedding 质量：
  1024 维 → 完整精度
  512 维  → 前 512 维也是好的 Embedding
  256 维  → 前 256 维也能用
  128 维  → 前 128 维也行（牺牲少量精度）

好处：部署时可以按需选维度 → 平衡精度和速度
OpenAI 的 text-embedding-3 系列支持此特性
```


## 8. 面试常问

### Q1: Embedding 模型是怎么训练的？

**答**：基于对比学习。用正样本对（如 query 和相关文档）和负样本对训练，目标是让正样本的 Embedding 余弦相似度高，负样本的低。核心损失函数是 InfoNCE。训练中，硬负样本挖掘是提升效果的关键。

### Q2: 为什么 RAG 用余弦相似度而不是点积？

**答**：余弦只比较方向不比较长度 → 不受文本长度影响。但如果向量已经 L2 归一化，余弦等价于点积。实际向量数据库中通常存归一化向量 + 用点积（更快且等价）。

### Q3: Embedding 维度怎么选？

**答**：768-1024 对大多数场景够了。维度越高表达力越强但存储和检索成本也越高。可以用 Matryoshka Embedding 根据场景灵活选维度。

### Q4: Bi-Encoder 和 Cross-Encoder 的区别？

**答**：
- **Bi-Encoder**：句子独立编码 → 快（毫秒级检索）→ 用于初筛
- **Cross-Encoder**：两个句子拼在一起编码 → 准（交叉注意力）→ 用于重排
- RAG 标准流程：Bi-Encoder 初筛 top-100 → Cross-Encoder 重排 top-10
