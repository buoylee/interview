## 1. 分类评估指标

### 1.1 混淆矩阵 (Confusion Matrix) ⭐

```
                    预测
                正 (Positive)    负 (Negative)
实际  正 (P)      TP               FN
      负 (N)      FP               TN

TP = True Positive  → 预测正，实际正 ✅（真阳性）
FP = False Positive → 预测正，实际负 ❌（假阳性，误报）
FN = False Negative → 预测负，实际正 ❌（假阴性，漏报）
TN = True Negative  → 预测负，实际负 ✅（真阴性）
```

### 1.2 Accuracy（准确率）

```
Accuracy = (TP + TN) / (TP + TN + FP + FN) = 对的 / 全部
```

**问题**：类别不平衡时 Accuracy 没意义！

```
例：99% 是负样本，模型全猜「负」→ Accuracy = 99%，但完全没用
```

### 1.3 Precision（精确率）⭐

```
Precision = TP / (TP + FP) = 预测为正的里面真正是正的比例
```

**直觉**：「模型说的阳性可信吗？」

- 高 Precision → 很少误报
- 重要场景：垃圾邮件过滤（不能误把正常邮件当垃圾）

### 1.4 Recall（召回率）⭐

```
Recall = TP / (TP + FN) = 实际为正的里面被找出来了多少
```

**直觉**：「阳性样本有漏网之鱼吗？」

- 高 Recall → 很少遗漏
- 重要场景：癌症检测（不能漏诊）

### 1.5 Precision vs Recall 权衡 ⭐⭐ 面试高频

```
Precision ↑ → Recall ↓（更严格，宁可漏也不要错）
Recall ↑    → Precision ↓（更宽松，宁可错也不要漏）
```

| 场景 | 优先 | 原因 |
|------|------|------|
| 癌症检测 | Recall | 宁可误诊也不能漏诊 |
| 垃圾邮件 | Precision | 宁可收到垃圾也不能删正常邮件 |
| 信息检索(RAG) | Recall@K | 确保有用的文档被检索到 |
| 内容审核 | 取决于策略 | 太严影响体验，太松有风险 |

### 1.6 F1 Score

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
   = Precision 和 Recall 的调和平均
```

**直觉**：平衡 Precision 和 Recall 的综合分数。

- F1 高 → Precision 和 Recall 都不差
- 只有两者都高时 F1 才高（一个为 0 则 F1 为 0）

### 1.7 AUC-ROC

```
ROC 曲线：不同阈值下的 TPR(Recall) vs FPR(1-Specificity) 曲线
AUC = ROC 曲线下面积

AUC = 0.5 → 随机猜（对角线）
AUC = 1.0 → 完美分类
AUC > 0.9 → 通常认为很好
```

**优点**：不受阈值选择和类别不平衡影响。

## 2. 回归评估指标

| 指标 | 公式 | 直觉 |
|------|------|------|
| **MSE** | Σ(y-ŷ)²/n | 平均「误差的平方」，对大误差惩罚更重 |
| **RMSE** | √MSE | MSE 开根号，和原数据同量纲 |
| **MAE** | Σ\|y-ŷ\|/n | 平均「绝对误差」，对异常值更鲁棒 |
| **R²** | 1 - SS_res/SS_tot | 模型解释了数据多少方差。R²=1 完美，R²=0 和猜均值一样 |

## 3. 正则化

### 3.1 为什么需要正则化？

```
损失函数 = 数据拟合损失 + λ × 正则化项

数据拟合：让模型在训练数据上好 → 偏差 ↓
正则化：  限制模型复杂度        → 方差 ↓（防止过拟合）
λ：       控制正则化强度        → λ 大 → 更正则化
```

### 3.2 L1 正则化（Lasso）

```
L1 = λ × Σ|wᵢ|
```

**效果**：倾向于让某些权重变为**精确的 0** → **稀疏性！**

```
特征选择：L1 会自动把不重要的特征的权重变成 0
适用场景：特征很多但只有少数真正有用
```

### 3.3 L2 正则化（Ridge）⭐

```
L2 = λ × Σwᵢ²
```

**效果**：让所有权重都**变小但不为 0** → 权重更均匀、更平滑。

```
概率解释：L2 正则化 = 假设权重服从高斯先验 N(0, 1/λ)（MAP 估计）
```

> 🔑 **ML 关联**：
> - **Weight Decay = L2 正则化**（几乎等价，AdamW 中做了修正）
> - LLM 训练中 AdamW 的 Weight Decay 就是 L2 正则化

### 3.4 L1 vs L2 对比

| | L1 (Lasso) | L2 (Ridge) |
|--|-----------|-----------|
| 效果 | 稀疏（部分权重=0） | 平滑（全部权重变小） |
| 特征选择 | ✅ 自动 | ❌ |
| 概率解释 | 拉普拉斯先验 | 高斯先验 |
| 常用于 | 特征选择 | 防过拟合（更常用）|

### 3.5 ElasticNet

```
ElasticNet = α × L1 + (1-α) × L2

结合两者优点
```

### 3.6 Dropout ⭐

```
训练时：随机把一些神经元的输出设为 0（概率 p）
推理时：不 Dropout，但输出乘以 (1-p) 或训练时除以 (1-p)
```

**直觉**：
- 防止神经元之间过度依赖（「不能总指望同事帮忙」）
- 每次训练相当于用不同的子网络 → 类似集成学习
- 迫使每个神经元学到更鲁棒的特征

> 🔑 **LLM 关联**：
> - 预训练 LLM 通常**不用 Dropout**（数据量够大，不容易过拟合）
> - 微调时有时加 Dropout（数据少）
> - LoRA 中也可以对 A、B 矩阵加 Dropout

## 4. 交叉验证

### 4.1 K-Fold 交叉验证

```
数据分成 K 份（通常 K=5 或 10）

轮次 1: [验证] [训练] [训练] [训练] [训练]
轮次 2: [训练] [验证] [训练] [训练] [训练]
轮次 3: [训练] [训练] [验证] [训练] [训练]
轮次 4: [训练] [训练] [训练] [验证] [训练]
轮次 5: [训练] [训练] [训练] [训练] [验证]

最终得分 = 5 次验证分数的平均
```

### 4.2 为什么用交叉验证？

- 数据少时，单次划分的评估不稳定（运气好分到简单的验证集）
- 交叉验证利用了所有数据，评估更可靠
- 可以估计模型性能的方差（稳定性）

### 4.3 其他变体

| 方法 | 说明 | 适用 |
|------|------|------|
| Leave-One-Out | K = 数据量 | 数据极少 |
| Stratified K-Fold | 保持每折的类别比例 | 类别不平衡 |
| Time-Series Split | 按时间顺序划分 | 时序数据 |

### 4.4 LLM 时代：交叉验证不常用

```
传统 ML：数据少 → 交叉验证很重要
LLM：    数据多 → 简单的 train/val/test 划分就够了
         但 benchmark 评估很重要（MMLU、HumanEval 等）
```

## 5. 超参数调优

### 5.1 超参数 vs 参数

```
参数 (Parameters)：模型自己学到的（权重 W、偏置 b）
超参数 (Hyperparameters)：人手动设置的（学习率、层数、隐藏维度）
```

### 5.2 调优方法

| 方法 | 思想 | 效率 |
|------|------|------|
| **Grid Search** | 穿举所有组合 | 极慢，维度高不可用 |
| **Random Search** | 随机组合 | 比 Grid 好！因为不同超参重要性不同 |
| **Bayesian Optimization** | 建模目标函数，智能选择下一组 | ⭐ 效率最高 |
| **学习率查找** | 指数增加，看哪里损失下降最快 | 针对学习率 |

### 5.3 常见超参数

#### 传统 ML
| 算法 | 关键超参数 |
|------|-----------|
| 随机森林 | 树的数量、最大深度 |
| XGBoost | 学习率、树数量、最大深度、正则化 |
| SVM | C（惩罚强度）、核函数、γ |

#### 深度学习 / LLM
| 超参数 | 典型值 | 影响 |
|--------|--------|------|
| 学习率 | 1e-5 ~ 1e-3 | 最重要的超参数！ |
| Batch Size | 32 ~ 4096 | 显存 vs 训练稳定性 |
| 权重衰减 | 0.01 ~ 0.1 | 正则化强度 |
| Warmup 步数 | 总步数的 1-10% | 训练初期稳定性 |
| Dropout | 0 ~ 0.3 | 正则化 |

## 7. 面试常问

### Q1: Precision 和 Recall 怎么取舍？

**答**：取决于业务成本：
- 误报成本高（如垃圾邮件误报正常邮件）→ 要高 Precision
- 漏报成本高（如癌症漏诊）→ 要高 Recall
- 可以通过调整分类阈值来权衡
- F1 Score 是两者的折中

### Q2: AUC-ROC 是什么？为什么比 Accuracy 好？

**答**：
- AUC = ROC 曲线下面积，衡量模型区分正负样本的能力
- **不受阈值影响**：Accuracy 只看一个阈值，AUC 看所有阈值
- **不受类别不平衡影响**：99% 负样本时 Accuracy=99% 没意义，但 AUC 能反映真实能力
- AUC = 0.5 = 随机，AUC = 1.0 = 完美

### Q3: L1 和 L2 正则化的区别？

**答**：
- **L1**：让权重稀疏（部分为 0）→ 自动特征选择
- **L2**：让权重均匀变小 → 平滑、防过拟合
- L1 用拉普拉斯先验，L2 用高斯先验（MAP 解释）
- LLM 训练中通常用 L2（Weight Decay）

### Q4: 为什么 Random Search 比 Grid Search 好？

**答**：
- 不同超参数的重要性不同（学习率可能比 batch size 重要 10 倍）
- Grid Search 在不重要的维度上浪费了大量尝试
- Random Search 在每个维度上都能探索更多不同的值
- 同样的计算预算下，Random Search 更可能找到好的超参数组合

### Q5: 如何评估一个 RAG 的检索效果？

**答**：
| 指标 | 含义 |
|------|------|
| **Recall@K** | 前 K 条检索结果中包含正确答案的比例 |
| **MRR** (Mean Reciprocal Rank) | 正确答案第一次出现的位置的倒数 |
| **NDCG** | 考虑排序质量的综合指标 |
| Hit Rate | 前 K 条中有没有至少一个正确答案 |

→ 知道这些指标是什么就好，RAG 检索评估和模型评估安全主题会继续展开。
