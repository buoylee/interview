## 1. 线性回归 (Linear Regression)

### 1.1 核心思想

```
y = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
  = Wx + b

目标：找到最优的 W 和 b，让预测尽可能接近真实值
```

**直觉**：在数据点之间拟合一条最合适的线（高维就是超平面）。

### 1.2 训练过程

```
1. 初始化随机权重 W, b
2. 前向计算: ŷ = Wx + b
3. 计算损失: L = MSE = Σ(y - ŷ)² / n
4. 反向传播: ∂L/∂W, ∂L/∂b
5. 更新参数: W = W - α × ∂L/∂W
6. 重复 2-5 直到收敛
```

> 🔑 **ML 关联**：这就是梯度下降的第一个完整例子！从线性回归到 GPT，训练流程本质相同，只是模型复杂度不同。

### 1.3 闭式解 vs 梯度下降

```
闭式解 (Normal Equation): W = (XᵀX)⁻¹Xᵀy
- 优点：一步到位
- 缺点：矩阵求逆 O(n³)，数据大时不可用

梯度下降：
- 优点：大规模数据都能用
- 缺点：需要迭代
```

## 2. 逻辑回归 (Logistic Regression)

### 2.1 核心思想

```
线性回归 + sigmoid → 分类

z = Wx + b           (线性部分)
ŷ = σ(z) = 1/(1+e⁻ᶻ)  (sigmoid 把实数压缩到 [0, 1])
```

**直觉**：先做线性变换，再用 sigmoid 把结果变成「概率」。

### 2.2 sigmoid 函数 ⭐

```
σ(z) = 1 / (1 + e⁻ᶻ)

当 z >> 0 → σ ≈ 1
当 z = 0  → σ = 0.5
当 z << 0 → σ ≈ 0
```

**性质**：
- 输出在 (0, 1) 之间，天然可以解释为概率
- 导数 σ'(z) = σ(z)(1-σ(z))，最大值只有 0.25 → **梯度消失的根源！**

### 2.3 决策边界

```
σ(Wx + b) = 0.5 → Wx + b = 0 → 这就是决策边界（一个超平面）
```

> 🔑 **ML 关联**：
> - 逻辑回归 = 最简单的「神经网络」（单个神经元）
> - sigmoid 的梯度消失问题就是后来 ReLU 被发明的原因
> - 多分类时 sigmoid → softmax

## 3. 决策树 (Decision Tree)

### 3.1 核心思想

```
像玩「20 个问题」猜谜游戏：
  天气是晴天吗？ → 是 → 温度 > 30°？ → 是 → 不出门
                                       → 否 → 出门
                 → 否 → 会下雨吗？ → ...
```

**直觉**：通过一系列 if-else 规则将数据分类。

### 3.2 怎么选分裂特征？

| 方法 | 用在 | 原理 |
|------|------|------|
| **信息增益** (Information Gain) | ID3 | 选能最大减少熵的特征 |
| **信息增益比** | C4.5 | 信息增益 / 特征自身的熵 |
| **基尼系数** (Gini Index) | CART | 选能最大减少基尼不纯度的特征 |

```
基尼不纯度 = 1 - Σ pᵢ²

例：一个节点有 50% A 类和 50% B 类 → Gini = 1 - 0.5² - 0.5² = 0.5 (最不纯)
    一个节点全是 A 类 → Gini = 1 - 1² = 0 (最纯，完美分裂) ✅
```

### 3.3 优缺点

| 优点 | 缺点 |
|------|------|
| 可解释性强 | 容易过拟合（特别是深树） |
| 不需要归一化 | 不稳定（数据小变化，树大变化） |
| 能处理非线性 | 缺乏平滑性 |

> 🔑 **ML 关联**：决策树的信息增益直接用了信息论的熵！(→ 回顾 [04-信息论](../00-math-foundations/04-information-theory.md))

## 4. 集成学习 (Ensemble Learning)

### 4.1 核心思想

**一个模型不够好 → 用一群模型投票/取平均**

### 4.2 Bagging vs Boosting ⭐ 面试常考

| | Bagging | Boosting |
|--|---------|----------|
| 思想 | 多个模型**独立**训练 + 投票 | 模型**串行**训练，每个纠正前一个的错误 |
| 减少 | **方差**（让不稳定模型变稳定）| **偏差**（让弱模型变强）|
| 代表 | **随机森林** | **XGBoost / LightGBM** |
| 类比 | 问 100 个人投票（三个臭皮匠）| 学生做错的题老师重点讲 |
| 训练速度 | 可并行 | 必须串行 |
| 过拟合 | 不容易 | 可能（需要控制轮数）|

### 4.3 随机森林 (Random Forest)

```
1. 随机采样数据（bootstrap，有放回抽样）→ 训练 N 棵决策树
2. 每棵树分裂时只看随机选的部分特征
3. 预测时所有树投票（分类）或取平均（回归）
```

**为什么有效**：每棵树都过拟合了，但方式不同 → 平均后过拟合被抵消。

### 4.4 XGBoost / LightGBM (Gradient Boosting)

```
1. 训练第一棵树
2. 计算预测误差（残差）
3. 第二棵树专门学习这些残差
4. 第三棵树学习 1+2 的残差
5. ...
最终预测 = 所有树的预测之和
```

| 特点 | XGBoost | LightGBM |
|------|---------|----------|
| 分裂方式 | 逐层 (level-wise) | 逐叶 (leaf-wise) ← 更快 |
| 速度 | 快 | 更快 |
| 适用 | 表格数据竞赛标配 | 工业界也广泛使用 |

> 🔑 **面试关联**：「表格数据(Tabular)上，XGBoost/LightGBM 至今仍然比深度学习好」→ 这是一个经典面试话题。

## 5. SVM 支持向量机

### 5.1 核心思想

```
找到一个超平面，使得两类数据之间的「间隔」(margin) 最大

         |        |
  ○ ○ ○  | margin |  × × ×
  ○ ○    |←──────→|    × ×
         |        |
       支持向量  支持向量
```

**直觉**：不只是找到能分开两类的线，而是找「最宽的分隔带」→ 泛化能力更强。

### 5.2 核函数 (Kernel Trick) ⭐

当数据在原始空间不可线性分开时：

```
原始 2D 空间：无法画直线分开
      ↓ 核函数映射
高维空间：可以用超平面分开

常用核：
- 线性核: K(x,y) = x·y
- RBF 核: K(x,y) = exp(-γ||x-y||²)  ← 最常用
- 多项式核: K(x,y) = (x·y + c)^d
```

**核函数的妙处**：不需要真的计算高维映射，只需要计算核函数值 → 效率高。

### 5.3 现在还用 SVM 吗？

- 小数据 + 高维特征：SVM 仍然有竞争力
- 大数据：深度学习更好
- **核函数的思想**影响深远：隐式特征变换

## 6. 无监督学习算法

### 6.1 K-Means 聚类

```
1. 随机选 K 个中心点
2. 每个数据点分配到最近的中心
3. 重新计算每组的中心
4. 重复 2-3 直到稳定

K 怎么选？ → 肘部法则 (Elbow Method)
```

### 6.2 PCA 主成分分析

```
高维数据 → 找到方差最大的方向 → 投影到这些方向 → 降维

数学原理：对协方差矩阵做特征分解，取最大特征值对应的特征向量
```

> 🔑 **ML 关联**：PCA 用到了线性代数的特征值/特征向量 (→ 回顾 [01-线性代数](../00-math-foundations/01-linear-algebra.md))

### 6.3 t-SNE / UMAP 高维可视化

```
PCA：线性降维 → 适合保留全局结构
t-SNE：非线性降维 → 更好保留局部结构（邻居关系）
UMAP：更快的非线性降维

常用场景：把 768 维的 Embedding 降到 2D 可视化
```

> 🔑 **实用关联**：
> - 检查 Embedding 模型质量：同类文档的 Embedding 应该在可视化中聚在一起
> - 发现数据中的簇结构

## 7. 算法选择指南

```
                    ├── 有标签？
                    │   ├── 是 → 监督学习
                    │   │   ├── 输出连续值？→ 回归
                    │   │   │   ├── 特征少/线性关系 → 线性回归
                    │   │   │   └── 特征多/非线性  → XGBoost / 神经网络
                    │   │   └── 输出类别？→ 分类
                    │   │       ├── 需要解释性 → 决策树 / 逻辑回归
                    │   │       ├── 表格数据   → XGBoost / LightGBM
                    │   │       ├── 文本/图像   → 深度学习
                    │   │       └── 小数据高维  → SVM
                    │   └── 否 → 无监督学习
                    │       ├── 分组   → K-Means
                    │       ├── 降维   → PCA
                    │       └── 可视化 → t-SNE / UMAP
```

## 8. 面试常问

### Q1: Bagging 和 Boosting 的区别？

**答**：
- **Bagging**：多个模型独立并行训练 + 投票，降低**方差**（如随机森林）
- **Boosting**：模型串行训练，每个纠正前一个的错误，降低**偏差**（如 XGBoost）
- Bagging 不容易过拟合；Boosting 可能过拟合需要早停

### Q2: XGBoost 为什么比随机森林好？

**答**：
- 随机森林减方差，XGBoost 减偏差 → XGBoost 通常精度更高
- XGBoost 有正则化防止过拟合
- XGBoost 有二阶梯度信息（Newton 方法）→ 收敛更快
- 但随机森林更鲁棒、更不容易过拟合

### Q3: 逻辑回归和 SVM 的区别？

**答**：
| | 逻辑回归 | SVM |
|--|----------|-----|
| 目标 | 最大化似然 | 最大化间隔 |
| 输出 | 概率 | 分类标签 |
| 对异常值 | 较敏感 | 只关注支持向量 |
| 核函数 | 不直接支持 | 天然支持 |
| 多分类 | 天然支持 (softmax) | 需要 OvO/OvR |

### Q4: 为什么表格数据上树模型比深度学习好？

**答**：
- 表格特征之间通常没有空间/时间上的局部结构（不像图像/文本）
- 树模型天然适合处理特征之间的非线性交互
- 深度学习需要大量数据才能发挥优势，表格数据通常没那么多
- 树模型不需要归一化、对缺失值更鲁棒
- 但如果表格中有文本/图像列，多模态方案可能更好

### Q5: K-Means 的局限性？

**答**：
- 需要预先指定 K（不知道应该分几类）
- 对初始中心点敏感（K-Means++ 改进）
- 只能发现球形的簇（不规则形状不行 → DBSCAN 更好）
- 对异常值敏感
