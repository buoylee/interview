## 1. 学习范式

### 1.1 四大范式

| 范式 | 有标签？ | 一句话理解 | 代表 | 面试关联 |
|------|---------|-----------|------|---------|
| **监督学习** (Supervised) | ✅ 全部有 | 给答案让模型学 | 分类、回归 | 传统 ML 主流 |
| **无监督学习** (Unsupervised) | ❌ 没有 | 自己发现数据规律 | 聚类、降维 | K-Means、PCA |
| **半监督学习** (Semi-supervised) | 🔶 部分有 | 少量标签 + 大量无标签 | 伪标签 | 数据标注成本高时 |
| **自监督学习** (Self-supervised) | 🔄 自造标签 | 从数据本身构造任务 | BERT(MLM), GPT(CLM) | ⭐ **大模型预训练核心！** |

### 1.2 自监督学习详解 ⭐

为什么自监督学习是大模型的基础？

```
传统监督学习：需要人工标注 → 贵、慢、规模有限
自监督学习：  从数据中自动构造标签 → 便宜、快、规模无限

BERT 的自监督方式（MLM）：
  输入：「今天天气[MASK]好」
  标签：「真」               ← 标签来自数据本身！

GPT 的自监督方式（CLM）：
  输入：「今天天气」
  标签：「真」               ← 下一个词就是标签！
```

> 🔑 这就是为什么 LLM 可以在 TB 级数据上训练——不需要人标注，数据本身就是标签。

### 1.3 其他学习方式

| 方式 | 说明 | 关联 |
|------|------|------|
| 强化学习 (RL) | 试错 + 奖励信号 | **RLHF 的基础** |
| 迁移学习 (Transfer Learning) | 在 A 任务学的知识迁移到 B | **预训练 + 微调范式** |
| 元学习 (Meta Learning) | 学会学习 | Few-shot learning 理论 |
| 主动学习 (Active Learning) | 模型选择最该标注的样本 | 降低标注成本 |

## 2. 数据划分

### 2.1 三种集合

```
全部数据
├── 训练集 (Training Set)   60-80%  → 模型在这上面学习
├── 验证集 (Validation Set) 10-20%  → 调超参数、选模型
└── 测试集 (Test Set)       10-20%  → 最终评估，只看一次！
```

### 2.2 类比

```
训练集 = 平时做练习题     (学习)
验证集 = 模拟考           (调整策略)
测试集 = 高考             (最终评价，不能提前看！)
```

### 2.3 为什么要分？

| 不分会怎样？ | 问题 |
|------------|------|
| 不分验证集 | 不知道超参数好不好，容易过拟合 |
| 不分测试集 | 模型在「见过的数据」上表现好不代表在新数据上好 |
| 测试集提前看 | **数据泄漏** (data leakage)，评估结果不可信 |

### 2.4 LLM 时代的数据划分

```
传统 ML：数据少，三七分很常见
LLM 时代：
  - 预训练数据：TB 级，划分比例不同
  - 数据去重（deduplication）更重要 → 训练数据和测试 benchmark 不能重叠
  - 数据污染（contamination）：如果 benchmark 题目出现在训练数据中，评测结果虚高
```

## 3. 过拟合 vs 欠拟合

### 3.1 核心概念

| | 欠拟合 (Underfitting) | 过拟合 (Overfitting) |
|--|------|------|
| 类比 | 没学会，考试都不及格 | 死记硬背，做过的题全对但新题全错 |
| 训练集表现 | ❌ 差 | ✅ 好 |
| 测试集表现 | ❌ 差 | ❌ 差 |
| 原因 | 模型太简单/训练不够 | 模型太复杂/数据太少 |

### 3.2 判断方法

```
训练误差高,  测试误差高  → 欠拟合
训练误差低,  测试误差高  → 过拟合 ⭐ 最常见问题
训练误差低,  测试误差低  → 刚刚好 ✅
```

### 3.3 解决方案

#### 欠拟合的解决

| 方法 | 说明 |
|------|------|
| 更复杂的模型 | 线性模型 → 非线性模型 → 深度网络 |
| 更多特征 | 增加输入信息 |
| 减少正则化 | 给模型更多自由度 |
| 训练更久 | 还没收敛 |

#### 过拟合的解决 ⭐ 面试高频

| 方法 | 原理 | ML/LLM 应用 |
|------|------|------------|
| **更多数据** | 见更多样的例子 | 数据增强、合成数据 |
| **正则化** | 限制模型复杂度 | L1/L2/Dropout/Weight Decay |
| **早停 (Early Stopping)** | 验证集误差开始上升就停 | 训练时的通用技巧 |
| **Dropout** | 随机关掉一些神经元 | DL 中广泛使用 |
| **数据增强** | 对数据做变换增加多样性 | 图像翻转旋转、文本同义替换 |
| **交叉验证** | 多次划分数据评估 | 小数据集时 |
| **集成学习** | 多个模型取平均 | 降低方差 |

> 🔑 **LLM 关联**：
> - LLM 的预训练数据量极大，一般**不太会过拟合**（欠拟合更常见）
> - 但**微调 (Fine-tuning)** 时数据少，非常容易过拟合 → 这就是为什么需要 LoRA（参数更少，降低过拟合风险）

## 4. 偏差-方差权衡

### 4.1 定义

```
模型总误差 = 偏差² + 方差 + 不可约误差(噪声)

偏差 (Bias)：模型的预测和真实值的系统性偏离 → 准不准
方差 (Variance)：模型对不同训练数据的敏感度 → 稳不稳
```

### 4.2 直觉：打靶类比

```
          低方差          高方差
低偏差   [集中在靶心]     [分散但围绕靶心]   ← 理想 vs 过拟合
高偏差   [集中但偏了]     [又偏又散]          ← 欠拟合 vs 最差
```

### 4.3 与过拟合/欠拟合的关系

| | 偏差 | 方差 | 对应 |
|--|------|------|------|
| 简单模型 | 高 | 低 | 欠拟合 |
| 复杂模型 | 低 | 高 | 过拟合 |
| **合适的模型** | **适中** | **适中** | **刚好** |

### 4.4 不同模型的偏差-方差

```
线性回归       → 高偏差、低方差
决策树(深)     → 低偏差、高方差
随机森林       → 低偏差、低方差（集成降方差）✅
深度神经网络   → 低偏差、可能高方差 → 正则化来控制
```

> 🔑 **LLM 关联**：
> - 大模型参数极多 → 偏差很低（能拟合复杂模式）
> - 但方差不一定高（大量数据 + 正则化效果）
> - **Scaling Law** 研究的本质：增大模型/数据如何影响偏差-方差权衡

## 5. 损失函数

### 5.1 什么是损失函数？

**损失函数 = 衡量模型预测和真实值差了多少的函数**

```
L(y_true, y_pred) → 一个数字，越小越好
```

### 5.2 常见损失函数

#### 回归任务

| 损失函数 | 公式 | 特点 |
|---------|------|------|
| MSE (均方误差) | Σ(y - ŷ)² / n | 对大误差惩罚更重 |
| MAE (平均绝对误差) | Σ\|y - ŷ\| / n | 对异常值更鲁棒 |
| Huber Loss | 小误差用 MSE，大误差用 MAE | 两者折中 |

#### 分类任务

| 损失函数 | 用途 | 面试关联 |
|---------|------|---------|
| **交叉熵 (Cross-Entropy)** | 多分类 | ⭐ LLM 预训练目标 |
| 二元交叉熵 (BCE) | 二分类 | 逻辑回归 |
| Hinge Loss | SVM | 最大间隔分类 |
| Focal Loss | 类别不平衡 | 目标检测 |

> 🔑 **关键理解**：
> - 最小化交叉熵 = 最大化似然 (MLE) = 让模型给正确答案更高概率
> - LLM 预训练就是最小化 next token 的交叉熵

### 5.3 损失函数的选择

```
回归问题 → MSE / MAE
分类问题 → 交叉熵
排序问题 → Pairwise Loss（后续 DPO 会用到）
生成问题 → 交叉熵 + KL 约束（RLHF）
```

## 6. 梯度下降

### 6.1 核心思想

```
目标：找到让损失函数最小的参数
方法：沿梯度反方向（下坡方向）一步步走

θ_new = θ_old - α × ∇L(θ)

α = 学习率（步长大小）
∇L = 损失对参数的梯度（坡的方向和陡度）
```

### 6.2 三种变体

| 变种 | 每次用多少数据 | 优缺点 |
|------|-------------|--------|
| **BGD** (Batch GD) | 全部数据 | 准确但慢，内存消耗大 |
| **SGD** (Stochastic GD) | 1 条数据 | 快但噪声大，震荡 |
| **Mini-batch SGD** | 一批(32/64/128) | ⭐ **实际最常用**，平衡速度和稳定性 |

### 6.3 优化器演进

```
SGD → SGD+Momentum → RMSProp → Adam → AdamW
```

| 优化器 | 改进 | 直觉 |
|--------|------|------|
| SGD | 基础版 | 闭眼下山 |
| **Momentum** | 加上「惯性」 | 下山时滚雪球，越滚越快 |
| **RMSProp** | 自适应学习率 | 陡的方向小步走，平的方向大步走 |
| **Adam** | Momentum + RMSProp | 综合两者优点 |
| **AdamW** | Adam + 正确的权重衰减 | ⭐ **LLM 训练标配** |

### 6.4 学习率调度 (Learning Rate Schedule)

```
固定学习率 → 容易不收敛或太慢

常用策略：
1. Warmup：开始时学习率从 0 慢慢增大（避免初期震荡）
2. Cosine Decay：按余弦函数逐渐减小（平滑衰减）
3. Linear Decay：线性减小
4. Step Decay：每 N 步减半

LLM 标配：Warmup + Cosine Decay
```

> 🔑 **为什么需要 Warmup？**
> - 训练初期参数随机，梯度不稳定
> - 大学习率会让参数跑偏
> - 先小步探索，再大步学习，最后小步精调

## 7. 特征工程

### 7.1 什么是特征工程？

```
原始数据 → 提取/构造有用信息 → 模型能理解的数字特征
```

### 7.2 传统 ML 时代

| 方法 | 说明 |
|------|------|
| 数值归一化 | Min-Max / Z-Score 标准化 |
| 类别编码 | One-Hot / Label Encoding |
| 特征交叉 | 组合多个特征 |
| 特征选择 | 去掉无关/冗余特征 |
| 文本特征 | TF-IDF / BoW |

### 7.3 深度学习时代的变化

```
传统 ML：特征工程是最重要的环节，需要大量领域知识
深度学习：模型自动学特征（表示学习 / Representation Learning）

Word2Vec / BERT / GPT 本质上都在做自动特征提取：
  原始文本 → [模型] → 有意义的向量表示 (Embedding)
```

> 🔑 **但特征工程没有消失！**
> - LLM 时代的「特征工程」变成了 **Prompt Engineering**
> - 如何组织输入（上下文、示例、指令）→ 本质上还是在构造更好的「特征」

## 8. 面试常问

### Q1: 监督学习和自监督学习的区别？为什么大模型用自监督？

**答**：
- **监督学习**：需要人工标注的标签（贵、慢、规模有限）
- **自监督学习**：从数据本身构造训练信号（免费、快、可大规模）
- GPT 用下一个词作为标签，BERT 用被遮住的词作为标签
- 这使得在互联网规模的数据上训练成为可能

### Q2: 过拟合怎么判断？怎么解决？

**答**：
- **判断**：训练损失低但验证损失高（gap 大）
- **解决**（按优先级）：
  1. 加更多数据（最有效但最贵）
  2. 正则化（L2/Dropout）
  3. 早停（Early Stopping）
  4. 减小模型复杂度
  5. 数据增强

### Q3: Adam 优化器为什么好？为什么 LLM 用 AdamW？

**答**：
- **Adam** 结合了 Momentum（惯性加速）和 RMSProp（自适应学习率），对大多数问题都表现好
- **AdamW** 修复了 Adam 的权重衰减实现错误
  - Adam 中的 L2 正则化与自适应学习率交互有问题
  - AdamW 将权重衰减从梯度更新中解耦
  - LLM 训练必须用 AdamW

### Q4: 什么是梯度下降的局部最小值问题？深度学习中严重吗？

**答**：
- 非凸函数可能有多个局部最小值
- 但在高维空间（深度学习中参数上亿），局部最小值很少见
- 更常见的是**鞍点**（一个方向是最小值，另一个方向是最大值）
- 实践中，SGD 的噪声反而帮助逃离鞍点
- 大多数局部最小值的损失值和全局最小值很接近

### Q5: 偏差和方差分别対应什么？怎么一起降低？

**答**：
- 偏差高 = 欠拟合（模型太简单）→ 增加模型复杂度
- 方差高 = 过拟合（模型太敏感）→ 正则化 / 更多数据
- **同时降低**：集成学习（如随机森林）、增大模型 + 增大数据（Scaling Law 的思路）
