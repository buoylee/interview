## 1. 层级地图

顶层是**人工智能（AI）**，机器学习是它的子集：

```
人工智能 (AI)                         ← 顶层目标："让机器表现出智能"
 ├─ 符号主义 / 专家系统                ← 手写规则和逻辑推理（80年代主流，已式微）
 ├─ 搜索与规划                        ← 比如下棋的博弈树搜索
 └─ 机器学习 (ML)                     ← "从数据学规律"这条路线（当今主流）
     ├─ 经典机器学习                  ← 和深度学习同层
     │   ├─ 线性回归 / 逻辑回归
     │   ├─ 决策树 / 随机森林 / XGBoost
     │   ├─ SVM
     │   └─ kNN、朴素贝叶斯...
     └─ 深度学习 (DL)                 ← 模型是多层神经网络
         └─ Transformer               ← 深度学习里的一种网络架构
             └─ LLM                   ← 把 Transformer 放大到极致、喂海量文本
```

一句话串起来：**LLM 是用深度学习这套方法论、基于 Transformer 架构、在海量文本上训练出来的超大号神经网络。**

后端类比：

| 概念 | 类比 |
|------|------|
| 深度学习 | 框架 + 运行时（规定通用机制：分层、前向传播、反向传播、梯度下降） |
| Transformer | 框架下的一种架构模式（如同微服务是一种架构模式；CNN/RNN 是别的模式） |
| LLM | 用这套架构部署出来的具体应用（GPT、Claude、LLaMA） |

> 🔑 AI 比 ML 大，是因为历史上「让机器智能」不止「从数据学」一条路。早期主流是专家系统——本质是手写几万条 if-else 规则，败因工程师秒懂：规则维护不动、边界情况穷举不完。今天口语里的 "AI" 几乎默认指 ML/DL，但严格说 AI 是最外圈的目标。

## 2. 和深度学习同一层的是什么

答案：**经典机器学习**。划分标准是「**模型本体是什么**」：

| | 经典 ML | 深度学习 |
|---|---|---|
| 模型本体 | 浅的数学模型（一棵树、一条分割线、一个概率公式） | 多层神经网络 |
| 特征 | 人工设计（特征工程） | 模型自己学（表示学习） |
| 强项数据 | 表格类数据（风控、推荐排序） | 原始信号（图像、语音、文本） |
| 资源 | CPU 可跑、训练快、可解释 | 靠 GPU 算力堆 |

经典 ML 至今没死：表格类数据上 XGBoost 经常打赢神经网络，且训练快、可解释、不需要 GPU。深度学习碾压的是图像/语音/文本这种「特征人工根本设计不出来」的原始信号。

反过来，深度学习也不等于 LLM：图像识别、AlphaGo、AlphaFold、自动驾驶感知都是深度学习，但都不是 LLM。

## 3. 常见混淆一：监督/无监督不是层级里的一层

「监督 / 无监督 / 强化学习」是**另一个正交的维度**——划分标准是「**训练信号从哪来**」，不是「用什么模型」：

|  | 经典 ML | 深度学习 |
|---|---|---|
| 监督学习 | SVM 做分类 | ResNet 做图像分类 |
| 无监督/自监督 | k-means 聚类 | LLM 预训练 |
| 强化学习 | Q-learning 查表 | AlphaGo、RLHF |

所以「深度学习」和「监督学习」不冲突，一个模型可以同时是两者。很多教材把两个维度画进同一棵树，就把人讲糊涂了。

## 4. 常见混淆二：深度学习「包含」经典 ML 吗

直觉陷阱：深度学习里也有损失函数、梯度下降、过拟合、训练/测试划分……这些不都是经典 ML 的内容吗？那 DL 不就包含了经典 ML？

**不对。这些共享概念不属于经典 ML，它们属于上一层——机器学习本身。**

```
机器学习 (ML)
 │  ← 公共框架放在这一层：
 │     数据 → 模型 → 损失函数 → 优化 → 评估
 │     监督/无监督范式、过拟合、泛化……
 │
 ├─ 经典 ML 独有：决策树的分裂规则、SVM 的核函数、kNN 的距离投票
 └─ 深度学习独有：多层网络、反向传播、attention、卷积
```

用继承来说就清楚了：

```java
abstract class MachineLearning {
    // 公共契约：所有子类共享
    abstract void fit(data);      // 用损失函数 + 优化器从数据学参数
    abstract Prediction predict(input);
}

class ClassicalML extends MachineLearning { /* 树分裂、核函数…… */ }
class DeepLearning extends MachineLearning { /* 多层网络、反向传播…… */ }
```

`DeepLearning` 和 `ClassicalML` 共享大量词汇，是因为**共同的父类**，不是因为一个包含另一个。深度学习里没有树分裂逻辑、没有核技巧——就像 Postgres 和 MySQL 都支持 SQL，但 Postgres 不包含 MySQL。

> 🔑 **边界本来就是模糊的**：逻辑回归既被归为经典 ML，又恰好就是「单层神经网络」——叠多层、加非线性激活就成了深度学习。这棵分类树是为教学方便画的，真实含义是两个方法族的风格差异，不是数学上严格不重叠的划分。

## 5. 人工参与程度的梯子

区分「经典 ML vs 深度学习」最准的说法不是「人工 vs 黑盒」，而是看**机器自动化的范围一级级往上爬**：

| | 特征怎么来 | 规则/参数怎么来 |
|---|---|---|
| 专家系统（符号主义） | 人工 | **人工**（手写 if-else） |
| 经典 ML | **人工设计** | 机器从数据学 |
| 深度学习 | **机器自己学** | 机器从数据学 |

两个关键澄清：

1. **经典 ML 的「人工」指特征，不是规则。** 你不会手写「年收入低于 X 且负债高于 Y 则拒贷」——模型自己从数据学出来。人工做的是把原始数据加工成「年收入、负债比、逾期次数」这些有意义的字段（特征工程）。手写规则的是专家系统，别混。
2. **深度学习把特征这一步也自动化了**：直接喂原始像素/文本，网络前几层自己学出「边缘 → 纹理 → 眼睛 → 脸」的特征层级。这就是**表示学习（representation learning）**，是 DL 革命性的点。

「黑盒」是副作用，不是定义：机器学出的特征以几百万个浮点数存在，人类读不懂，**所以**黑盒。两个边界修正：

- 经典 ML 不全是白盒：单棵决策树能逐条看懂，几百棵树的 XGBoost 集成实际也接近黑盒。
- DL 不是绝对不可解释：interpretability 是一整个研究方向（如 Anthropic 研究 LLM 内部哪些「回路」对应哪些概念），只是远未到完全读懂。

> 🔑 **修正后的总结（背这个版本）**：
> 经典 ML：人工设计特征，机器学参数；模型相对浅、相对可解释。
> 深度学习：特征和参数全部由机器从数据学出；代价是内部表示读不懂（黑盒），收益是能处理人类设计不出特征的原始信号。

## 6. LLM 相比传统深度学习特殊在哪

如果 LLM 只是「大一点的神经网络」，不会引发这场革命。它和经典深度学习模型（如 ResNet）有三个质的区别：

| 区别 | 传统 DL（如 ResNet） | LLM |
|------|---------------------|-----|
| **训练方式** | 需要人工标注（这张图是猫/狗） | 自监督：「预测下一个 token」，互联网文本天然是训练数据，解锁数据量天花板 |
| **规模效应** | 做大主要是精度提升一点 | 做大到一定程度出现训练目标里没有的能力——推理、翻译、写代码（涌现） |
| **任务切换** | 学会一个任务只能做那个，换任务要重新训练 | In-context learning：prompt 里给几个例子就能执行新任务，**参数不变**——相当于运行时改行为、不需要重新编译部署 |

> 🔑 机器人方向呼应：Embodied AI 正在把 LLM 这一支（语言/推理）和深度学习其他支（视觉感知、运动控制）拼回一个系统——即 VLA（Vision-Language-Action）模型。深度学习基础两边都用得上。

## 7. 面试常问

### Q1: LLM 和深度学习是什么关系？

- LLM 是深度学习的子集实例：用深度学习方法论（多层网络 + 梯度下降 + 反向传播），基于 Transformer 架构，在海量文本上以自监督方式（next token prediction）训练出来的超大模型。
- 层级：AI ⊃ ML ⊃ DL ⊃ Transformer 系 ⊃ LLM。

### Q2: 深度学习和传统机器学习的区别？

- 同层关系（都是 ML 的子集），划分标准是模型本体。
- 核心区别一句话：**特征是人工设计还是机器学出来的**（特征工程 vs 表示学习）。
- 别忘了补：表格数据上 XGBoost 至今常打赢神经网络——体现工程判断。

### Q3: 监督学习和深度学习什么关系？

- 正交维度，不冲突：监督/无监督按「训练信号来源」分，经典 ML/DL 按「模型本体」分。
- 举例：ResNet 做图像分类 = 深度学习 + 监督学习，同时是两者。

### Q4: 为什么说 LLM 不只是「更大的神经网络」？

- 三点：自监督解锁数据规模、规模涌现新能力、in-context learning 让模型不改参数就能换任务。
