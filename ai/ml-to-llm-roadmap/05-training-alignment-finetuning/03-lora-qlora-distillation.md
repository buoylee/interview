# LoRA、QLoRA、蒸馏与模型合并

## 这篇解决什么问题

全量微调大模型很贵，也有风险：显存需求高、训练时间长、容易破坏通用能力，还会带来多版本权重管理问题。这一篇解决的问题是：当你想让模型适配某个任务、领域或部署约束时，为什么常用 LoRA、QLoRA、蒸馏和模型合并，而不是直接全量训练。

## 学前检查

读这篇前，最好理解：

- 线性层里的权重矩阵是什么：[神经元、MLP 与激活函数](../foundations/deep-learning/01-neuron-mlp-activation.md)
- 预训练和 SFT 的关系：[预训练与 SFT](./01-pretraining-sft-overview.md)
- 如果你还没读 Transformer，可以先知道：LLM 里大量计算都来自矩阵乘法和线性投影。

## 概念为什么出现

业务里经常遇到的问题不是“训练一个新大模型”，而是：

- 让通用模型适应某类客服话术、代码规范或行业文档。
- 在有限显存上训练或部署。
- 把大模型能力迁移到更小模型。
- 把多个适配器整理成可发布的模型版本。

全量微调会更新所有参数，成本和破坏面都大。参数高效微调和蒸馏出现，是为了用更低成本完成“适配”或“压缩”。

## 最小心智模型

LoRA 的核心是：不要改原始大矩阵，只学习一个小的低秩增量。

```text
原权重 W 不动
只训练一个低秩增量 ΔW = A @ B
推理时使用 W + ΔW
```

QLoRA 再进一步：把 base model 量化后冻结，只训练 LoRA 适配器，从而降低训练显存。蒸馏则是让小模型学习大模型的输出、解释或偏好，把能力从 teacher 迁移到 student。

## 最小例子

假设模型某层有一个很大的权重矩阵 `W`。全量微调会直接更新 `W` 的每个参数。LoRA 不动 `W`，只添加两个更小的矩阵 `A` 和 `B`：

```text
输入 x
原路径: x @ W
LoRA 路径: x @ (W + A @ B)
```

如果 `W` 是 `4096 x 4096`，全量更新参数很多。LoRA 选择一个小 rank，例如 `r = 8`，只训练 `4096 x 8` 和 `8 x 4096` 两个矩阵，训练参数量大幅下降。

## 原理层

LoRA 假设很多任务适配不需要在完整参数空间里大幅移动，只需要一个低秩方向的增量。训练时冻结 base model，只更新 adapter 参数。实际使用中，LoRA 通常挂在选定的投影层上，例如 attention 的 Q/K/V/O projection 或 FFN projection，而不是必须覆盖每一个矩阵。这样显存、存储和多任务版本管理都更轻：同一个 base model 可以配多个 LoRA adapter。

QLoRA 解决的是训练显存问题。它把 base model 以低比特量化形式加载并冻结，再通过 LoRA adapter 训练少量参数。注意，QLoRA 不是“量化后全量训练”，而是“量化 base + LoRA 训练”的组合。

蒸馏解决的是模型压缩和能力迁移问题。Teacher model 可能是一个更大、更强或更昂贵的模型；student model 学习 teacher 的答案、步骤、分布或偏好，从而在更低成本下接近特定能力。蒸馏常用于端侧部署、低延迟场景或垂直任务小模型。

模型合并通常指把权重或 adapter 组合成一个新模型版本，例如把 LoRA 权重 merge 回 base，或合并多个 adapter。它能简化部署，但也可能引入能力冲突、遗忘或评估不可控，所以合并后必须重新评估。

## 和应用/面试的连接

选择方法时，先问问题属于哪一类：

| 场景 | 更常见选择 | 原因 |
|------|------------|------|
| 只是换输出格式、语气或短任务约束 | Prompt 或结构化输出 | 成本低，迭代快 |
| 需要使用外部私有知识 | RAG 或工具 | 知识常变，不适合写入权重 |
| 需要稳定学习某类任务风格或标签体系 | LoRA/QLoRA | 适配行为，成本低于全量微调 |
| 需要极致能力迁移且资源充足 | 全量微调 | 改动空间大，但风险和成本高 |
| 需要更小、更快模型 | 蒸馏 | 用 teacher 迁移能力到 student |

面试里不要把“微调”当成万能答案。很多 AI Engineer 场景优先用 prompt、结构化输出、RAG、工具或评估闭环，只有当行为模式需要被稳定写入模型时才考虑 LoRA/QLoRA。

## 常见误区

| 误区 | 更准确的说法 |
|------|--------------|
| LoRA 一定不影响原模型能力 | Adapter 会改变推理路径，仍可能带来能力退化 |
| QLoRA 就是推理量化 | QLoRA 是训练方案：量化 base model，并训练 LoRA adapter |
| 蒸馏只复制答案 | 蒸馏可以学习答案、过程、logits、偏好或任务分布 |
| 模型合并后不用评估 | 合并会改变权重行为，必须重新跑任务和安全评估 |
