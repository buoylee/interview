## 1. 导数（Derivative）

### 1.1 定义

```
f'(x) = lim[h→0] (f(x+h) - f(x)) / h
```

### 1.2 直觉

**导数 = 变化率 = 斜率**

- f'(x) > 0 → 函数在这一点上升
- f'(x) < 0 → 函数在这一点下降
- f'(x) = 0 → 极值点（极大或极小）

**生活例子**：
- 位移的导数 = 速度（位置变化率）
- 速度的导数 = 加速度

### 1.3 导数的三种记号（别被符号吓到）

`d/dx` 是一个**操作符**，意思就是「对 x 求导」，它作用在后面的函数上——类比代码里接收函数、返回函数的高阶函数：

```
d/dx f(x)   ≈   derivative(f)     # 把 f 求导
```

斜杠下面那个字母表示**对谁求导**（把谁当变量）。现在只有一个变量 x 还看不出区别，等函数有多个变量时，`d/dx` 和 `d/dy` 就不一样了——那就是下一节的偏导数。先记住：**斜杠下面的字母 = 你正在动的那个变量**。

同一件事你会看到三种等价写法，别懵：

| 写法 | 读法 | 说明 |
|------|------|------|
| `d/dx f(x)` | 「对 x 求导」 | 莱布尼茨写法，强调对 x 这个变量求导 |
| `df/dx` | 同上 | 同样是莱布尼茨，上面放函数、下面放变量 |
| `f'(x)` | 「f prime」 | 拉格朗日写法，右上角加一撇，最简洁 |

三个完全是一个意思：f 的导数。

### 1.4 常用导数

```
d/dx (x²) = 2x
d/dx (eˣ) = eˣ          ← 指数函数导数是自己！
d/dx (ln x) = 1/x
d/dx (sigmoid(x)) = sigmoid(x)(1 - sigmoid(x))  ← 完整推导见 1.5
```

> 🔑 **ML 关联**：我们要找损失函数的最小值 → 求导 → 看哪个方向能让损失下降。

### 1.5 sigmoid 导数完整推导（面试高频）

先说清 `σ(x)` 是什么：**σ 是希腊字母（读 sigma），机器学习里约定俗成用它代表 sigmoid 函数**。所以 `σ(x)` 就是 `sigmoid(x)`，它的「函数体」是：

```
        1
σ(x) = ─────────
       1 + e⁻ˣ
```

`σ(x)` 和 `1/(1+e⁻ˣ)` 是同一个东西的两种写法——σ 只是个简写，就像把一段逻辑抽成函数。

目标：证明 `d/dx σ(x) = σ(x)(1 - σ(x))`。分两部分，**第一部分是机械的链式法则，第二部分是纯代数改写**。

#### 第一部分：链式法则（机械操作）

把 σ 看成一层层嵌套的函数调用：

```python
a = -x          # 第1层：取负
b = exp(a)      # 第2层：e 的 a 次方 = e⁻ˣ
c = 1 + b       # 第3层：加 1
d = 1 / c       # 第4层：取倒数 → 这就是 σ
```

链式法则说：整体导数 = 每层导数**连乘**。

| 层 | 是什么 | 这层导数 | 为什么 |
|----|--------|----------|--------|
| 1 | a = -x | -1 | -x 求导是 -1 |
| 2 | b = eᵃ | eᵃ = e⁻ˣ | eˣ 的导数是它自己 |
| 3 | c = 1+b | 1 | 常数 1 求导为 0 |
| 4 | d = 1/c = c⁻¹ | -1/c² | 幂法则：c⁻¹ → -1·c⁻² |

四个连乘（注意两个 -1 相乘变正号）：

```
dσ/dx = (-1/c²) · 1 · e⁻ˣ · (-1) = e⁻ˣ / c² = e⁻ˣ / (1+e⁻ˣ)²
```

到这里导数其实已经算完了，只是形式很丑，没法直接用。

#### 第二部分：改写成 σ(1-σ)（纯代数）

把分母的平方 (1+e⁻ˣ)² 拆成两个分数相乘：

```
  e⁻ˣ              1                e⁻ˣ
─────────  =  ───────────  ×  ───────────
(1+e⁻ˣ)²       (1+e⁻ˣ)          (1+e⁻ˣ)
                  ↑                 ↑
               = σ(x)          = 1 - σ(x)
```

- **第一块** `1/(1+e⁻ˣ)` 就是 σ 的定义，等于 `σ(x)`。
- **第二块**用一个小技巧：把分子 `e⁻ˣ` 改写成 `(1+e⁻ˣ) - 1`（两者相等，只是换个长相），再拆开：

```
   e⁻ˣ          (1+e⁻ˣ) - 1       (1+e⁻ˣ)         1
─────────  =  ───────────────  =  ─────────  -  ─────────  =  1 - σ(x)
(1+e⁻ˣ)          (1+e⁻ˣ)           (1+e⁻ˣ)       (1+e⁻ˣ)
                                   上下相同=1    = σ(x)
```

合起来：

```
d/dx σ(x) = σ(x) · (1 - σ(x))   ✓
```

#### 为什么这个形式重要（工程视角）

导数**只用到 σ(x) 自己的输出**，完全不含 e⁻ˣ。反向传播时，前向传播已经算过并缓存了 `a = σ(x)`，求梯度时直接 `a * (1 - a)`——一次乘法一次减法，不用重算指数。就像值被 memoize 了，梯度几乎是「白送」的。tanh 也有同样性质（`1 - tanh²`）。

#### 埋个伏笔：最大值 0.25

`σ(1-σ)` 在 x=0（此时 σ=0.5）时取最大值 **0.25**。也就是说 sigmoid 的梯度最大才 0.25，多层连乘会迅速衰减——这正是第 7 节「梯度消失」的根因，也是深层网络后来改用 ReLU 的原因。

## 2. 偏导数（Partial Derivative）

### 2.1 定义

当函数有多个变量时，对其中一个变量求导，把其他变量当常数：

```
f(x, y) = x² + 3xy + y²

∂f/∂x = 2x + 3y      (y 当常数)
∂f/∂y = 3x + 2y      (x 当常数)
```

### 2.2 直觉

**偏导数 = 只在一个方向上看变化率**

想象你站在山坡上：
- ∂f/∂x = 往东走，地面有多陡
- ∂f/∂y = 往北走，地面有多陡

> 🔑 **ML 关联**：神经网络有成百万/上十亿个参数，损失函数对每个参数都有一个偏导数。

## 3. 梯度（Gradient）

### 3.1 定义

梯度 = 所有偏导数组成的向量：

```
∇f = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]
```

### 3.2 关键性质 ⭐

**梯度指向函数增长最快的方向**

因此：
- 要最小化损失 → 沿梯度的**反方向**走 → 这就是**梯度下降**！

```
θ_new = θ_old - α × ∇L(θ)

θ = 参数
α = 学习率（步长）
∇L = 损失函数的梯度
```

### 3.3 梯度下降的直觉

想象一个山谷：
1. 你站在某处（初始参数）
2. 看看哪个方向最陡（计算梯度）
3. 往下坡方向走一步（更新参数）
4. 重复直到到达最低点（损失最小）

```
局部最小值 vs 全局最小值：
- 凸函数 → 只有一个最低点，梯度下降保证找到
- 非凸函数 → 可能有很多局部最低点（深度学习常见）
- 实践中效果依然很好（高维空间中鞍点比局部最小值更常见）
```

## 4. 链式法则（Chain Rule）

### 4.1 定义

复合函数的导数 = 各步导数的乘积：

```
如果 y = f(g(x))，那么：

dy/dx = (df/dg) × (dg/dx)
```

**怎么读这个公式（很多人就卡在 `dg/dx`）**

`f`、`g` 不是新概念，只是给 `y` 拆出来的两块**起的名字**：

```
y = f(g(x))
        ▲ 内层函数，叫它 g：u = g(x)
     ▲ 外层函数，叫它 f：y = f(u)
```

`g` 本身就是一个和 `y` 一样的函数（输入 x、输出一个值），所以 `dg/dx` 和 `dy/dx` 是**完全同一种操作**——都是"对 x 求导，问 x 变一点点、这东西变多少"，只是作用对象不同（一个是整体 y，一个是中间的 g）。

把内层输出改名叫 `u`，公式立刻顺眼：

```
y = f(u)，  u = g(x)

dy/dx = (dy/du) × (du/dx)
                ▲ du 一上一下，像分数一样"约掉"，正好剩 dy/dx
```

> 💡 `d` = "一点点变化"。`dy/dx` = y 的微小变化 ÷ x 的微小变化 = 变化的比例（斜率）。`df/dg`、`dg/dx` 全是这同一个意思，只是换了对象。

### 4.2 直觉

**把嵌套看成数据管道**，每一步都是一次"放大"：

```
x ──[g]──▶ u ──[f]──▶ y
      放大 dg/dx 倍   放大 df/dg 倍
```

给 x 推一点点：先被 `g` 放大 `dg/dx` 倍变成 u 的变化，再被 `f` 放大 `df/dg` 倍变成 y 的变化。**连续两次缩放，总倍数 = 相乘**，所以 `dy/dx = df/dg × dg/dx`。

> 类比：自行车是走路的 3 倍速，汽车是自行车的 2 倍速 → 汽车是走路的 `2×3=6` 倍。链条上的倍率是乘起来的，不是加起来的。

**拿数字验一遍**（设 `g(x)=2x`，`f(u)=u²`）：

```
y = (2x)² = 4x²
直接求导：  dy/dx = 8x

链式法则：  dg/dx = 2,   df/dg = 2u = 4x
           相乘 = 4x × 2 = 8x   ✅ 对上了
```

**剥洋葱记忆法**：先对外层函数求导，再乘内层函数的导数，一层层往里剥。

### 4.3 多层例子

```
y = sigmoid(W₃ × relu(W₂ × relu(W₁ × x + b₁) + b₂) + b₃)

dy/dW₁ = (dy/dh₃) × (dh₃/dh₂) × (dh₂/dh₁) × (dh₁/dW₁)

            第3层       第2层       第1层       输入层
```

> 🔑 **这就是反向传播的全部原理！** 从输出层开始，往回一层层乘导数。

> ⚠️ **符号预告**：下一节会把这里的 `dy/dW₁` 写成 `∂L/∂W₁`，两处改动、同一个链式法则：
> - **`d → ∂`**：`W₁` 只是上百万参数之一，对它求导时其余参数全按住不动，这种「多变量里只动一个」严格要写偏导 `∂`（见 §2）。4.3 用 `d` 只是图省事，运算没变。
> - **`y → L`**：训练真正要降的是**损失 L**（错多少），不是网络输出 `y`。而 `L = 损失函数(y)`，只是在 `y` 外面又套一层，所以 `∂L/∂W₁` 就是上式最前面再多乘一个 `∂L/∂y`——`x→…→y→L` 链条多接一节而已。

## 5. 反向传播 = 链式法则

### 5.1 前向传播 (Forward Pass)

从输入到输出，逐层计算：

```
Input x → h₁ = f₁(x) → h₂ = f₂(h₁) → ... → Loss L
```

> 📖 **先认三个符号**（第 5 节起一直用）：
> - **`hᵢ`** = 第 i 层的输出（hidden，隐藏层激活值）。**就是第 4 节那个中间量 `u`，多层时按层编号**：`h₁`=第1层输出，`h₂`=第2层输出……数据流 `x → h₁ → h₂ → h₃ → L`。`h₃` 是最后一层、离损失最近的输出。
> - **`L`** = 损失（Loss），**一个标量数字**，衡量网络输出和正确答案差多少；训练的全部目的就是让 `L` 变小（即第 3 节的 `∇L`）。
> - **`∂`** = 偏导，就是第 2 节学的那个——多变量时"只动一个、其余当常数"。所以 `∂L/∂h₃` 读法和 `dy/dx` 完全一样：**"h₃ 变一点点，L 变多少"**，只因 L 同时依赖很多东西，才用花体 `∂` 标记"其余按住不动"。

### 5.2 反向传播 (Backward Pass / Backpropagation)

从损失 L 开始，往回逐层计算梯度：

```
∂L/∂W₃ = ∂L/∂h₃ × ∂h₃/∂W₃
∂L/∂W₂ = ∂L/∂h₃ × ∂h₃/∂h₂ × ∂h₂/∂W₂
∂L/∂W₁ = ∂L/∂h₃ × ∂h₃/∂h₂ × ∂h₂/∂h₁ × ∂h₁/∂W₁
```

**关键：看出三行的「共享前缀」**——这才是「反向」和「全部原理」的来源。注意这里要的是**所有**权重的梯度（拿去更新 `W ← W − lr·∂L/∂W`），不是单独某一个：

```
∂L/∂W₃ = [ ∂L/∂h₃ ] × ∂h₃/∂W₃
∂L/∂W₂ = [ ∂L/∂h₃ × ∂h₃/∂h₂ ] × ∂h₂/∂W₂
∂L/∂W₁ = [ ∂L/∂h₃ × ∂h₃/∂h₂ × ∂h₂/∂h₁ ] × ∂h₁/∂W₁
          └──── 中括号 = 共享的「上游梯度」，越往深越长 ────┘
```

每深一层，中括号只是在上一层基础上**多乘一个局部导数**。所以不必每个权重都从头连乘（那是 O(层²) 的浪费）——从输出端出发，维护一个累积的 `grad`，一层层往回推（就是带累加器的 DP / 记忆化）：

```
grad = ∂L/∂h₃                # 出发：loss 对最后一层输出的梯度
往回每一层 i：
    ∂L/∂Wᵢ = grad × ∂hᵢ/∂Wᵢ   # ① 用当前 grad 算这层权重的梯度
    grad    = grad × ∂hᵢ/∂hᵢ₋₁ # ② grad 再往前推一层（乘进本层局部导数）
```

被一层层传下去的 `grad`（上游梯度）就是「**传播**」的东西；从输出走向输入就是「**反向**」。**一趟回扫，所有权重的梯度全到手**——这就是为什么它配叫"全部原理"：除了链式法则没有任何新数学，唯一的机关是**从后往前算、复用共享前缀**。

> 🔁 **为什么往后扫而不是往前？** loss 只有 1 个标量输出，权重却有几百万个。从这唯一的输出往回扫，一趟就把共享前缀摊给所有权重；要是从每个权重正向推，得扫几百万趟。这就是深度学习用「反向模式」自动微分的原因。

### 5.3 计算图视角

```
x ──→ [×W₁+b₁] ──→ [ReLU] ──→ [×W₂+b₂] ──→ [Softmax] ──→ [Loss]
         ↑                         ↑
       W₁,b₁                    W₂,b₂

前向：从左到右计算输出
反向：从右到左计算梯度（链式法则）
```

PyTorch / TensorFlow 就是自动帮你做这件事（自动微分 Autograd）。

### 5.4 为什么叫「反向」？

- **不是**倒着跑一遍网络
- **是**从后往前传播梯度信息
- 因为链式法则是从外向内（即从输出向输入）展开的

### 5.5 一个能手算的完整例子：看每个 W、b 怎么更新

整个训练就是**三步一循环**，对每个参数（W 和 b 一视同仁）：

```
① 前向：用当前参数算出预测 y，再算损失 L
② 反向：给每个参数算出一个梯度数 ∂L/∂参数   ← 链式法则那一趟回扫
③ 更新：参数 ← 参数 − lr × ∂L/∂参数        ← 每个参数都套这同一行
```

第 ③ 步是死的，所有参数共用一行。难的只有第 ② 步，而它机械到只靠两个事实：

> 对一层 `z = W·输入 + b`：
> - `∂z/∂W = 输入`　→ **某权重的梯度 = 上游梯度 × 它当初乘的那个输入**
> - `∂z/∂b = 1`　　 → **某偏置的梯度 = 上游梯度原样穿过**（b 就是个普通参数，无特殊）

拿一个能手算的最小网络跑完整一步：

```
输入 x = 1.5     正确答案 t = 2.0     学习率 lr = 0.1
初始参数：W₁=0.8  b₁=0.1   W₂=0.5  b₂=0.2

第1层：z₁ = W₁·x + b₁ ,  a₁ = relu(z₁)
第2层：z₂ = W₂·a₁ + b₂,  y  = z₂
损失： L  = ½(y − t)²
```

> 📖 **看懂 `a₁ = relu(z₁)` 和 `y = z₂`**：每一层其实是**两小步**——
> - `z = W·输入 + b`：线性加权求和，`z` 叫**预激活**；
> - `a = 激活函数(z)`：套一个非线性函数，`a` 才是这层**真正的输出**（就是 §5.1 的 `hᵢ`，"a"取自 activation 激活值，同一个东西）。
>
> **ReLU** 是最常用的激活函数：`relu(z) = max(0, z)`——正数原样留、负数压成 0，所以 `relu(1.3)=1.3`。它给网络加「拐弯」：没有它，多层线性会塌缩成一层（`W₂(W₁x+b₁)+b₂` 仍是线性），学不了复杂关系。
>
> **末层 `y = z₂` 为何不套激活**？输出层通常不套（或套任务专用的）：本例是预测数值（回归），要的就是原始线性值，故 `y = z₂`。若是分类，这里会换成 sigmoid（二分类）/ softmax（多分类）。

**① 前向**（顺着算，并记住每个中间值，反向要用）：

```
z₁ = 0.8×1.5 + 0.1 = 1.3
a₁ = relu(1.3)      = 1.3
z₂ = 0.5×1.3 + 0.2  = 0.85
y  = 0.85
L  = ½(0.85 − 2.0)² = 0.661       ← 现在错得挺多
```

**② 反向**（从 L 出发，grad 一路往回乘，每个参数领一个梯度）：

```
起点  ∂L/∂y  = y − t = 0.85 − 2.0 = −1.15        ← ½(y−t)² 对 y 的导数

第2层 z₂ = W₂·a₁ + b₂：
  ∂L/∂W₂ = ∂L/∂y × a₁ = −1.15 × 1.3 = −1.495     （上游 × 它乘的输入 a₁）
  ∂L/∂b₂ = ∂L/∂y × 1  = −1.15                    （上游原样穿过）
  ∂L/∂a₁ = ∂L/∂y × W₂ = −1.15 × 0.5 = −0.575      ← 传给第1层的新 grad

第1层 a₁ = relu(z₁)，z₁ = W₁·x + b₁：
  ∂L/∂z₁ = ∂L/∂a₁ × relu′(z₁) = −0.575 × 1 = −0.575   （z₁>0，relu导数=1）
  ∂L/∂W₁ = ∂L/∂z₁ × x = −0.575 × 1.5 = −0.8625    （上游 × 它乘的输入 x）
  ∂L/∂b₁ = ∂L/∂z₁ × 1 = −0.575                    （上游原样穿过）
```

**③ 更新**（每个参数都套同一行 `新 = 旧 − lr × 梯度`）：

```
W₂: 0.5 − 0.1×(−1.495)  = 0.6495
b₂: 0.2 − 0.1×(−1.15)   = 0.315
W₁: 0.8 − 0.1×(−0.8625) = 0.88625
b₁: 0.1 − 0.1×(−0.575)  = 0.1575
```

梯度是负的 → 参数往上调（朝让损失下降的方向）。

**验证**：用新参数再前向一次，损失确实降了：

```
z₁=0.886×1.5+0.158=1.487 → a₁=1.487
z₂=0.6495×1.487+0.315=1.281 → y=1.281
L = ½(1.281−2.0)² = 0.259         ← 0.661 → 0.259，错误小了一大半 ✅
```

这就是**一步**训练。三步对成千上万个样本、重复几万次，参数被一点点推到损失最小——网络就「学会」了。

> 🔑 **三个让你踏实的点**
> 1. **b 没有特殊待遇**：和 W 一样是参数，只因 `∂z/∂b = 1`，它的梯度就是上游梯度原样落下来，更新公式照用同一行。
> 2. **「反向」在算什么**：`grad`（例子里 −1.15 → −0.575）是真东西，一层层往回乘局部导数；每经过一个参数，用当时的 grad 乘「该参数当初乘的输入」，得到它的梯度。
> 3. **为什么前向要缓存中间值**：算 `∂L/∂W₂` 用到了 `a₁`、算 `∂L/∂W₁` 用到了 `x`。所以框架前向时缓存各层激活值，反向才取得出来——这也是训练比推理更吃显存的原因。

PyTorch 里这整套就是 `loss.backward()`（②）+ `optimizer.step()`（③）两行，自动微分替你做完上面的回扫。

## 7. 面试常问

### Q1: 什么是反向传播？用一句话解释。

**答**：反向传播就是用链式法则，从损失函数开始，逐层计算损失对每个参数的梯度，然后用这些梯度更新参数。

### Q2: 为什么会梯度消失？怎么解决？

**答**：
- **原因**：反向传播时梯度是一连串导数的**乘积**。如果每层导数 < 1（如 sigmoid 最大导数只有 0.25），多层连乘后梯度趋向零。
- **解决**：
  1. **ReLU 激活函数**：正区间导数 = 1，不会衰减
  2. **Residual Connection**：跳跃连接让梯度可以直通，不经过那么多乘法
  3. **Layer Normalization**：稳定中间层的分布
  4. **合理的权重初始化**（Xavier/He）

### Q3: 梯度下降有哪些变种？

**答**：
| 变种 | 每次用多少数据 | 特点 |
|------|-------------|------|
| BGD (Batch GD) | 全部数据 | 准确但慢 |
| SGD | 1 条数据 | 快但噪声大 |
| Mini-batch SGD | 一批数据（32/64/128...） | **实际最常用** |

### Q4: 学习率为什么重要？太大/太小会怎样？

**答**：
- **太大** → 在最低点附近来回跳，无法收敛（甚至发散）
- **太小** → 收敛太慢，可能卡在局部最小值
- **实践**：使用 Warmup（先慢后快）+ Cosine Decay（逐渐减小）
- **Adam 优化器**：自适应学习率，自动为不同参数调整步长
