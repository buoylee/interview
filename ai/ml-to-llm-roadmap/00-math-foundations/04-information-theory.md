# 0.4 信息论（Day 6-7）

> **一句话定位**：信息论提供了衡量「不确定性」和「信息量」的数学工具。交叉熵是 ML 最常用的损失函数，KL 散度是 RLHF 的核心约束。

## 1. 信息量与信息熵

> 🔢 **看不懂 `log` / `-log`?先花 30 秒补这个**(本章全靠它，看懂这台「机器」后面就顺了)
>
> **`log₂(x)` 只回答一个问题:「2 的几次方 = x?」** 它是「次方」的逆运算。程序员视角:`log₂(x)` = x 要几个 bit / 二分几次 / 从 1 翻几次倍到 x。
>
> ```
> 2³=8     → log₂(8)=3        2⁰=1     → log₂(1)=0     ← 分界线
> 2¹⁰=1024 → log₂(1024)=10    2⁻¹=0.5  → log₂(0.5)=-1  (负次方=倒数)
>                             2⁻²=0.25 → log₂(0.25)=-2
> ```
>
> **关键:0~1 之间的数，log 是负的**——数越小、log 越负，趋近 0 时奔 -∞。概率都 ≤1，所以 `log(概率)` 永远是负数或 0。
>
> **前面的负号，就是把它翻成正数**。于是 `-log(p)` 是一台机器:喂进概率(0~1)、吐出一个非负数——**p=1 吐 0,p 越小吐得越大,p→0 吐 +∞**。这个吐出来的数，下面叫「信息量/惊讶度」，第 2 节当「损失」。
>
> **下标 = 底数 = 「几次方」里那个数**:`log₂`=「2 的几次方」、`ln`=`logₑ`=「e 的几次方」。没下标的 `log` 是「底数看语境」——本章第 1 节讲 bit 默认底数 2(写 `log₂`),第 2 节讲损失按 ML 惯例默认底数 e(即 `ln`,写 `log`)。**换底数只是把所有数值乘一个固定倍数**(`ln ≈ log₂ × 0.69`),曲线形状、0 点(都在 x=1)、正负趋势全不变——所以理解时一律当 `log₂` 就好,看到没下标的 `log` 读成 `ln` 即可,不影响任何结论。

> 📈 **想把 `-log` 的图刻进脑子**:打开 [Desmos](https://www.desmos.com/calculator) 输入 `-log_2(x)` 拖着看(或 Google 搜 `plot -log2(x)`);记不住就抓三个锚点连起来——`(p→0, 损失→∞)`、`(p=0.5, 损失=1)`、`(p=1, 损失=0)`,曲线**左边冲天、一路下滑、右端 x=1 落地归零**(像一条滑梯)。
>
> ⚠️ 这里 x 是**概率,只看 0~1 这一段**。Desmos 会把 x>1 也画出来(那里 y 变负),但概率不可能 >1,**那段用不到、无视它**;想眼不见心不烦,输入 `-log_2(x) {0<x<1}` 把右边裁掉。在 0~1 整段里,`-log` 永远 ≥ 0。

### 1.1 信息量（Information / Self-Information）

```
I(x) = -log₂ P(x)
```

**直觉**：一件事越不可能发生，一旦发生了，带来的信息量越大。所以「信息量」也叫 **惊讶度**——越意外的事，信息量越大。

> 🧩 **记住这个 `-log P(x)`(= 惊讶度)**：它是本章的「原子零件」。后面的信息熵、交叉熵、KL 散度，全是拿它搭出来的(求加权平均、或把概率换一个代进去)。以后在文档里看到 `-log(某概率)`，就读成「某件事的惊讶度」，它和这里的 `I(x)` 是同一个东西，只是省略了 `I(x) =` 和下标 `₂`。

| 事件 | 概率 | 信息量 |
|------|------|--------|
| 太阳从东边升起 | ≈ 1 | ≈ 0（毫不意外）|
| 今天下雨 | 0.5 | 1 bit |
| 彩票中奖 | 0.001 | ≈ 10 bits（很意外）|

### 1.2 信息熵（Entropy）⭐

```
H(X) = -Σ P(xᵢ) × log₂ P(xᵢ)
     = E[I(X)]
     = 信息量的期望值
```

**直觉**：一个分布的「不确定性」有多大。

> 🧩 **接 1.1 节**：把每个事件的惊讶度 `-log P(xᵢ)`，按它出现的概率 `P(xᵢ)` 加权平均，就是熵——所以 **熵 = 惊讶度的平均值**(`E[I(X)]` 说的就是这个)。记住这个「加权平均」的套路，第 2 节交叉熵用的是一模一样的套路，只是把算惊讶用的概率换成模型的 q。

```
公平硬币: P(正)=0.5, P(反)=0.5 → H = 1 bit（最不确定）
作弊硬币: P(正)=0.99, P(反)=0.01 → H ≈ 0.08 bit（几乎确定）
```

**性质**：
- 均匀分布时，熵最大（最不确定）
- 完全确定时，熵 = 0（没有不确定性）
- 总是非负的

> 🔑 **ML 关联**：
> - 决策树用信息增益（熵减少量）来选择最佳分裂特征
> - LLM 生成时，高熵 = 模型对下一个 token 很不确定

## 2. 交叉熵（Cross-Entropy）⭐⭐⭐

### 2.1 定义

```
H(p, q) = -Σ p(xᵢ) × log q(xᵢ)

p = 真实分布
q = 模型预测的分布
```

### 2.2 直觉

**用模型预测 q 去编码真实分布 p 的数据，需要多少「比特」。**

- 如果 q 完美匹配 p → 交叉熵 = 熵 H(p)（最小值）
- q 和 p 差距越大 → 交叉熵越大

#### 从 `-log P(x)` 看懂这个 Σ（关键）

`H(p,q) = -Σ p(xᵢ)·log q(xᵢ)` 不是新东西，它就是 1.1 节那个**惊讶度原子 `-log P(x)` 的加权平均**。按角色拆开：

```
H(p, q) =  Σ      p(xᵢ)    ×   ( -log q(xᵢ) )
          └─┬─┘   └──┬──┘        └────┬────┘
          所有事件   出现频率        这个事件的惊讶度
          求和=平均  (现实 p 说了算)  (用模型 q 算 = 那个原子)
```

- **`-log q(xᵢ)`** = 1.1 节的原子，只是概率换成模型的 q ——**单个事件的惊讶度**。
- **`p(xᵢ) ×`** = 权重 = 这个事件**实际多常发生**(真实分布 p 决定)。
- **`Σ`** = 扫过所有事件加起来 = **求加权平均**。

所以交叉熵 = 「**惊讶度的平均值**」。它和熵 `H(p)` 长得一模一样，区别只在**用谁的概率算惊讶**：

```
熵     H(p)   = Σ p(x)·( -log p(x) )   ← 用真实概率 p 算惊讶
交叉熵 H(p,q) = Σ p(x)·( -log q(x) )   ← 用模型概率 q 算惊讶
```

这就是「**交叉**」二字的来源：**多常发生听现实 p 的，多惊讶听模型 q 的**——两个分布交叉着用。

**具体走一遍(天气，3 种结果)**：

```
现实 p = [晴 0.7, 阴 0.2, 雨 0.1]   ← 决定每种天气多常出现(权重)
模型 q = [晴 0.6, 阴 0.3, 雨 0.1]   ← 决定每种天气来了模型多惊讶

每个事件的惊讶度(用 q 算)：
  晴 -log₂(0.6)≈0.74   阴 -log₂(0.3)≈1.74   雨 -log₂(0.1)≈3.32

按现实频率 p 加权平均：
H(p,q) = 0.7×0.74 + 0.2×1.74 + 0.1×3.32 ≈ 1.20 bit
```

#### 训练时 Σ 塌缩回单个 `-log`

训练**一条样本时，现实只发生一件事**——真实标签 / token。所以那条样本的 p 是 **one-hot**(正确答案=1，其余=0)，Σ 里 p=0 的项全归零，只剩正确那一项：

> **交叉熵损失 = 正确答案出现时，模型有多惊讶**(用模型自己给的概率算)。

```
模型给正确答案 0.9  → 不惊讶 → -log(0.9) ≈ 0.11  → 损失小 ✅
模型给正确答案 0.01 → 很惊讶 → -log(0.01) ≈ 4.6  → 损失大 ❌
```

训练 = 让模型对「真实发生的答案」尽量不惊讶 = 逼它给正确答案高概率。LLM 预训练每一步：看到真实的下一个 token → 算 `-log(模型给这个 token 的概率)` → 把它压小。

**整条线闭环了**：

```
-log P(x)            ← 1.1 节：一个事件的惊讶度(原子)
   ↓ 按真实频率 p 加权求平均(Σ p·)
Σ p(x)·(-log q(x))   ← 交叉熵一般式：平均惊讶度
   ↓ 训练时 p 是 one-hot，Σ 塌缩
-log q(正确答案)      ← 又回到原子，就是你的及格线公式
```

那个吓人的 Σ 只是给「真实答案本身也不确定」的情况准备的(如知识蒸馏，p 是老师的软标签)；你做分类 / LLM 预训练永远是 one-hot，所以永远只剩一个 `-log`。

### 2.3 作为损失函数

在分类问题中（p 是 one-hot）：

```
真实标签 p = [0, 0, 1, 0, 0]  (第 3 类)
模型预测 q = [0.1, 0.1, 0.6, 0.1, 0.1]

H(p, q) = -log(0.6) ≈ 0.51

如果模型更准:
q = [0.01, 0.01, 0.95, 0.01, 0.02]
H(p, q) = -log(0.95) ≈ 0.05  → 更小，更好！
```

**简化**：当 p 是 one-hot 时，交叉熵 = **-log(模型给正确类的概率)**

> 🔑 **ML 关联**：
> - **分类任务**的标准损失函数
> - **LLM 预训练**的训练目标 = 交叉熵（即最大化正确 next token 的概率）
> - 最小化交叉熵 = 最大化似然（MLE）

### 2.4 二元交叉熵（Binary Cross-Entropy）

```
BCE = -[y × log(ŷ) + (1-y) × log(1-ŷ)]

y = 真实标签 (0 或 1)
ŷ = 模型预测的概率
```

用于二分类问题（逻辑回归等）。

## 3. KL 散度（KL Divergence）⭐⭐⭐

### 3.1 定义

```
D_KL(p || q) = Σ p(xᵢ) × log(p(xᵢ) / q(xᵢ))
             = H(p, q) - H(p)
             = 交叉熵 - 真实熵
```

### 3.2 直觉

**KL 散度 = 用 q 近似 p 的「信息损失」= 两个分布的「距离」**

- D_KL ≥ 0（永远非负）
- D_KL = 0 当且仅当 p = q（两个分布完全一样）
- **不对称**：D_KL(p||q) ≠ D_KL(q||p)

### 3.3 不对称性的含义

```
D_KL(p || q):  "前向 KL" → q 必须覆盖 p 的所有可能（mode-covering）
D_KL(q || p):  "反向 KL" → q 倾向于集中在 p 的峰值（mode-seeking）
```

这个区别在 RLHF/DPO 等对齐方法中很关键。

### 3.4 在 ML/LLM 中的重要应用

| 应用场景 | 具体用法 |
|---------|---------|
| **RLHF** | KL 约束：让微调后的模型不要偏离原始预训练模型太远 `reward - β × D_KL(π_θ \|\| π_ref)` |
| **DPO** | 隐式 KL 约束，直接优化偏好而非训练奖励模型 |
| **知识蒸馏** | 让学生模型的 soft label 分布接近教师模型 |
| **VAE** | 让编码器输出的分布接近标准正态分布 |
| **信息瓶颈** | 压缩无关信息，保留有用信息 |

> 🔑 **RLHF 中的关键作用**：
> ```
> 目标 = 最大化 [奖励模型分数] - β × D_KL(当前策略 || 参考策略)
> ```
> - 前半部分鼓励模型生成人类偏好的回答
> - KL 惩罚防止模型为了讨好而「跑偏」（说奇怪的话只为得高分）

## 4. 互信息（Mutual Information）

### 4.1 定义

```
I(X; Y) = H(X) - H(X|Y)
        = 知道 Y 后 X 的不确定性减少了多少
```

### 4.2 直觉

**互信息 = X 和 Y 共享了多少信息**

- I(X; Y) = 0 → X 和 Y 完全独立
- I(X; Y) 越大 → 知道一个对预测另一个越有帮助

### 4.3 应用

| 场景 | 用法 |
|------|------|
| 特征选择 | 选择和目标变量互信息最大的特征 |
| 对比学习 | InfoNCE Loss 最大化正样本之间的互信息 |
| 信息瓶颈 | 压缩表示同时保留和目标的互信息 |

> 🔑 **ML 关联**：
> - **对比学习** (CLIP, SimCLR) 的理论基础就是最大化正样本对之间的互信息
> - 这和 Embedding 模型的训练直接相关

## 5. 困惑度（Perplexity）

### 5.1 定义

```
PPL = 2^H = 2^(-1/N × Σ log₂ P(wᵢ))
    = exp(交叉熵)  (当使用自然对数时)
```

### 5.2 直觉

**困惑度 = 模型在每一步平均有多「困惑」（有多少等可能的选项）**

```
PPL = 1    → 模型完全确定下一个词是什么（完美预测）
PPL = 10   → 模型在 10 个等可能的词之间犹豫
PPL = 1000 → 模型在 1000 个词之间犹豫（很困惑）
```

### 5.3 为什么重要？

- **语言模型的标准评估指标**
- PPL 越低 → 模型越好（越不困惑）
- 但 PPL 低不一定等于生成质量好（可能过拟合）

> 🔑 **面试常问**：GPT-4 的 PPL 大约是多少？怎么理解这个数字？

## 6. 在 ML/LLM 中的应用

| 信息论概念 | ML/LLM 应用 | 阶段 |
|-----------|------------|------|
| 信息熵 | 决策树分裂标准、不确定性度量 | 1 |
| 交叉熵 | **分类损失函数、LLM 预训练目标** | 1, 2, 6 |
| KL 散度 | **RLHF/DPO 约束、知识蒸馏、VAE** | 6 |
| 互信息 | **对比学习 (CLIP)、特征选择** | 2, 3 |
| 困惑度 | **语言模型评估** | 3, 6 |

### 概念关系图

```
          信息熵 H(p)
         ↙        ↘
  交叉熵 H(p,q)    互信息 I(X;Y)
      ↓                ↓
  KL 散度           对比学习
 D_KL = H(p,q) - H(p)
      ↓
  困惑度 PPL = exp(H)
```

## 7. 面试常问

### Q1: 交叉熵和 KL 散度的关系？

**答**：
```
H(p, q) = H(p) + D_KL(p || q)
交叉熵 = 真实分布的熵 + KL 散度
```
- 当 p 固定时（训练中真实标签不变），最小化交叉熵等价于最小化 KL 散度
- 这就是为什么训练用交叉熵就够了

### Q2: 为什么 RLHF 需要 KL 约束？

**答**：
- 没有 KL 约束 → 模型会过度优化奖励模型的分数
- 奖励模型不完美（有盲点），模型可能找到「hack」方式得高分但实际生成很差（reward hacking）
- KL 约束确保微调后的模型不会偏离预训练模型太远，保留通用能力
- β 参数控制约束强度：β 大 → 更保守；β 小 → 更激进

### Q3: 什么是 Perplexity？怎么解释给非技术人？

**答**：
- 「困惑度就像一个选择题的选项数。如果模型的困惑度是 10，就意味着模型在每个位置平均要在 10 个等可能的词之间做选择。」
- 困惑度越低，模型越好
- 数学上是交叉熵的指数形式

### Q4: 信息增益（Information Gain）是什么？

**答**：
```
IG(Y, X) = H(Y) - H(Y|X)
         = 知道 X 后 Y 的不确定性减少了多少
         = 互信息 I(X; Y)
```
- 决策树用信息增益来选择分裂特征
- 选信息增益最大的特征 = 选能最大程度减少不确定性的特征

### Q5: 交叉熵损失和 MSE 损失什么时候用哪个？

**答**：
| | 交叉熵 | MSE |
|--|--------|-----|
| 适用 | 分类问题 | 回归问题 |
| 输出 | 概率分布 (softmax) | 连续值 |
| 梯度特点 | 错误越大梯度越大 → 学得快 | 可能梯度平坦 |
| LLM | ✅ 预训练用交叉熵 | ❌ 不适合 |
