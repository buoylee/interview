## 1. 向量（Vector）

### 1.1 什么是向量？

**直觉**：向量就是一组有序的数字，可以理解为空间中的一个「箭头」或一个「点」。

```
2D 向量: [3, 4]      → 平面上的一个点/方向
3D 向量: [1, 2, 3]   → 三维空间中
768D 向量: [0.12, -0.34, ..., 0.56]  → Embedding 空间中！
```

### 1.2 向量运算

#### 加法：对应元素相加

```
[1, 2] + [3, 4] = [4, 6]
```

**直觉**：两个力的合成，两个方向的叠加。

#### 数乘：缩放

```
3 × [1, 2] = [3, 6]
```

**直觉**：拉长或缩短箭头，方向不变。

#### 点积（Dot Product）⭐ 极其重要

```
[1, 2] · [3, 4] = 1×3 + 2×4 = 11
```

**直觉**：衡量两个向量的「相似度」。

- 点积 > 0 → 方向大致相同
- 点积 = 0 → 互相垂直（正交），完全不相关
- 点积 < 0 → 方向大致相反

> 🔑 **ML 关联**：Attention 机制中 Q·K 就是点积，计算 query 和 key 的相似度！

#### 余弦相似度（Cosine Similarity）⭐

```
cos(θ) = (A · B) / (||A|| × ||B||)
```

- 值域 [-1, 1]
- 1 = 完全相同方向
- 0 = 正交（不相关）
- -1 = 完全相反

> 🔑 **ML 关联**：RAG 中向量检索用余弦相似度比较 query embedding 和 document embedding！

**为什么 RAG 用余弦而不是点积？**
- 余弦只看「方向」，不看「长度」
- 同一概念的文本，不管长短，方向应该相似
- 点积会受到向量长度影响，长文本天然长度更大，会 bias 结果

### 1.3 向量的范数（Norm）

范数 = 向量的「长度」：

```
L1 范数（曼哈顿）:   ||v|| = |v₁| + |v₂| + ... + |vₙ|
L2 范数（欧几里得）: ||v|| = √(v₁² + v₂² + ... + vₙ²)
```

**直觉：两种不同的「量长度」方式**

以向量 v = (3, 2) 为例：

```
L1 = |3| + |2| = 5        ← 在城市里沿街道走：横走3格 + 竖走2格 = 5步
L2 = √(3² + 2²) = √13 ≈ 3.6  ← 小鸟直线飞过去（就是勾股定理）
```

**什么时候用哪个？**
- **L2**：你觉得每个特征都有用，不想丢掉任何一个（如预测房价，面积/楼层/朝向都有影响）
- **L1**：你觉得大部分特征是噪音，想自动筛出关键的几个（如 1000 个基因里只有 5 个跟某种病有关）

**为什么范数能做到「丢特征」还是「全保留」？** 训练时会把范数当成「罚款」加进目标里，惩罚过大的权重（罚款力度由一个人为设的旋钮 λ 控制）。两种范数的罚款「拉力」规律不同：

- **L1**（绝对值）的拉力**恒定**，不管权重多小都一直往 0 拽 → 没用的权重被压到**正好 0** → 等于丢掉特征（稀疏）。
- **L2**（平方）的拉力**随权重变小而减弱**，越接近 0 越拽不动 → 只能把权重压得很小、**到不了 0** → 所有特征都保留（只是变温和）。

> 🔑 **ML 关联**：这就是 L1/L2 正则化——对权重加范数惩罚来防止过拟合。**这里记住「L1 筛特征、L2 全保留」的结论即可**；完整机制（loss 是什么、罚款怎么算账、λ 怎么调）在 01-ml-basics 阶段细讲。

## 2. 矩阵（Matrix）

### 2.1 什么是矩阵？

**直觉**：矩阵就是一个数字的表格，更本质地说，矩阵是一个「变换」（transformation）。

```
W = | 2  0 |    → 把 x 方向拉伸 2 倍
    | 0  3 |    → 把 y 方向拉伸 3 倍
```

### 2.2 矩阵的维度

```
(m × n) 矩阵 = m 行 × n 列

权重矩阵 W: (768 × 768)  → 768 维输入变换到 768 维输出
Embedding 矩阵: (50000 × 768) → 50000 个词，每个词 768 维向量
```

> 🔑 **ML 关联**：
> - 神经网络的每一层本质就是 `y = Wx + b`（矩阵乘法 + 偏置）
> - Embedding 层本质就是一个大的查找矩阵

## 3. 矩阵乘法的几何直觉

### 3.1 矩阵 × 向量 = 变换

```
| a  b | × | x |   =   | ax + by |
| c  d |   | y |       | cx + dy |
```

**直觉**：把向量从一个空间「变换」到另一个空间。

> 🔑 **ML 关联**：Attention 中的 Q = X·Wq, K = X·Wk, V = X·Wv 就是把输入 X 通过不同的矩阵变换到不同的子空间。

#### 换个读法：矩阵 × 向量 = 一堆点积 ⭐ 连接神经网络

上面是「变换」视角。同一个式子换个角度看更关键：**矩阵的每一行，各和向量做一次点积，得到输出里的一个数字。**

```
            语文 数学 英语                x = 小明成绩
W = | 0.5  0.1  0.4 |  ← 第1行 = 文科权重    x = [80, 90, 70]
    | 0.2  0.7  0.1 |  ← 第2行 = 理科权重

文科分 = 第1行·x = 0.5×80 + 0.1×90 + 0.4×70 = 77   ← 一次点积 → 1 个输出
理科分 = 第2行·x = 0.2×80 + 0.7×90 + 0.1×70 = 86   ← 又一次点积 → 又 1 个输出

y = Wx = [77, 86]
```

**核心一句话**：W 有几行，就做几次点积，就有几个输出。想要 768 个输出，就是用 768 行权重，对同一个 x 做 768 次点积。

用代码看，就是一个双层循环 —— 内层是「点积」，外层转几次就出几个输出：

```
for i in 输出个数:          # 外层：每个输出转一圈
    y[i] = (W[i] · x) + b[i]   # 内层这个点积，就是一个神经元
```

> 🔑 **ML 关联**：神经网络一层 `y = Wx + b` 本质就是这样一「堆点积」。这就是为什么「点积」是理解整个网络的最小积木——后面 Attention、FFN 全是它堆出来的。
> 展开见 [神经元、MLP 与激活函数](../foundations/deep-learning/01-neuron-mlp-activation.md)。

### 3.2 矩阵 × 矩阵 = 组合变换

```
(m × k) × (k × n) = (m × n)

注意：内侧维度 k 必须匹配！
```

**直觉**：把两步运算合成一步，就像手机修图「先旋转再放大」可以合成一个操作一步到位。

**为什么内侧维度必须匹配？** 3 门课的成绩就需要 3 个权重，数量必须对上，否则没法算。

### 3.3 批量处理视角

```
X (batch_size × input_dim) × W (input_dim × output_dim) = Y (batch_size × output_dim)
```

**翻译成人话**：

```
假设 2 个学生，各有 3 门课成绩，要算 1 个加权总分：

         语文 数学 英语               权重
小明  =  [80, 90, 70]    ×    [0.3]    =  [83  ]  ← 小明总分
小红  =  [60, 85, 95]         [0.5]       [79.5]  ← 小红总分
                               [0.2]

  X (2×3)             ×   W (3×1)     =  Y (2×1)
  (2个学生×3门课)        (3个权重)       (2个总分)
```

- **batch_size** = 几条数据一起算（2 个学生）
- **input_dim** = 每条数据有几个特征（3 门课）
- **output_dim** = 算出几个结果（1 个总分）

关键点：**所有学生共用同一套权重，一次性全算完** → 这就是 GPU 并行快的原因。

## 4. 转置与逆矩阵

### 4.1 转置（Transpose）

行变列，列变行 —— 就是你已经懂的那个「沿对角线翻转」：

```
A = | 1  2  3 |      Aᵀ = | 1  4 |
    | 4  5  6 |           | 2  5 |
                          | 3  6 |
```

原来 A 是 2 行 3 列 (2×3)，转置后变成 3 行 2 列 (3×2)。规律：原来第 i 行第 j 列的数，跑到第 j 行第 i 列。

**但「会翻」不重要，重要的是「翻了拿来干嘛」。** 转置在 ML 里几乎只为一件事服务：**让两堆向量能两两做点积。**

#### 为什么转置能「凑出点积」？

回忆第 3 节：矩阵乘法 = 一堆点积，规则是「**左矩阵的行 · 右矩阵的列**」。

问题来了——如果我手上有两堆向量、而且**都是按行存的**（每行一个向量），想让它们两两点积，右边那堆的「方向」就不对：乘法要拿它的「列」去点积，可我的向量是按行躺着的。**转置就是把右边那堆从「按行躺着」翻成「按列站着」**，乘法这才能把它们一一点积。

#### 🔑 ML 关联：看懂 Attention 的 QKᵀ

这是转置在 LLM 里最重要的一次出场，拆开看：

```
Q（n 个 query，每行一个 d 维向量）：形状 (n × d)
K（n 个 key，  每行一个 d 维向量）：形状 (n × d)

目标：算「每个 query 和每个 key 有多像」 → 一张 n×n 的相似度表
      表里每一格 = 一个 query 向量 · 一个 key 向量（点积 = 相似度）

但 (n×d) × (n×d) 没法乘（内侧 d 和 n 对不上，见 3.2）。
把 K 转置：Kᵀ 形状变成 (d × n) —— key 们从「按行躺着」变成「按列站着」。

   Q (n×d)  ×  Kᵀ (d×n)  =  相似度表 (n×n)
                             第 i 行第 j 列 = Q 的第 i 行 · K 的第 j 行
                                          = 第 i 个 query 跟第 j 个 key 的相似度
```

所以转置在这里**一箭双雕**：① 让维度对上、乘法合法；② 顺手把 key 摆成列，乘出来正好是我们要的两两点积表。

#### 性质：(AB)ᵀ = BᵀAᵀ（🔍 知道就行）

转置一个乘积，顺序要**反过来**。用维度记最快：AB 是 (m×n)，转置后是 (n×m)；只有 `Bᵀ(n×k) × Aᵀ(k×m)` 才乘得出 (n×m)，顺序不反就对不上。

> 类比穿脱衣服：先穿袜子后穿鞋，脱的时候必须**先脱鞋后脱袜**——撤销一串操作，顺序一定反过来。

### 4.2 逆矩阵（Inverse）

#### 先用普通数字理解

数字 5 的「逆」是 1/5，因为 `5 × 1/5 = 1`。**乘以逆 = 撤销原来的乘法**（先 ×5 再 ×1/5，等于啥也没干）。

矩阵也一模一样，只是把「1」换成了**单位矩阵 I**：

```
A × A⁻¹ = I
```

单位矩阵 I 就是矩阵世界里的「1」：对角线全 1、其余全 0。任何向量乘它都不变（`I × v = v`，相当于「原地不动」这个变换）。

```
I = | 1  0 |
    | 0  1 |
```

#### 直觉：逆矩阵 = 撤销变换

如果 A 做的是「旋转 45° 再放大 2 倍」，那 A⁻¹ 做的就是「缩小一半、再反向旋转 45°」——把 A 干的事一步步倒回去。（注意顺序也反过来了，和上面穿脱衣服一个道理。）

#### 为什么有些矩阵没有逆？

关键一句：**有些变换会丢信息，丢了就回不去。**

想象一个矩阵把整个 2D 平面「拍扁」到一条直线上：

```
原来平面上无数个不同的点  ──拍扁──▶  全挤到一条线上

很多原本不同的点，现在落在同一个位置 → 你已经分不出谁是谁
```

信息丢了 → 没法撤销 → **没有逆矩阵**（这种矩阵叫奇异矩阵 / 不可逆矩阵）。

怎么判断丢没丢？看**行列式 det(A)**：

- det(A) = 这个变换把面积/体积放大了几倍
- det(A) = 0 → 面积被压成了 0（拍扁了，丢了一个维度）→ **不可逆**

#### 🔍 在 ML 里会遇到吗？

深度学习的热路径里**几乎不直接求逆**（权重矩阵动辄几千维，求逆又慢又数值不稳）。你主要在这些地方碰到它：

- 线性回归的闭式解 `w = (XᵀX)⁻¹Xᵀy`（经典 ML）
- 理解「为什么训练用梯度下降、而不是直接求逆解出最优权重」——因为大矩阵求逆代价太高、还可能不可逆

记住「**逆 = 撤销变换、det=0 就拍扁了回不去**」这个直觉即可。

## 5. 特征值与特征向量

### 5.1 先建立直觉（公式先别看）

记住第 2 节那句话：**矩阵 = 一个「变换」**。你喂给它一个向量（箭头），它通常把箭头**又转方向、又改长短**。

但对**某些特殊的箭头**，这个矩阵会客气一点：**只把它拉长或压短，不转它的方向**（变换后还躺在原来那条直线上）。

- 这种「不被转方向」的特殊箭头 → 叫这个矩阵的**特征向量（v）**
- 它被拉长/压短的倍数 → 叫对应的**特征值（λ）**

> ⚠️ 关键：特征向量永远是**「某个矩阵的」**特征向量。同一个箭头，是 A 的特征向量，换成矩阵 B 就不一定是了——这是「箭头」和「矩阵」之间的一种**关系**，不是箭头自己的属性。所以正确说法是「**A 的**特征向量」，不能光说「特征向量」。

### 5.2 一个看得见的例子

拿第 2 节那个「x 拉 2 倍、y 拉 3 倍」的矩阵：

```
A = | 2  0 |
    | 0  3 |
```

挨个喂三个箭头进去，看谁的方向被转歪了：

```
箭头 [1, 1]（斜 45°）：  A×[1,1] = [2, 3]  → 歪到约 56° 了  ✗ 方向变了，不是特征向量
箭头 [1, 0]（躺 x 轴）：  A×[1,0] = [2, 0]  → 还躺在 x 轴    ✓ 方向没变！长度×2 → λ=2
箭头 [0, 1]（躺 y 轴）：  A×[0,1] = [0, 3]  → 还躺在 y 轴    ✓ 方向没变！长度×3 → λ=3
```

所以这个 A 有两个特征向量：`[1,0]`（λ=2）和 `[0,1]`（λ=3）。

**这下再回头读公式 `A × v = λ × v`：**

- 左边 `A × v`：动用**整个矩阵** A 去变换 v（本来是一堆乘法加法）。
- 右边 `λ × v`：只拿**一个数字** λ 去乘 v（单纯把箭头拉长）。

等号在说一件神奇的事：**对这个特殊的 v，「整个矩阵的作用」塌缩成了「乘一个数字」。** 代入验证：`A×[1,0]=[2,0]`，而 `2×[1,0]=[2,0]`，两边相等 ✓，这里 λ=2。

> 📌 小补充：λ 可以是负数。λ=-2 表示箭头被**翻转再拉长**，但它仍躺在原直线上，**这也算特征向量**。所以严谨说法是「方向不被转歪、还在原直线上」，正反向都算。

### 5.3 一个秒懂类比：地球自转轴

想象**地球自转**。转动时地面上几乎每个点都在移动——**唯独南北极那根自转轴上的点待在原地**。

- 那根**自转轴** = 特征向量（整个旋转里唯一方向不被转歪的方向）
- 轴上的点连长度都没变 → **特征值 λ=1**

「在一个变换里，找出那个不被转歪的特殊方向」——这就是在找特征向量。

> 反例印证：一个**纯旋转矩阵**（比如转 90°）**根本没有实数特征向量**，因为它把每一个方向都转歪了，没有箭头能幸免。可见「有没有特征向量、有几个」也是**矩阵自己的性质**，不同矩阵答案不同。

### 5.4 为什么重要？—— 从 PCA 降维说起

**PCA（Principal Component Analysis，主成分分析）= 砍掉重复/没用的列，保留关键信息**

具体例子：你招聘，每个候选人有 5 个指标：

```
       数学  物理  化学  编程  英语
小明   90   88   91   85   60
小红   70   68   72   95   90
小王   80   78   82   90   75
```

你发现数学、物理、化学三科总是同高同低（都是理科），用 1 列「理科综合」就能代替：

```
       理科综合  编程  英语       ← 从5列降到了3列，信息几乎没丢
小明     90     85   60
小红     70     95   90
小王     80     90   75
```

**特征向量和特征值在 PCA 里的角色**：
- **特征向量** = 从哪个方向看数据差异最大（最佳观察方向）
- **特征值** = 那个方向上能看到多少信息（值越大 = 信息越多，越值得保留）

```
5 个方向的特征值：  80, 15, 3, 1.5, 0.5

前 2 个方向就捕获了 80+15 = 95% 的信息
后 3 个方向才 5%，扔了无所谓

5 维 → 降到 2 维，信息只丢了 5%
```

> 🔑 **ML 关联**：
> - PCA 在高维 Embedding 可视化中使用（768 维降到 2 维画图）
> - 理解数据的主要变化方向

## 6. SVD 奇异值分解

### 6.1 一句话理解 ⭐⭐⭐ 必须理解

**SVD = 把一个大矩阵拆开，找出里面哪些信息重要、哪些是废话，然后扔掉废话来压缩。**

就像音乐里的音量：主旋律音量大（重要），背景噪音音量小（不重要）。把小音量静音，歌还是那首歌，但文件小了很多。

### 6.2 具体例子

5 个学生，3 门课成绩：

```
       语文  数学  英语
学生A   90   91   40
学生B   80   79   70
学生C   60   61   90
学生D   70   69   60
学生E   50   51   80
```

语文和数学几乎一模一样！SVD 发现了这个规律：

```
你的数据其实只有 2 个真正不同的"模式"：

模式1：理科能力（语文和数学高度一致） → 重要性 = 300（很大）
模式2：英语能力（和理科无关）         → 重要性 = 100（也挺大）

这些"重要性"数字就是奇异值（Σ 里的值），从大到小排列。
```

**为什么大的值 = 有用信息？** 因为大值代表数据里反复出现的强模式，小值代表随机噪音。

### 6.3 公式（🔍 知道就行，不用背）

```
A = U × Σ × Vᵀ

A: 原始矩阵（你的数据）
U: 描述"行的模式"（哪些学生表现相似）
Σ: 每个模式有多重要（从大到小排，大的重要，小的不重要）
V: 描述"列的模式"（哪些科目高度相关）
```

怎么拆的？**电脑算的，你不用管**，就像 JPEG 压缩图片你不用自己写算法。

### 6.4 低秩近似（⭐ 需要理解思想）

**先搞懂「秩（rank）」——这一节和 6.5 的 LoRA 全靠它。** 文档前面一直在用「秩」却没解释过，先补上：

> **秩 = 一个矩阵里，真正「独立」、互相替代不了的方向（维度）有几个。**

很多行/列其实是别人的翻版或组合，不带新信息；把这些重复的去掉，剩下真正独立的有几个，秩就是几。

- 回看 6.2 那张表：3 列里语文≈数学，**真正独立的只有 2 列** → 秩 = 2（不是 3）。
- 再看一个极端例子：

```
| 1  2  3 |    第2行 = 第1行×2
| 2  4  6 |    第3行 = 第1行×3   → 真正独立的方向只有 1 个 → 秩 = 1
| 3  6  9 |
```

这张 3×3 的表，其实是**两个小向量拼出来的**（9 个格子，真信息只有这俩 3 维向量）：

```
| 1 |                  | 1  2  3 |
| 2 | × [1  2  3]  =   | 2  4  6 |
| 3 |                  | 3  6  9 |
列向量            行向量        整张表
```

**「低秩」= 秩远小于矩阵尺寸** = 一张大表没多少真信息、能用两个瘦矩阵重建出来。秩越低 = 用越少的零件就能拼回整张表 = 越能压缩——这正是下面「低秩近似」和 6.5 LoRA 的全部底气。**下面要保留的「r 个奇异值」，这个 r 就是你决定留下的秩。**

只保留最大的 r 个奇异值，扔掉小的：

```
原始: 存 1,000,000 个数
只保留前 10 个最重要的模式: 存约 20,000 个数

信息保留了 95%，体积省了 50 倍！
```

### 6.5 为什么重要？⭐⭐⭐ 必须理解 → 这是 LoRA 的基础！

**LoRA (Low-Rank Adaptation)** 直接用了 SVD 的"扔掉不重要部分"思想：

```
微调大模型，权重变化量 ΔW 是 4096 × 4096 = 1600万参数
但 ΔW 里大部分值很小（不重要）

LoRA：直接用 2 个小矩阵代替

ΔW (4096×4096)  ≈  A (4096×16) × B (16×4096)
  1600万参数     →        13万参数            省了 100 倍！
```

> 👆 这就是 6.4 那个「大表 = 两个小向量拼出来」的放大版——只是把秩从 1 放宽到 16，把两个向量换成两个瘦矩阵 A、B。**ΔW 低秩 → 拆成 A×B → 省参数**，一条线串起来。

为什么行得通？微调只是让模型从"通用"稍微偏向某个任务，需要的变化很少（r=4~16 就够）。

**其他应用**（🔍 知道就行）：
- **推荐系统**：用户-物品矩阵的低秩分解
- **数据压缩**：图像、文本的降维
- **降噪**：去掉小奇异值（噪声）

## 8. 面试常问

### Q1: 为什么 Attention 要除以 √d_k？

**答**：Q·Kᵀ 的结果量级和维度 d_k 成正比。假设 Q 和 K 的每个元素独立且均值 0、方差 1，则 Q·K 的方差约为 d_k。如果不除以 √d_k，当 d_k 很大时，点积值会很大，导致 softmax 输出趋近 one-hot（一个接近 1，其他接近 0），梯度接近零，训练不稳定。

**一句话**：归一化，防止点积值过大导致 softmax 饱和。

### Q2: 为什么 LoRA 有效？低秩假设合理吗？

**答**：
- 经验观察表明，预训练模型在微调到下游任务时，权重变化量 ΔW 确实近似低秩
- 直觉上：微调只是让模型从「通用」稍微「偏向」某个任务，需要的变化很少
- 实验表明 r=4~16 就够了，说明任务适应确实只需要很少的「自由度」

### Q3: 余弦相似度和点积的区别？什么场景用哪个？

**答**：
- **余弦**：只看方向，忽略大小 → 文本语义相似度（RAG 检索）
- **点积**：同时考虑方向和大小 → Attention 计算（需要考虑向量的「强度」）
- 如果向量已归一化（单位向量），二者等价

### Q4: 什么是正交？为什么多头注意力要投影到不同子空间？

**答**：
- 正交 = 互相垂直 = 点积为 0 = 完全不相关
- 多头注意力把输入投影到多个子空间，每个头关注不同的「方面」
- 类比：看一幅画从不同角度看（色彩、构图、主题），综合起来更全面
