## 1. 向量（Vector）

### 1.1 什么是向量？

**直觉**：向量就是一组有序的数字，可以理解为空间中的一个「箭头」或一个「点」。

```
2D 向量: [3, 4]      → 平面上的一个点/方向
3D 向量: [1, 2, 3]   → 三维空间中
768D 向量: [0.12, -0.34, ..., 0.56]  → Embedding 空间中！
```

### 1.2 向量运算

#### 加法：对应元素相加

```
[1, 2] + [3, 4] = [4, 6]
```

**直觉**：两个力的合成，两个方向的叠加。

#### 数乘：缩放

```
3 × [1, 2] = [3, 6]
```

**直觉**：拉长或缩短箭头，方向不变。

#### 点积（Dot Product）⭐ 极其重要

```
[1, 2] · [3, 4] = 1×3 + 2×4 = 11
```

**直觉**：衡量两个向量的「相似度」。

- 点积 > 0 → 方向大致相同
- 点积 = 0 → 互相垂直（正交），完全不相关
- 点积 < 0 → 方向大致相反

> 🔑 **ML 关联**：Attention 机制中 Q·K 就是点积，计算 query 和 key 的相似度！

#### 余弦相似度（Cosine Similarity）⭐

```
cos(θ) = (A · B) / (||A|| × ||B||)
```

- 值域 [-1, 1]
- 1 = 完全相同方向
- 0 = 正交（不相关）
- -1 = 完全相反

> 🔑 **ML 关联**：RAG 中向量检索用余弦相似度比较 query embedding 和 document embedding！

**为什么 RAG 用余弦而不是点积？**
- 余弦只看「方向」，不看「长度」
- 同一概念的文本，不管长短，方向应该相似
- 点积会受到向量长度影响，长文本天然长度更大，会 bias 结果

### 1.3 向量的范数（Norm）

范数 = 向量的「长度」：

```
L2 范数（欧几里得）: ||v|| = √(v₁² + v₂² + ... + vₙ²)
L1 范数（曼哈顿）:   ||v|| = |v₁| + |v₂| + ... + |vₙ|
```

> 🔑 **ML 关联**：L1/L2 正则化就是对权重向量加范数惩罚，防止过拟合。

## 2. 矩阵（Matrix）

### 2.1 什么是矩阵？

**直觉**：矩阵就是一个数字的表格，更本质地说，矩阵是一个「变换」（transformation）。

```
W = | 2  0 |    → 把 x 方向拉伸 2 倍
    | 0  3 |    → 把 y 方向拉伸 3 倍
```

### 2.2 矩阵的维度

```
(m × n) 矩阵 = m 行 × n 列

权重矩阵 W: (768 × 768)  → 768 维输入变换到 768 维输出
Embedding 矩阵: (50000 × 768) → 50000 个词，每个词 768 维向量
```

> 🔑 **ML 关联**：
> - 神经网络的每一层本质就是 `y = Wx + b`（矩阵乘法 + 偏置）
> - Embedding 层本质就是一个大的查找矩阵

---

## 3. 矩阵乘法的几何直觉

### 3.1 矩阵 × 向量 = 变换

```
| a  b | × | x |   =   | ax + by |
| c  d |   | y |       | cx + dy |
```

**直觉**：把向量从一个空间「变换」到另一个空间。

> 🔑 **ML 关联**：Attention 中的 Q = X·Wq, K = X·Wk, V = X·Wv 就是把输入 X 通过不同的矩阵变换到不同的子空间。

### 3.2 矩阵 × 矩阵 = 组合变换

```
(m × k) × (k × n) = (m × n)

注意：内侧维度 k 必须匹配！
```

**直觉**：先做一个变换，再做另一个变换 = 做一个组合变换。

```
旋转 × 缩放 = 旋转后缩放
```

### 3.3 批量处理视角

```
X (batch_size × input_dim) × W (input_dim × output_dim) = Y (batch_size × output_dim)
```

**直觉**：一次性把一整批数据都做同样的变换。这就是 GPU 并行计算的基础。

## 4. 转置与逆矩阵

### 4.1 转置（Transpose）

行变列，列变行：

```
A = | 1  2  3 |      Aᵀ = | 1  4 |
    | 4  5  6 |           | 2  5 |
                          | 3  6 |
```

**性质**：(AB)ᵀ = BᵀAᵀ

> 🔑 **ML 关联**：Attention 计算 QKᵀ，就是 Q 矩阵和 K 的转置相乘，得到每对 token 之间的相似度矩阵。

### 4.2 逆矩阵（Inverse）

```
A × A⁻¹ = I（单位矩阵）
```

**直觉**：逆矩阵 = 撤销变换。如果 A 是「旋转 45°」，那 A⁻¹ 就是「反向旋转 45°」。

- 不是所有矩阵都有逆（奇异矩阵/不可逆矩阵）
- 行列式 det(A) = 0 → 矩阵不可逆（把空间压扁了，信息丧失，回不去了）

## 5. 特征值与特征向量

### 5.1 定义

```
A × v = λ × v
```

- **v** = 特征向量：被矩阵 A 变换后，方向不变的向量
- **λ** = 特征值：变换后的缩放系数

### 5.2 直觉

想象一个变换（比如拉伸）：
- 大多数向量在变换后方向会改变
- 但有些特殊方向，变换后只改变了长度，方向没变 → 这就是特征向量
- 改变了多少倍 → 这就是特征值

### 5.3 为什么重要？

- **PCA 降维**的核心：找到数据方差最大的方向 = 协方差矩阵的最大特征值对应的特征向量
- 帮助理解矩阵的本质行为

> 🔑 **ML 关联**：
> - PCA 在高维 Embedding 可视化中使用
> - 理解数据的主要变化方向

## 6. SVD 奇异值分解

### 6.1 定义

任何矩阵都可以分解为三个矩阵的乘积：

```
A = U × Σ × Vᵀ

A: (m × n) 原始矩阵
U: (m × m) 左奇异向量（正交矩阵）
Σ: (m × n) 对角矩阵，对角线上是奇异值（从大到小排列）
V: (n × n) 右奇异向量（正交矩阵）
```

### 6.2 直觉

任何线性变换都可以拆解为三步：
1. **Vᵀ** — 旋转
2. **Σ** — 沿各轴缩放（奇异值就是缩放系数）
3. **U** — 再旋转

### 6.3 低秩近似

关键洞察：如果只保留最大的 r 个奇异值，就得到了原矩阵的最佳 r 秩近似：

```
A ≈ U_r × Σ_r × V_rᵀ

原始: (m × n) → 存储 m×n 个数
近似: (m × r) + (r × r) + (r × n) → 存储远少于 m×n 个数（当 r 很小时）
```

### 6.4 为什么重要？⭐⭐⭐

> 🔑 **这是 LoRA 的数学基础！**

**LoRA (Low-Rank Adaptation)** 的核心假设：
- 微调时权重的变化量 ΔW 是低秩的
- 不需要存储完整的 ΔW (d × d)
- 而是分解为 ΔW = A × B，其中 A (d × r)，B (r × d)，r << d

```
原始参数:  W (4096 × 4096) = 约 1600 万参数
LoRA 增量: A (4096 × 16) × B (16 × 4096) = 约 13 万参数 → 节省 100x!
```

**其他应用**：
- **推荐系统**：用户-物品矩阵的低秩分解
- **数据压缩**：图像、文本的降维
- **降噪**：去掉小奇异值（噪声）

## 8. 面试常问

### Q1: 为什么 Attention 要除以 √d_k？

**答**：Q·Kᵀ 的结果量级和维度 d_k 成正比。假设 Q 和 K 的每个元素独立且均值 0、方差 1，则 Q·K 的方差约为 d_k。如果不除以 √d_k，当 d_k 很大时，点积值会很大，导致 softmax 输出趋近 one-hot（一个接近 1，其他接近 0），梯度接近零，训练不稳定。

**一句话**：归一化，防止点积值过大导致 softmax 饱和。

### Q2: 为什么 LoRA 有效？低秩假设合理吗？

**答**：
- 经验观察表明，预训练模型在微调到下游任务时，权重变化量 ΔW 确实近似低秩
- 直觉上：微调只是让模型从「通用」稍微「偏向」某个任务，需要的变化很少
- 实验表明 r=4~16 就够了，说明任务适应确实只需要很少的「自由度」

### Q3: 余弦相似度和点积的区别？什么场景用哪个？

**答**：
- **余弦**：只看方向，忽略大小 → 文本语义相似度（RAG 检索）
- **点积**：同时考虑方向和大小 → Attention 计算（需要考虑向量的「强度」）
- 如果向量已归一化（单位向量），二者等价

### Q4: 什么是正交？为什么多头注意力要投影到不同子空间？

**答**：
- 正交 = 互相垂直 = 点积为 0 = 完全不相关
- 多头注意力把输入投影到多个子空间，每个头关注不同的「方面」
- 类比：看一幅画从不同角度看（色彩、构图、主题），综合起来更全面
