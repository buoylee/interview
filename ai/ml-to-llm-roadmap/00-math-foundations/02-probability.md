# 0.2 概率与统计（Day 3-4）

> **一句话定位**：语言模型本质是概率模型 P(下一个词|前面的词)。理解概率是理解 ML 几乎所有算法的基础。

> **📌 看不懂也能往下走——本章至少带走这 4 条**（其余等后面章节用到时自然就懂，不必在此硬啃）：
> 1. **LLM 本质 = 条件概率** `P(下一个 token | 前文)`。
> 2. **模型输出 = 词表上的一个概率分布**（softmax），生成 = 从这个分布里采样（温度 / top-k / top-p 都是采样策略）。
> 3. **训练 = MLE**：拧模型的旋钮，让它对"训练数据里真实发生过的答案"最有信心。
> 4. **正则化 = MAP**：加一条"旋钮别太极端"的规矩，防止模型死记硬背（过拟合）。
>
> 这 4 条能点头，本章目的就达到了。MLE / 似然 / 交叉熵的细节会在 [04-information-theory](./04-information-theory.md) §2 和 `02-deep-learning/07-loss-functions` 用到时再讲透——到那时回看 §5 会顺很多。

## 1. 基础概念

### 1.1 概率是什么？

**直觉**：一件事发生的可能性大小，值域 [0, 1]。

```
P(A) = 事件 A 发生的可能性
P(A) = 0 → 不可能
P(A) = 1 → 必然发生
```

### 1.2 联合概率 vs 边际概率

```
联合概率:  P(A, B)  = A 和 B 同时发生
边际概率:  P(A)     = 不管 B 怎样，A 发生的总概率
          P(A) = Σ P(A, B=bᵢ)  （对 B 所有可能求和）
```

### 1.3 独立性

```
如果 A 和 B 独立:  P(A, B) = P(A) × P(B)
```

**直觉**：A 发生不影响 B 的概率。

> 🔑 **ML 关联**：朴素贝叶斯假设所有特征独立（虽然现实中很少成立，但有时效果出奇好）。

## 2. 条件概率与贝叶斯

### 2.1 条件概率

```
P(A|B) = P(A, B) / P(B)
```

**直觉**：已知 B 发生了，A 发生的概率。

> 🔑 **ML 关联**：**语言模型的核心就是条件概率！**
> ```
> P("猫" | "我家有一只可爱的") = ?
> GPT 做的就是：给定前面的词，预测下一个词的概率分布
> ```

### 2.2 贝叶斯定理 ⭐

```
P(A|B) = (P(B|A) × P(A)) / P(B)

后验 = (似然 × 先验) / 证据
```

**这个公式不用背 —— 它就是 2.1 的条件概率定义推出来的。** 关键只用到一个不起眼的事实：联合概率是对称的，`P(A, B) = P(B, A)`（"A 和 B 同时发生" 和 "B 和 A 同时发生" 当然是同一件事）。

把条件概率定义对称地写两遍，各自解出那个联合概率：

```
P(A|B) = P(A, B) / P(B)   ⟹   P(A, B) = P(A|B) × P(B)
P(B|A) = P(A, B) / P(A)   ⟹   P(A, B) = P(B|A) × P(A)
```

两个右边等于的是**同一个** P(A, B)，所以它们彼此相等：

```
P(A|B) × P(B) = P(B|A) × P(A)
```

两边同除以 P(B)，贝叶斯定理就掉出来了：

```
P(A|B) = (P(B|A) × P(A)) / P(B)
```

一张图记住本质 —— **同一个联合概率，两种拆法，中间画等号**：

```
            P(A, B)
          ↙        ↘
   P(A|B)·P(B)  =  P(B|A)·P(A)
          ↘        ↙
      一除，贝叶斯就出来了
```

> 🔑 **为什么要这样翻转？** 因为现实中 `P(A, B)` 你往往不知道，但 `P(B|A)` 和 `P(A)` 你知道。贝叶斯让你用"已知的方向 P(B|A)"算出"想要的方向 P(A|B)"——这就是它被叫做"翻转条件"的原因（下面流感例子里 P(有病→阳性) 已知、P(阳性→有病) 才是你真正想要的，正是这种翻转）。
>
> 以后看到 LLM/ML 里 `posterior ∝ likelihood × prior` 那种省掉分母的写法（见 [5.2 MAP](#52-最大后验估计map)），也能立刻还原成这张图——分母 P(B) 对所有假设是常数，所以经常被丢掉。

**具体例子：你去做流感检测，测出了阳性，你真有流感的概率是多少？**

先定义 A 和 B：

```
A = 你真的有流感（事实上有没有病）
B = 测试结果显示阳性（试剂给你的结果）
```

已知信息：

```
P(A)   = 1%    ← 先验：流感发病率1%，没测之前你大概率没事
P(B|A) = 99%   ← 似然：如果你真有流感，测出阳性的概率（试剂很准）
误报率  = 5%   ← 如果你没流感，也有5%几率误测为阳性（试剂不完美）
```

想象 10000 人一起测：

```
10000 人
  ├── 100 人真有流感（1%）
  │     └── 其中 99 人测出阳性 ✅（99%准确率）
  │
  └── 9900 人没有流感（99%）
        └── 其中 495 人也测出阳性 ❌（5%误报）

测出阳性的总人数 P(B) = (99 + 495) / 10000 = 5.94%
其中真有流感的 = 99 人
```

代入公式：

```
P(A|B) = (P(B|A) × P(A)) / P(B)
       = (99% × 1%) / 5.94%
       = 16.7%

测出阳性后，你真有流感的概率只有 16.7%！
因为本来得流感的人就很少（1%），误报的人反而更多。
```

**一句话总结**：贝叶斯就是"拿到新证据（阳性结果）后，更新你原来的判断（从 1% 升到 16.7%）"。

**贝叶斯解决的核心问题**：你知道 P(B|A)（有病→测出阳性 = 99%），但你真正想知道的是反过来的 P(A|B)（测出阳性→真有病 = 只有 16.7%）。这两个概率完全不同！

### 2.3 贝叶斯在 ML 中的应用

| 场景 | 贝叶斯怎么用 |
|------|------------|
| 朴素贝叶斯分类 | P(类别\|特征) ∝ P(特征\|类别) × P(类别) |
| 正则化 | L2 正则化 ≈ 高斯先验（后面 MAP 会解释）|
| 贝叶斯优化 | 超参数搜索时建模目标函数 |
| LLM 生成 | 本质上在做 P(next_token \| context) |

## 3. 常见概率分布

### 3.1 离散分布

#### 伯努利分布 (Bernoulli)

```
只有两个结果：成功(1) 或 失败(0)
P(X=1) = p,  P(X=0) = 1-p
```

**例子**：抛一次硬币、二分类问题的输出。

#### 多项式分布 (Categorical / Multinomial)

```
多个类别中选一个
P(X=k) = pₖ,  Σpₖ = 1
```

> 🔑 **ML 关联**：LLM 每次生成 token 就是从一个多项式分布中采样！
> ```
> 词表大小 = 50000
> 每个词都有一个概率 → softmax 输出就是这个多项式分布
> ```

### 3.2 连续分布

#### 正态分布 / 高斯分布 (Normal / Gaussian) ⭐

```
N(μ, σ²)
μ = 均值（中心位置）
σ = 标准差（分散程度）
```

**为什么到处都是正态分布？**
- **中心极限定理**：大量独立随机变量的均值趋向正态分布
- 权重初始化通常用正态分布
- 很多自然现象近似正态

> 🔑 **ML 关联**：
> - 权重初始化：N(0, 1/√n)（Xavier/He 初始化）
> - VAE 的潜空间假设为正态分布
> - Dropout 也可理解为一种噪声注入

## 4. 期望、方差、协方差

### 4.1 期望（Expectation / Mean）

```
E[X] = Σ xᵢ × P(xᵢ)         (离散)
E[X] = ∫ x × f(x) dx        (连续)
```

**直觉**：「平均来看会怎样」，加权平均。

### 4.2 方差（Variance）

```
Var(X) = E[(X - μ)²] = E[X²] - (E[X])²
标准差 σ = √Var(X)
```

**直觉**：数据有多「分散」。方差大 = 不稳定。

> 🔑 **ML 关联**：
> - **偏差-方差权衡**：模型的误差 = 偏差² + 方差 + 噪声
> - Batch Normalization 就是让每层输出的均值=0、方差=1

### 4.3 协方差（Covariance）

```
Cov(X, Y) = E[(X - μ_X)(Y - μ_Y)]
```

**直觉**：两个变量一起变化的趋势。
- 正协方差 → 一起增大
- 负协方差 → 一个增大另一个减小
- 零协方差 → 没有线性关系

**协方差矩阵**：多个变量之间的协方差组成矩阵 → PCA 就是对协方差矩阵做特征分解。

## 5. MLE vs MAP

> **这一节在解决什么?** 第 3 节讲的每个分布都留了"参数"没填:伯努利的 `p`、高斯的 `μ` 和 `σ`。现实里**没人会告诉你这些参数是多少,你手上只有一堆数据**。MLE 和 MAP 就是同一个问题的两种答案——**怎么从数据反推出参数?** 这件事,数学上就是"训练一个模型"。
>
> 用后端的话说:一个分布 = 一个带 config 的函数;你只有生产日志(数据),config 文件丢了;MLE/MAP 就是逆推出"最可能生成这批日志的 config"。

**先把 θ 说清楚。** `θ`(读 theta)是"参数"的统称,理解成**模型的旋钮**:

| 模型 | 旋钮 θ 是什么 | 几个 |
|------|------------|------|
| 抛硬币(伯努利) | 正面概率 `p` | 1 个 |
| 一个高斯分布 | 均值 `μ`、标准差 `σ` | 2 个 |
| 一个 LLM | 全部权重 | 几十亿个 |

旋钮有几个不重要,**任务永远是同一个:给定数据,把旋钮拧到"最合理"的位置。** 两种拧法就是下面的 MLE 和 MAP:

- **MLE(最大似然)**:只信数据。"哪个旋钮位置,让我手里这批数据最可能出现?"
- **MAP(最大后验)**:数据 + 先验。"我事先对旋钮有点猜测(比如它不该太大),把猜测和数据合起来一起拧。"

带着这个框架往下看,5.1 里那行公式 `argmax P(数据|θ)` 就只是一句话:把旋钮 θ 拧到"让观测数据最可能出现"的位置。

### 5.1 最大似然估计（MLE）⭐

把模型想成**一台有几百万个旋钮的机器**。给它一个输入，它不吐一个答案，而是给**每个可能的答案打一个信心分(概率)**。

举例，输入半句话 `我家有一只可爱的___`，数据里真实的下一个字是 `猫`。机器现在(旋钮乱设)是这样的：

```
猫   5%
狗   4%
桌子 3%
...(5万个字，基本在瞎猜)
```

**训练 = 拧旋钮，让机器对"数据里真实发生的那个答案(猫)"的信心越来越高：**

```
拧之前： 猫 5%
拧之后： 猫 5% → 30% → 80%
```

数据里每一条都这样拧一遍，就是训练。一句话：**让机器对"实际发生过的东西"最有信心**——这就叫 MLE(最大似然)。

写成公式就是一行，读法跟着上面的故事走：

```
θ_MLE = argmax P(数据 | θ)
 ↑        ↑            ↑
找旋钮   在 θ 上找最大   让这批数据出现的概率
```

> 下面三块是"想深一点再看"的，**第一遍可以直接跳到 5.2**；面试前或学完信息论再回来补。

#### 📐 选读 · "似然"到底是什么——和"概率"的区别

似然和概率是**同一个公式 `P(数据|θ)`，只是"谁固定、谁在变"反过来了**：

```
概率：参数 θ 固定，问"会出现什么数据？"
      硬币公平(p=0.5)，抛10次出7正的概率多大？   ← p 钉死，数据在变

似然：数据固定(已经看到了)，问"哪个参数最可能产生它？"
      已看到 7 正 3 反，是 p=0.5 还是 p=0.7 更可能产生它？  ← 数据钉死，p 在变
```

对固定的"7 正"代入不同 p，得到一条曲线：

```
p=0.3 → 很小   (偏反面的硬币，很难凑出 7 正)
p=0.7 → 最大   ← MLE 取这个峰
p=0.9 → 又变小 (太偏正面，本该出 ~9 正)
```

所以**似然不是"参数的概率"，而是"这个参数把数据解释得有多好"**。MLE 就是在 θ 上滑动，找最能解释数据的那个峰。

#### 📐 选读 · 例子里的 0.7 是算出来的，不是拍脑袋

抛 10 次硬币，7 正 3 反。模型只有一个旋钮：`p = P(正面)`。

**第一步，写出似然**——模型生成"恰好这批数据"的概率。每个正面贡献 `p`，每个反面贡献 `1−p`，独立就相乘：

```
L(p) = p⁷ × (1−p)³
```

**第二步，取 log 把连乘变连加**。log 单调，最大值位置不变，但求导容易得多——**这就是"对数似然 (log-likelihood)"无处不在的原因**：

```
ℓ(p) = log L(p) = 7·log p + 3·log(1−p)
```

**第三步，求导=0 找最大值**：

```
7/p − 3/(1−p) = 0  ⟹  7(1−p) = 3p  ⟹  p = 7/10 = 0.7
```

所以 0.7 是**最大化算出来的结果**，不是"正面频率的定义"。MLE 在这里刚好等于经验频率，是推出来的，不是假设的。

#### ⏭️ 进阶 · 从 MLE 到交叉熵：三步桥（面试必走这条链）

> ⏭️ **第一遍可跳过**：这里的"交叉熵"要到本章 [04-information-theory](./04-information-theory.md) §2 才正式定义。建议先看完下面 5.2/5.3，等学完信息论那节再回来——那时这条桥会顺很多。MLE 本身（上面"0.7 怎么算"那部分）不依赖交叉熵，已经够用。

文档里"最小化交叉熵 = 最大化似然"中间其实有三个台阶：

**台阶 1：整批数据的似然 = 每个样本概率连乘。** 模型给第 i 个样本打的概率是 `P(xᵢ|θ)`：

```
L(θ) = Π P(xᵢ|θ)
```

**台阶 2：取 log 变连加，再加负号变成"可最小化的损失"。** 优化器习惯最小化，就把"最大化 ℓ"翻成"最小化 −ℓ"，这个 `−ℓ` 叫**负对数似然 (NLL)**：

```
max  Σ log P(xᵢ|θ)   ⟺   min  −Σ log P(xᵢ|θ)   ← 负对数似然 NLL
```

到这里已经能讲"训练 = MLE = 最小化 NLL"了。

**台阶 3：one-hot 标签下，单样本 NLL 就等于交叉熵。** 真实标签是 one-hot 的 `p`（正确类=1，其余 0），模型 softmax 输出是分布 `q`：

```
H(p,q) = −Σ p(x)·log q(x)
       = −log q(正确类)             ← p 是 one-hot，只剩这一项非零
       = −log P(正确 token|context)  ← 正是这个样本的 NLL
```

整条链就没有跳跃了：

```
似然(连乘) → 取log变连加 → 加负号=NLL(可最小化) → one-hot下 NLL=交叉熵
```

> 🔑 **ML/LLM 关联**：大多数 ML 训练本质都是 MLE。
> - LLM 每个训练 token = 一次"在 5 万词表上的分类"：正确的下一个词是 one-hot 标签，softmax 是 `q`。
> - 所以 LLM 预训练损失 = 平均交叉熵 = NLL = MLE，被累加的就是一堆 `−log P(正确 token)`。

### 5.2 最大后验估计（MAP）

先看 MLE 的毛病：让旋钮**随便拧、拧到底**去死磕训练数据，模型会**死记硬背**——像一个本该通用的函数，为了通过每条测试用例拼命塞：

```java
if (input == 训练样本1) return 答案1;
if (input == 训练样本2) return 答案2;
...  // 把训练集背下来
```

训练集 100 分，**线上新数据全错**——它没学到规律，只背了答案。这叫**过拟合**，表现就是某些旋钮被拧到极端值去迁就怪样本。

**正则化 = 加一条规矩："旋钮别拧太大、别太极端。"** 逼机器用一组**温和、简单**的旋钮去解释数据——简单的解才是真规律，才扛得住新数据(奥卡姆剃刀)。

那这跟 MAP、先验有什么关系？**先验 = 你动手之前就有的信念**，这里的信念是"对的旋钮八成不会太极端"。把信念和数据**合起来**用，就从"纯迁就数据"(MLE)变成"迁就数据、但保持旋钮温和"(MAP)。一句话：**正则化 = MAP = MLE + "别太大"的先验**。

> 现在记住上面这句就够。下面的公式和细节是选读。

#### 📐 选读 · 公式与细节

写成公式，MAP 比 MLE 只多乘一个先验 `P(θ)`：

```
θ_MAP = argmax P(θ | 数据) = argmax P(数据 | θ) × P(θ)
                              似然          先验
```

**先验怎么"拉回"估计**——还是抛 10 次得 7 正：

```
纯 MLE：p = 0.7          ← 完全听数据
带"硬币通常公平"的先验：MAP 把估计往 0.5 拉 → 可能落在 0.6
   (因为"10次里7次"还不足以推翻"硬币该公平"的信念)
若抛 1000 次得 700 正：证据压倒先验 → MAP ≈ MLE ≈ 0.7
```

这就是 5.3 表里"数据少时先验帮忙约束、数据多时先验影响变小"的真实含义。用后端的话：**先验 = sane defaults，数据少时信默认值，数据多时信实测。**

**先验的形状决定正则化的类型**：
- 如果先验是高斯 N(0, σ²) → **等价于 L2 正则化**
- 如果先验是拉普拉斯 → **等价于 L1 正则化**

> 🔑 **ML 关联**：正则化的概率解释！加正则项 = 加先验假设（参数不应该太大）。

### 5.3 MLE vs MAP 对比

| | MLE | MAP |
|--|-----|-----|
| 公式 | argmax P(D\|θ) | argmax P(D\|θ)P(θ) |
| 直觉 | 只看数据 | 数据 + 先验 |
| 数据少时 | 容易过拟合 | 先验帮助约束 |
| 数据多时 | 两者趋同 | 先验影响变小 |
| ML 对应 | 无正则训练 | L1/L2 正则化 |

> 🔑 **一句话收尾**:"训练模型"在数学上就是 MLE/MAP——把旋钮 θ 拧到让数据最合理的位置;不加约束是 MLE,加了"参数别太大"这种先验就是 MAP(= 正则化)。后面 `02-deep-learning/07-loss-functions`、`06-llm-core/01-training-pipeline` 都默认你已经懂这件事。

## 7. 面试常问

### Q1: 为什么语言模型用交叉熵损失？

**答**：
- 语言模型的目标是最大化 P(正确 token | context)
  - ⚠️ 这里 P 是**模型参数 θ 的函数**（写全是 `P(正确 token | context ; θ)`，分号 `;` 读作"在参数 θ 下"——`;` 后是固定参数/旋钮，区别于 `|` 后的"条件"）：同一个 context，旋钮 θ 不同，给"正确 token"打的概率就不同。"最大化"= 调 θ 让这个概率尽量大——也就是在 θ 上做 MLE，不是去最大化一个固定的数。
- 最大化似然 = 最小化负对数似然 = 最小化交叉熵
- 交叉熵 H(p, q) = -Σ p(x) log q(x)
- 当 p 是 one-hot（真实标签），q 是模型预测时，这就等于 -log q(正确 token)
- 直觉：让模型给正确答案分配尽可能高的概率

### Q2: softmax 是什么？为什么用它？

**答**：
```
softmax(zᵢ) = e^zᵢ / Σⱼ e^zⱼ
```
- 把任意实数向量变成概率分布（所有值 > 0，和 = 1）
- 保持原来的大小关系（更大的 logit → 更高的概率）
- **Temperature 控制**：softmax(z/T)
  - T > 1 → 分布更均匀（更随机）
  - T < 1 → 分布更尖锐（更确定）
  - T → 0 → 等价于 argmax（贪婪）

> 🔑 这就是 LLM 中 Temperature 参数的数学原理！

### Q3: MLE 和 MAP 的区别？什么时候用哪个？

**答**：见上文 5.3 的对比表。简短版：
- 数据充足 → MLE 就够
- 数据少或容易过拟合 → 用 MAP（加正则化）
- 实际中：几乎所有训练都加了正则化，所以本质上都是 MAP

### Q4: 什么是采样 (Sampling) vs 贪婪解码 (Greedy Decoding)？

**答**：
- **贪婪**：每步选概率最高的 token → 确定性，但无聊重复
- **采样**：按概率分布随机选 → 多样性，但可能不连贯
- **Top-k**：只从前 k 个最高概率中采样
- **Top-p (Nucleus)**：只从累积概率 ≥ p 的最小集合中采样
- 这些都是在多项式分布上的不同采样策略
