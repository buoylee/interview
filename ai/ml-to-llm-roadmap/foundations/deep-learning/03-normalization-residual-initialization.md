# Normalization、Residual 与初始化

## 这篇解决什么问题

深层网络不只是“层数更多”。信息要穿过很多层到达输出，梯度也要从损失函数一路传回前面的层。层数变深以后，信息流可能逐层变形或丢失，梯度流可能逐层放大或衰减。

这篇只解释 Transformer Block 前需要的三个基础零件:Residual、LayerNorm/RMSNorm、初始化。它们共同解决一个问题:让深层网络在训练开始和训练过程中保持稳定的信息流和梯度流。

本篇刻意每个概念都配一个能照着算的小例子——不是为了你背数字,而是因为这些概念离开数字就只剩口号。

## 学前检查

读这篇前,你只需要知道:

- 神经网络由多层函数组合而成。
- 训练时用反向传播更新参数;梯度在层之间是“连乘”传播的(不熟先看 [反向传播、梯度消失与梯度爆炸](./02-backprop-gradient-problems.md))。
- Attention 和 FFN 是 Transformer Block 里的两个主要子层。
- 向量可以表示一个 token 当前携带的信息。

## 一个真实问题:为什么“深”这么难

上一篇说过,梯度穿过很多层时是“连乘”。光说“小于 1 会消失、大于 1 会爆炸”太抽象,我们代个数。

假设有 **100 层**,每层把回传的梯度乘上一个固定系数。理想情况是每层都乘 `1.0`,梯度原样传到底。但训练里很难恰好是 1.0,假设每层只差了 10%:

```text
每层 ×1.1  →  1.1^100 ≈ 13780        梯度爆炸:回到第 1 层时被放大近 1.4 万倍
每层 ×1.0  →  1.0^100 = 1            理想:原样传回
每层 ×0.9  →  0.9^100 ≈ 0.0000266    梯度消失:回到第 1 层时几乎归零
```

这就是后端最熟的“复利”:**每层 10% 的微小偏差,在 100 层上被指数放大成灾难。** 信号正向穿过 100 层时是同一个道理——稍微放大就爆,稍微缩小就没。

所以深层网络的核心难题不是“算不出来”,而是“数值在 100 次连乘后控制不住”。Residual、Normalization、初始化就是三个把这个连乘“稳在 1 附近”的工具。

## 核心概念

### Residual:给梯度留一条 ×1 的高速路

形式很简单:

```text
y = x + F(x)
```

`F(x)` 是某个子层(Attention 或 FFN)学到的新变换,`x` 是原始输入。先看**正向**它帮了什么:

```text
没有 residual:  y = F(x)         F 必须从零重建“整个”输出
有   residual:  y = x + F(x)     F 只需学“在 x 上改动什么”(一个 delta)
```

像 `git diff` 而不是每次重写整个文件——子层只负责改动量,原信息走直达通道,不会被层层重搬时弄丢。

但 residual 真正的杀手锏在**反向**。梯度回传时,`y = x + F(x)` 对 `x` 的导数是:

```text
∂y/∂x = 1 + ∂F/∂x
        ↑
        这个 "1" 是关键
```

回到上面 100 层的连乘:没有 residual,梯度得指望每层的 `∂F/∂x` 都恰好接近 1,否则连乘后非爆即消。有了 residual,**每层至少保底有个 `1`**——就算 `∂F/∂x` 缩到几乎为 0,梯度仍能沿着这个 `1` 原样穿过去。

这里有个容易看岔的点:那个 `1` **不是**事后在连乘结果旁边加上去的,而是**每一层被连乘的那个数,从 `∂F/∂x` 变成了 `1 + ∂F/∂x`**:

```text
没 residual:  梯度 = ∂F₁ × ∂F₂ × … × ∂F₁₀₀            被连乘的是 ∂F,中心在 0
有 residual:  梯度 = (1+∂F₁) × (1+∂F₂) × … × (1+∂F₁₀₀)   被连乘的是 1+∂F,中心在 1
```

梯度消失的病根,是 `∂F` 缩到接近 0(某层暂时没学到东西、“堵住了”)。盯着这个坏层,看它的乘数干了什么:

```text
某一层 ∂F = 0(这层卡住了)
没 residual:  乘数 = ∂F   = 0  →  × 0  →  整条链被它清零,前面全收不到梯度
有 residual:  乘数 = 1+∂F = 1  →  × 1  →  原样穿过,链没断
```

`∂F` 缩到 0 正是病发的时刻,而那一刻 `1+∂F` 恰好等于 1——**这个 `1` 像一份保险,正好在 `∂F` 失效的瞬间顶上来。** 拿 5 层、中间一层卡住算一眼:

```text
各层 ∂F = [0.5, 0.5, 0(卡住), 0.5, 0.5]
没 residual:  0.5 × 0.5 × 0 × 0.5 × 0.5 = 0        中间一个 0,全链清零
有 residual:  1.5 × 1.5 × 1 × 1.5 × 1.5 ≈ 5.06     那个卡住层只是 ×1 透传,链活着
```

同一个卡住层:没 residual 时它 `×0` 杀掉整条链,有 residual 时它 `×1` 只是让信号原样通过。`∂F` 没变,但它从“链杀手”降级成了“透明的一节”。

用后端视角:residual 就是在每层旁边并了一条**默认直达短路线**。主逻辑 `F(x)` 哪怕暂时没学好、贡献接近 0,梯度和信息也能从短路线无损通过,不会因为某一层“堵了”就让整条链断掉。这就是为什么加了 residual 之后,几百层也能训得动。

**再点破一层分工:** 没 residual 时 `∂F` 一个人兼两份工——既要**学习**(这层到底学到什么),又要**恰好停在 1 附近撑住梯度**;后者极难,任何一层失手整条链就崩。`1` 出现后,它专管“保命”,`∂F` 就只管“学习”了,缩到 0 也无所谓。

但要划清一个边界:**residual 的 `1` 只建“地板”(防消失),不建“天花板”(防暴增)。** `1+∂F` 若一直偏正,`(1+∂F)^100` 照样爆(就是开头那个 `1.1^100 ≈ 13780`)。residual 真正顺手帮的忙,是把**正向信号从一遍遍“乘”改成一块块“加”**:`y = x + F(x)` 是累加而非相乘。相乘两头都是指数(`0.9^100` 消失、`1.1^100` 暴增),累加只会线性涨——暴增就从“指数灾难”降成了“线性缓涨”。剩下把每层尺度摁回单位、别让反向梯度被 `∂F` 越乘越大,交给下面两节的 **Norm 和初始化**。

### LayerNorm:把一个 token 的向量拉回标准尺度

LayerNorm 做的事:**对一个 token 向量内部的各个维度**,算均值和方差,再归一化。注意是“一行之内”,不跨其他 token、不跨 batch。

> 30 秒补底层运算:**均值** = 把这些数加起来除以个数(平均);**方差** = 每个数偏离均值多少,平方后再平均(衡量“散得有多开”);**标准差** = 方差开根号,和原数据同量纲。

拿一个夸张点的 4 维 token 向量算一遍 `x = [2, 100, -40, 8]`:

```text
1. 均值     mean = (2 + 100 - 40 + 8) / 4 = 17.5
2. 方差     var  = [(2-17.5)² + (100-17.5)² + (-40-17.5)² + (8-17.5)²] / 4
                 = [240.25 + 6806.25 + 3306.25 + 90.25] / 4 = 2610.75
3. 标准差   std  = √2610.75 ≈ 51.1
4. 归一化   (xᵢ - mean) / std:
            [(2-17.5)/51.1, (100-17.5)/51.1, (-40-17.5)/51.1, (8-17.5)/51.1]
          = [-0.30, 1.61, -1.13, -0.19]
```

看结果:原来 `-40` 到 `100` 忽大忽小的一行,被压成了**中心在 0、散布约为 1** 的一行。下一个子层无论拿到哪个 token,输入尺度都差不多,不会因为某个 token 数值特别大就把后面冲爆——这正是上面“连乘”要的“稳在 1 附近”。

Transformer 用 LayerNorm 而不是 BatchNorm,是因为生成时 batch 大小和序列长度会变,依赖“跨样本统计”很不方便;而 LayerNorm 只看自己这一行,和 batch 怎么组无关。

真实实现里还会加 `epsilon`(防止除以 0),并带可学习的 `scale`(γ)和 `shift`(β),让模型可以在归一化之后再把尺度/中心微调回有用的范围。

### RMSNorm:省掉“减均值”,只管尺度

RMSNorm 是更轻量的变体,**不减均值**,直接用均方根(root-mean-square)控制整体尺度:

```text
rms(x) = √(各维度平方的平均)
normalized_x = x / rms(x)
```

同一个 `x = [2, 100, -40, 8]` 算一遍:

```text
1. rms  = √[(2² + 100² + (-40)² + 8²) / 4]
        = √[(4 + 10000 + 1600 + 64) / 4] = √2917 ≈ 54.0
2. x / rms = [2/54, 100/54, -40/54, 8/54]
           = [0.04, 1.85, -0.74, 0.15]
```

和 LayerNorm 对比一眼就看出区别:RMSNorm 结果的均值**不是 0**(没有第 1 步的“减均值”那步)。

```text
LayerNorm:  先减均值(挪中心)  再除标准差(调尺度)   → 中心和尺度都管
RMSNorm:    ──(跳过)──        除以 rms(调尺度)     → 只管尺度
```

很多 LLM(LLaMA 系列等)改用 RMSNorm,就是因为它少算一遍均值、少存一组 bias,**更便宜**,而实践中只控尺度通常就够稳了。

### 初始化:开局的尺度 + 打破对称

初始化管两件事,都很具体。

**第一,控制开局尺度。** 训练第一步还没学任何东西,如果权重普遍偏大,激活值会像上面 `1.1^100` 那样逐层放大;偏小则逐层衰减。所以像 Xavier/He 这类初始化,本质是按“输入维度”挑一个合适的随机方差,让信号穿过每层后尺度大致还是 1。

**第二,打破对称。** 这点最容易被一句“随机给个值”带过,代个例子就清楚了。设某一层有 3 个神经元,如果它们**初始权重完全相同**:

```text
同样的输入 x
  → 神经元 A、B、C 算出完全一样的输出
  → 反向传播给它们完全一样的梯度
  → 更新后,A、B、C 还是完全一样
  → 下一步继续一样…… 永远一样
```

3 个神经元退化成 1 个,另外 2 个白占参数。就像你起了 3 个 worker 线程、却给了同一个种子做逐字节相同的活——纯浪费。**随机初始化让它们从不同起点出发,才可能分工学不同的模式。**

所以初始化负责“开局尺度对、起点各不相同”;Residual 负责“给梯度留一条永不消失的 `×1` 直达路”(防消失);Norm 负责“开局之后把每层尺度摁回单位、不让它在很多 block 之间指数涨”(防暴增)。

### Pre-Norm:把 Norm 放在子层之前,别挡住 residual 高速路

把 Norm 放哪,直接决定 residual 那条 `×1` 高速路干不干净。对比两种放法:

```text
Post-Norm:  x = Norm( x + Attention(x) )      Norm 套在“加完之后”的整条 residual 流上
Pre-Norm:   x = x + Attention( Norm(x) )      Norm 只套在支路上,residual 流原样穿过
```

差别在哪:**Post-Norm 每层都把累加起来的 residual 流重新归一化一遍**,等于在那条高速路上每隔一层就架一道收费站,深了之后信号被反复压缩,梯度高速路被切断;**Pre-Norm 只在支路入口做归一化**,主干那条 `x + …` 的高速路从头到尾不被 Norm 碰,上一节说的 `+1` 直达性质一路保到底。所以深层 Transformer 普遍用 Pre-Norm——不是摆放细节,是它保住了 residual 的命根子。

还要注意 Pre-Norm 下 residual stream 是**逐步更新**的:

```text
x = x + Attention(Norm(x))     # Attention 先更新 x
x = x + FFN(Norm(x))           # FFN 拿“更新后的 x”继续加
```

不是 Attention 和 FFN 从同一个旧 `x` 并行跑,而是串行地往同一条 residual 流上累加。

## 最小心智模型

把深层网络想成一条很长的传送带,目标是让信号“稳在 1 附近”地走完几百层:

- **Residual** = 每层旁边并一条默认直达短路线:信息不必每层重搬,梯度永远有条 `×1` 的路可走。
- **LayerNorm/RMSNorm** = 每隔一段给信号做尺度校准:把忽大忽小的一行拉回“中心 0、散布 1”,不让它在连乘里失控。
- **初始化** = 开机时的力度设定:不推爆、不传不动,且每个零件起点都不同,能分工。
- **Pre-Norm** = 把校准器装在支路上、别装在主传送带上:主干高速路保持畅通。

一句话分工:**Residual 管下限(防消失)——把连乘的地板钉在 `1`;Norm + 初始化管上限(防暴增)——把每层尺度摁回单位、不让它指数涨。**

## 和 Transformer 的连接

Transformer Block 通常由 Attention、FFN、Residual Add 和 Norm 组合而成。抓住这条关系即可:

```text
稳定的 token 表示 -> Norm 校准 -> Attention/FFN 变换 -> Residual 加回 -> 下一层继续
```

现代 LLM block 经常使用 Pre-Norm,并在 LayerNorm / RMSNorm 中选一种。读 [Transformer Block](../../04-transformer-foundations/05-transformer-block.md) 时,重点盯三件事:Norm 放在子层前面还是后面、Residual 从哪里加回去、Attention/FFN 怎么被包在这个稳定结构里。

## 常见误区

| 误区 | 修正 |
|------|------|
| Residual 只是“把输入加回来” | 它给梯度留了条 `∂y/∂x = 1 + ∂F/∂x` 的 `×1` 直达路,深层才训得动 |
| Residual 也能挡住梯度暴增 | 它只建“地板”防消失;暴增归 Norm + 初始化(+ 梯度裁剪)管。residual 把正向信号从“乘”改成“加”,顺手把暴增从指数降到线性 |
| LayerNorm 是跨 batch 做归一化 | LayerNorm 是在一个 token 这“一行”内部归一化,和 batch 怎么组无关 |
| RMSNorm 和 LayerNorm 完全无关 | RMSNorm 是省掉“减均值”的轻量版,只用均方根控尺度 |
| 初始化只是随机给个值 | 它要控开局尺度,还要打破对称——权重全相同的神经元会永远学成一样 |
| Pre-Norm 是无关紧要的摆放细节 | Norm 放支路(Pre)能保住 residual 高速路,放主干(Post)会层层切断它 |

## 自测

1. 100 层、每层梯度 ×0.9,回到第 1 层大约剩多少?这说明了什么问题?
2. 在 `y = x + F(x)` 里,反向传播时那个保底的 `1` 是从哪来的、为什么重要?
3. 对 `x = [4, 4, 4, 4]` 做 LayerNorm 会得到什么?(提示:先算均值和方差)这暴露了什么边界情况,为什么需要 `epsilon`?
4. RMSNorm 相比 LayerNorm 省掉了哪一步?所以它的输出和 LayerNorm 比,少了什么性质?
5. 一层里所有神经元初始权重相同会怎样?为什么这等于浪费参数?
6. 为什么 Pre-Norm 比 Post-Norm 更能保住 residual 的梯度高速路?
7. residual 的 `1` 解决了梯度消失,那梯度/激活的**暴增**靠什么解决?为什么说 residual 把“乘”改成“加”,顺手也帮了暴增一把?
