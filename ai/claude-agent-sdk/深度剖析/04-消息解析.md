# 04 - 消息解析：从 JSON 到类型安全

> `parse_message()` 是 SDK 的"翻译官"。
> CLI 输出的 raw JSON dict 经过它变成类型化的 Python dataclass。
> 本文逐行走读消息解析器和类型系统。

---

```

InternalClient.process_query()      ← 或 ClaudeSDKClient.receive_messages()
```

## parse_message() — 主入口

源码：`_internal/message_parser.py:29-251`

```python
def parse_message(data: dict[str, Any]) -> Message | None:
    if not isinstance(data, dict):
        raise MessageParseError(f"Invalid type (expected dict, got {type(data).__name__})")

    message_type = data.get("type")
    if not message_type:
        raise MessageParseError("Message missing 'type' field")

    match message_type:
        case "user":          ...
        case "assistant":     ...
        case "system":        ...
        case "result":        ...
        case "stream_event":  ...
        case "rate_limit_event": ...
        case _:
            logger.debug("Skipping unknown message type: %s", message_type)
            return None
```

**7 种消息类型**：

| type | 返回类型 | 说明 |
|------|---------|------|
| `user` | `UserMessage` | 用户消息（含工具结果回传） |
| `assistant` | `AssistantMessage` | 助手消息（文本/思考/工具调用） |
| `system` | `SystemMessage` / `TaskStartedMessage` / `TaskProgressMessage` / `TaskNotificationMessage` | 系统通知 |
| `result` | `ResultMessage` | 执行结果（费用、轮次、session_id） |
| `stream_event` | `StreamEvent` | 流式事件（部分消息更新） |
| `rate_limit_event` | `RateLimitEvent` | 速率限制状态变更 |
| 其他 | `None` | 前向兼容——跳过未知类型 |

> **前向兼容设计**：未知 `type` 返回 `None` 而不是抛异常。
> 这样新版 CLI 增加消息类型时，老版 SDK 不会崩溃。

## 分支 1：user — 用户消息

```python
case "user":
    parent_tool_use_id = data.get("parent_tool_use_id")
    tool_use_result = data.get("tool_use_result")
    uuid = data.get("uuid")

    if isinstance(data["message"]["content"], list):
        user_content_blocks = []
        for block in data["message"]["content"]:
            match block["type"]:
                case "text":
                    user_content_blocks.append(TextBlock(text=block["text"]))
                case "tool_use":
                    user_content_blocks.append(ToolUseBlock(
                        id=block["id"], name=block["name"], input=block["input"]
                    ))
                case "tool_result":
                    user_content_blocks.append(ToolResultBlock(
                        tool_use_id=block["tool_use_id"],
                        content=block.get("content"),
                        is_error=block.get("is_error"),
                    ))
        return UserMessage(content=user_content_blocks, uuid=uuid, ...)
    # content 是 str → 直接使用
    return UserMessage(content=data["message"]["content"], uuid=uuid, ...)
```

**UserMessage 的 content 有两种形式**：
1. `str` — 简单文本
2. `list[ContentBlock]` — 结构化内容（包含 text、tool_use、tool_result）

> **为什么 user 消息会包含 tool_use？** 当 CLI 使用 `--replay-user-messages` 时，
> 用户消息会包含完整的对话历史，其中可能有工具调用的上下文。

## 分支 2：assistant — 助手消息

```python
case "assistant":
    content_blocks = []
    for block in data["message"]["content"]:
        match block["type"]:
            case "text":
                content_blocks.append(TextBlock(text=block["text"]))
            case "thinking":
                content_blocks.append(ThinkingBlock(
                    thinking=block["thinking"],
                    signature=block["signature"],
                ))
            case "tool_use":
                content_blocks.append(ToolUseBlock(
                    id=block["id"], name=block["name"], input=block["input"]
                ))
            case "tool_result":
                content_blocks.append(ToolResultBlock(
                    tool_use_id=block["tool_use_id"],
                    content=block.get("content"),
                    is_error=block.get("is_error"),
                ))

    return AssistantMessage(
        content=content_blocks,
        model=data["message"]["model"],
        parent_tool_use_id=data.get("parent_tool_use_id"),
        error=data.get("error"),
        usage=data["message"].get("usage"),
    )
```

**4 种 content block**：

| block.type | dataclass | 说明 |
|-----------|-----------|------|
| `text` | `TextBlock(text)` | 文本输出 |
| `thinking` | `ThinkingBlock(thinking, signature)` | 扩展思考（带签名验证） |
| `tool_use` | `ToolUseBlock(id, name, input)` | 工具调用请求 |
| `tool_result` | `ToolResultBlock(tool_use_id, content, is_error)` | 工具执行结果 |

**AssistantMessage 的额外字段**：
- `model`：使用的模型（如 `claude-sonnet-4-20250514`）
- `parent_tool_use_id`：如果消息是子 Agent 的回复，指向父工具调用
- `error`：认证失败、限流、无效请求等错误类型
- `usage`：token 使用量

> **ThinkingBlock.signature**：这是 Claude 扩展思考的签名，用于验证思考内容的完整性。
> SDK 只是原样传递，不做验证。

## 分支 3：system — 系统消息

```python
case "system":
    subtype = data["subtype"]
    match subtype:
        case "task_started":
            return TaskStartedMessage(
                subtype=subtype, data=data,
                task_id=data["task_id"],
                description=data["description"],
                uuid=data["uuid"],
                session_id=data["session_id"],
                tool_use_id=data.get("tool_use_id"),
                task_type=data.get("task_type"),
            )
        case "task_progress":
            return TaskProgressMessage(
                subtype=subtype, data=data,
                task_id=data["task_id"],
                description=data["description"],
                usage=data["usage"],  # {total_tokens, tool_uses, duration_ms}
                ...
            )
        case "task_notification":
            return TaskNotificationMessage(
                subtype=subtype, data=data,
                task_id=data["task_id"],
                status=data["status"],  # "completed" | "failed" | "stopped"
                output_file=data["output_file"],
                summary=data["summary"],
                ...
            )
        case _:
            return SystemMessage(subtype=subtype, data=data)
```

**系统消息的子类层次**：

```
SystemMessage(subtype, data)
  ├── TaskStartedMessage(task_id, description, uuid, session_id, ...)
  ├── TaskProgressMessage(task_id, description, usage, ...)
  └── TaskNotificationMessage(task_id, status, output_file, summary, ...)
```

> **继承设计**：Task* 消息继承自 `SystemMessage`，所以 `isinstance(msg, SystemMessage)`
> 仍然匹配所有系统消息。这保持了向后兼容——老代码处理 SystemMessage 不会遗漏 Task 消息。

## 分支 4：result — 执行结果

```python
case "result":
    return ResultMessage(
        subtype=data["subtype"],          # "success" | "error"
        duration_ms=data["duration_ms"],   # 总耗时
        duration_api_ms=data["duration_api_ms"],  # API 调用耗时
        is_error=data["is_error"],
        num_turns=data["num_turns"],       # 对话轮次
        session_id=data["session_id"],
        stop_reason=data.get("stop_reason"),
        total_cost_usd=data.get("total_cost_usd"),
        usage=data.get("usage"),           # token 使用详情
        result=data.get("result"),         # 文本结果
        structured_output=data.get("structured_output"),  # 结构化输出
    )
```

**ResultMessage 是会话结束的标志**。`ClaudeSDKClient.receive_response()` 收到 ResultMessage 就停止迭代。

> **structured_output**：当 `ClaudeAgentOptions.output_format` 设置了 JSON Schema 时，
> CLI 会验证输出并返回结构化数据到这个字段。

```python
case "rate_limit_event":
    info = data["rate_limit_info"]
    return RateLimitEvent(
        rate_limit_info=RateLimitInfo(
            status=info["status"],             # "allowed" | "allowed_warning" | "rejected"
            resets_at=info.get("resetsAt"),     # Unix 时间戳
            rate_limit_type=info.get("rateLimitType"),  # "five_hour" | "seven_day" | ...

            overage_status=info.get("overageStatus"),
            overage_resets_at=info.get("overageResetsAt"),
            overage_disabled_reason=info.get("overageDisabledReason"),

        session_id=data["session_id"],

```

> **三种速率限制状态**：
> - `allowed`：正常
> - `allowed_warning`：接近限额，可以提前告知用户
> - `rejected`：已达限额，请求被拒绝

## 类型系统 — Content Blocks

源码：`types.py:734-767`

```python
@dataclass
class TextBlock:
    text: str

@dataclass
class ThinkingBlock:
    thinking: str
    signature: str

@dataclass
class ToolUseBlock:
    id: str
    name: str
    input: dict[str, Any]

@dataclass
class ToolResultBlock:
    tool_use_id: str
    content: str | list[dict[str, Any]] | None = None
    is_error: bool | None = None

ContentBlock = TextBlock | ThinkingBlock | ToolUseBlock | ToolResultBlock
```

**ContentBlock 的使用场景**：

```python
async for msg in query(prompt="..."):
    if isinstance(msg, AssistantMessage):
        for block in msg.content:
            if isinstance(block, TextBlock):
                print(block.text)                    # 文本输出
            elif isinstance(block, ThinkingBlock):
                print(f"[思考] {block.thinking}")    # 扩展思考
            elif isinstance(block, ToolUseBlock):
                print(f"[工具] {block.name}({block.input})")  # 工具调用
            elif isinstance(block, ToolResultBlock):
                print(f"[结果] {block.content}")     # 工具结果
```

## 类型系统 — Message Union

源码：`types.py:950-957`

```python
Message = (
    UserMessage
    | AssistantMessage
    | SystemMessage
    | ResultMessage
    | StreamEvent
    | RateLimitEvent
)
```

**使用 pattern matching**：

```python
async for msg in query(prompt="..."):
    match msg:
        case AssistantMessage(content=blocks):
            for block in blocks:
                ...
        case ResultMessage(is_error=False, total_cost_usd=cost):
            print(f"完成，花费 ${cost}")
        case ResultMessage(is_error=True):
            print("执行出错")
        case SystemMessage(subtype="init"):
            print("初始化完成")
        case RateLimitEvent(rate_limit_info=info) if info.status == "rejected":
            print("已达速率限制")
```

## 错误处理

```python
class MessageParseError(ClaudeSDKError):
    def __init__(self, message: str, data: dict | None = None):
        self.data = data     # 保留原始数据用于调试
        super().__init__(message)
```

**三种错误场景**：
1. `data` 不是 dict → `MessageParseError`
2. 缺少 `type` 字段 → `MessageParseError`
3. 某种类型内缺少必需字段 → `MessageParseError`（携带原始 data）

> **注意**：未知 message type 不会抛错，而是返回 None。
> 只有**已知类型但字段不完整**才抛 MessageParseError。

## 与 OpenAI Agents SDK 的对比

| 维度 | OpenAI `process_model_response` | Claude `parse_message` |
|------|-------------------------------|----------------------|
| **输入** | LLM 原始 response.output 列表 | CLI stdout JSON dict |
| **输出** | ProcessedResponse（分类 + 执行计划） | 单个 Message 对象 |
| **复杂度** | 400+ 行，15+ 种 output type | 250 行，7 种 message type |
| **职责** | 分类 + 决定下一步 | 纯类型转换 |
| **工具处理** | 识别工具调用并安排执行 | 只记录工具调用（执行在 CLI 内） |
| **同步/异步** | 同步函数 | 同步函数 |
| **错误处理** | ModelBehaviorError | MessageParseError |

**根本区别**：OpenAI 的 `process_model_response` 是"分拣中心"——它决定哪些需要执行；
Claude 的 `parse_message` 是"翻译官"——它只做格式转换。工具执行、循环控制都在 CLI 内部。
