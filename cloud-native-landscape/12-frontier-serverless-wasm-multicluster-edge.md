## 1. Serverless / Eventing(Knative)

- **是什麼**:在 k8s 上提供 FaaS 式體驗——**請求驅動、scale-to-zero、事件觸發**。**Knative Serving**(自動擴縮到 0、按請求拉起)+ **Knative Eventing**(基於 **CloudEvents** 的事件路由)。
- **解決什麼**:突發/低頻 workload 不想常駐燒資源;事件驅動架構(消息來了才跑)。你在 ch10 見過它——**KServe 的 scale-to-zero 底層就是 Knative**。
- **該不該關心**:✅ 中等。有「閒置就該縮到 0」的服務(實驗模型、低頻 webhook、批觸發)就用;常駐高頻服務用不上。
- **取捨**:scale-to-zero 省錢 vs 冷啟動延遲(同 ch10 的權衡)。

---

## 2. WASM(WebAssembly)

- **是什麼**:比容器**更輕、更快冷啟動、更強沙箱**的運行單元。生態:**runwasi**(用 containerd shim 跑 WASM,回扣 ch02)、**SpinKube**、**wasmCloud**;標準是 **WASI**。
- **解決什麼**:容器冷啟動還是太慢/太重的場景——邊緣、插件系統、海量短任務、不可信代碼沙箱。**毫秒級冷啟動 + 跨平台字節碼**。
- **該不該關心**:🟡 觀望偏早。生態仍在成熟,系統調用/生態庫支持有限;**不是容器的普遍替代**,而是特定場景補充。知道它存在、它解決冷啟動與沙箱即可。
- **和容器的關係**:不是取代,是**多一種運行時選項**——k8s 通過 RuntimeClass(回扣 ch02)也能調度 WASM workload。

---

## 3. 多集群 / Fleet

- **是什麼**:管理**多個 k8s 集群**的生命週期與跨集群 workload 分發。**Cluster API(CAPI)**:用聲明式 CRD 管「集群本身」的創建/升級(集群也成了被 reconcile 的對象,回扣 01);**Karmada / Fleet**:跨集群調度與策略。
- **解決什麼**:
  - **規模超單集群上限**(回扣 ch01:~5000 節點 / etcd 限制)→ 拆多集群。
  - **爆炸半徑隔離**:一個集群炸了不連坐全部。
  - **地理就近 / 數據主權 / 多雲**:不同區域/雲各一套。
- **該不該關心**:✅ 規模/合規到了就必須。中小規模單集群 + 多 namespace 足矣——**別過早多集群**(運維複雜度陡增)。
- **取捨**:隔離與彈性 vs 運維複雜度、跨集群一致性難題(又回到 system-design 的分布式一致性)。

---

## 4. 邊緣(Edge)

- **是什麼**:把 k8s 伸到**資源受限、網絡不穩、地理分散**的邊緣節點。**K3s**(輕量 k8s,單二進制,kine 替 etcd,回扣 ch01)、**KubeEdge**(雲邊協同,邊緣自治)。
- **解決什麼**:IoT / 門店 / 工廠 / 車載——要在靠近數據源處算,且斷網也能自治。**這和你的 robotics 方向強相關**:機器人/具身智能的邊緣推理就是這格。
- **該不該關心**:✅ 若你做 robotics/邊緣 AI,這是必修;純後端可暫緩。
- **取捨**:邊緣自治 vs 中心管控;弱網下的最終一致;邊緣資源受限(故 K3s/WASM 在這格吃香)。

---

## 🏛 架構師視角(這環只給「何時看這格」)

| 信號 | 該看哪格 |
|---|---|
| 服務閒置時想縮到 0 / 事件驅動 | Serverless(Knative) |
| 容器冷啟動/體積仍嫌重、要強沙箱 | WASM(觀望) |
| 撞單集群規模上限 / 要隔離爆炸半徑 / 多雲多區 | 多集群(Cluster API + Karmada) |
| 要在門店/工廠/機器人/弱網處算 | 邊緣(K3s / KubeEdge) |

> **共同判斷**:這四格都是「**有了明確驅動力才上**」——別因為新潮就引入,每一格都顯著增加複雜度。前十環是大多數團隊的主戰場;這環是按需岔路。

---

## 🧵 示例服務在這一環(各取一例)

- **Serverless**:order 系統的「生成訂單 PDF」是低頻突發任務 → 做成 Knative 服務,沒請求時縮到 0,不常駐燒資源。
- **多集群**:order 系統做大了,華東/華南各一套集群(就近 + 隔離),用 Karmada 統一分發 `order-api`。
- **邊緣**:若 order 系統延伸到門店自助機,用 K3s 在門店本地跑一個輕量副本,斷網也能下單,聯網再同步。
