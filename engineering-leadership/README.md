# 資深開發的領導力與行為面試

> 投開發崗(IC),卻被問「帶過人嗎 / 怎麼分配工作 / 怎麼拆需求 / 工作流程 / 衝突怎麼處理」?
> 這個 track 教你**兩件事一次拿到**:① 把這些事**真的做好**(工作週一能用);② 被問到時**講成加分故事**(照著練就能上場)。
>
> 寫給沒正式帶過人的資深 IC——不教你假裝當經理,教你把已有的經驗**翻譯成 leadership 訊號**。

## 怎麼用

- **第一次**:按 00→09 順序讀,每章末的 **drill 一定動手填**(不填就退化成嘴上說說)。
- **面試前**:讀 `面试卡-cheatsheet.md` + 你自己建的故事銀行。
- **想練**:任何時候跟我說「mock X 題」,我當面試官問你、追問、用 5 維度當場打分。

## 一條貫穿全程的元原則

**縮短反饋環、讓風險早而便宜地暴露**——拆活(03)、對齊(07)、流程(05)底層是同一件事。看懂這條,整個 track 就串起來了。
