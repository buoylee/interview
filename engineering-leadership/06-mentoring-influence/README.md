# 06 · 帶人 & 影響力

> 02 章教你把「沒帶過人」翻譯成訊號;這章教你**真的把人帶好、真的影響別人**——方法論層面。學完,你工作中帶新人/做 review/推動事情會更有章法,面試講「帶人」時也有真貨。
>
> 核心一句:**你沒有頭銜,但你可以有影響力——而影響力比頭銜更值錢。** 這章就講影響力到底怎麼來。

---

# Part A · 方法論(工作真戰力)

## A0 收口地圖:帶人與影響的 5 條核心原語

| # | 原語 | 一句話 |
|---|---|---|
| 1 | **教釣魚,不給魚** | 好 mentor 讓對方下次不用你,不是當他的拐杖 |
| 2 | **code review 是最高頻的教學/影響槓桿** | review 傳遞 why、建標準、共享 ownership,不只挑錯 |
| 3 | **無頭銜影響力 = 可信度 + 利他 + 講清楚** | 靠專業和信任,不靠權力 |
| 4 | **先理解,再被理解** | 先聽懂對方/現狀,你的建議才進得去 |
| 5 | **影響力是複利** | 靠長期一致的可信積累,不是單次說服技巧 |

> **收口金句**:帶人不是「指揮」,是「**讓別人變強**」;影響不是「說服贏」,是「**讓對的方向發生**」。兩者都不需要你有權力,只需要你可信、利他、能把話講清。

---

## A1 黑盒內幕:影響力到底從哪來?

### 為什麼「無頭銜影響力」是更高級的能力
有頭銜的人讓你做事,你照做是因為**他有權力**(不做有後果)。沒頭銜的人讓你做事,你照做是因為**你信他**(他通常是對的、他是為了把事做好、他講得有道理)。後者難得多,也值錢得多——它證明你的影響來自**專業資本**,走到哪都帶得走,不依賴位置。00 章說的「leadership-readiness」,核心就是這個。

### 影響力的三根支柱
別人為什麼會被你影響?拆開就三樣:
1. **可信度(credibility)**:你過去說的話、做的判斷,被驗證是靠譜的。可信度是**存款**——平時一次次靠譜地交付、靠譜地預警,存進去;要影響別人時才提得出來。
2. **利他(intent)**:對方相信你是**為了把事做好 / 為了他好**,不是為了你自己出風頭 / 甩鍋。意圖一旦被懷疑,再對的話也進不去。
3. **講清楚(clarity)**:把「為什麼該這樣」講得讓對方能懂、能算明白賬(07 章對 PM 算返工成本那招)。

> **這對你超關鍵**:你沒頭銜,但這三樣**你全都能攢**——靠譜地做事(可信)、真心為事好(利他)、把道理講清(clarity)。這就是你的領導力地基。

### 「先理解再被理解」——為什麼你的好建議常常沒人聽
工程師最常犯的影響錯誤:**還沒搞懂對方/現狀,就急著給自己的方案**。對方感覺「你根本不懂我的處境就來指點」,本能防衛,你再對也白搭。

正確順序(Stephen Covey 的習慣):**先理解(聽懂他的處境、他的顧慮、他為什麼這麼做),再被理解(這時你的建議才接得進去)。** 07 章「挖 interest 不懟 position」是這條的應用——先懂他要什麼,再給方案。

## A2 標準動作

### 帶人(mentoring)的標準動作

**動作一:用提問代替給答案。** 新人來問「這個怎麼弄」,別直接給代碼。問他:「你卡在哪?你試過什麼?你覺得可能是什麼?」——引導他自己走到答案。**他自己想通的,才是他的;你直接給的,下次他還來問。**(原語 1:教釣魚)

**動作二:給挑戰 + 安全網**(和 04 章 stretch task 同源)。給新人一個夠得著的難一點的任務,讓他扛,但讓他知道「卡住隨時找我、出事我兜」。成長發生在「踮腳夠得著」的地方。

**動作三:具體、對事、即時的反饋**(用 01 章 SBI)。表揚要具體(「你這個邊界處理考慮得很周到」比「你真棒」有用),批評用 SBI(對行為不對人)。

### code review 當教學槓桿(你最現成的工具)

你說你「偶爾 review」。把 review 從「挑錯」升級成「教 + 影響」,你就在帶人了:
- **講 why,不只 what**:不說「這裡改成 X」,說「**這裡用 X 會更好,因為**……」——對方學到的是可遷移的原則,不是一次性的修改。
- **分清「必須改」和「建議」**:用 `nit:`(小建議)、`question:`(我想懂)、`blocking:`(這個得改)標清楚,別讓對方猜哪些是硬要求。
- **也問問題,不只下指令**:「這裡為什麼用這個方案?」有時是你不懂(你學到),有時引導他自己發現問題。
- **態度對事**:review 的是代碼不是人。「這個函式」別說成「你這人寫得」。

### 無頭銜影響力的標準動作
1. **先用小事攢可信度**:別等大事才出手,平時靠譜地交付、準確地預警,存款。
2. **影響前先理解**:聽懂現狀和對方顧慮(原語 4)。
3. **用利弊講,不用立場壓**:「這樣做我們能省 X / 避免 Y」,而不是「應該這樣」(07 章對 PM 算賬)。
4. **先做出來給人看**:沒人聽你說?做一個小原型/小改進證明它行——**demo 比 argue 有說服力**。

## A3 走一遍:帶一個卡了三天的新人

**情境**:新人 B 一個任務卡了三天,來找你,眼看要 delay。

- **❌ 反面**:你嫌他慢,直接把代碼寫給他/告訴他答案。→ 任務是趕上了,但他啥也沒學到,下個任務繼續卡、繼續來找你,你變成他的拐杖。
- **✅ 正面**:
  1. **先理解**:「你卡在哪一步?說說你的思路。」——發現他卡在不會用調試工具定位問題。
  2. **教釣魚**:不直接給答案,陪他用一次調試工具走一遍,讓他自己看到問題在哪。
  3. **留安全網**:「以後這類定位問題你應該能自己來了,真卡住再找我。」
  4. **SBI 鼓勵**:「你今天學會 debug 這套,以後能省很多時間,進步很快。」
- **結果**:任務趕上了,**而且**他學會了一個可遷移的技能,下次不用你了——你真正放大了一個人。**這才是「帶人」,也是面試裡最香的故事。**

## A4 自評量表(rubric)

判斷「你帶人/影響做得好不好」:
- ☐ 別人來問,我是**給答案**還是**引導他自己想通**?
- ☐ 我的 code review 講了 **why**,還是只標了 what?
- ☐ 我影響別人前,有沒有**先理解**他的處境/顧慮?
- ☐ 我靠的是**講清利弊 + 先做出來**,還是「我覺得應該」這種空話?
- ☐ 對方有沒有因為我**真的變強 / 真的改變了方向**(而不只是當下聽了)?

# Part B · 面試轉化(應答戰力)

## B1 這題在考什麼

「你怎麼帶新人 / 怎麼影響別人 / 怎麼推動一件沒人聽的事」——主探 **Influence + Collaboration**,外加 leadership-readiness。配合 02 章的誠實定位一起用:02 解決「你帶過人嗎」的開場,06 提供「你具體怎麼帶/怎麼影響」的細節故事。

## B2 STAR:薄 vs 厚

**事件**:推動團隊加上 code review(原本沒有)。

- **❌ 薄版**:「我覺得我們該做 code review,我就跟大家說了,後來就做了。」(沒有影響過程、像是被動發生)
- **✅ 厚版**:
  - **S**:「我們團隊當時沒有 code review,線上 bug 偏多,但提『加 review』大家會嫌慢、抵觸。」
  - **T**:「我想推動 review 文化,但我沒有權力強制,只能靠影響。」
  - **A**:「**我**沒有開會要求大家做。**我**先自己對幾個同事的 PR 主動寫了高質量 review——只標 why、態度對事、抓到了兩個會上線的 bug。大家**看到了好處**。然後**我**在 retro 輕量提議『要不要試兩週互相 review』,並把規則訂得很輕(只 review 核心邏輯、不糾結風格)。」
  - **R**:「兩週後線上 bug 明顯少了,團隊自己決定保留下來。後來 review 成了我們的標配,新人也透過 review 學得很快。」

**5 維度**:Influence ✓✓(無頭銜、先做出來、用效果說話)、Collaboration ✓、Ownership ✓、Judgment ✓(規則訂輕降低阻力)。

## B3 常見追問 & 接法
1. **「如果你帶的人就是學不會 / 不上進呢?」** → 「我先區分是不會還是不願。不會,我調整教法、拆更小步;不願,我先搞懂他卡在哪(沒興趣?有顧慮?),而不是硬推。但我也誠實:我能提供環境和引導,真正的內驅得靠他自己,我不會替他成長。」(成熟、不全攬也不甩鍋)
2. **「你影響別人,對方就是不買賬怎麼辦?」** → 「argue 沒用時我會先做出來給他看,demo 比嘴有力;還不行可能是我沒真正理解他的顧慮,我會回去先聽。實在是職責邊界內他拍板,我 disagree-and-commit(07 章)。」
3. **「帶人花你時間,影響你自己產出嗎?」** → 「短期會,但這是投資——他起來了,我被放大了(04 章委派邏輯)。我不會因為『自己做更快』就不帶人。」

## B4 紅線
- 🚫 帶人 = 「替他把事做完」(拐杖型,反成長)。
- 🚫 影響 = 「我跟他們說了」(沒有真正的影響過程/效果)。
- 🚫 靠抱怨/情緒/拉關係影響,而不是可信度+利他+講清楚。
- 🚫 把帶人說成負擔(「帶人耽誤我」)——暴露你沒有 multiplier 意識。

## 本章小結

- 你沒頭銜,但你能有**影響力**,而影響力比頭銜值錢——它證明你靠**專業資本**,不靠位置。
- 影響力三支柱:**可信度(平時攢的存款)+ 利他(意圖被信任)+ 講清楚**。這三樣你全攢得起。
- 帶人核心:**教釣魚不給魚**、給挑戰+安全網、用 SBI 反饋;**code review 是你最現成的教學/影響槓桿**(講 why、分清必須改/建議、對事不對人)。
- 影響前**先理解再被理解**;argue 沒用就**先做出來給人看**(demo > 嘴)。
