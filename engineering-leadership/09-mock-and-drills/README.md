# 09 · 模擬問答 & 追問演練

> 你有了方法論(03-07)、有了故事銀行(08)。這章把它們**練成肌肉**:完整題庫讓你掃盲、面試當天 playbook 讓你臨場不慌、追問演練讓你經得起挖。
>
> 讀法:先用題庫自測哪些題會卡,回去補對應的卡;再把 playbook 練熟;最後找人(或對著手機)真演一遍。

---

## 1. 完整題庫(按 01 章 6 大題類)

每題先想:**這在考哪 2 條維度?我從銀行裡掏哪張卡?** 掏不出的,就是你的缺口。

### 領導 / 影響(Influence + Collaboration)
- 你帶過人嗎?怎麼帶的?
- 你沒有頭銜,怎麼影響別人 / 推動事情?
- 講一個你 mentor 別人的例子。
- 你怎麼讓一個團隊變得更好?

### 協作 / 衝突(Collaboration + Communication)
- 講一個你和同事 / 產品經理的衝突,怎麼處理的?
- 和你合作最難的人是什麼樣?你怎麼應對?
- 你被別人誤解 / 不配合時怎麼辦?

### 當責 / 模糊(Ownership + Judgment)
- 講一個你 ownership 很強的例子。
- 需求很模糊 / 沒人負責時,你怎麼辦?
- 講一個你主動扛下責任的事。

### 規劃 / 優先級(Judgment + Communication)
- 給你一個複雜需求,你怎麼拆 / 怎麼規劃?
- 同時很多事,你怎麼排優先級?
- 你怎麼估時?估不準怎麼辦?
- 怎麼分配工作給團隊?

### 失敗 / 學習(Ownership + Maturity)
- 講一個你失敗 / 犯錯的經歷。
- 你做過最後悔的技術決定?
- 你收到過最難接受的反饋?怎麼消化的?

### 分歧 / 說服(Influence + Judgment)
- 你技術上和別人強烈不合,怎麼辦?
- 你怎麼說服別人接受你的方案?
- 你被否決過嗎?怎麼處理的?

### 流程 / 其它
- 你們的工作流程是怎樣的?你怎麼看?
- 你改進過什麼流程?
- 你理想的團隊協作方式是?

> **自測法**:掃一遍,每題標 ✅(有卡能答)/ ❓(勉強)/ ❌(裸奔)。**❌ 和 ❓ 就是你回 08 章要補的卡。**

## 2. 面試當天 playbook:聽到題,腦子怎麼跑

實時四步,練到變反射:

```
聽到題
  ↓
① 識別題類 + 維度:這題在考哪 2 條軸?(00/01 章的地圖)
  ↓
② 選卡:從銀行掏一張主打那 2 條軸的故事卡
  ↓
③ STAR 講出來:S/T 短 → A 厚(全「我」)→ R 給結果
  ↓
④ 收尾留鉤子:帶一句反思 / 外溢影響,等他追問
```

**節奏控制**:
- 主答案控制在 **1.5-2 分鐘**。太短沒料,太長(超 3 分鐘)面試官會走神、覺得你抓不住重點。
- 先給**一句話結論/概括**,再展開(「這讓我學到對齊要前置——具體是這樣……」)。

**遇到完全沒準備的題怎麼辦?(必學)**
別慌、別硬編。兩招:
1. **爭取 2 秒**:「讓我想一個具體的例子」——停頓思考是專業,不是丟臉。
2. **臨場套骨架**:就算沒對應的卡,也用 **5 維度**反推——「這題在考 Ownership,我有沒有哪件扛責任的事?」+ 套 STAR 講。**有框架,就不會語無倫次。**

**結尾反問(加分項)**:面試官問「你有什麼想問我的」時,別說「沒有」。問跟協作/成長相關的,反向展示你在乎這些:「團隊現在 code review / 知識共享是怎麼做的?」「這個角色最希望我頭三個月帶來什麼改變?」

## 3. 完整 mock 示範(一題走全程)

**面試官**:「講一個你和產品經理有分歧的例子。」

**你(識別:協作/衝突 + 分歧,考 Collaboration/Communication/Maturity → 掏 PM 對齊那張卡)**:

> **(結論先行)** 「有的。我和產品在需求驗收上反覆對不齊,後來我意識到責任在我去建機制,就解決掉了。**(S)** 具體是有段時間,我常常做完了才發現格式、範圍這些細節和 PM 理解不同,得返工。**(T)** 我決定不再等需求變完美,讓對齊在動手前發生。**(A)** 我給自己定了個習慣:接到需求先回一條消息,用我的話複述目的、列可檢驗的驗收標準、把我的假設攤開讓他否決,再請他確認;另外做出主流程就先 demo,不憋到最後。**(R)** 那之後返工明顯減少,PM 說跟我合作很省心、願意更早拉我進需求討論,這套做法組裡幾個同事也用起來了。」

**面試官追問**:「如果 PM 嫌麻煩、不配合確認呢?」
**你**:「我不會要求他配合一套流程,我把賬算給他看——花 10 分鐘確認,省的是幾天返工;我也把消息寫得很好回,他圈一下或回 OK 就行。用一兩次嘗到甜頭,他就主動了。」

**面試官再追問**:「那需求中途又變了呢?」
**你**:「需求變很正常,我反對的不是變,是各自默默地變。所以我把驗收標準寫成文件,變更時更新它再確認一次,有基準線,變更就是可控的增量。」

> **拆解為什麼這輪贏了**:結論先行(Communication)、全程「我」+ 具體動作(Ownership/Judgment)、沒說 PM 一句壞話(Collaboration 紅線守住)、結果量化+外溢(Influence)、追問都接得住(真經歷)。**這就是你練的目標形狀。**

## 4. 追問通用應對模式

不管被追問什麼,這幾個模式能接住大部分:

| 追問類型 | 它想挖 | 你的接法 |
|---|---|---|
| 「具體你做了什麼?」 | 剝離團隊、定位你 | 立刻回到「我」+ 一個具體動作/判斷 |
| 「如果再來一次?」 | 反思力、成熟度 | 給故事卡裡備好的「一句話反思」 |
| 「有人反對你嗎?」 | 協作、影響力 | 講分歧點 + 你怎麼理解對方再推動 |
| 「結果怎麼衡量?」 | 結果是否真實 | 給數字 / 反饋 / 避免的損失 |
| 「失敗 / 做不好的呢?」 | 真實、自省 | 給真實低風險的失敗 + 學習(08 §5) |
| 「為什麼這麼做?」 | 判斷的依據 | 講你當時的 trade-off,不是「就這麼做了」 |

## 5. 紅線總清單(各章彙總,面試前掃一遍)

- 🚫 用「我們」掩蓋個人貢獻(頭號)。
- 🚫 說 PM / 同事 / 前公司的壞話(致命)。
- 🚫 純受害者敘事、甩鍋。
- 🚫 假裝帶過團隊 / 編造經歷(追問必穿幫)。
- 🚫 結果含糊(「後來就好了」)。
- 🚫 只描述不表態(尤其流程題)、只抱怨不改進。
- 🚫 失敗題答「我想不到」或凡爾賽假失敗。
- 🚫 妄自菲薄(「我就是個寫代碼的」)。

## 6. 怎麼練(別只在腦子裡過)

1. **錄音自評**:掃題庫,每題口頭答、錄下來、回放。聽 S 長不長、A 有沒有「我們」、R 有沒有結果。(最低成本、最有效)
2. **對鏡 / 找人**:找朋友扮面試官,**讓他往死裡追問**——經得起追問才算真會。
3. **計時**:練到主答案穩定在 1.5-2 分鐘。
4. **回到本 track 的互動 mock**:任何時候你可以跟我說「mock X 題」(X = 衝突/帶人/拆活/失敗…),我用面試官口吻問你、追問、用 5 維度當場打分。**這是把紙上變肌肉最快的路。**

## 本章小結

- 題庫就 6 大類,逐題自測 ✅/❓/❌,❌ 回 08 章補卡。
- 當天 playbook:**識別題類→選卡→STAR→留鉤子**;主答案 1.5-2 分鐘、結論先行。
- 沒準備的題:**爭取 2 秒 + 用 5 維度反推套 STAR**,別硬編。
- 追問有固定模式,守住紅線(別「我們」、別說壞話、別甩鍋)。
- **練法靠錄音 + 找人追問 + 互動 mock**,腦內預演不算數。
