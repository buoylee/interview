# 第 7 章 · 安全:網路分區、最小權限與認證授權

> 🔬 你一開始問的「**DMZ / 業務 / 管理 / 儲存 四區隔離、最小權限**」就是這章前半。後端面試安全是**中頻**(不像韌性/一致性必問),但能講清「網路為什麼這麼分區」「認證和授權差在哪」「JWT 怎麼運作」就夠用、不露怯。這章分兩半:**網路/系統層**(你的心結)+ **應用層認證授權**。

---

## Part A · 網路分區與最小權限(你的心結)🔬

**為什麼分區**:**攻擊面最小化 + 縱深防禦**。把系統切成多個區、區間嚴格管控,**一個區被攻破不波及其他區**(和 L3 艙壁隔離同一個思想)。

**經典多區隔離**:
```
外網 ──► [DMZ 隔離區] ──► [業務區] ──► [數據區]
         網關/反向代理     應用服務      資料庫/存儲
         對外暴露的        不直接對外    只收業務區,絕不對外
                              ▲
                        [管理區] 運維/監控/堡壘機
   逐層收緊:越往裡,能碰到的人/網路越少
```

- **DMZ(非軍事區)**:夾在外網和內網之間,放對外暴露的東西(網關、反向代理)。即使被攻破,也碰不到內網核心。
- **數據區**:資料庫/存儲,**只接受業務區訪問,絕不直接對外**——這是最該死守的一層。
- **最小權限(least privilege)** 🔬:每個主體只給「完成任務所需的最小權限」。防火牆/安全組**默認拒絕、白名單放行**。
- **零信任(zero trust)** 🔬:**不靠「網路位置」信任**(內網 ≠ 可信),每次訪問都驗證身份 + 權限。是傳統「邊界防禦」的進化。
- **堡壘機(跳板機)**:運維訪問生產的**唯一入口**,所有操作審計留痕。

> 面試一句話收束:**「網路分區是縱深防禦,最小權限是默認拒絕,零信任是不靠網路位置來信任。」**

## Part B · 認證 vs 授權(高頻陷阱)🔬

兩個詞常被搞混,務必分清:

| | **認證 Authentication** | **授權 Authorization** |
|---|---|---|
| 回答 | **你是誰?** | **你能做什麼?** |
| 驗證 | 身份(密碼/token/OAuth) | 權限(RBAC/ABAC) |
| 順序 | 在前 | 在後(先認證才談授權) |
| HTTP | 401 Unauthorized(未認證) | 403 Forbidden(未授權) |

> 陷阱:`401` 其實是「未**認證**」,`403` 才是「未**授權**」——名字起反了,常考。

## Part C · 認證方案 🔬

| | **Session-Cookie** | **Token / JWT** |
|---|---|---|
| 狀態 | **有狀態**(服務端存 session) | **無狀態**(服務端不存) |
| 怎麼work | cookie 帶 sessionId,服務端查 session | JWT 自帶簽名,服務端驗簽即可 |
| 多實例 | session 要**共享**(放 Redis,回扣 redis 選型) | 天然支持(無需共享) |
| 跨域/跨服務 | 麻煩 | 好 |
| 主動失效 | 容易(刪 session) | **難**(沒到期就一直有效) |

**JWT 結構** 🔬:`header.payload.signature` 三段。服務端用密鑰**驗簽**確認沒被篡改,所以不用存。
- **坑**:① 無法主動失效(要靠短過期 + refresh token 或黑名單)② payload 別放敏感信息(只是 base64,不是加密)③ 要防重放。
- **OAuth2 / OIDC**:第三方授權(「用 Google 登錄」)——你不給 App 密碼,而是讓 Google 發一個授權令牌。
- **SSO 單點登錄**:一次登錄,多個系統通用。

## Part D · 授權模型 + 生產安全清單

- **授權模型** 🔬:
  - **RBAC(基於角色,最常用)**:用戶 → 角色 → 權限。改一個角色,該角色所有人權限齊變。
  - **ABAC(基於屬性)**:按屬性(部門/時間/資源標籤)動態判權,更靈活更複雜。
- **生產安全清單(各一句話,面試能列出來就顯全面)**:
  - **傳輸加密**:TLS(HTTPS),防竊聽。
  - **靜態加密**:資料庫/磁碟加密,防拖庫。
  - **密鑰管理**:KMS / Vault,別把密鑰寫死在代碼/配置。
  - **防注入**:SQL 注入(參數化查詢)、XSS、CSRF。
  - **WAF**:Web 應用防火牆,擋常見攻擊。
  - **限流防刷**:防撞庫/爬蟲(回扣 L3 限流)。
  - **審計日誌**:誰在什麼時候做了什麼(回扣 L8 / `observability/`)。
  - **敏感數據脫敏**:日誌/展示時遮蔽手機號、身份證。

## 本章小結

- **網路分區**(DMZ/業務/數據/管理)= 縱深防禦,攻擊面最小化,一區破不波及全局;**數據區絕不對外**。
- **最小權限**=默認拒絕白名單放行;**零信任**=不靠網路位置信任、每次都驗。
- **認證(你是誰)vs 授權(你能做什麼)**:認證在前,401 未認證 / 403 未授權(名字起反,易考)。
- **Session(有狀態,多實例要共享)vs JWT(無狀態,但難主動失效、payload 別放敏感)**;OAuth 第三方授權、SSO 單點登錄。
- **授權 RBAC(角色,最常用)vs ABAC(屬性)**;生產安全清單:TLS/靜態加密/密鑰管理/防注入/WAF/限流/審計/脫敏。
- **L7 完成 → 主線六個空洞層全部寫完。**
