# 生產級分布式系統 · L0–L9 master 索引

> 面試導向 · 資深後端視角 · **這是一張索引,不是一堆重寫**
>
> 你的 repo 裡分布式材料的真相不是「散」,是「**有深礦但沒有地圖**」:`financial-consistency/`(~80 檔講一致性/事務)、`performance-tuning-roadmap/10·11·13`(18 個深檔講韌性排查/架構性能/SRE)、`observability/`(可觀測)——都是世界級深礦,卻互相孤立、頭上沒有導航。這份文件就是那張地圖:把「生產級分布式系統」拆成 L0–L9 十層,**每一層只做兩件事之一:① 指進既有深礦,或 ② 填一個真空洞**。絕不重寫已有的。

---

## 先解開心結:你不需要「做過」,你需要「能推理」

| 角色 | 要求深度 | 一句話 |
|---|---|---|
| 一般工程師 | 聽得懂、不擋路 | 知道有這回事,討論跟得上 |
| **資深工程師(你的目標)** | **能 reason about tradeoff** | 不配置,但能說清「為什麼要、代價是什麼、何時不該用」 |
| 架構師 | **擁有這個決策** | 「要 RPO<15min → 必須同步複製 → 換來寫延遲上升 → 這業務值不值」這句話是架構師說的 |

> 面試考的就是中間那層。「沒做過」不丟人,**「講不出 why 和 tradeoff」才致命**——而後者可補。這張地圖就是把每一層的 why/tradeoff 給你補上。

## 一句話總綱(背起來)

> **生產級分布式系統 = L0 理論地基(CAP/共識/一致性)撐起 L1 服務怎麼組合通信、L2 數據怎麼架構,再用 L3 韌性扛流量、L4 容災扛機房、L5 看得見、L6 發得安全、L7 攻不破、L8 人能協作、L9 跑在容器上。**
> 你**不需要每層都親手做過**,但要對每層講得出「它防什麼失效、付什麼代價、何時不該用」——這就是資深與架構師的分水嶺。這張圖把你已有的深礦(financial / perf / observability)接成一個整體,只在真空洞處補新章。
