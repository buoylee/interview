## 本阶段主线

分布式问题要按链路思考：

```text
入口请求慢
→ Trace 找最长 Span
→ 判断是下游慢、重试放大、连接池排队还是超时配置错误
→ 先止血：限流、降级、断路器
→ 再修根因：慢查询、资源瓶颈、依赖异常
```

---

## 最小完成标准

学完后应该能做到：

- 用 Jaeger 找出一条慢请求的关键 Span
- 画出一次请求的超时预算链
- 解释重试风暴如何放大流量
- 设计一个弱依赖降级方案
- 给 MQ 消费写出幂等处理思路

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 每层都设置很长超时 | 总超时预算要自上而下递减 |
| 失败就立即重试 | 使用指数退避和抖动，并控制重试次数 |
| 降级只在服务端做 | 客户端、网关、服务端都要考虑 |
| Trace 断了就放弃 | 检查上下文传播和采样配置 |
