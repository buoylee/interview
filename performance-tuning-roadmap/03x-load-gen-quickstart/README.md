# 阶段 3.5：压测快速上手学习指南

> 本阶段目标：获得最小负载生成能力。不是完整压测课程，只是让后续 profiling、监控和排查练习有稳定请求来源。

| 2 | [02-locust-quickstart.md](./02-locust-quickstart.md) | 用 Python 描述用户行为，做简单场景压测 |

---

## 工具分工

| 工具 | 适合 | 不适合 |
|------|------|--------|
| wrk | 单接口、快速验证、制造稳定压力 | 多步骤业务流程、复杂参数化 |
| Locust | 用户行为建模、阶梯加压、场景压测 | 极限吞吐探测 |

初学建议：先用 wrk 打一个接口，再用 Locust 写同一个接口的脚本，对比输出差异。

---

## 最小完成标准

学完后应该能做到：

- 用 wrk 对一个接口持续压 30 秒
- 读懂 QPS、Latency、P99、Socket errors
- 写一个最小 Locust 用户脚本
- 压测时打开 Grafana 观察 QPS、错误率、延迟、CPU
- 知道压测客户端本身也可能成为瓶颈

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 直接用最大并发打爆服务 | 先小并发确认基线，再逐步增加 |
| 只看 QPS | 同时看延迟和错误率 |
| 忽略预热 | JVM、连接池、缓存都需要预热 |
| 客户端 CPU 打满还以为是服务瓶颈 | 同时观察压测机资源 |
