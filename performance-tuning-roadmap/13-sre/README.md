---

## 本阶段主线

SRE 实践围绕可靠性闭环：

```text
定义目标 → 持续监控 → 发现异常 → 快速止血 → 复盘改进 → 验证韧性
```

性能调优在这里不再只是“更快”，而是服务是否稳定满足用户承诺。

---

## 最小完成标准

学完后应该能做到：

- 为一个接口定义可用性和延迟 SLI
- 设置一个合理 SLO，并计算 Error Budget
- 根据压测结果估算容量和安全系数
- 写一份事故响应 Playbook
- 写一份无责复盘

---

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 追求 100% 可用 | 用 SLO 和 Error Budget 平衡可靠性与迭代 |
| 告警越多越安全 | 告警必须可行动，减少疲劳 |
| 事故中先找根因 | 先止血恢复，再深入根因 |
| 复盘变追责 | 聚焦系统改进和可执行 action item |
