## 本阶段主线

一次完整压测必须能回答：

```text
测什么？
为什么这么测？
环境是否可信？
负载模型是否接近目标场景？
结果说明什么？
下一步是扩容、优化，还是继续定位瓶颈？
```

---

## 最小完成标准

学完后应该能做到：

- 设计基线、阶梯、浸泡、尖峰中的至少两类测试
- 固定数据、环境、并发和持续时间
- 输出 P50/P95/P99、QPS、错误率、资源利用率
- 找到一次 QPS-延迟曲线拐点
- 写一份结构化压测报告

---

## 本阶段产物

压测报告至少包含：

```text
目标：
环境：
数据规模：
工具和参数：
结果摘要：
关键图表：
瓶颈判断：
优化建议：
复测计划：
```

---

## 常见误区

| 误区 | 正确做法 |
|------|----------|
| 压一下看看 | 先定义目标和成功标准 |
| 只报告 QPS | 同时报告延迟分布和错误率 |
| 空库压测 | 数据规模和分布要接近真实场景 |
| 忽略压测机瓶颈 | 同时监控客户端和服务端 |
| 优化后换了压测参数 | 复测必须保持同样条件 |
