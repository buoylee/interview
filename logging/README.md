# 怎麼把日誌寫好(Logging Done Right)— 一張地圖

> 面試導向 · 開發者視角(不是運維)· 跨語言:**Python / Go / Java**
>
> 這份文件只解決一件事:**怎麼把「打日誌」這個動作做對**,讓未來在 debug 的你,真的找得到想要的那一行。

---

## 這份文件要治的病

你 debug 時的所有痛苦,其實是日誌從**兩個相反方向**壞掉。兩邊都讓你找不到東西:

| 方向 | 症狀 | 真正的根因 |
|---|---|---|
| **太少 / 太弱** | 完全沒有日誌 · 只有一行(`error: xxx` 就沒了)· 等級不夠看不到 | 該記的沒記、記了沒上下文/沒堆疊、門檻設錯 |
| **太多 / 太吵** | 一長串撈不到 · 同一條重複 N 次 | 過度記錄 / 倒整個物件 / 在熱迴圈記、傳播重複發射 + 逐層 `log-and-rethrow` |

所以目標**不是「多記」也不是「少記」**,而是這一句:

> ## 在對的高度,給對的訊號(right signal at the right altitude)

整個 track 就在教你怎麼做到這句。

---

## 核心框架:5 個症狀 → 6 條紀律

任何「找不到日誌」的場景,先對到下面這張表,就知道是哪條紀律壞了、去哪章修:

| 你遇到的症狀 | 根因屬於哪條紀律 | 去哪章 |
|---|---|---|
| 等級不夠 / 太多無關的也在刷 | ① **等級判準**(什麼該 ERROR、什麼該 DEBUG) | `01` |
| 該有日誌的地方完全沒有 · 一長串撈不到 | ② **該在哪記 / 記什麼**(邊界 × 決策點) | `02` |
| 只有一行、撈不出整個故事 | ③ **結構化 + 關聯 ID**(一行帶你找到整串) | `03` |
| 只有一行 `getMessage()` · 異常被吞 · 同一錯重複 | ④ **異常怎麼記**(log-or-throw、完整堆疊) | `04` |
| 程式明明有 log 卻沒輸出 · 一條印兩三次 | ⑤ **配置讓它真的吐**(各語言的坑) | `05` |
| 一長串、刷屏、把祕密也印出來 | ⑥ **效能與衛生**(惰性求值、取樣、PII) | `06` |

## 一個原則:同一批紀律,三語言只是換原語

這 6 條紀律 **Python / Go / Java 完全一樣**,差別只在「用哪個原語去解」。看懂這張對照表,三語言就不是三套知識,是一套知識的三種寫法:

| 紀律 | Python | Go | Java |
|---|---|---|---|
| 分級 | `logging` 五級 | `log/slog` 四級 | SLF4J 五級 |
| 結構化 | `structlog` / `extra` + JSON formatter | `slog` 原生 key=value | Logback + logstash JSON encoder |
| 關聯 ID 隨上下文走 | `contextvars` | `context.Context` | `MDC`(ThreadLocal) |
| 異常帶堆疊 | `logging.exception()` / `exc_info=True` | `fmt.Errorf("...: %w", err)` 頂層記一次 | `log.error("msg", e)`(例外當最後一參) |
| 配好才會吐 | `dictConfig` + logger 階層 | 全域 `slog.SetDefault` + handler | `logback-spring.xml` + 一個 binding |
| 惰性求值 | `log.debug("%s", x)`(非 f-string) | `slog` 天生惰性 | `log.debug("{}", x)` / `isDebugEnabled` |

> 🔬 標記的段落是**開發者內幕** —— 「黑盒裡到底發生什麼」(為何 Python no-handler 會靜默、為何 Java 要 facade、level 過濾到底在哪一步短路)。這些是你面試能多撐 5 分鐘、線上能真正 debug 的地方。

## 章節地圖

每章格式固定:**原理 → Python / Go / Java 並排 → 反例**。

| 章 | 檔案 | 讀完你能做到 | 殺哪個症狀 |
|---|---|---|---|
| `00` | `00-為什麼你總是找不到日誌.md` | 拿到「找不到日誌」的診斷流程,知道去哪章修 | 總綱 |
| `01` | `01-日誌等級-rubric.md` | 用「3am 會不會被叫醒」判準分級,設對 prod 門檻 | 等級看不到 |
| `02` | `02-該在哪記-記什麼.md` | 在邊界與決策點記、不在熱迴圈刷、記聚合而非逐筆 | 沒日誌 · 一長串 |
| `03` | `03-結構化與關聯ID.md` | 用結構化 + `request_id`/`trace_id`,一行撈出整串 | 只有一行 · 撈不到 |
| `04` | `04-異常怎麼記.md` | log-or-throw、帶完整堆疊與 cause chain、不重複不吞 | 一行 · 沒日誌 · 重複(cascade) |
| `05` | `05-配置讓它真的吐.md` | 修掉 Py no-handler/propagate、Go 無 level、Java binding 的坑 | 沒日誌 · 等級擋 · 重複(propagate) |
| `06` | `06-效能與衛生.md` | 惰性求值、取樣、欄位截斷、不漏祕密/PII | 一長串 |
| `07` | `07-面試自檢與落地清單.md` | 對白板題自我抽考 + 上線前 checklist | 複習 |
| `08` | `08-生產級完整範例-三語言.md` | 看同一個服務三語言「全部紀律都對」的完整實作 + 真實輸出 | 壓軸 |

`08` 的可實際執行程式碼放在 `examples/{python,go,java}/`。

## 這個 track 和隔壁兩個的關係

你 repo 裡有三個相關的地方,別搞混:

| 目錄 | 管什麼 | 一句話 |
|---|---|---|
| **`logging/`(這份)** | 怎麼**寫**日誌 | 埋點紀律,跨語言。日誌「離開你程式之前」的事 |
| `observability/` | 日誌怎麼被**收集 / 存 / 查** | 工具與管線(OTel / LGTM / ELK / SkyWalking)。日誌「離開你程式之後」的事 |
| `log/` | Python / FastAPI 的具體落地 | 單語言深切範例,被本 track `05`、`08` 引用 |

**銜接點**:本 track `03` 章的 `trace_id`,正是把日誌接進 `observability/` 那條管線的鉤子 —— 你先在這裡學會「每行帶 trace_id」,到 `observability/` 才有辦法「從一條告警點到出問題的那一行 log」。

## 一句話總綱(背起來)

> 日誌從**兩個方向**壞:**太少**(沒記/沒堆疊/被等級擋)和**太多**(刷屏/重複)。
> 修法是 6 條紀律 —— **分級、在對的點記、結構化帶關聯 ID、異常 log-or-throw、配好才會吐、惰性又乾淨** ——
> 三語言同一套紀律、只換原語,目標是**在對的高度給對的訊號**。
