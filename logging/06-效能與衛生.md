## 一、效能:日誌不該是熱路徑上的成本

### ① 惰性求值:被擋掉的 DEBUG 也別花錢(呼應 `01` 🔬)

回顧 `01` 的內幕:`log.debug(...)` 即使被 level 擋掉,**它的引數還是會先被算好**。兩個層次的浪費:

**層次 1:字串拼接** —— 用參數化,別用字串拼接 / f-string:

```python
log.debug("user=%s order=%s", uid, oid)   # ✅ 被擋時不拼字串
log.debug(f"user={uid} order={oid}")       # ❌ f-string 進函式前就拼好了
```

```go
slog.Debug("checkout", "user", uid, "order", oid)   // ✅ slog 結構化,被擋時不格式化
```

```java
log.debug("user={} order={}", uid, oid);   // ✅ {} 佔位,被擋時不呼叫 toString()
log.debug("user=" + uid + " order=" + oid); // ❌ 字串先拼好了
```

**層次 2:昂貴的引數計算** —— 參數化只省「拼字串」,**省不了**「算出那個值」。如果引數本身很貴(序列化、查 DB、走網路),要主動 guard 或延遲:

```python
if log.isEnabledFor(logging.DEBUG):       # ✅ 整段昂貴計算只在 DEBUG 開時才跑
    log.debug("snapshot=%s", expensive_dump())
```

```go
// slog:用 LogValuer 把昂貴計算延遲到「真的要輸出時」才執行
type lazyDump struct{ obj *Big }
func (l lazyDump) LogValue() slog.Value { return slog.StringValue(l.obj.Serialize()) }
slog.Debug("snapshot", "data", lazyDump{obj})   // Serialize() 只在這條真的被輸出時才呼叫
```

```java
// Log4j2 的 lambda;或 SLF4J 2.x 的 fluent API
log.atDebug().setMessage("snapshot={}").addArgument(() -> expensiveDump()).log();
```

### ② 熱迴圈記聚合(呼應 `02`)

迴圈裡逐筆記,既是「一長串」也是效能殺手(每圈一次 IO)。→ 迴圈外記 `total/ok/failed` 摘要。

### ③ 取樣(sampling):高頻事件只記一部分

有些事件每秒成千上萬次(每個 cache 命中、每個健康檢查)。全記 = 淹沒 + 拖慢。做法:

- **抽樣**:只記 1/N(例:每 100 次命中記 1 次)。
- **限速**:每秒最多記 N 條,超過丟棄並標註「省略了 M 條」。
- `zap` / `zerolog` 內建 sampler;Python/Java 可用自訂 filter 或結構化處理器實作。

### ④ 截斷:別把大 payload 整個倒進去

```python
log.info("response received, size=%d body_head=%s", len(body), body[:200])  # ✅ 只記大小+前 200 字
```

別 `log.info("response=%s", huge_body)` —— 一條日誌幾 MB,既撈不動也燒儲存。記 **size + 摘要 + 必要時 hash**。

### ⑤ 異步輸出:別讓寫 IO 卡住請求

同步寫日誌(尤其寫遠端)會把延遲加到請求路徑上。兩個方向:

- **寫 stdout** 交給平台收集(見 🔬):stdout 是 buffered、很快,把 rotation/傳輸交給收集器(Fluentbit/Promtail)。**容器時代的預設答案。**
- **AsyncAppender**(Logback):把寫入丟到背景佇列。代價見 🔬。

## 二、衛生與安全:日誌是資料外洩的常見破口

日誌常被當成「內部的、安全的」,於是什麼都往裡塞 —— 然後日誌進了 ELK、進了第三方 SaaS、被截圖貼進 ticket。**日誌要當成會外流的資料對待。**

### 絕不記:祕密與憑證

密碼、token、API key、session、信用卡號、私鑰 —— **一律不記**。即使是 DEBUG 也不行(DEBUG 會在出事時被打開)。

### 遮罩:PII(個資)

姓名、身分證、手機、email、地址等,依法規(GDPR / PCI-DSS / 個資法)**遮罩或雜湊**:`user@example.com` → `u***@example.com`,卡號只留末四碼。

### 三語言怎麼做「集中遮罩」

別指望每個呼叫點自己記得遮 —— 在**輸出管線**統一攔截:

```python
# Python:structlog processor 或 logging filter,統一改寫敏感欄位
def mask_pii(logger, method, event_dict):
    if "card" in event_dict:
        event_dict["card"] = "****" + event_dict["card"][-4:]
    return event_dict
```

```go
// Go:slog HandlerOptions.ReplaceAttr,在屬性寫出前統一改寫
opts := &slog.HandlerOptions{
    ReplaceAttr: func(groups []string, a slog.Attr) slog.Attr {
        if a.Key == "password" || a.Key == "token" {
            return slog.String(a.Key, "***")
        }
        return a
    },
}
```

```xml
<!-- Java:logback 的 mask pattern,或自訂 converter / logstash encoder 的欄位過濾 -->
<encoder class="net.logstash.logback.encoder.LogstashEncoder">
  <!-- 自訂 JsonProvider / value masker 把敏感欄位改寫 -->
</encoder>
```

## 三、🔬 內幕

### 為什麼「寫 stdout」比「寫檔案」好(12-factor)

雲原生/容器時代的最佳實踐(12-factor App 第 XI 條):**程式把日誌當事件流寫到 stdout,不自己管檔案。** 原因:

- 容器是**短命且唯讀**的,在裡面寫檔、做 rotation,容器一掛日誌就沒了。
- 把「收集、rotation、傳輸、儲存」交給平台(k8s 把 stdout 收走 → Fluentbit → Loki/ES)。**關注點分離**:你的程式只負責「產生」,管線負責「搬運」(正好對應 `observability/` 的階段表)。
- 你的程式因此不需要任何 log 檔路徑設定,跨環境一致。

→ 結論:**容器裡別寫 log 檔、別在 app 裡做 rotation**,寫 stdout 就好。(傳統 VM 部署才需要 app 內 rotation。)

### 異步 appender 的取捨

`AsyncAppender` 用一個 `BlockingQueue` 把寫入丟背景執行緒,好處是不阻塞請求。代價:

- **佇列滿時**:預設會**丟棄** TRACE/DEBUG/INFO(`discardingThreshold`),保 WARN/ERROR;或設 `neverBlock` 直接丟。所以高壓時你可能**掉日誌**。
- **硬當機時**:佇列裡還沒寫出的會**丟失**。
- → 取捨:吞吐 vs 不丟。關鍵的 ERROR 路徑可考慮同步;一般日誌異步。很多人乾脆「同步寫 stdout(夠快)+ 平台收集」避開這題。

## 四、反例

- ❌ `log.debug(f"dump={huge.serialize()}")` —— f-string + 昂貴計算,DEBUG 關了也照跑。✅ guard + 參數化。
- ❌ `log.info("user=%s", user)` 而 `user` 物件 `__str__` 含密碼。✅ 只記白名單欄位 / 集中遮罩。
- ❌ 迴圈裡逐筆 log。✅ 聚合(`02`)。
- ❌ 容器裡 `FileHandler` 寫 `/var/log/app.log` 還自己 rotation。✅ 寫 stdout,交給平台。
- ❌ 同步把每條 log 寫到遠端 ES。✅ 寫 stdout / 異步,讓收集器負責傳輸。

## 五、落地清單

- [ ] DEBUG 用**參數化**(`%s` / `{}` / slog kv),不用 f-string / 字串拼接。
- [ ] 昂貴引數有 **guard**(`isEnabledFor`)或**延遲**(`LogValuer` / `Supplier`)。
- [ ] 高頻事件有**取樣 / 限速**;大 payload **截斷**(記 size + 摘要)。
- [ ] **絕不記**密碼 / token / key / 卡號;PII **集中遮罩**(processor / ReplaceAttr / converter)。
- [ ] 容器裡**寫 stdout**,不寫檔、不在 app 內做 rotation;傳輸交給收集器。
- [ ] 想清楚異步 appender 的**丟失取捨**;關鍵 ERROR 路徑考慮同步。
