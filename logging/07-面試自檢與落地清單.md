## 一、白板題自測(先自己答,再看提示)

**Q1. 線上某服務「完全沒有日誌」,你怎麼查?**
> 提示:照 `00` 的診斷流程樹。先分「任何日誌都沒有(配置壞:`05`)」還是「別的有、這條沒有(沒寫 `02` / 被等級擋 `01`)」。配置面:Python 沒掛 handler(lastResort 只收 WARNING+)、Go 用了無 level 的老 `log`、Java SLF4J 沒綁 backend(NOP)。

**Q2. 同一條 log 印了兩三次,兩種完全不同的根因是什麼?**
> 提示:**配置重複發射**(propagate/additivity/多次 SetDefault,內容一模一樣,`05`)vs **程式碼 cascade**(每層 catch-log-rethrow,堆疊分散在不同層,`04`)。內容全同 → 查配置;堆疊分層 → 查 catch-rethrow。

**Q3. `log.error` 到底該記什麼?什麼情況才算 ERROR?**
> 提示:**3am 測試**(`01`)—— 願意為它被叫醒才是 ERROR;查無資料、400 那種是正常業務,記 INFO/WARN。記 ERROR 一定要帶**完整堆疊 + cause chain**(`04`),不是 `getMessage()`。

**Q4. 怎麼從「一條告警」一路點到「出問題的那一行 code」?**
> 提示:每行帶 `trace_id`(`03`)→ 用同一個 trace_id 把 metrics/logs/traces 三支柱串起來(`observability/`)→ 從 trace 看是哪一段 span 慢/錯 → 跳到該 span 對應的 logs。關聯 ID 是這一切的鉤子。

**Q5. `log.debug(...)` 被 level 擋掉了,還會花錢嗎?**
> 提示:會。**引數在進函式前就算好了**(`01` 🔬)。參數化(`%s`/`{}`/slog kv)只省「拼字串」,省不了「算昂貴引數」—— 後者要 `isEnabledFor` guard 或 `LogValuer`/`Supplier` 延遲(`06`)。

**Q6. log-or-throw 是什麼?為什麼重要?**
> 提示:一個異常要嘛在能處理的那層**記一次**,要嘛**往上拋**(可 `%w`/`from e`/`new X(msg,e)` 加上下文),**別又記又拋**。又記又拋 → cascade 重複;吞掉不記不拋 → 靜默失敗。中間層只包裝往上傳,**頂層記一次**(`04`)。

**Q7. 為什麼要結構化日誌,不用字串就好?**
> 提示:結構化(JSON / key=value)讓日誌**可被機器 filter / aggregate**(`user_id=1001 AND status=failed`、`count by status`),字串只能正則硬撈。重點:`msg` 要固定,變數放成欄位(`03`)。

**Q8. Python `logging` / Go `slog` / Java SLF4J 的核心差異?**
> 提示:Python 樹狀階層 + propagate(易雙重發射);Go slog 無階層、context 顯式傳、原生結構化、`SetDefault` 設全域;Java SLF4J 是 facade 要綁 backend、用 MDC(ThreadLocal)帶上下文、additivity 同 Python propagate。三者解同一批紀律(`00` 對照表)。

**Q9. 上下文(request_id)怎麼自動出現在每一行?三語言的並發陷阱?**
> 提示:Python `contextvars`(async 安全)、Java `MDC`(ThreadLocal,**執行緒池會串味、@Async 不跟過去,必須 clear()**)、Go `context.Context`(顯式傳、最不會錯)(`03` 🔬)。

**Q10. 線上問題但日誌不夠細,又不想重新部署,怎麼辦?**
> 提示:**動態調等級**(`01` 🔬)—— Java `/actuator/loggers` 或 logback `scan`、Go `slog.LevelVar` 接端點、Python 內部端點 `setLevel`。把某模組臨時開到 DEBUG 撈完再關。

**Q11. 容器裡日誌該寫檔還是 stdout?為什麼?**
> 提示:**stdout**(12-factor,`06` 🔬)。容器短命唯讀,app 不該管檔案/rotation;寫 stdout 交給平台收集(k8s → Fluentbit → Loki/ES)。關注點分離。

**Q12. 日誌的安全/合規要注意什麼?**
> 提示:**絕不記**密碼/token/key/卡號(DEBUG 也不行);PII 依 GDPR/PCI **集中遮罩**(processor/ReplaceAttr/converter),別靠每個呼叫點自律(`06`)。

## 二、上線前總落地清單(六章收口成一張)

**等級(`01`)**
- [ ] 有那把尺(FATAL>ERROR>WARN>INFO>DEBUG>TRACE),ERROR 過「3am 測試」、INFO 過「還原現場測試」。
- [ ] prod 門檻 = INFO;能**不重啟**把某模組臨時開 DEBUG。

**位置與內容(`02`)**
- [ ] 請求進/出、每個外部呼叫(目標+耗時+成敗)、重要決策/狀態轉換都有記。
- [ ] 迴圈裡沒 log,改記聚合;沒有 enter/leave 噪音;不重複框架已記的。

**結構化與關聯 ID(`03`)**
- [ ] 輸出結構化 JSON;`msg` 固定、變數成欄位;有最小欄位集。
- [ ] 入口生成/接收 request_id|trace_id,自動帶進每行;跨服務用 header 傳;Java 有 `MDC.clear()`。

**異常(`04`)**
- [ ] log-or-throw:中間層只包裝往上拋,頂層異常處理器記一次完整堆疊 + cause。
- [ ] 沒有空 catch;可預期業務狀況不走 exception-logging。

**配置(`05`)**
- [ ] Python:dictConfig、handler 只掛 root、業務 logger 不 addHandler。
- [ ] Go:用 slog、啟動 SetDefault、一律 InfoContext。
- [ ] Java:只一個 backend、appender 只掛 root(或 additivity=false)、用 logback-spring.xml。
- [ ] 啟動後實際打一條確認 stdout 有 JSON 輸出。

**效能與衛生(`06`)**
- [ ] DEBUG 參數化;昂貴引數 guard/延遲;高頻取樣;大 payload 截斷。
- [ ] 不記祕密;PII 集中遮罩;容器寫 stdout;想清楚異步丟失取捨。
