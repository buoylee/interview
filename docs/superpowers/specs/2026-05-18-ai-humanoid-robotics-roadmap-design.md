## 0. 这份文档是什么

把一段"要不要走 IoT 转 AI 机器人"的纠结,整理成一条**清晰、可执行、不碎片化**的学习路线。
全文用大白话写,机器人术语第一次出现时都给出软件类比。

---

## 1. 目标定位(一句话锚点)

> **我是一个资深全栈 + LLM 应用工程师,正在成为 "Embodied AI 应用工程师" —— 能把语言和视觉智能接到机器人本体上,让机器人听懂任务、看懂环境、调用技能完成多步操作。**

关键含义:

- 目标行业 = **AI 人形机器人**(Figure / Optimus / Unitree 那类人样子的机器人)。
- 不是"从零转行学硬件",而是**给现有能力加一个机器人本体的出口**。
- 核心战场 = 机器人的**"大脑 / 决策层"**,不是运动控制、不是底层驱动。
- "人形机器人"是目标**行业**;"Embodied AI 应用"是进入这个行业的**那一片**。

### 为什么是"大脑层"

人形机器人内部大致三层:

| 层 | 负责什么 | 谁来做 |
|---|---|---|
| 🦿 身体层 | 平衡、关节转动、马达出力 | 别人做(博士级控制理论) |
| 🛠️ 技能层 | 现成能力包:"走到那边"、"把手伸过去抓" | 大多别人做好,**你只要会调用** |
| 🧠 大脑层 | 听懂任务 → 拆步骤 → 决定先做什么 → 调用技能 → 检查 → 出错重来 | **← 你的层** |

大脑层 ≈ 现在做 LLM 应用的本质(给 AI 任务、给工具、让它一步步完成),
只是"工具"从"调 API"变成"让机器人动手"。**这是能力迁移,不是转行。**

## 2. 能力地图

### 已有(直接复用,不重学)

后端架构 · API · 数据库 · DevOps · Docker · 系统设计 · 前端 dashboard · **LLM 应用层(RAG / agent / function calling / prompt)**

### 要补(按"离你多远"排序)

| 能力 | 学到什么程度 | 为什么 | 大白话 |
|---|---|---|---|
| ROS 2 | 熟练 | 入行硬门槛 | 机器人的"操作系统 + 微服务框架",机器人通用语 |
| 仿真(Gazebo / Isaac Sim) | 够用 | 没硬件也能做出可展示系统 | 机器人版"本地开发环境" |
| 感知(perception) | 只当"消费者" | 把画面变成机器人能用的位姿 | 用现成 AI 模型,不自己训练 |
| MoveIt 2 / Nav2 | 只当"调用者" | 它们是大脑调用的"技能库" | 现成的"手臂怎么动 / 怎么走"技能包 |
| VLA / 模仿学习 | 中等 | 人形机器人的核心 AI | 吃"画面+一句话"吐"怎么动"的 AI |
| C++ | 只到"读得懂" | 你命令的驱动/规划器是 C++ | 大脑用 Python 写,C++ 只需读懂 |
| 边缘部署 | 够用 | 把大脑部署上机器人 | Jetson / TensorRT,你熟悉的"部署上线" |
| 硬件 | 素养级,手段 | 不心虚、能端到端理解 | 阶段 3 买真机械臂时自然获得 |

### 明确砍掉

传统 IoT 全套(MQTT 平台 / LoRa / Zigbee / NB-IoT)、PCB、RF、深度 RTOS、裸机韧体。

**为什么砍 IoT:** IoT 和机器人不是"楼上楼下",是隔壁两栋楼。IoT 想教的——设备通信、
身份安全、遥测监控、边缘计算——要么你全栈已会,要么 ROS 2 路线本来就教(只是换方言),
要么是你本来就不想要的纯硬件部分。**没有一根横档是只有 IoT 能给的。**
(例外:若改做工业自动化 Modbus/OPC-UA,IoT 知识才有用——但那不是人形机器人。)

## 3. 五阶段路线图

- 节奏假设:每周约 **8 小时**,总长约 **15 个月(~1.5 年)**。
- 每阶段三要素:**主干资源**(只认一条主线)、**做什么**、**毕业考**(不查资料能做到就过)。

### 阶段 0 ｜ 学机器人的"通用语"(ROS 2 基础) · 约 2.5 个月

- **主干资源:** ROS 2 官方教程(docs.ros.org,Jazzy 发行版),Beginner 系列从头走完。
- **做什么:** 学 ROS 2 —— 让几个小程序按机器人的方式互相通话。纯软件,无硬件。
  概念:节点(node)、话题(topic)、服务(service)、动作(action)、tf2、launch、RViz、rosbag。
- **类比:** 像当年入行学 web 后端,先搞懂 HTTP、请求/响应、服务间调用。
- **毕业考:** 不看教程,从零写出两个能互相通话的 ROS 2 程序(一个发命令、一个收到后反应),
  并在 RViz 里看到效果。

### 阶段 1 ｜ 在虚拟世界里摆一台机器人(仿真 + 调用技能) · 约 2.5 个月

- **主干资源:** Gazebo 官方教程 + MoveIt 2 官方教程(机械臂方向,与阶段 3 真机械臂衔接)。
- **做什么:** 在仿真里放一台机器人(机械臂为主),装上虚拟摄像头,学会**调用**现成的
  "手臂怎么动"技能包(MoveIt 2);理解 URDF(机器人结构描述)。
- **类比:** 像用现成第三方库/SDK —— 不自己造,学会怎么正确调用。
- **毕业考:** 仿真里的机械臂能听一个简单命令,把手伸到指定位置 / 完成一次简单抓取。

### 阶段 2 ｜ 把"AI 大脑"接上去 · 约 3.5 个月 ← 你的主场

- **主干资源:** 你自己的 LLM/agent 经验 + ROS 2 action 接口;参考 LLM-as-robot-planner 类思路。
  这一阶段主资源是"你边做我边讲"。
- **做什么:** 用 LLM 把一句话拆成步骤;用现成视觉模型(VLM/检测)找目标位置;
  让大脑按顺序调用阶段 1 的技能;做错了能发现并重试;加安全护栏。
- **类比:** 跟做 LLM agent 几乎一样 —— 给 AI 任务、给工具、让它一步步完成。
- **毕业考(简历核心作品雏形):** 对仿真机器人说一句**教程里没有的**新指令
  (例:"把红色方块放进盒子"),它能看 → 规划 → 抓 → 放,完成多步任务。

### 阶段 3 ｜ 碰真东西 + 学人形机器人核心 AI(VLA) · 约 4 个月

- **主干资源:** Hugging Face LeRobot 项目(低成本机械臂 + 数据集 + 模型工具);OpenVLA 了解性阅读。
- **做什么:** 买一台便宜的真实机械臂(LeRobot 低成本臂,几百美元级);手把手"教"它动作、录数据;
  了解 VLA 模型,跑通一个开源的。
- **硬件的位置:** 这就是"慢慢往硬件方向扩展"自然发生的地方 —— 摸真机械臂、真传感器、
  真 AI 计算板。被 AI 目标"拉"出来,不是单独的硬件支线。
- **毕业考:** 真机械臂能在你录的数据上学会一个小任务;你能向别人讲清楚"VLA 是怎么回事"。

### 阶段 4 ｜ 部署 + 整理作品集 + 开始面试 · 约 3 个月

- **主干资源:** NVIDIA Jetson 文档 + 你的全栈老本行。
- **做什么:** 把"大脑"部署到机器人计算板(Jetson)上跑;做网页 dashboard 展示机器人状态
  与摄像头画面(全栈强项,直接复用);整理 GitHub、录 demo 视频、改简历。
- **类比:** 你熟悉的"项目部署上线 + 做后台界面"。
- **毕业考:** 一个完整、能演示的 AI 机器人项目 + 一份能自信讲出来的简历;开始投
  人形机器人公司的"Embodied AI 应用"岗。

### 路线逻辑

通用语 → 虚拟环境练手 → 接上 AI 大脑(主场)→ 碰真硬件 + 核心 AI → 部署上线找工作。
每阶段都踩在前一阶段上,中间无断档。

## 4. 学习系统 —— 怎么保证"系统、不卡壳"

自学最常见的失败不是不够聪明,是**碎片化**:东一个教程西一个视频都学一半;
不知道"学到什么程度算够";卡住不知道怎么脱困;学过的东西互不相连。

五个内建机制对抗碎片化:

1. **每阶段只认一条主干** —— 每阶段锁定一个主资源走完,其它只在卡住时当补充。不开 20 个 tab。
2. **每阶段一个毕业考** —— 具体、可检验、不查资料能做到才算过。永远知道自己在哪、还差多少。
3. **项目驱动** —— 为了做出那个小成果才去学。学是手段,做是目的。
4. **接力** —— 上一阶段产出 = 下一阶段零件。不会每阶段推倒重来。
5. **卡壳协议** —— 同一个坎硬钻超过约 2 小时就停,改成问 Claude / 查官方文档 / 先跳过下一块再回头。
   卡壳是协议触发器,不是失败。

**随身导师机制:** 学习在这个 repo 里进行,Claude 当随身导师 ——
每阶段拆成一节节小课,每节先用大白话讲清"为什么、是什么"(讲明白再练,不一上来塞公式),
再带做、考毕业考。笔记进 repo(像现有 LangChain / MySQL 笔记一样)。

> 教学原则:叙述先行 —— 先把"为什么、怎么来的"讲成故事,再给练习,
> 不在讲清输入之前抛出"预测 X"之类的空格题。

"系统、流畅"= 路线文档(地图)+ 每阶段主干资源(干道)+ 毕业考(里程碑)+ Claude(讲解 / 解卡)。

## 5. 作品集

每阶段的毕业考产出**本身就是作品集素材**,无需额外造。重点两件:

1. **核心作品(阶段 2 起,阶段 4 打磨):** "AI 机器人助手" ——
   自然语言指令 → LLM 拆解 → 视觉找目标 → 调用技能完成多步操作 →
   网页 dashboard 显示状态与画面。仿真起步,阶段 3 后接入真机械臂片段。
2. **GitHub 呈现:** README、架构图、demo 视频、每阶段毕业考记录。
   让看的人一眼看出:不是只会做 demo,而是懂端到端的 AI 机器人系统。

作品集叙事主线:**"我把 LLM 应用能力,接到了真实机器人身体上。"**

## 6. 明确取舍备忘

| 主题 | 决定 | 理由 |
|---|---|---|
| IoT | 不专门学 | 隔壁楼,不是地基;有用的交集全栈已会或 ROS 2 顺带 |
| 硬件 | 是手段,阶段 3 自然接触 | 用于不心虚、端到端理解;非独立支线 |
| C++ | 只学到"读得懂" | 大脑用 Python;C++ 仅需读驱动/规划器 |
| 控制理论 | 不深入 | 身体层是别人的活 |
| 强化学习 | 暂不深钻 | 投入产出比低于"ROS+大脑+VLA+部署" |
| 机器人类型 | 入行切大脑层,长期向 embodied AI 核心生长 | "完整人形"是行业不是岗位 |
