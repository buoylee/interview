## 1. 背景与动机

学习者读 `linux-handson/04-memory-model` 时,被 `malloc`/`mmap` 卡住——文档把它们当**已懂**直接用。这不是孤例:他是 Java/Go 背景(已生疏),从没系统碰过 C/OS,卡点**反复出现在「原语层」而非「高层概念层」**(这次 malloc/mmap,之前撞过中断/队列/唤醒)。

诊断结论:仓库里所有 OS/性能相关 track(`linux-handson`、`os-for-architects`、`performance-tuning-roadmap`…)都**假设你已懂一批底层词**(malloc/mmap/syscall/fd/page fault/指针/虚拟地址/中断…),但**没有任何一篇教这些词本身**。这是一层真空——「所有人都引用、没人定义」的底层词汇底座。

`linux/` 目录现在是**空壳**(只有一个重定向 README 指向 `linux-handson/`,旧笔记已归档到 `_archive/`)。语义上「Linux/OS 底层」本就该住在 `linux/` 名下。本 track 复活它,填这层真空。

## 2. 定位与差异化(防重复硬契约)

**一句话定位**:教「所有 OS 课假设你懂、但没人教的底层原语」,每个原语爬三层深度,当其他 track 的**词汇底座**。

它在课程拓扑里的位置:

```
        os-for-architects     ← 原语 → 架构决策(薄镜头)
              ▲ 坐其上
        linux-handson         ← 机制·能答·能排查(动手课,假设原语已懂)
              ▲ 坐其上
        linux/(本 track)      ← 底层原语本身:它是什么、黑盒里发生什么、怎么亲眼看到
```

与最容易混淆的两个邻居的边界(必须守住):

| | `linux-handson` | `os-for-architects` | **`linux/`(本 track)** |
|---|---|---|---|
| 教什么 | 机制怎么用、怎么排查 | 原语→设计决策的反射 | **原语本身是什么** |
| 海拔 | 命令/现象层 | 架构决策层 | **词汇/内核原理底座** |
| 对 malloc | 用它解释 RSS 不降现象 | 由它推 allocator 选型决策 | **讲 malloc 内部 arena/brk/mmap 怎么运作** |
| 形态 | 七段式 + VM lab | 薄反射 + 指针 | **三层模板(见 §4)** |

**重复检测规则**:本 track 只讲「原语内部如何运作 + 为什么这样设计」。凡涉及「怎么排查某现象」一律回链 `linux-handson`;凡涉及「由此推出什么架构决策」一律回链 `os-for-architects`,不在本 track 展开。Review 时按此规则逐段检查。

## 4. 三层深度模板(每个原语统一爬三层)

```
① 你视角   → Java/Go 等价物,30 秒落地(已有的桥,防悬空)
② 黑盒内部 → 内核/runtime 里真实发生什么 + 架构取舍(资深层·必有)
③ 砸实     → 命令连同【预期输出】直接印在文档里(透明证据,非作业)
```

**关键契约:**

- **② 是核心,每个原语必有。** 这是资深深度所在——「填黑盒里发生什么」,不 defer。
- **③ 是透明证据,不是作业。** 命令 + 预期输出一起印在页面上(学习者**读**就懂②是真的,**想验**随时能亲手敲)。形态对标 `cli-toolbox` 的「⚡验证用例(带 # 预期输出)」。理由:学习者从没碰过 C,②的内部说法对他是悬空记忆,③把它砸实成亲眼可见;且 ③ 本身就是面试谈资(「我会 strace 看 brk/mmap 区分大小块」比「我知道有 arena」值钱)。
- **③ 只在有干净复现的原语放。** 中断、用户态↔内核态切换这类没法 5 行复现的,③ 跳过,② 讲透即可。**不强求每原语都有 ③。**
- **① 始终利用 Java/Go 桥**(学习者背景),把陌生原语挂到他已有的概念上。

每个原语末尾**回链**到「假设它的那个 handson 章」(例:malloc → `linux-handson/04-memory-model`),坐实「底座」定位。

## 6. 写作顺序(01 先发)

**01 → 02 → 03 → 04**,`01-内存原语` 先发。理由:

1. **01 是学习者此刻的真痛**(malloc/mmap 是触发本 track 的原点)。
2. **01 是地基**:虚拟地址/页/page fault 被 02(缺页走内核)、03(mmap 文件映射)反复踩。
3. **syscall(02)被 03 的 fd 依赖**;并发(04)最抽象,压轴。
4. 顺带是**难度爬坡**,不一上来劝退。

## 7. 成功标准

- 学习者读完 `01` 能独立讲清:VSZ≫RSS 的根因、free 后 RSS 为何不降、为什么换 jemalloc——且每条都有 ② 的内部依据 + ③ 的一条可验证命令。
- 每个原语三层齐(② 必有;③ 有干净复现则有),无「只给名词解释」的悬空条目。
- 回链成网:从 `linux-handson` 撞到原语时,能一跳到本 track 对应条目;反之本 track 每原语能跳回假设它的 handson 章。
- 范围守住:无链接装载内容混入;无「排查现象」「架构决策」越界展开(各自回链邻居 track)。
