# 資深開發的領導力與行為面試 track 設計

- **日期**:2026-06-25
- **目錄**:`engineering-leadership/`(根目錄,與 `engineering-handbook/` 正交)
- **語言**:繁體中文(可調)
- **狀態**:已對齊,開工中

## 1. 緣起與目標

使用者(8 年軟體工程師,資深 IC)投遞**開發崗**,但面試常被問:是否帶過人、如何分配工作、如何規劃/拆分工作、工作流程。他自評對這塊**沒有系統概念**,且**目前工作中經常與他人衝突**。

這些題目橫跨三個範疇,本 track 統一覆蓋:

| 被問的題 | 真正在考 | 範疇 |
|---|---|---|
| 帶過人嗎、怎麼帶 | leadership / people | 技術領導力 |
| 怎麼分配/規劃/拆分 | execution / planning | 工程項目管理 |
| 工作流程 | process | 軟體工程流程 |

**目標(staged)**:先搭面試應答骨架(短期能上場),再往下挖方法論(長期變真戰力)。但採**逐章雙層**交付(見 §3),不分兩批寫。

**定位基調**:投 IC,故這些是 **seniority / maturity / leadership-readiness 訊號題**,不是 management 題。**不假裝當過經理**;方法論只挖到「資深 IC / 準 Tech Lead」深度,不塞 EM 的招聘/績效/解雇內容。

## 2. 「足夠細」契約(驗收標準)

使用者要求:學完能**輕鬆面對面試**,且**工作中能用**。據此每個方法論章必須交付:

- **方法論層(工作真戰力)**:核心原語收口地圖 → 黑盒內幕 → 標準動作/公式 → 走一個具體例子。標準:**週一上班就能用**。
- **面試層(應答戰力)**:完整範例 **STAR 逐字稿** + 面試官追問清單與接法 + 會扣分的紅線/反模式。標準:**照著練就能上場**。
- **你的 drill**:每章逼出一個真實產物(WBS / RACI / 故事卡),進故事銀行。

**學習風格錨點**(來自使用者既有偏好):

- 底層知識寫進**正文教學**,問答題只當最後的複習自檢,不承載新知。
- **不預設底層名詞**(STAR、RACI、Scrum、CI/CD 等)他懂;卡點補 30 秒程序員視角 primer。
- 主題若顯雜亂,先給「**幾條核心原語**」收口地圖再展開散招。
- 公式/框架先在敘事中教清楚輸入,才出練習題;不在教之前留空白。

## 3. 整體結構

### 脊柱(10 章 + 1 張面試卡)

**Part 0 · 地基**
- `00-overview` — 這套題在考什麼:**5 個評分維度收口地圖**(ownership / influence / judgment / collaboration / communication)、IC vs Tech Lead 標準線、四種被問法都是同 5 維度的不同窗口、地圖怎麼用
- `01-behavioral-interview` — 行為面試運作原理:為何問過去、**STAR 與 SBI 框架透徹**、追問機制(挖真假)、signal vs noise、行為題**完整 taxonomy**(帶人/協作/衝突/當責/模糊/失敗/優先級)
- `02-honest-positioning` — 「沒正式帶過人」的誠實定位法:把 IC 工作翻成 leadership-readiness 訊號、一套誠實有結構的開場框架

**Part 1 · 核心方法論(每章雙層)**
- `03-work-breakdown` — 規劃 & 拆分工作:WBS、垂直切片、估時、definition of done
- `04-delegation` — 分配 & 委派工作:delegation、RACI、人事匹配、ownership vs 幫忙的界線
- `05-ways-of-working` — 工作流程:Scrum/Kanban/code review/CI-CD 的「為什麼存在」而非儀式,讓你對流程有觀點
- `06-mentoring-influence` — 帶人 & 影響力:mentoring、用 code review 教人、unblock、無頭銜的影響力
- `07-conflict` — **衝突 & 分歧處理**(獨立章,方法論層偏「工作立刻能用」,因使用者當前痛點):對事不對人、SBI 給反饋、interest vs position、disagree-and-commit、升級路徑、情緒管理

**Part 2 · 實戰合成**
- `08-story-bank` — 自我盤點 & 故事銀行:把真實專案挖成 **6-8 張結構化故事卡**,覆蓋完整 taxonomy(含衝突/失敗/模糊/優先級)
- `09-mock-and-drills` — 模擬問答 & 追問演練:分類題庫、追問 drill、扣分紅線、面試當天 playbook

**面試卡** `cheatsheet` — STAR 模板、故事卡模板、四題型一句話應答骨架、5 維度速查、題庫精選

### 每個方法論章(03-07)的固定雙層模板

1. **上層 · 方法論(工作真戰力)** — 核心原語地圖 + 黑盒內幕 + 標準動作/公式 + 具體例子走一遍
2. **下層 · 面試轉化(應答戰力)** — 這題型在考什麼 → STAR 套此主題模板 → 1-2 個範例逐字稿 → 常見追問 & 怎麼接 → 紅線/反模式
3. **你的 drill** — 拿自己一個真實專案套一遍,產出一張故事卡

`07-conflict` 特例:上層加厚,偏現用;明確標「本章方法論下週就能用在當前衝突上」。

## 4. 建置順序

`00 → 01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09 → 面試卡`,一次一章,每章寫完可獨立閱讀。08 故事銀行需先有前面方法論詞彙,故不前置。

## 5. 範圍邊界(YAGNI)

- **不含** EM 管理學(招聘流程、績效考核、解雇、團隊預算)。
- **不含** 與 `engineering-handbook/` 重疊的團隊治理規範;本 track 是「個人如何展現 + 應答」。
- **不含** 通用面試(算法/系統設計);那些在 repo 其他 track。
