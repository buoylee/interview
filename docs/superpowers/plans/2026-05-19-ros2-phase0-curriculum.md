# 阶段 0:ROS 2 通用语 · 课程表(实施计划)

> **给执行者:** 这是一份**学习课程表**,不是代码实施计划。执行模型见文末"如何使用"。
> 每节课用 checkbox(`- [ ]`)跟踪;每节课**必须**以"更新进度地图"收尾。

**目标:** 学会机器人软件的通用语 ROS 2 —— 能让多个程序按机器人的方式互相通话。

**做法:** 22 节课,按顺序走;每节课 = 一到两次约 2 小时的坐下学习;以官方教程为主干,本课程表负责排序、加动手练习、加检验关卡。

**技术栈:** Ubuntu 24.04(虚拟机)· ROS 2 Jazzy · Python · RViz / rqt / rosbag

**主干资源:** ROS 2 Jazzy 官方教程 —— <https://docs.ros.org/en/jazzy/Tutorials.html>

**毕业考(整个阶段 0):** 不看教程,从零写出两个能互相通话的 ROS 2 程序(一个发命令、一个收到后反应),并在 RViz 里看到效果。= 课 22。

**总量:** 约 63 小时纯学习 + 卡壳缓冲 ≈ 每周 8 小时下约 9~10 周(~2.5 个月)。

### 课 2:装好 ROS 2 开发环境

**时长:** ~4h(2 次坐下)｜ **前置:** 课 1

**目标:** 学完你有一个能跑 ROS 2 的 Ubuntu 环境,`ros2` 命令可用。

- [ ] **步骤 1:装虚拟机**

下载 UTM(<https://mac.getutm.app>),安装 Ubuntu 24.04 LTS(Apple Silicon 用 ARM64 镜像,Intel 用 AMD64)。给虚拟机至少 4 核 / 8GB 内存 / 40GB 磁盘。

- [ ] **步骤 2:装 ROS 2 Jazzy**

在 Ubuntu 里跟随官方安装文档 <https://docs.ros.org/en/jazzy/Installation/Ubuntu-Install-Debs.html>,用 Debian 包方式安装 `ros-jazzy-desktop`(desktop 版含 RViz、turtlesim、demo)。

- [ ] **步骤 3:配置环境**

跟随官方教程 "Configuring environment"。把 `source /opt/ros/jazzy/setup.bash` 加进 `~/.bashrc`。

- [ ] **步骤 4:验证(小检查)**

新开终端,运行 `ros2 run turtlesim turtlesim_node` —— 弹出一个有小乌龟的窗口,即成功。再运行 `ros2 doctor` 没有致命报错。

- [ ] **步骤 5:收尾**

笔记记下安装中踩的坑(给未来的自己)。更新进度地图 + 交接笔记,commit。

### 课 3:用 turtlesim 建立心智模型

**时长:** ~2h ｜ **前置:** 课 2

**目标:** 学完你能用大白话讲清 node / topic / service / action 各是什么。

- [ ] **步骤 1:跟官方教程 "Using turtlesim, ros2, and rqt"**

开 turtlesim,再开 `ros2 run turtlesim turtle_teleop_key` 用键盘控制乌龟。

- [ ] **步骤 2:对照心智模型**

边玩边对照:`turtlesim_node` 和 `teleop` 是两个**节点(node,小程序)**;键盘控制走的是**话题(topic,持续的数据流)**;"重置/传送"走的是**服务(service,一问一答)**;"转到某角度"走的是**动作(action,有过程的长任务)**。

- [ ] **步骤 3:小检查**

在笔记里用你自己的大白话写下这 4 个词各是什么、举一个 turtlesim 里的例子。不准抄教程原文。

- [ ] **步骤 4:收尾**

更新进度地图 + 交接笔记,commit。

### 课 4:命令行工具集

**时长:** ~2h ｜ **前置:** 课 3

**目标:** 学完你能用命令行观察一个运行中的 ROS 2 系统,并手动驱动它。

- [ ] **步骤 1:跟官方教程 "Understanding nodes" / "Understanding topics"**

练 `ros2 node list`、`ros2 topic list`、`ros2 topic echo`、`ros2 topic info`。

- [ ] **步骤 2:手动让乌龟动**

只用一条 `ros2 topic pub` 命令让乌龟转圈(向 `/turtle1/cmd_vel` 发消息)。

- [ ] **步骤 3:小检查**

不开 teleop,只用命令行:(a) 让乌龟动起来;(b) `echo` 出乌龟当前位置数据。

- [ ] **步骤 4:收尾**

更新进度地图 + 交接笔记,commit。**模块 A 完成。**

## 模块 B · 话题 Topic(单向数据流)(课 5–8,约 12h)

### 课 5:话题概念 + 第一个发布者

**时长:** ~3h ｜ **前置:** 课 4

- [ ] **步骤 4:小检查**

另开终端 `ros2 topic echo` 你的话题,能看到你的节点在持续发消息。

更新进度地图 + 交接笔记,commit。

## 如何使用这份课程表(执行模型)
