## 📖 讲解

### 消息类型是什么

消息类型 = 话题里流动的那份数据的**结构定义**(有哪些字段、各是什么类型)。

类比:一个 DTO / 一个 protobuf schema / 一个 TypeScript `interface`。

前两课你用的 `std_msgs/msg/String`、课 4 用的 `geometry_msgs/msg/Twist`,都是 ROS 2 **内置**的现成类型。但真实项目里,你常常要传自己的数据结构 —— 这就要自定义。

### `.msg` 文件

自定义一个消息,就是写一个 `.msg` 文件,每行一个字段:`类型 名字`。例如 `Temperature.msg`:

```
float32 temperature
string location
```

可用的基础类型:`bool int32 int64 float32 float64 string` 等。

### 一个反直觉的约定:接口要放在单独的 CMake 包里

新手直觉是"把 `.msg` 放进我的 Python 包里"。**但 ROS 2 的惯例不是这样:**

`.msg`(消息)和 `.srv`(服务接口,课 10 用)通常放在一个**单独的、`ament_cmake` 类型的包**里(不是 `ament_python`)。原因:接口定义要被编译成多种语言(Python、C++)都能用的代码,这套机制用的是 CMake。

所以标准做法:
1. 建一个专门的接口包(`ament_cmake` 类型),里面放 `.msg`。
2. 你的 Python 包**依赖**这个接口包,然后就能 `import` 用。

具体怎么建、`CMakeLists.txt` 和 `package.xml` 怎么配,**严格照官方教程**(下方链接)—— 这块配置细节多,照抄官方最稳。

---

## 🛠️ 动手

跟官方教程 **"Creating custom msg and srv files"**:
<https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Custom-ROS2-Interfaces.html>

1. **建接口包**:在 `~/ros2_ws/src` 里建一个 `ament_cmake` 类型的包(例:`my_interfaces`)。
2. **加 `.msg` 文件**:在包里建 `msg/Temperature.msg`,内容如上面讲解所示。
3. **配置**:照官方页改 `CMakeLists.txt`(加 `rosidl_generate_interfaces(...)`)和 `package.xml`(加相关依赖)。
4. **构建**:
   ```bash
   cd ~/ros2_ws
   colcon build --packages-select my_interfaces
   source install/setup.bash
   ```
5. **验证类型生成成功**:
   ```bash
   ros2 interface show my_interfaces/msg/Temperature
   ```
   应打印出你定义的两个字段。

6. **在节点里用上它**:把课 6 的发布者/订阅者,从 `std_msgs/String` 改成你的 `my_interfaces/msg/Temperature`。
   - 改 import:`from my_interfaces.msg import Temperature`
   - 发布者里给 `msg.temperature` 和 `msg.location` 赋值。
   - **你的 Python 包的 `package.xml` 要加一条对 `my_interfaces` 的依赖。**
   - 重新 `colcon build` 整个工作区 + source。

---

## ⚠️ 踩坑预警

- **`.msg` 文件名首字母必须大写**:`Temperature.msg` ✅,`temperature.msg` ❌。
- **改了接口包,要重新 build**:接口包变了,用它的 Python 包也要重新 build + source。
- **Python 包忘了加依赖**:`import my_interfaces.msg` 会失败 —— `package.xml` 里补上依赖。
- **接口包用错类型**:它必须是 `ament_cmake`,不能是 `ament_python`。
