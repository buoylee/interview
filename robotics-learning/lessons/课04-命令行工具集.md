## 📖 讲解:你的调试本能

ROS 2 自带一套命令行工具(就是 `ros2` 开头的命令),让你**不写代码**就能看见、并戳一个正在运行的系统。这是你以后最重要的调试本能 —— 系统不对劲时,你就是靠这套命令查。

类比:这套命令 ≈ 你调试 web 系统时用的 `curl`(发请求)+ `ps`(看什么在跑)+ 看日志。

### 命令的规律:`ros2 <对象> <动作>`

ROS 2 命令很有规律,记住对象和动作就行:

| 对象 | 常用动作 | 干什么 |
|---|---|---|
| `ros2 node` | `list` / `info` | 有哪些节点 / 某节点的详情 |
| `ros2 topic` | `list` / `info` / `echo` / `pub` / `hz` | 话题相关 |
| `ros2 service` | `list` / `type` / `call` | 服务相关 |
| `ros2 param` | `list` / `get` / `set` | 参数相关(课 11 细讲) |

### 话题最常用的三个动作(重点)

- `ros2 topic list` —— 当前有哪些话题。
- `ros2 topic echo <话题名>` —— **把这个话题里流过的数据打印出来给你看**。调试神器。
- `ros2 topic pub <话题名> <消息类型> <数据>` —— **手动往话题发一条消息**。让你不写发布者也能测试。

## 🛠️ 动手

跟官方教程 **"Understanding nodes"** 和 **"Understanding topics"**:
- <https://docs.ros.org/en/jazzy/Tutorials/Beginner-CLI-Tools/Understanding-ROS2-Nodes/Understanding-ROS2-Nodes.html>
- <https://docs.ros.org/en/jazzy/Tutorials/Beginner-CLI-Tools/Understanding-ROS2-Topics/Understanding-ROS2-Topics.html>

1. **先开着乌龟**(终端 1):`ros2 run turtlesim turtlesim_node`

2. **观察节点**(终端 2):
   ```bash
   ros2 node list
   ros2 node info /turtlesim
   ```

3. **观察话题**:
   ```bash
   ros2 topic list
   ros2 topic info /turtle1/cmd_vel
   ros2 topic echo /turtle1/pose
   ```
   最后一条会持续打印乌龟的位置 —— 这就是"看见数据流"。`Ctrl+C` 停。

4. **手动让乌龟动**(关键练习):只用一条命令让乌龟一边前进一边转圈:
   ```bash
   ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "{linear: {x: 2.0}, angular: {z: 1.8}}"
   ```
   解释:往 `/turtle1/cmd_vel` 话题发 `Twist` 类型的消息;`linear.x` 是前进速度,`angular.z` 是旋转速度。`Ctrl+C` 停。

---

## ⚠️ 踩坑预警

- **`ros2 topic pub` 的数据格式**:那串 `"{...}"` 是 YAML 格式,引号、大括号、空格(`x:` 后面要有空格)都不能错。
- **echo 没输出**:确认话题名拼对了(用 `ros2 topic list` 复制),且确实有节点在往里发。
- **命令找不到**:又是没 source —— 见课 2 讲解。
