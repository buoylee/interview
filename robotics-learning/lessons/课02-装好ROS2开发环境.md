## 📖 讲解:先搞懂你在装什么

### 为什么要装虚拟机,不能直接在 macOS 上装?

ROS 2 是为 **Linux** 生的。它官方支持、几乎所有教程和公司用的,都是 **Ubuntu**。macOS 上虽然能勉强装,但坑多、和教程对不上、出问题没人能帮你。

所以标准做法:在你的 Mac 里跑一台**虚拟机**(虚拟机 = 用软件模拟出来的一台完整电脑)。这台虚拟"电脑"装 Ubuntu,ROS 2 装在它里面。你的 Mac 不受影响,虚拟机搞坏了删掉重来即可。

- **UTM** = Mac 上的虚拟机软件(类比:VMware / VirtualBox 的 Mac 版,免费)。
- **Ubuntu** = 跑在虚拟机里的 Linux 操作系统。

### 为什么是 Ubuntu 24.04 + ROS 2 Jazzy 这一对?

ROS 2 每年出一个新版,每个版本"配对"一个特定的 Ubuntu 版本——**装错对子,一路报错**。

- **ROS 2 Jazzy**(全名 Jazzy Jalisco)= 当前的长期支持版(LTS),配对 **Ubuntu 24.04**。
- 这就是为什么下面要你装 Ubuntu **24.04**,不是别的版本。

把它记成铁律:**Jazzy ↔ Ubuntu 24.04,绑死。**

### "ros-jazzy-desktop" 是什么

ROS 2 不是一个程序,是一大堆包。`ros-jazzy-desktop` 是一个"全家桶",装它会一起带上:核心通信库 + 你后面要用的 RViz(可视化)、turtlesim(练习用的小乌龟)、demo 程序。装这个就够阶段 0 用了。

### 什么叫"source 环境"(这点很多新手卡)

装完 ROS 2 后,你打开一个新终端,直接敲 `ros2` 会说"命令找不到"。**这不是装坏了。**

原因:ROS 2 装在 `/opt/ros/jazzy/` 里,但终端默认不知道去那里找。你需要先运行:

```bash
source /opt/ros/jazzy/setup.bash
```

`source` 这个动作 = "把 ROS 2 的位置信息加载进当前终端"。运行后,这个终端才认识 `ros2` 命令。

麻烦的是:**每开一个新终端都要 source 一次**。所以标准做法是把这行写进 `~/.bashrc`(`.bashrc` = 每次新开终端会自动执行的脚本),这样就一劳永逸。下面步骤 3 会做这件事。

> 类比:`source` 就像每次进项目前 `nvm use` / 激活虚拟环境。写进 `.bashrc` = 让它自动激活。

## 🛠️ 动手

### 步骤 1:装虚拟机 + Ubuntu 24.04(~2h,大部分是等待)

1. **装 UTM**:浏览器打开 <https://mac.getutm.app> → 点 "Download"(网站直接下载是免费的;App Store 版要付费,是同一个 app,省钱就用网站版)。下载后拖进"应用程序"。

2. **下载 Ubuntu 24.04 LTS Desktop 镜像(ISO)**:去 <https://ubuntu.com/download/desktop>。
   - **Apple Silicon 芯片的 Mac**(M1/M2/M3/M4):需要 **ARM64** 版镜像。Ubuntu 桌面版 ARM64 ISO 见 <https://cdimage.ubuntu.com/noble/daily-live/current/>(选 `noble-desktop-arm64.iso`)。
   - **Intel 芯片的 Mac**:用上面 desktop 页面默认的 64-bit(AMD64)版即可。
   - 不确定自己芯片?左上角  → "关于本机" 看"芯片/处理器"。

3. **在 UTM 里建虚拟机**:UTM → "Create a New Virtual Machine" → "Virtualize" → "Linux" → 选你下载的 Ubuntu ISO。配置给足:
   - CPU:4 核 ｜ 内存:8 GB(8192 MB)｜ 磁盘:40 GB
   - (给太少会非常卡,RViz 之类会很难受)

4. **装 Ubuntu**:启动虚拟机,会进入 Ubuntu 安装界面。一路按引导:选语言 → "Install Ubuntu" → 键盘 → "Normal installation" → "Erase disk and install Ubuntu"(注意:这个"擦除"只擦**虚拟机的虚拟磁盘**,不碰你真实的 Mac)→ 设用户名密码 → 等装完 → 重启。

✅ **步骤 1 完成标准**:虚拟机里进入 Ubuntu 桌面,能打开"Terminal"(终端)。

### 步骤 2:装 ROS 2 Jazzy(~1h)

> ⚠️ 这一步的具体命令(加 apt 源、装 GPG key)**会随时间变化**。我不在这里抄一份可能过期的——请直接照**官方页面**操作,它是持续维护的、最新最准:
>
> **官方安装页:** <https://docs.ros.org/en/jazzy/Installation/Ubuntu-Install-Debs.html>

在 Ubuntu 的终端里,照官方页面顺序做这几件事(下面是"地图",具体命令以官方页为准):

1. **设置 locale** —— 确保系统语言环境是 UTF-8(ROS 2 要求)。
2. **启用 Ubuntu Universe 软件源**。
3. **添加 ROS 2 的 apt 软件源**(官方现在用一个 `ros2-apt-source` 包来帮你装好 key + 源)。
4. **安装**:`sudo apt update` 然后 `sudo apt install ros-jazzy-desktop`(注意选 **desktop**,带 RViz/turtlesim)。
5. 官方页面还会建议装 `ros-dev-tools`(开发工具,后面写代码要用),一起装上。

这一步主要在下载,会等一会儿。

✅ **步骤 2 完成标准**:`apt install` 跑完没有红色致命错误。

### 步骤 3:配置环境(~10min)

让每个新终端自动认识 `ros2`(原理见上面讲解):

```bash
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

第一行:把 source 命令追加进 `.bashrc`。第二行:让当前终端立刻生效(不用关掉重开)。

### 步骤 4:验证(~20min)

1. **检查命令在了**:
   ```bash
   ros2 --help
   ```
   应打印一堆 ros2 子命令的帮助,而不是 "command not found"。

2. **体检**:
   ```bash
   ros2 doctor
   ```
   会输出系统检查结果。允许有 warning(警告),但不应有 fatal/error 致命问题。

3. **跑出小乌龟**(下节课的主角):
   ```bash
   ros2 run turtlesim turtlesim_node
   ```
   应弹出一个蓝色窗口、中间一只小乌龟。**看到乌龟 = 环境彻底通了。** 按 `Ctrl+C` 关掉。

## ⚠️ 踩坑预警

- **下错 Ubuntu 镜像架构**:Apple Silicon 必须用 ARM64 镜像,用错架构装不上或极慢。装前再确认一次芯片。
- **虚拟机很卡**:多半是内存/CPU 给少了。关掉虚拟机,在 UTM 设置里调到 4 核 / 8GB 以上。
- **`ros2: command not found`**:几乎都是没 source(见讲解)。确认步骤 3 做了,且终端是 source 之后新开的。
- **turtlesim 窗口出不来 / 报显示错误**:确认你装的是 Ubuntu **Desktop**(桌面版,带图形界面),不是 Server 版。
- **apt 报 GPG / 源错误**:别照网上老教程,严格照上面那个官方页面当前的命令——这块官方改过几次。
