## 📖 讲解

### 工作区 / 包 / colcon —— 你写代码的地方

写代码前要先搭好"项目结构"。三个词:

- **包 package** = 一个功能模块,你的代码 + 配置打成一个单元。
  类比:一个 npm 包 / 一个 Maven module。
- **工作区 workspace** = 一个目录,里面的 `src/` 放你所有的包。习惯叫 `ros2_ws`。
  类比:一个 monorepo。
- **colcon** = ROS 2 的构建工具,把工作区里的包编译、安装好。
  类比:`npm run build` / Maven 的构建。

流程:写代码 → `colcon build` → `source install/setup.bash` → `ros2 run` 跑。

> 注意有**两个 source**:课 2 那个 `source /opt/ros/jazzy/setup.bash` 加载的是 ROS 2 本体;
> 这里的 `source install/setup.bash` 加载的是**你自己工作区**的包。两个都要。

### 发布者 Publisher

发布者 = 一个节点,定时往某个话题发消息。回顾课 3:话题是单向广播流。

ROS 2 用 Python 写节点的库叫 **rclpy**。一个节点的代码骨架永远是这个套路:

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String          # 用到的消息类型

class MinimalPublisher(Node):             # 1. 定义一个类,继承 Node
    def __init__(self):
        super().__init__('minimal_publisher')          # 2. 给节点起名
        self.publisher_ = self.create_publisher(String, 'topic', 10)  # 3. 建发布者:消息类型、话题名、队列长度
        self.timer = self.create_timer(0.5, self.timer_callback)      # 4. 每 0.5 秒触发一次回调
        self.i = 0

    def timer_callback(self):             # 5. 定时执行:造一条消息并发出去
        msg = String()
        msg.data = 'Hello World: %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)   # 打日志
        self.i += 1

def main(args=None):
    rclpy.init(args=args)                 # 6. 初始化
    node = MinimalPublisher()
    rclpy.spin(node)                      # 7. 让节点一直转(否则程序立刻结束)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

七个套路记住,以后写任何节点都是它的变体。**完整、最新的官方示例以官方教程页为准**(下方链接)。

## 🛠️ 动手

### 步骤 1:建工作区和包

跟官方 **"Creating a workspace"** 和 **"Creating a package"**:
- <https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Creating-A-Workspace/Creating-A-Workspace.html>
- <https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Creating-Your-First-ROS2-Package.html>

大致:
```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python --license Apache-2.0 py_pubsub
```
(`ament_python` = 这是个 Python 包。)

### 步骤 2:写发布者

跟官方 **"Writing a simple publisher and subscriber (Python)"** —— **只做发布者部分**:
<https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Writing-A-Simple-Py-Publisher-And-Subscriber.html>

在包里新建发布者的 `.py` 文件(代码就是上面讲解里那个骨架)。

### 步骤 3:登记两个配置文件(新手最容易错的地方 ⚠️)

- **`package.xml`** —— 声明依赖(用到 `rclpy`、`std_msgs` 就要在这里写上)。
- **`setup.py`** —— 在 `entry_points` 里登记一个可执行名,`ros2 run` 才找得到你的节点。

这两个文件怎么改,**严格照上面那个官方教程页**(它每个字段都写了)。

### 步骤 4:构建并运行

```bash
cd ~/ros2_ws
colcon build --packages-select py_pubsub
source install/setup.bash
ros2 run py_pubsub <你在 setup.py 里登记的可执行名>
```
应该看到终端每 0.5 秒打印一行 "Publishing: ..."。

---

## ⚠️ 踩坑预警

- **`ros2 run` 找不到可执行**:99% 是 `setup.py` 的 `entry_points` 没写对,或者改完没重新 `colcon build`。
- **改了代码不生效**:每次改完都要重新 `colcon build` + `source install/setup.bash`。
- **`colcon build` 报缺依赖**:`package.xml` 里漏了 `rclpy` / `std_msgs`。
- **在哪个目录 build**:`colcon build` 要在工作区根目录(`~/ros2_ws`)跑,不是在 `src` 里。
