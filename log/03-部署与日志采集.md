# 部署与日志采集

## 1. 环境变量一览

应用通过以下环境变量控制日志行为，代码不需要改动：

| 变量 | 可选值 | 默认值 | 说明 |
|------|--------|--------|------|
| `SERVICE_NAME` | 任意字符串 | `my-service` | 服务名称 |
| `LOG_LEVEL` | DEBUG/INFO/WARNING/ERROR | `INFO` | 日志级别 |
| `LOG_OUTPUT` | stdout / file | `stdout` | 输出目标 |
| `LOG_FILE_PATH` | 文件路径 | `/var/log/app/app.log` | 仅 file 模式生效 |
| `LOG_FORMAT` | json / console | `json` | 输出格式 |

切换环境只需要改环境变量：

```bash
# VM 部署：写文件
LOG_OUTPUT=file  LOG_FILE_PATH=/var/log/app/app.log  LOG_FORMAT=json

# Docker 部署：写 stdout
LOG_OUTPUT=stdout  LOG_FORMAT=json

# 本地开发：写 stdout + 可读格式
LOG_OUTPUT=stdout  LOG_FORMAT=console
```

## 2. VM 部署

### systemd 服务配置

```ini
# /etc/systemd/system/myapp.service

[Unit]
Description=My FastAPI App
After=network.target

[Service]
User=app
EnvironmentFile=/etc/myapp/env
WorkingDirectory=/opt/myapp
ExecStart=/opt/myapp/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
# /etc/myapp/env

SERVICE_NAME=order-service
LOG_LEVEL=INFO
LOG_OUTPUT=file
LOG_FILE_PATH=/var/log/myapp/app.log
LOG_FORMAT=json
```

### 日志轮转

应用代码中已配置 `RotatingFileHandler`（100MB 单文件，保留 5 个），通常够用。

如果想用系统级 logrotate 替代（比如统一管理所有服务的日志），配置如下：

```
# /etc/logrotate.d/myapp

/var/log/myapp/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
}
```

> 用 `copytruncate` 是因为 Python logging 不像 nginx 那样支持 reopen 信号，truncate 方式不需要重启应用。

## 3. Docker 部署

### docker-compose.yml

```yaml
services:
  order-service:
    image: order-service:latest
    environment:
      - SERVICE_NAME=order-service
      - LOG_LEVEL=INFO
      - LOG_OUTPUT=stdout       # Docker 环境只需这一个变化
      - LOG_FORMAT=json
    logging:
      driver: json-file
      options:
        max-size: "100m"        # 单文件最大 100MB
        max-file: "3"           # 最多 3 个文件，总共最多 300MB
    ports:
      - "8000:8000"

  user-service:
    image: user-service:latest
    environment:
      - SERVICE_NAME=user-service
      - LOG_LEVEL=INFO
      - LOG_OUTPUT=stdout
      - LOG_FORMAT=json
    logging:
      driver: json-file
      options:
        max-size: "100m"
        max-file: "3"
    ports:
      - "8001:8000"
```

也可以在 Docker daemon 层面全局配置日志轮转，这样每个 service 不用重复写：

```json
// /etc/docker/daemon.json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
```

修改后重启 Docker：`sudo systemctl restart docker`

## 4. Loki + Grafana 日志采集

### 架构

```
VM 上的服务  ── 写文件 ──┐
                         ├── Promtail ──→ Loki ──→ Grafana
Docker 服务  ── stdout ──┘
```

三个组件，一个 docker-compose 跑起来，部署在任意一台 VM 上即可。

### docker-compose.logging.yml

```yaml
services:
  loki:
    image: grafana/loki:3.0.0
    command: -config.file=/etc/loki/local-config.yaml
    ports:
      - "3100:3100"
    volumes:
      - ./loki-config.yaml:/etc/loki/local-config.yaml
      - loki-data:/loki

  promtail:
    image: grafana/promtail:3.0.0
    volumes:
      - ./promtail-config.yaml:/etc/promtail/config.yml
      - /var/log:/var/log:ro
      - /var/lib/docker/containers:/var/lib/docker/containers:ro
    depends_on:
      - loki

  grafana:
    image: grafana/grafana:11.0.0
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=changeme     # 首次登录后立即修改
    volumes:
      - grafana-data:/var/lib/grafana

volumes:
  loki-data:
  grafana-data:
```

### loki-config.yaml

```yaml
auth_enabled: false

server:
  http_listen_port: 3100

common:
  path_prefix: /loki
  storage:
    filesystem:
      chunks_directory: /loki/chunks
      rules_directory: /loki/rules
  replication_factor: 1
  ring:
    kvstore:
      store: inmemory

limits_config:
  retention_period: 720h          # 保留 30 天

schema_config:
  configs:
    - from: 2024-01-01
      store: tsdb
      object_store: filesystem
      schema: v13
      index:
        prefix: index_
        period: 24h

compactor:
  working_directory: /loki/compactor
  retention_enabled: true
```

### promtail-config.yaml

```yaml
server:
  http_listen_port: 9080

positions:
  filename: /tmp/positions.yaml

clients:
  - url: http://loki:3100/loki/api/v1/push

scrape_configs:
  # ---- VM 上直接部署的服务 (读日志文件) ----
  - job_name: vm-apps
    static_configs:
      - targets: [localhost]
        labels:
          env: production
          __path__: /var/log/myapp/*.log
    pipeline_stages:
      - json:
          expressions:
            level: level
            service: service
            trace_id: trace_id
      - labels:
          level:
          service:

  # ---- Docker 容器日志 ----
  - job_name: docker
    docker_sd_configs:
      - host: unix:///var/run/docker.sock
        refresh_interval: 5s
    relabel_configs:
      - source_labels: ["__meta_docker_container_name"]
        regex: "/(.*)"
        target_label: "container"
      - source_labels: ["__meta_docker_container_name"]
        regex: "/(.*)"
        target_label: "service"
    pipeline_stages:
      - docker: {}
      - json:
          expressions:
            level: level
            service: service
            trace_id: trace_id
      - labels:
          level:
          service:
```

### 启动

```bash
docker compose -f docker-compose.logging.yml up -d
```

启动后访问 `http://<your-ip>:3000`，用 admin / changeme 登录 Grafana，添加 Loki 数据源（URL 填 `http://loki:3100`），即可在 Explore 页面查询日志。

### Grafana 常用查询

```logql
# 查看某个服务的所有 ERROR 日志
{service="order-service"} | json | level="error"

# 按 trace_id 追踪一个请求的完整链路
{env="production"} | json | trace_id="abc123def456"

# 最近 1 小时 ERROR 数量趋势
count_over_time({service="order-service"} | json | level="error" [5m])
```

### 告警配置

在 Grafana 中创建 Alert Rule：

- 条件：`count_over_time({service=~".+"} | json | level="error" [5m]) > 10`
- 含义：5 分钟内任意服务 ERROR 超过 10 条则告警
- 通知渠道：配置 Contact Point 发送到企微/钉钉/飞书群机器人 Webhook

## 5. 资源预估

以 5 个 FastAPI 服务、每个日均 50 万条 INFO 日志计算：

| 组件 | 内存 | 磁盘 |
|------|------|------|
| Loki | ~256MB | ~50GB / 30天 |
| Promtail | ~64MB | 忽略 |
| Grafana | ~128MB | ~1GB |
| **合计** | **~512MB** | **~51GB** |

现有 VM 上随便找一台就能跑。
