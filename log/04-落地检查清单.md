# 落地检查清单

## 应用层

- [ ] 安装 structlog 并完成日志初始化（`setup_logging()` 在启动时调用）
- [ ] 请求中间件已注册，每个请求自动注入 trace_id
- [ ] 生产环境输出结构化 JSON，包含 timestamp / level / service / trace_id / caller / message
- [ ] timestamp 使用 UTC 时区，ISO 8601 格式
- [ ] message 不拼接动态变量，动态数据放独立字段
- [ ] ERROR 日志必须包含 error 字段（完整错误信息）
- [ ] 敏感字段（password / token / secret）自动脱敏
- [ ] 手机号字段自动脱敏为 `138****8888`
- [ ] 调用其他服务时通过 `x-trace-id` Header 传递 trace_id

## 级别使用

- [ ] 参数校验失败、用户输入错误 → WARNING，不是 ERROR
- [ ] 第三方服务超时重试成功 → WARNING
- [ ] 非预期异常、需要人工排查 → ERROR
- [ ] 循环内不逐条打 INFO，汇总后打一条
- [ ] 大请求/响应体截断后再记录（最多 1KB）

## 部署配置

- [ ] 环境变量 `LOG_OUTPUT` 控制输出目标（stdout / file）
- [ ] VM 部署：LOG_OUTPUT=file，已配置 RotatingFileHandler 或 logrotate
- [ ] Docker 部署：LOG_OUTPUT=stdout，docker-compose 或 daemon.json 已配置 max-size / max-file
- [ ] 生产环境日志级别默认 INFO，支持通过接口动态调整
- [ ] 动态调整日志级别的接口已做鉴权

## 日志采集（可选，建议尽早搭建）

- [ ] Loki + Grafana + Promtail 已部署
- [ ] Promtail 同时采集 VM 日志文件和 Docker 容器日志
- [ ] Grafana 已配置 Loki 数据源
- [ ] 告警规则：5 分钟内 ERROR > 10 条触发通知
- [ ] 告警通知已对接团队 IM（企微/钉钉/飞书）
- [ ] Loki 日志保留 30 天
