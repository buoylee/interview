# 資深 CLI 速查(cli-toolbox)

> 給「會寫程式、但 shell 是半路出家」的資深工程師 / 架構師,一份**反查得到、記得住**的命令速查層。

## 這份速查不是什麼

- **不是教材**。它不從零教 Linux;深度需求(「為什麼會這樣」)一律連回對應的教學 track。
- **不是命令大全**。不收冷僻參數;只收**資深日常 / 救火 / 面試**真正反覆用得到、且初級不會的那批。

## 它是什麼:坐在教學 track 之上的「反查層」

```
教學 track(教「為什麼」)          這份(查「敲什麼 + 底層一句」)
linux-handson / nginx /     ──►   cli-toolbox
observability / mysql-handson      症狀/意圖 → 一行命令 → 底層一兩句
```

`linux-handson` 等負責**深教**;這份只負責**「一行查到 + 一句說透」**,要挖深就順著每章末尾的連結回去。

## 怎麼用(重要:別背命令)

每章都是同一個結構:

1. **收口地圖** —— 開頭幾條「核心原語」。**你要記的是這個**,不是命令。
2. **一行命令表** —— `命令 | 作用 | 底層一兩句`。
3. **深挖連結** —— 想懂原理 → 回對應教學 track。

> 心法:命令會忘,**原語不會**。每章開頭那張地圖記住了,具體命令用到再查——這份就是給你查的。

## 3 條跨域心法(整份表的地基)

這三條原語會在**好幾章**反覆出現。先吃透它們,後面很多命令就不用硬記,能「推」出來。

### 原語 1 —— 進程是分層歸屬的,信號沿這條鏈傳

```
終端(controlling terminal)
  └─ 會話 session
       └─ 進程組 process group
            └─ 進程 process
```

`Ctrl+C`、`Ctrl+Z`、斷線時的 `SIGHUP`、`kill` 整個進程組——**全是「對某一層發信號」**。理解了這條鏈,job control、`setsid`、`kill -- -PGID` 一次全通。(細節見 **01**)

### 原語 2 —— 找東西看「程式名 vs 命令列」

工具預設只比對**程式名**(`bash`、`sleep`);要按**命令列裡的字串**找(`/tmp/foo.log`、`--port=8080`),得切到「整條命令列」模式:

- `pgrep` / `pkill` → 加 **`-f`**
- `ps` → 改用 `ps -ef | grep '[x]xx'`(`ps -C` 永遠只認程式名)

**今天有人在這裡卡過**:`ps -C setsid-test.log` 撲空,就因為那是參數不是程式名。(見 **01**、**05**)

### 原語 3 —— 一切皆檔案,小工具靠管道組合

Linux 裡進程、socket、設備都被當「檔案 / fd」對待——所以 `lsof`、`/proc/<pid>/`、`> file 2>&1` 能一招通吃。而 Unix 的威力在**組合**:`|` 串起小工具,`xargs` 把「上一個的輸出」變「下一個的參數」。**不存在一個萬能命令,只有能拼起來的小積木**。(見 **04**、**05**)
