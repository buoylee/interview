## 指令

### `:` 绑定（v-bind 缩写）

把 JS 表达式的值绑定到 HTML 属性上，属性值是**动态**的。

```html
<!-- 完整写法 -->
<img v-bind:src="imgUrl">
<!-- 缩写 -->
<img :src="imgUrl">

<!-- 不加 : 是纯字符串 -->
<img src="imgUrl">        <!-- 浏览器会请求 "imgUrl" 这个字符串路径 -->
<img :src="imgUrl">       <!-- 会请求 imgUrl 变量的值，如 "https://xxx.png" -->
```

常见用法：

```html
<!-- 绑定属性 -->
<div :class="{ active: isActive }">
<div :style="{ color: textColor }">
<input :disabled="isLoading">

<!-- 向子组件传 prop -->
<MyComp :title="pageTitle">      <!-- 传变量 -->
<MyComp title="hello">           <!-- 传固定字符串，不需要 : -->
<MyComp :count="3">              <!-- 传数字，需要 :，否则 "3" 是字符串 -->
<MyComp :show="true">            <!-- 传布尔值，需要 : -->
<MyComp :list="[1, 2, 3]">      <!-- 传数组/对象，需要 : -->
```

**总结**：只要值不是纯字符串，就需要 `:`。

---

### `@` 监听（v-on 缩写）

监听 DOM 事件或子组件自定义事件。

```html
<!-- 完整写法 -->
<button v-on:click="handleClick">
<!-- 缩写 -->
<button @click="handleClick">
```

常见用法：

```html
<!-- DOM 原生事件 -->
<button @click="count++">           <!-- 内联表达式 -->
<button @click="handleClick">       <!-- 方法名 -->
<button @click="handleClick(item)"> <!-- 传参 -->
<input @input="onInput">            <!-- 输入事件 -->
<input @keyup.enter="onSubmit">     <!-- 按键修饰符 -->
<form @submit.prevent="onSubmit">   <!-- .prevent 阻止默认行为 -->

<!-- 子组件自定义事件 -->
<!-- 子组件内: this.$emit('change', newVal) -->
<MyComp @change="handleChange">

<!-- 等同于 v-model 拆写 -->
<MyComp :value="msg" @input="msg = $event">
```

---

### `:` 和 `@` 的对比

| 缩写 | 完整写法 | 方向 | 作用 |
|------|---------|------|------|
| `:` | `v-bind` | 父 → 子（传数据） | 绑定属性/prop |
| `@` | `v-on` | 子 → 父（传事件） | 监听事件 |

配合使用就是 `v-model`：

```html
<MyComp v-model="msg">
<!-- 等价于 -->
<MyComp :value="msg" @input="msg = $event">
<!--      ↑ 父传子       ↑ 子传父           -->
```

### 命名转换规则（kebab-case vs camelCase）

**Props（`:` 绑定）— 自动转换**

HTML 属性不区分大小写，所以模板中必须写 kebab-case，Vue 内部自动转为 camelCase 匹配 prop：

```html
<!-- 父组件：kebab-case -->
<MyComp :agent-name="name">

<!-- 子组件：camelCase 接收 -->
props: {
  agentName: String   // 自动匹配 agent-name
}
```

**Events（`@` 监听）— 不转换**

事件名不做任何转换，触发和监听必须完全一致：

```html
<!-- 子组件触发 -->
this.$emit('update-name', val)

<!-- 父组件 -->
<MyComp @update-name="handleUpdate">   <!-- OK -->
<MyComp @updateName="handleUpdate">    <!-- 不生效! -->
```

| | 模板中写法 | JS 中写法 | 自动转换 |
|---|---|---|---|
| `:` props | `:agent-name` (kebab) | `agentName` (camel) | 有 |
| `@` events | `@event-name` (kebab) | `$emit('event-name')` (kebab) | 无 |

## 时间总线

通过 Vue 原型对象实现bus, 把某个component的**某个事件注册到 bus**, 第2个component 可以通过**触发bus上这个事件**, 传递参数到 某个component.

### 初始化bus

```
new Vue({
...
	beforeCreate(){
		Vue.prototype.$bus = this //安装全局事件总线，$bus就是当前应用的vm
	},
...
})
```

### 注册事件

```
methods(){
	demo(data){...}
}
...
mounted(){
	this.$bus.$on('eventName'， this.funcName)
}
```

### 触发事件

```
this.$bus.$emit('xxxx'，数据)
```

### 记得解绑

最好在`beforeDestroy`钩子中，用 $off 去解绑当前组件所用到的事件。



## $nexttick (dom下一轮(dom更新后))

有个需求, 需要把一个text框变为input, 然后focus这个input(focus是实际标签操作), 
在一个方法内有序地执行这2个动作, 
因为虚拟dom的缘故, 即使model改了, 页面上的text实际还没来得及变化, 却调用了docus,
导致focus失败, 所以,需要延迟执行, 
这里的nexttick 就等于延迟执行.
