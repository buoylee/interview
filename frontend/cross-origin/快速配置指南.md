# 本地调试跨域 Cookie 登录 - 快速配置指南

## TL;DR（太长不看版）

**问题**：本地开发无法获取 `.cocorobo.cn` 域的 Cookie
**原因**：`localhost` 和 `.cocorobo.cn` 域名不匹配
**解决**：使用 HTTPS + 本地子域名 `local.cocorobo.cn`

## 一键配置脚本

```bash
# 1. 安装工具并生成证书
brew install mkcert
mkcert -install
cd /path/to/your/project
mkcert local.cocorobo.cn localhost 127.0.0.1

# 2. 配置 hosts
echo "127.0.0.1  local.cocorobo.cn" | sudo tee -a /etc/hosts

# 3. 启动服务
npm run serve
```

## Shadowrocket 配置（一次性）

在配置的 `[Rule]` 部分最上面添加：

```
DOMAIN-SUFFIX,local.cocorobo.cn,DIRECT
DOMAIN,localhost,DIRECT
IP-CIDR,127.0.0.1/32,DIRECT
```

## 使用方式

### 快速开发（推荐）
```
https://localhost:8081/standalone.html#/?id=xxx&type=mutiagent
```
- 自动使用测试用户
- 无需登录

### 测试真实登录
```
https://local.cocorobo.cn:8081/standalone.html#/?id=xxx&type=mutiagent&testRealLogin=true
```
- 完整登录流程
- 测试 Cookie 传递

## 常见错误速查

| 错误 | 原因 | 解决 |
|------|------|------|
| ERR_EMPTY_RESPONSE | 用 http 访问 https 服务器 | 改成 `https://` |
| ERR_TUNNEL_CONNECTION_FAILED | 代理拦截了本地域名 | 配置 Shadowrocket 直连 |
| 登录后页面不消失 | 使用了 localhost | 改用 `local.cocorobo.cn` |
| 401 Unauthorized | Cookie 未传递 | 检查域名、HTTPS、代理配置 |

## 核心原理（一句话）

**浏览器只会将 `.cocorobo.cn` 域的 Cookie 发送给 `*.cocorobo.cn` 的域名，localhost 无法接收。**
