# 跨域速查表

## 🎯 30 秒速记

### 同源判断
```
https://www.example.com:443/page.html

协议：https
域名：www.example.com
端口：443

三者完全相同 = 同源 ✅
任一不同 = 跨域 ❌
```

### 跨域解决方案选择

| 场景 | 推荐方案 | 代码 |
|------|---------|------|
| API 请求 | CORS | `res.header('Access-Control-Allow-Origin', '*')` |
| 开发调试 | Proxy | `devServer.proxy: { '/api': { target: 'xxx' } }` |
| 生产环境 | Nginx | `proxy_pass http://backend;` |
| iframe 通信 | postMessage | `window.postMessage(data, origin)` |
| 实时通信 | WebSocket | `new WebSocket('wss://api.com')` |
| 老浏览器 | JSONP | `<script src="api?callback=fn">` |

## 🔥 高频考点

### 考点 1：同源策略限制什么？

```
✅ 限制：
  - DOM 访问（iframe）
  - AJAX/Fetch 请求
  - Cookie/LocalStorage

❌ 不限制：
  - <script>、<img>、<link>、<iframe> 加载
  - 表单提交
```

### 考点 2：CORS 简单请求 vs 预检请求

**简单请求（直接发送）**：
- 方法：`GET`、`POST`、`HEAD`
- Content-Type：
  - `text/plain`
  - `multipart/form-data`
  - `application/x-www-form-urlencoded`

**预检请求（先发 OPTIONS）**：
- 其他方法：`PUT`、`DELETE`、`PATCH`
- 自定义头
- `Content-Type: application/json`

### 考点 3：跨域携带 Cookie

```javascript
// 前端
fetch(url, { credentials: 'include' })

// 后端
res.header('Access-Control-Allow-Credentials', 'true')
res.header('Access-Control-Allow-Origin', 'https://example.com') // 不能用 *
```

### 考点 4：Cookie 域名策略

```javascript
Cookie: Domain=.example.com

可访问：
✅ example.com
✅ www.example.com
✅ api.example.com
✅ local.example.com

不可访问：
❌ localhost
❌ other.com
```

**重点**：同源策略 ≠ Cookie 域名策略

### 考点 5：SameSite 属性

```javascript
// Strict：完全禁止跨站
Set-Cookie: id=a; SameSite=Strict

// Lax：允许安全导航（默认）
Set-Cookie: id=a; SameSite=Lax

// None：允许所有（需 HTTPS）
Set-Cookie: id=a; SameSite=None; Secure
```

## 💻 常用代码片段

### 1. CORS 服务端配置（Node.js）

```javascript
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', 'https://example.com');
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE');
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  res.header('Access-Control-Allow-Credentials', 'true');

  if (req.method === 'OPTIONS') {
    return res.sendStatus(200);
  }
  next();
});
```

### 2. webpack-dev-server 代理

```javascript
module.exports = {
  devServer: {
    proxy: {
      '/api': {
        target: 'https://api.example.com',
        changeOrigin: true,
        pathRewrite: { '^/api': '' }
      }
    }
  }
}
```

### 3. Nginx 反向代理

```nginx
location /api/ {
    proxy_pass https://api.example.com/;
    proxy_set_header Host api.example.com;
    proxy_set_header X-Real-IP $remote_addr;
}
```

### 4. postMessage 通信

```javascript
// 发送
targetWindow.postMessage(data, 'https://target.com');

// 接收
window.addEventListener('message', (event) => {
  if (event.origin !== 'https://trusted.com') return; // 验证来源
  console.log(event.data);
});
```

### 5. JSONP 实现

```javascript
function jsonp(url, callback) {
  const callbackName = 'jsonp_' + Date.now();
  window[callbackName] = (data) => {
    callback(data);
    delete window[callbackName];
    script.remove();
  };

  const script = document.createElement('script');
  script.src = `${url}?callback=${callbackName}`;
  document.body.appendChild(script);
}
```

## 🐛 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `No 'Access-Control-Allow-Origin'` | 服务端未设置 CORS | 添加响应头 |
| `Credentials flag is true, but...` | 携带凭证但服务端未配置 | 添加 `Allow-Credentials: true` |
| `Origin must not be '*'` | 携带凭证时 Origin 用了 * | 指定具体 Origin |
| `Tainted canvas` | Canvas 跨域图片污染 | 设置 `img.crossOrigin = 'anonymous'` |

## 📝 面试回答模板

### Q: 什么是跨域？

**答**：
跨域是指浏览器从一个域名的网页去请求另一个域名的资源，由于浏览器的同源策略（协议、域名、端口必须完全相同）限制而被阻止的现象。

这是浏览器的安全机制，防止恶意网站读取其他网站的敏感数据，比如防止恶意网站通过 AJAX 读取用户在银行网站的账户信息。

### Q: 如何解决跨域？

**答**：
常用方案有：

1. **CORS**（最推荐）：服务端设置 `Access-Control-Allow-Origin` 响应头，支持所有 HTTP 方法，可携带凭证。

2. **代理服务器**：开发环境用 webpack-dev-server 代理，生产环境用 Nginx 反向代理。

3. **postMessage**：用于不同窗口/iframe 之间的通信。

4. **JSONP**：利用 script 标签不受限制的特性，但只支持 GET 请求，已过时。

5. **WebSocket**：不受同源策略限制的实时通信协议。

具体选择哪种方案要根据实际场景，一般 API 请求用 CORS，开发调试用代理。

### Q: CORS 携带 Cookie 需要注意什么？

**答**：
需要同时配置前端和后端：

**前端**：
```javascript
fetch(url, { credentials: 'include' })
```

**后端**：
```javascript
res.header('Access-Control-Allow-Credentials', 'true')
res.header('Access-Control-Allow-Origin', 'https://example.com') // 不能用 *
```

**重点**：携带凭证时，`Access-Control-Allow-Origin` 不能设置为 `*`，必须指定具体的源。

## 🎓 加分项

### 1. 提及安全

- CORS 配置不要随意使用 `*`
- postMessage 必须验证 `origin`
- 注意 CSRF 攻击，使用 `SameSite`
- 不要在客户端暴露敏感信息

### 2. 性能优化

- 使用 `Access-Control-Max-Age` 缓存预检请求
- 避免不必要的跨域请求
- 考虑使用 HTTP/2 多路复用
