# 闭包 (Closure)

## 一句话定义

**闭包 = 返回的函数 + 它引用的外层变量。** 只要返回的函数还被持有，它引用的外层变量就不会被 GC 回收。

## 闭包 vs 非闭包

```js
// 非闭包：返回值，复制后断开联系，变量被回收
function createIndex() {
  let currentIndex = -1
  currentIndex = 5
  return currentIndex  // 返回数字 5，跟 currentIndex 再无关系
}
const idx = createIndex()  // idx = 5，currentIndex 已死

// 闭包：返回函数，函数引用了外层变量，变量活着
function createCounter() {
  let count = 0
  return {
    increment() { count++ },
    getCount() { return count }
  }
}
const counter = createCounter()
// createCounter 执行完了，但 count 没被回收
counter.increment()
counter.increment()
counter.getCount()  // 2，count 一直活着
```

## 判断标准

**返回的对象里有没有函数，这个函数有没有用到外层的变量。** 有就是闭包，没有就不是。

跟返回的数据类型无关：

```js
// 返回对象但不是闭包 → 值复制
function createObj() {
  let name = 'hello'
  return { name }  // { name: 'hello' }，只是把值拷进对象
}
const obj = createObj()
// name 变量已死，obj.name 只是字符串 'hello'

// 返回对象且是闭包 → 函数引用了外层变量
function createObj2() {
  let name = 'hello'
  return {
    getName() { return name },
    setName(n) { name = n }
  }
}
const obj2 = createObj2()
obj2.setName('world')
obj2.getName()  // 'world'，name 还活着
```

## 容易搞混的点

### 1. "返回了内部变量" ≠ 闭包

```js
const idx = createIndex()  // idx 拿到的是数字 5
```

这不是引用，是**值复制**。`idx` 和 `currentIndex` 没有任何关系，改 `idx` 不会影响 `currentIndex`，`currentIndex` 已经被回收了。

### 2. 外层变量的范围：整条作用域链，但只保留被引用的

```js
function grandparent() {
  let a = 1
  function parent() {
    let b = 2
    function child() {
      return a + b  // 引用了 grandparent 的 a 和 parent 的 b
    }
    return child
  }
  return parent
}

const p = grandparent()
const c = p()
c()  // 3

// a 活着 ← child 引用了
// b 活着 ← child 引用了
```

现代 JS 引擎只保留闭包**实际引用的变量**，没被引用的正常回收。

## 实际应用：Vue Composable

Vue composable 就是典型的闭包模式：

```js
function useStreamingTTS(options) {
  // 内部变量 —— 外部无法直接访问
  let streamingTTS = null
  let processChain = Promise.resolve()

  // 返回的函数引用了内部变量 —— 闭包
  function addStreamingText(text) {
    processChain = processChain.then(() => {
      streamingTTS.addText(text)
    })
  }

  function stopAll() {
    processChain = Promise.resolve()
    streamingTTS?.stop()
  }

  return { addStreamingText, stopAll }
}

const tts = useStreamingTTS()
// useStreamingTTS 执行完了
// 但 streamingTTS、processChain 还活着
// 因为 addStreamingText 和 stopAll 在引用它们
```

用闭包代替 class 的 this，内部变量天然私有，外部只能通过返回的方法访问。
