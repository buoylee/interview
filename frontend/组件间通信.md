## 总览

分清: 事件, 还是 状态.



## store

全局

一般存放 业务数据

持久数据.

一份数据在多处共享, 响应式更新.

非组件代码中访问, API 调用、路由处理等

## dispatch 事件传递

由事件来触发, 再到 数据更新...

1. 临时的事件通知：按钮点击、表单提交、文件上传等
2. 用户交互响应：输入变化、选择变化等
3. 组件间的动作通知：子组件告诉父组件"我完成了某个动作"

## props

直接父 -> 子



## content

### 使用场景:

i18n, 更像是一个**工具配置**，而**不是业务状态**, 确保只有需要国际化的组件才会获取 i18n 对象

测试, 如果直接 import，测试时需要 mock 整个 store, 用 context 可以只 mock 需要的部分.

### 原因

避免全局污染
